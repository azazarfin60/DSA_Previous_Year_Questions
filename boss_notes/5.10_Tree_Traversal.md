# 5.10 Tree Traversal

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 03–06 | **Section:** A/B  
> **Course Outcome:** Maps to CO2

---

## 1. Prerequisite Knowledge
- **Binary Tree:** A tree where each node has at most two children (Left and Right).
- **Node Structure:**
  ```
  Node:
      DATA                     // the actual value stored
      LEFT (pointer)           // address of the left child
      RIGHT (pointer)          // address of the right child
  ```
- **Recursion:** Traversal algorithms heavily rely on recursive function calls.

---

## 2. Core Definitions
**Tree Traversal** is the process of visiting every node in a tree exactly once. Unlike arrays or linked lists (which are linear), trees can be traversed in multiple ways. 

There are two main categories:
1. **Depth-First Search (DFS):** Goes deep down a branch before backtracking. Includes Preorder, Inorder, and Postorder.
2. **Breadth-First Search (BFS):** Visits nodes level by level. Includes Level Order Traversal.

---

## 3. Depth-First Traversals (DFS)

### 3.1 Inorder Traversal (Left, Root, Right)
**Concept:** Visit the left subtree, then the root node, then the right subtree.
*Note: In a Binary Search Tree (BST), Inorder traversal yields the elements in sorted (ascending) order.*

**Pseudocode:**
**C++ Style:**
```cpp
void inorder(Node* root) {
    if (root == nullptr) {
        return;
    }
    inorder(root->left);           // 1. Visit Left subtree
    cout << root->data << " ";     // 2. Visit Root
    inorder(root->right);          // 3. Visit Right subtree
}
```

**OR, Textbook Style:**
```
Procedure INORDER(ROOT)
    If ROOT = NULL Then
        Return
    End If
    INORDER(LEFT[ROOT])            // 1. Visit Left subtree
    Print INFO[ROOT]               // 2. Visit Root
    INORDER(RIGHT[ROOT])           // 3. Visit Right subtree
End Procedure
```

### 3.2 Preorder Traversal (Root, Left, Right)
**Concept:** Visit the root node first, then the left subtree, then the right subtree.
*Use Case: Used to create a copy of the tree.*

**Pseudocode:**
**C++ Style:**
```cpp
void preorder(Node* root) {
    if (root == nullptr) {
        return;
    }
    cout << root->data << " ";     // 1. Visit Root
    preorder(root->left);          // 2. Visit Left subtree
    preorder(root->right);         // 3. Visit Right subtree
}
```

**OR, Textbook Style:**
```
Procedure PREORDER(ROOT)
    If ROOT = NULL Then
        Return
    End If
    Print INFO[ROOT]               // 1. Visit Root
    PREORDER(LEFT[ROOT])           // 2. Visit Left subtree
    PREORDER(RIGHT[ROOT])          // 3. Visit Right subtree
End Procedure
```

### 3.3 Postorder Traversal (Left, Right, Root)
**Concept:** Visit the left subtree, then the right subtree, then the root node.
*Use Case: Used to delete the tree (you must delete children before the parent).*

**Pseudocode:**
**C++ Style:**
```cpp
void postorder(Node* root) {
    if (root == nullptr) {
        return;
    }
    postorder(root->left);         // 1. Visit Left subtree
    postorder(root->right);        // 2. Visit Right subtree
    cout << root->data << " ";     // 3. Visit Root
}
```

**OR, Textbook Style:**
```
Procedure POSTORDER(ROOT)
    If ROOT = NULL Then
        Return
    End If
    POSTORDER(LEFT[ROOT])          // 1. Visit Left subtree
    POSTORDER(RIGHT[ROOT])         // 2. Visit Right subtree
    Print INFO[ROOT]               // 3. Visit Root
End Procedure
```

## 4. Breadth-First Traversal (Level Order)

**Concept:** Visit every node on a level before going to a lower level.
*Implementation:* Requires a **Queue** data structure.

**Algorithm:**
1. Create an empty queue and enqueue the `root` node.
2. Loop while the queue is not empty:
   a. Dequeue a node and print its data.
   b. If the node has a left child, enqueue the left child.
   c. If the node has a right child, enqueue the right child.

**Pseudocode:**
**C++ Style:**
```cpp
#include <queue>
using namespace std;

void levelOrder(Node* root) {
    if (root == nullptr) {
        return;
    }

    queue<Node*> q;
    q.push(root);

    while (!q.empty()) {
        Node* current = q.front();
        q.pop();
        cout << current->data << " ";

        if (current->left != nullptr) {
            q.push(current->left);
        }
        if (current->right != nullptr) {
            q.push(current->right);
        }
    }
}
```

**OR, Textbook Style:**
```
Procedure LEVEL_ORDER(ROOT)
    If ROOT = NULL Then
        Return
    End If

    Create empty Queue Q
    ENQUEUE(Q, ROOT)

    While Q is not empty do
        Set CURRENT = DEQUEUE(Q)
        Print INFO[CURRENT]

        If LEFT[CURRENT] ≠ NULL Then
            ENQUEUE(Q, LEFT[CURRENT])
        End If
        If RIGHT[CURRENT] ≠ NULL Then
            ENQUEUE(Q, RIGHT[CURRENT])
        End If
    End While
End Procedure
```

## 5. Trace Example

**Given Tree:**
```text
       A
      / \
     B   C
    / \   \
   D   E   F
```

**Results:**
- **Inorder (L, Root, R):** `D B E A C F`
- **Preorder (Root, L, R):** `A B D E C F`
- **Postorder (L, R, Root):** `D E B F C A`
- **Level Order:** `A B C D E F`

---

## 6. Time & Space Complexity

| Traversal | Time Complexity | Space Complexity (Worst Case) | Space Complexity (Avg Case) |
|---|---|---|---|
| **Inorder** | O(N) | O(N) (Skewed Tree Stack) | O(log N) (Balanced Tree) |
| **Preorder** | O(N) | O(N) | O(log N) |
| **Postorder**| O(N) | O(N) | O(log N) |
| **Level Order**| O(N) | O(N) (Queue holds leaf level) | O(N/2) $\approx$ O(N) |

## 7. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Write the pseudocode for Preorder and Postorder traversal of a binary tree." (04 marks)**
   - *Strategy:* Write the node structure, followed by the two simple recursive procedures. Add inline comments labeling "Root, Left, Right".
2. **"Given a tree, write its Inorder, Preorder, and Postorder traversals." (05 marks)**
   - *Strategy:* Draw the tree on rough paper. Trace the contour: Left side = Preorder, Bottom = Inorder, Right side = Postorder. Box your final string sequences.
3. **"Construct a tree given Inorder and Preorder traversals." (05 marks)**
   - *Strategy:* The first element of Preorder is the Root. Find this root in the Inorder sequence. Elements left of it in Inorder form the left subtree; elements right form the right subtree. Recursively repeat.

### Practice Questions:
1. Given Inorder: `4 2 5 1 3 6`, Preorder: `1 2 4 5 3 6`. Construct the tree.
2. Why is Postorder traversal used for deleting a tree? *(Answer: You must free the memory of the left and right children before you free the parent node's memory. Postorder ensures the parent is visited last).*
