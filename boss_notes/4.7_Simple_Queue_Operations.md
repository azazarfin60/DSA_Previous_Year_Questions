# 4.7 Simple Queue Operations

> **Exam Frequency:** 🟡 High Frequency | **Typical Marks:** 03–05 | **Section:** A  
> **Course Outcome:** Maps to CO1 / CO2

---

## 1. Core Definitions
A **Queue** is a linear data structure that follows the **FIFO (First In, First Out)** principle. The element inserted first is the first one to be removed.
- **Real-world analogy:** People standing in a line at a ticket counter. The person who arrives first gets the ticket first.

### Key Pointers:
- **FRONT:** Tracks the index of the first element (where deletions happen).
- **REAR:** Tracks the index of the last inserted element (where insertions happen).
- **Initial State (Empty Queue):** `FRONT = -1`, `REAR = -1`.

---

## 2. Queue Representation (Array)

```
Declare queue[MAX]
Set front = -1
Set rear = -1

```

---

## 3. Enqueue Operation (Insertion)

### Concept
Adding an element to the **Rear** of the queue. Before adding, we must check if the queue is full (**Overflow**).

### Algorithm
1. Check if queue is full: `if (rear == MAX - 1)` $\rightarrow$ return Overflow.
2. If the queue is empty (`front == -1` and `rear == -1`), set `front = 0` and `rear = 0`.
3. Else, increment rear: `rear = rear + 1`.
4. Insert the value: `queue[rear] = value`.

### Pseudocode
```
Procedure ENQUEUE(value)
    If rear = MAX - 1 Then
        Print "Queue Overflow!"
        Return
    if (front == -1 && rear == -1) { // First element insertion
        Set front = 0
        Set rear = 0
    Else
        Set rear = rear + 1
    Set queue[rear] = value

```
**Time Complexity:** O(1)

---

## 4. Dequeue Operation (Deletion)

### Concept
Removing an element from the **Front** of the queue. Before removing, we must check if the queue is empty (**Underflow**).

### Algorithm
1. Check if queue is empty: `if (front == -1 || front > rear)` $\rightarrow$ return Underflow.
2. Store the value to be deleted: `value = queue[front]`.
3. If this was the last element in the queue (`front == rear`), reset the queue to empty: `front = -1`, `rear = -1`.
4. Else, increment front to point to the next element: `front = front + 1`.
5. Return the value.

### Pseudocode
```
Declare dequeue() {
    If front = -1 OR front > rear Then
        Print "Queue Underflow!"
        Return -1
    Set value = queue[front]

    if (front == rear) { // Queue now becomes empty
        Set front = -1
        Set rear = -1
    Else
        Set front = front + 1
    Return value

```
**Time Complexity:** O(1)

---

## 5. Peek / Front Operation
View the element at the front without removing it.

```
Declare peek() {
    If front = -1 OR front > rear Then
        Print "Queue is empty"
        Return -1
    Return queue[front]

```
**Time Complexity:** O(1)

---

## 6. The Major Drawback of Simple (Linear) Queue

In an array-based simple queue, once an element is dequeued from the `front`, that space is **wasted** and cannot be reused. 

**Example Trace:**
1. Insert 10, 20, 30, 40, 50. (`front = 0`, `rear = 4`). Queue is full.
2. Dequeue 10, 20. (`front = 2`, `rear = 4`).
3. Now attempt to Insert 60. 
   - `rear == MAX - 1` is TRUE. It will report **Overflow!**
   - However, indices 0 and 1 are empty!
   
*Solution to this problem:* **Circular Queue**.

---

## 7. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Define Queue. Write the algorithm for Enqueue and Dequeue operations." (05 marks)**
   - *Strategy:* Provide the FIFO definition. Write the full pseudocode. Crucially, don't forget the empty-queue reset logic `if (front == rear) { front=-1; rear=-1; }` in the Dequeue algorithm.
2. **"What is the main limitation of a linear array queue? How is it resolved?" (03 marks)**
   - *Strategy:* Explain the "wasted space" scenario. Show a quick diagram or state the indices as shown in Section 6. State that a Circular Queue resolves this by connecting the `rear` back to the `front`.
