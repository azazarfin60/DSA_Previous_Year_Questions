# 1.8 Linked List Operations

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 04–10 | **Section:** A/B  
> **Course Outcome:** Maps to CO1 / CO2

---

## 1. Prerequisite Knowledge
- Node Structure:
  ```
  Node:
      DATA                     // the actual value stored
      NEXT (pointer)           // address of the next node
  ```
- Memory allocation: Creating a new node and getting a free node from AVAIL.
- Understanding of the `HEAD` pointer.

---

## 2. Traversal and Searching

### Working Principle
To traverse or search, we start a temporary pointer `temp` at `head` and loop until `temp` becomes `NULL`.

### Algorithm: Search(head, key)
1. Initialize `temp = head`.
2. Repeat while `temp != NULL`:
   a. If `temp->data == key`, return "Found".
   b. `temp = temp->next`.
3. Return "Not Found".

### Pseudocode
```
Procedure SEARCH(HEAD, KEY)
    Set TEMP = HEAD
    While TEMP ≠ NULL do
        If INFO[TEMP] = KEY Then
            Print "Element found."
            Return
        End If
        Set TEMP = LINK[TEMP]
    End While
    Print "Element not found."
End Procedure
```
**Time Complexity:** O(N) | **Space Complexity:** O(1)

## 3. Insertion Operations

### 3.1 Insert at Beginning
**Concept:** Create a new node, point its `next` to the current `head`, and then update `head` to be the new node.

**Pseudocode:**
```
Procedure INSERT_BEGINNING(HEAD, AVAIL, ITEM)
    If AVAIL = NULL Then
        Print "OVERFLOW"
        Return
    End If
    Set NEW = AVAIL
    Set AVAIL = LINK[AVAIL]
    Set INFO[NEW] = ITEM
    Set LINK[NEW] = HEAD          // Link new node to current head
    Set HEAD = NEW                // Move head to new node
End Procedure
```
**Time Complexity:** O(1)

### 3.2 Insert at End
**Concept:** Create a new node. If the list is empty, make it the `head`. Otherwise, traverse to the last node and update its `next` pointer.

**Pseudocode:**
```
Procedure INSERT_END(HEAD, AVAIL, ITEM)
    If AVAIL = NULL Then
        Print "OVERFLOW"
        Return
    End If
    Set NEW = AVAIL
    Set AVAIL = LINK[AVAIL]
    Set INFO[NEW] = ITEM
    Set LINK[NEW] = NULL          // It will be the last node

    If HEAD = NULL Then           // Empty list case
        Set HEAD = NEW
        Return
    End If

    Set TEMP = HEAD
    While LINK[TEMP] ≠ NULL do   // Traverse to last node
        Set TEMP = LINK[TEMP]
    End While
    Set LINK[TEMP] = NEW          // Link last node to new node
End Procedure
```
**Time Complexity:** O(N)

### 3.3 Insert at Specific Position
**Concept:** Traverse to the `pos - 1` node. Link the new node to the `pos` node, then link the `pos - 1` node to the new node.

**Pseudocode:**
```
Procedure INSERT_AT_POSITION(HEAD, AVAIL, ITEM, POS)
    If AVAIL = NULL Then
        Print "OVERFLOW"
        Return
    End If
    Set NEW = AVAIL
    Set AVAIL = LINK[AVAIL]
    Set INFO[NEW] = ITEM

    If POS = 1 Then               // Insert at head
        Set LINK[NEW] = HEAD
        Set HEAD = NEW
        Return
    End If

    Set TEMP = HEAD
    For I = 1 to POS - 2 do      // Traverse to (POS-1)
        If TEMP = NULL Then
            Print "Position out of bounds"
            Return
        End If
        Set TEMP = LINK[TEMP]
    End For

    Set LINK[NEW] = LINK[TEMP]    // Point new node to next node
    Set LINK[TEMP] = NEW          // Point previous node to new node
End Procedure
```
**Time Complexity:** O(N)

## 4. Deletion Operations

### 4.1 Delete from Beginning
**Pseudocode:**
```
Procedure DELETE_BEGINNING(HEAD, AVAIL)
    If HEAD = NULL Then
        Print "UNDERFLOW — List is empty"
        Return
    End If
    Set TEMP = HEAD
    Set HEAD = LINK[HEAD]         // Move head to second node
    Set LINK[TEMP] = AVAIL        // Return freed node to AVAIL
    Set AVAIL = TEMP
End Procedure
```
**Time Complexity:** O(1)

### 4.2 Delete from End
**Pseudocode:**
```
Procedure DELETE_END(HEAD, AVAIL)
    If HEAD = NULL Then
        Print "UNDERFLOW — List is empty"
        Return
    End If

    If LINK[HEAD] = NULL Then     // Only one node
        Set AVAIL = HEAD
        Set HEAD = NULL
        Return
    End If

    Set TEMP = HEAD
    Set PREV = NULL
    While LINK[TEMP] ≠ NULL do   // Traverse to last node
        Set PREV = TEMP
        Set TEMP = LINK[TEMP]
    End While
    Set LINK[PREV] = NULL         // Second to last becomes last
    Set LINK[TEMP] = AVAIL        // Return freed node to AVAIL
    Set AVAIL = TEMP
End Procedure
```
**Time Complexity:** O(N)

## 5. Reverse a Linked List

This is a **frequently asked exam question**.

### Working Principle
We use three pointers: `prev`, `current`, and `next`. We iterate through the list, temporarily storing the next node, changing the current node's pointer to point to the previous node, and then advancing all pointers.

### Algorithm Trace
Initial State: `1 -> 2 -> 3 -> NULL`
- **Iter 1:** `curr = 1`. `next = 2`. `1->next = NULL`. `prev = 1`, `curr = 2`.
- **Iter 2:** `curr = 2`. `next = 3`. `2->next = 1`. `prev = 2`, `curr = 3`.
- **Iter 3:** `curr = 3`. `next = NULL`. `3->next = 2`. `prev = 3`, `curr = NULL`.
- Final `head` becomes `prev` (Node 3).

### Pseudocode
```
Procedure REVERSE(HEAD)
    Set PREV = NULL
    Set CURR = HEAD
    While CURR ≠ NULL do
        Set NEXT_NODE = LINK[CURR]     // Store next node
        Set LINK[CURR] = PREV          // Reverse the link
        Set PREV = CURR                // Move prev one step ahead
        Set CURR = NEXT_NODE           // Move current one step ahead
    End While
    Set HEAD = PREV                    // Update head to new first node
    Return HEAD
End Procedure
```
**Time Complexity:** O(N) | **Space Complexity:** O(1)

## 6. Split and Merge

### 6.1 Split a Linked List
**Concept:** Splitting a list into two halves. Best done using the "Slow and Fast Pointer" (Tortoise and Hare) approach to find the middle.

```
Procedure SPLIT_LIST(HEAD)
    If HEAD = NULL OR LINK[HEAD] = NULL Then
        Return                         // Cannot split empty or single-node list
    End If
    Set SLOW = HEAD
    Set FAST = LINK[HEAD]

    // Fast moves 2 steps, slow moves 1 step
    While FAST ≠ NULL AND LINK[FAST] ≠ NULL do
        Set SLOW = LINK[SLOW]
        Set FAST = LINK[LINK[FAST]]
    End While

    // Slow is now at the middle
    Set FRONT = HEAD
    Set BACK = LINK[SLOW]
    Set LINK[SLOW] = NULL              // Break the connection
End Procedure
```

### 6.2 Merge Two Sorted Linked Lists
**Concept:** Given two sorted lists, combine them into a single sorted list by comparing the heads.

```
Procedure SORTED_MERGE(A, B)
    // Base cases
    If A = NULL Then
        Return B
    End If
    If B = NULL Then
        Return A
    End If

    // Pick either A or B, and recur
    If INFO[A] ≤ INFO[B] Then
        Set RESULT = A
        Set LINK[RESULT] = SORTED_MERGE(LINK[A], B)
    Else
        Set RESULT = B
        Set LINK[RESULT] = SORTED_MERGE(A, LINK[B])
    End If
    Return RESULT
End Procedure
```
**Time Complexity:** O(N + M)

## 7. Exam Strategy & Past Questions

### Typical Exam Questions:
1. **"Write an algorithm to reverse a singly linked list. Show the trace." (05 marks)**
   - *Strategy:* Write the 3-pointer pseudocode. Draw a quick trace showing 3 nodes changing direction step-by-step.
2. **"Write a C function to insert a node at the end of a linked list." (04 marks)**
   - *Strategy:* Write the `insertAtEnd` pseudocode. Mention the O(N) time complexity. Don't forget the empty list edge case `if (*head_ref == NULL)`.
3. **"Explain how to merge two sorted linked lists." (04 marks)**
   - *Strategy:* Provide the recursive or iterative pseudocode and state the O(N+M) complexity.

### Practice Questions:
1. Trace the deletion of the middle node in the list: `10 -> 20 -> 30 -> 40`.
2. Write an algorithm to split a singly linked list at a specific node value $X$.
