# 3.3 Stack Representation

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 03–06 | **Section:** A  
> **Course Outcome:** Maps to CO1 / CO2

---

## 1. Core Definitions
A **Stack** is a linear data structure that follows the **LIFO (Last In, First Out)** principle. The element that is inserted last is the first one to be removed.
- **Real-world analogy:** A stack of plates in a cafeteria. You can only add or remove a plate from the top.

### Primary Operations:
- **PUSH:** Add an element to the top.
- **POP:** Remove the element from the top.
- **PEEK / TOP:** View the top element without removing it.

---

## 2. Representation 1: Array-Based Stack (Static)

### Concept
A fixed-size array is used to store elements. We maintain an integer variable `TOP` which keeps track of the index of the topmost element.
- **Initialization:** `TOP = -1` (indicates an empty stack).
- **Overflow:** Occurs if we try to PUSH when `TOP == MAX - 1`.
- **Underflow:** Occurs if we try to POP when `TOP == -1`.

### PUSH Algorithm
**C++ Style:**
```cpp
void push(int stack[], int& top, int max, int item) {
    if (top == max - 1) {
        cout << "OVERFLOW - Stack is full" << endl;
        return;
    }
    top = top + 1;
    stack[top] = item;
}
```

**OR, Textbook Style:**
```
Procedure PUSH(STACK, TOP, MAX, ITEM)
    If TOP = MAX - 1 Then
        Print "OVERFLOW — Stack is full"
        Return
    End If
    Set TOP = TOP + 1
    Set STACK[TOP] = ITEM
End Procedure
```

### POP Algorithm
**C++ Style:**
```cpp
int pop(int stack[], int& top) {
    if (top == -1) {
        cout << "UNDERFLOW - Stack is empty" << endl;
        return -1; // Or throw an exception
    }
    int item = stack[top];
    top = top - 1;
    return item;
}
```

**OR, Textbook Style:**
```
Procedure POP(STACK, TOP)
    If TOP = -1 Then
        Print "UNDERFLOW — Stack is empty"
        Return
    End If
    Set ITEM = STACK[TOP]
    Set TOP = TOP - 1
    Return ITEM
End Procedure
```

### Complexities (Array)
- PUSH: O(1)
- POP: O(1)
- PEEK: O(1)

---

## 3. Representation 2: Linked List-Based Stack (Dynamic)

### Concept
A dynamically allocated singly linked list is used. The `head` of the linked list acts as the `TOP` of the stack.
- **PUSH** is implemented as *Insert at Beginning*.
- **POP** is implemented as *Delete from Beginning*.
- **Initialization:** `top = NULL`.
- **Overflow:** Extremely rare, happens only if the system runs out of heap memory (`malloc` returns `NULL`).
- **Underflow:** Occurs if we try to POP when `top == NULL`.

### Node Structure
```
Node:
    DATA                     // the actual value stored
    NEXT (pointer)           // address of the next node
```

### PUSH Algorithm (Linked List)
**C++ Style:**
```cpp
void pushLL(Node*& top, int item) {
    Node* newNode = new Node();
    if (newNode == nullptr) {
        cout << "OVERFLOW (Heap memory full)" << endl;
        return;
    }
    newNode->data = item;
    newNode->next = top;          // Link new node to current top
    top = newNode;                // Update top to new node
}
```

**OR, Textbook Style:**
```
Procedure PUSH_LL(TOP, AVAIL, ITEM)
    If AVAIL = NULL Then
        Print "OVERFLOW (Heap memory full)"
        Return
    End If
    Set NEW = AVAIL
    Set AVAIL = LINK[AVAIL]
    Set INFO[NEW] = ITEM
    Set LINK[NEW] = TOP           // Link new node to current top
    Set TOP = NEW                 // Update top to new node
End Procedure
```

### POP Algorithm (Linked List)
**C++ Style:**
```cpp
int popLL(Node*& top) {
    if (top == nullptr) {
        cout << "UNDERFLOW - Stack is empty" << endl;
        return -1; // Or throw an exception
    }
    Node* temp = top;
    int item = top->data;
    top = top->next;              // Move top to the second node
    delete temp;                  // Free the old top node
    return item;
}
```

**OR, Textbook Style:**
```
Procedure POP_LL(TOP, AVAIL)
    If TOP = NULL Then
        Print "UNDERFLOW — Stack is empty"
        Return
    End If
    Set TEMP = TOP
    Set ITEM = INFO[TOP]
    Set TOP = LINK[TOP]           // Move top to the second node
    Set LINK[TEMP] = AVAIL        // Return freed node to AVAIL
    Set AVAIL = TEMP
    Return ITEM
End Procedure
```

### Complexities (Linked List)
- PUSH: O(1)
- POP: O(1)
- PEEK: O(1)

---

## 4. Array vs Linked Representation Comparison

| Feature | Array Representation | Linked Representation |
|---|---|---|
| **Memory Allocation** | Static (Fixed at compile time) | Dynamic (Grows at runtime) |
| **Overflow Condition** | Frequent (when array is full) | Very rare (only if RAM is full) |
| **Memory Overhead** | None (just the array and 1 integer) | Extra memory for pointers in each node |
| **Data Locality** | High (Cache friendly, contiguous) | Low (Scattered in memory) |

---

## 5. Exam Strategy & Past Questions

### Typical Exam Questions:
1. **"Define PUSH and POP operations with respect to a Stack. Write their algorithms." (04 marks)**
   - *Strategy:* Give the LIFO definition. Choose the Array representation (it's simpler). Clearly write the `if (top == MAX-1)` overflow condition in PUSH and `if (top == -1)` underflow condition in POP.
2. **"Compare array-based and linked-list-based stack representations." (03 marks)**
   - *Strategy:* Draw the table above. Highlight fixed vs dynamic size and the overhead of pointers.
3. **"Explain Stack Overflow and Underflow." (02 marks)**
   - *Strategy:* Overflow = pushing into a full stack (Array `top == MAX-1`). Underflow = popping from an empty stack (`top == -1` or `top == NULL`).

### Practice Questions:
1. Trace the value of `TOP` and the Array contents after: `PUSH 10`, `PUSH 20`, `POP`, `PUSH 30`.
2. Why is PUSH implemented as *Insert at Beginning* rather than *Insert at End* in a linked-list stack? *(Answer: To maintain O(1) time complexity. Inserting at the end requires O(N) traversal).*
