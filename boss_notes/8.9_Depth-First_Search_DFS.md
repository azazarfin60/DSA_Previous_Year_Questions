# 8.9 Depth-First Search (DFS)

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 04–06 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Prerequisite Knowledge
- **Stack Data Structure:** DFS strictly relies on a Stack (LIFO). This can be an explicit Stack data structure, or the implicit Call Stack used in recursion.
- **Visited Array:** To prevent infinite loops, we must keep an array marking nodes as visited.

---

## 2. Core Definition
**Depth-First Search (DFS)** is an algorithm for traversing or searching tree or graph data structures. It starts at a designated node and explores as far as possible along each branch before **backtracking**.
- **Analogy:** Exploring a maze. You keep walking down a path until you hit a dead end, then you turn back (backtrack) to the last intersection and try a new path.

---

## 3. Algorithm (Recursive)

The recursive approach is the most common and easiest to write in an exam.

1. Create a global `Visited` array initialized to `False`.
2. Define `DFS(node)`:
   a. Mark `node` as `Visited = True`.
   b. Print/Process `node`.
   c. For every adjacent neighbor of `node`:
      - If neighbor is not visited, recursively call `DFS(neighbor)`.

---

## 4. Pseudocode (C-Style)

### Recursive DFS
```
Set visited[V] = {0}; // Global or passed via pointer

Procedure DFS(graph[V][V], current)
    // 1. Mark current node as visited and print
    Set visited[current] = 1
    Print current

    // 2. Visit all unvisited neighbors recursively
    For i = 0 to V - 1 do
        If graph[current][i] = 1 AND visited[i] = 0 Then
            DFS(graph, i)

```

### Iterative DFS (Using explicit Stack)
```
Procedure DFS_ITERATIVE(graph[V][V], startNode)
    Set visited[V] = {0}
    Declare stack[MAX]
    Set top = -1

    // Push start node
    Set stack[++top] = startNode

    While top ≠ -1 do
        Set current = stack[top--]; // Pop

        If visited[current] = 0 Then
            Set visited[current] = 1
            Print current

            // Push all unvisited neighbors
            // Note: Pushing in reverse order ensures standard alphabetical traversal
            For i = V - 1, i >= 0, i-- do
                If graph[current][i] = 1 AND visited[i] = 0 Then
                    Set stack[++top] = i

```

## 5. Trace Example

**Graph:**
```text
      A
    / | \
   B  C  D
  / \    |
 E   F   G
```
*Start Node: A*

### Trace Table (Recursive Walkthrough):

| Node Visited | Action | Current Path |
|---|---|---|
| **A** | Mark Visited, Go to first neighbor (B) | `A` |
| **B** | Mark Visited, Go to first neighbor (E) | `A -> B` |
| **E** | Mark Visited, No unvisited neighbors. **BACKTRACK** | `A -> B -> E` |
| **B** | Back at B, Go to next neighbor (F) | `A -> B` |
| **F** | Mark Visited, No unvisited neighbors. **BACKTRACK** | `A -> B -> F` |
| **B** | Back at B, No unvisited neighbors. **BACKTRACK** | `A -> B` |
| **A** | Back at A, Go to next neighbor (C) | `A` |
| **C** | Mark Visited, No unvisited neighbors. **BACKTRACK** | `A -> C` |
| **A** | Back at A, Go to next neighbor (D) | `A` |
| **D** | Mark Visited, Go to first neighbor (G) | `A -> D` |
| **G** | Mark Visited, No unvisited neighbors. **BACKTRACK** | `A -> D -> G` |
| **D** | Back at D. **BACKTRACK** | `A -> D` |
| **A** | Back at A. All neighbors visited. DONE. | `A` |

> **Final DFS Output:** `A B E F C D G`

*(Note: Depending on the order you check neighbors, `A D G C B...` is also valid).*

---

## 6. Time & Space Complexity

### Time Complexity
- **O(V + E)** when using an Adjacency List.
- **O(V²)** when using an Adjacency Matrix.

### Space Complexity
- **O(V)** — In the worst-case (a completely skewed graph like a linked list), the recursive call stack will be $V$ frames deep.

---

## 7. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Write the recursive DFS algorithm." (04 marks)**
   - *Strategy:* Write the 3-line C pseudocode shown in Section 4. Ensure you define `visited` globally.
2. **"Trace DFS on the given graph." (05 marks)**
   - *Strategy:* Use the Trace Table or explicitly draw a tree showing the forward edges and dotted lines for the backtracking steps. State the final string sequence clearly.

### Practice Questions:
1. Can DFS be used to find the shortest path in an unweighted graph? *(Answer: No. DFS explores blindly deep. BFS is guaranteed to find the shortest path in an unweighted graph).*
