# 8.11 BFS for Shortest Path

> **Exam Frequency:** 🟢 Moderate Frequency | **Typical Marks:** 04–06 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Prerequisite Knowledge
- **Breadth-First Search (BFS):** Traverses the graph level by level using a Queue.
- **Unweighted Graphs Only:** This method ONLY works for graphs where every edge has the exact same weight (or no weight at all). If edges have different weights, you MUST use Dijkstra's or Bellman-Ford algorithms.

---

## 2. Core Concept
Because BFS explores the graph radially in uniform steps (level 1 neighbors, then level 2 neighbors), the very first time BFS encounters a target node, it is mathematically guaranteed that it took the minimum number of edges to reach it. 
- Therefore, the **path found by BFS is the shortest path** in an unweighted graph.

---

## 3. Algorithm Modification
To extract the actual shortest path (not just the traversal order), we must modify standard BFS to keep track of **Parents** and **Distances**.

1. Create a `Visited` array initialized to `False`.
2. Create a `Distance` array initialized to `Infinity` ($\infty$).
3. Create a `Parent` array initialized to `-1` (or `NULL`).
4. Set `Distance[startNode] = 0` and push it to the Queue.
5. While Queue is not empty:
   - Dequeue `current`.
   - For every unvisited `neighbor` of `current`:
     - Mark `neighbor` as Visited.
     - Set `Distance[neighbor] = Distance[current] + 1`
     - Set `Parent[neighbor] = current`
     - Enqueue `neighbor`.

---

## 4. Reconstructing the Path
Once BFS finishes, how do we find the path from `Start` to `Target`?
We trace backwards using the `Parent` array.

**Algorithm:**
1. Let `temp = Target`.
2. Push `temp` into a Stack (or an array you reverse later).
3. Loop while `Parent[temp] != -1`:
   - `temp = Parent[temp]`
   - Push `temp` into Stack.
4. Pop and print the Stack to get the path from Start to Target.

---

## 5. Trace Example

**Graph:**
```text
  1 --- 2 --- 3
  |           |
  4 --- 5 --- 6
```
*Goal:* Find shortest path from `1` to `6`.

**Trace Table (BFS Execution):**

| Queue | Dequeue | Unvisited Neighbors | Distance Updates | Parent Updates |
|---|---|---|---|---|
| `[1]` | **1** | 2, 4 | D[2]=1, D[4]=1 | P[2]=1, P[4]=1 |
| `[2, 4]` | **2** | 3 | D[3]=2 | P[3]=2 |
| `[4, 3]` | **4** | 5 | D[5]=2 | P[5]=4 |
| `[3, 5]` | **3** | 6 | D[6]=3 | P[6]=3 |
| `[5, 6]` | **5** | *(6 already visited)* | - | - |
| `[6]` | **6** | None | - | - |

**Reconstructing Path to 6:**
- `P[6] = 3`
- `P[3] = 2`
- `P[2] = 1`
- `P[1] = -1` (Stop)
- **Path Reverse:** 6 $\leftarrow$ 3 $\leftarrow$ 2 $\leftarrow$ 1
- **Actual Path:** `1 -> 2 -> 3 -> 6` (Length: 3 edges)
*(Note: `1 -> 4 -> 5 -> 6` is also a valid shortest path of length 3 depending on neighbor check order).*

---

## 6. Time & Space Complexity
- **Time Complexity:** O(V + E) (Using Adjacency List).
- **Space Complexity:** O(V) (For Queue, Visited, Distance, and Parent arrays).

---

## 7. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Can BFS be used to find the shortest path? Explain how." (05 marks)**
   - *Strategy:* Answer "Yes, but ONLY for unweighted graphs." Explain the level-by-level concept. Write the modified algorithm mentioning the `Distance` and `Parent` arrays.
2. **"Trace BFS shortest path from node A to node E." (05 marks)**
   - *Strategy:* Draw the Distance/Parent update table as shown in Section 5. Show the backward tracing step clearly to secure full marks.

### Practice Questions:
1. Why does DFS fail to find the shortest path in an unweighted graph? *(Answer: DFS goes deep randomly. It might find the target node by taking a massive 10-edge detour before it ever considers the direct 1-edge path).*
