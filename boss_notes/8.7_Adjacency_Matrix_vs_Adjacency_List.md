# 8.7 Adjacency Matrix vs Adjacency List

> **Exam Frequency:** 🟢 Moderate Frequency | **Typical Marks:** 03–05 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Prerequisite Knowledge
- **Graph ($G = (V, E)$):** A non-linear data structure consisting of Vertices ($V$, nodes) and Edges ($E$, lines connecting the nodes).
- To run graph algorithms, a computer needs a mathematical way to represent the graph in memory. The two primary methods are the **Adjacency Matrix** and the **Adjacency List**.

---

## 2. Adjacency Matrix

### Concept
An Adjacency Matrix is a 2D array of size $V \times V$ where $V$ is the number of vertices. 
- If there is an edge from vertex $i$ to vertex $j$, `Matrix[i][j] = 1` (or the weight of the edge).
- If there is no edge, `Matrix[i][j] = 0` (or $\infty$ for weighted graphs).

### Diagram & Representation
**Graph:**
```text
  1 --- 2
  |     |
  3 --- 4
```
*(Assume undirected, unweighted)*

**Matrix Representation:**
```text
    1  2  3  4
 1 [0  1  1  0]
 2 [1  0  0  1]
 3 [1  0  0  1]
 4 [0  1  1  0]
```

### Key Properties
- **Space Complexity:** O(V²) — A $10,000$ vertex graph takes massive memory, even if it has very few edges.
- **Time Complexity (Check Edge):** O(1) — `if(Matrix[i][j] == 1)`.
- **Symmetry:** For an *undirected* graph, the matrix is symmetric across the diagonal ($A[i][j] = A[j][i]$).

---

## 3. Adjacency List

### Concept
An Adjacency List uses an array of Linked Lists (or dynamic arrays). The size of the array is $V$. 
- `Array[i]` points to a linked list containing all the vertices that are directly connected to vertex $i$.

### Diagram & Representation
*(Using the same graph as above)*

**List Representation:**
```text
[1] -> 2 -> 3 -> NULL
[2] -> 1 -> 4 -> NULL
[3] -> 1 -> 4 -> NULL
[4] -> 2 -> 3 -> NULL
```

### Key Properties
- **Space Complexity:** O(V + E) — Only allocates memory for edges that actually exist.
- **Time Complexity (Check Edge):** O(Degree of V) — You must traverse the linked list of vertex $i$ to see if $j$ exists.
- **Time Complexity (Find all neighbors):** O(Degree of V) — Extremely fast compared to Matrix, which takes O(V) to scan the entire row.

---

## 4. Comparison Table (Crucial for Exam)

| Feature | Adjacency Matrix | Adjacency List |
|---|---|---|
| **Underlying Data Structure** | 2D Array | Array of Linked Lists |
| **Space Complexity** | O(V²) | O(V + E) |
| **Best Suited For** | **Dense Graphs** (Many edges, E $\approx$ V²) | **Sparse Graphs** (Few edges, E $\ll$ V²) |
| **Adding a Vertex** | O(V²) (Must resize the entire 2D array) | O(1) (Just add a new linked list) |
| **Checking if Edge (u,v) exists**| O(1) (Direct index lookup) | O(Degree of u) (Must traverse list) |
| **Finding all neighbors of u** | O(V) (Scan entire row) | O(Degree of u) (Just traverse its list) |

**Summary:** 
- Prefer **Adjacency Matrix** when the graph is dense or when quick edge-lookups are critical.
- Prefer **Adjacency List** when the graph is sparse (most real-world graphs) to save massive amounts of memory.

---

## 5. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Compare Adjacency Matrix and Adjacency List representations of a graph." (04 marks)**
   - *Strategy:* Draw the table from Section 4. Ensure you explicitly state the Big-O space complexities (O(V²) vs O(V+E)) and mention the Dense vs Sparse graph use case.
2. **"Draw the Adjacency Matrix and Adjacency List for the given directed graph." (05 marks)**
   - *Strategy:* Pay attention to whether the graph is **Directed** or **Undirected**. If directed $1 \rightarrow 2$, then `Matrix[1][2] = 1`, but `Matrix[2][1] = 0`. Same applies to the linked lists.

### Practice Questions:
1. If a graph has 1000 vertices but only 50 edges, how much memory (in integers) does an Adjacency Matrix waste compared to an Adjacency List? *(Answer: Matrix takes 1,000,000 slots. List takes roughly 1000 + 100 slots. Massive waste).*
