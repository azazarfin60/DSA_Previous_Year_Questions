# 8.17 Dijkstra's Algorithm

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 08–10 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Prerequisite Knowledge
- **Single-Source Shortest Path (SSSP):** Finding the shortest paths from ONE starting node to ALL other nodes in the graph.
- **Weighted Graph:** Dijkstra's algorithm works on weighted graphs.
- **CRITICAL LIMITATION:** Dijkstra **FAILS** if the graph contains **negative edge weights**. (Use Bellman-Ford for negative weights).

---

## 2. Working Principle (Greedy)
Dijkstra's algorithm maintains a set of visited nodes whose shortest distance from the source is permanently finalized. 
It uses a `Distance` array to keep track of the shortest known distance to every node. At each step, it greedily picks the unvisited node with the smallest known distance, finalizes it, and then **"relaxes"** all its neighboring edges.

### Edge Relaxation Formula:
If `Distance[u] + weight(u, v) < Distance[v]`:
   - `Distance[v] = Distance[u] + weight(u, v)`
   - `Parent[v] = u`

---

## 3. Algorithm

1. Initialize `Distance` array to $\infty$ for all nodes.
2. Initialize `Distance[startNode] = 0`.
3. Create a set of unvisited nodes.
4. While unvisited nodes exist:
   a. Pick the unvisited node `u` with the **minimum `Distance` value**.
   b. Mark `u` as visited (its shortest path is now finalized).
   c. For each unvisited neighbor `v` of `u`:
      - **Relax the edge:** Calculate `new_dist = Distance[u] + weight(u, v)`.
      - If `new_dist < Distance[v]`, update `Distance[v] = new_dist`.
5. Repeat until all nodes are visited.

---

## 4. Trace Example (Exam Essential)

**Given Graph:**
```text
       (4)
    A ----- B
    |      /|
 (2)|   (1) |(5)
    |  /    |
    C ----- D
       (8)
```
*Start Node:* **A**

### Trace Table:

| Iteration | Selected Node (Min Dist) | Distance Array `[A, B, C, D]` | Unvisited Set |
|---|---|---|---|
| Init | - | `[0, ∞, ∞, ∞]` | `{A, B, C, D}` |
| 1 | **A** (Dist: 0) | Relax A-B: $0+4 < \infty \rightarrow$ B=4<br>Relax A-C: $0+2 < \infty \rightarrow$ C=2<br>`[0, 4, 2, ∞]` | `{B, C, D}` |
| 2 | **C** (Dist: 2) | Relax C-B: $2+1 < 4 \rightarrow$ B=3 *(Updated!)*<br>Relax C-D: $2+8 < \infty \rightarrow$ D=10<br>`[0, 3, 2, 10]` | `{B, D}` |
| 3 | **B** (Dist: 3) | Relax B-D: $3+5 < 10 \rightarrow$ D=8 *(Updated!)*<br>`[0, 3, 2, 8]` | `{D}` |
| 4 | **D** (Dist: 8) | No unvisited neighbors.<br>`[0, 3, 2, 8]` | `{}` |

**Final Shortest Distances from A:**
- To A: 0
- To B: 3 (Path: A $\rightarrow$ C $\rightarrow$ B)
- To C: 2 (Path: A $\rightarrow$ C)
- To D: 8 (Path: A $\rightarrow$ C $\rightarrow$ B $\rightarrow$ D)

---

## 5. Time Complexity
- **Using Adjacency Matrix:** O(V²)
- **Using Adjacency List + Min-Priority Queue:** O((V + E) log V)
*(Extracting the minimum distance node is O(log V), and we relax E edges).*

---

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Find the shortest path from node X to all other nodes using Dijkstra's algorithm." (08-10 marks)**
   - *Strategy:* This is one of the highest-scoring questions. You MUST use a trace table similar to Section 4. Show the relaxation math (e.g., $2+1 < 4$, update!). Clearly state the final distance array at the end.
2. **"Why does Dijkstra fail with negative edges?" (02 marks)**
   - *Strategy:* Explain that Dijkstra is greedy and permanently finalizes a node once visited, assuming no shorter path can ever be found. A negative edge could later provide a "discount" that makes an already-finalized longer path actually shorter, but Dijkstra won't go back to check.

### Practice Questions:
1. Given a graph with a negative edge weight $-2$, trace Dijkstra to show exactly where it produces the wrong shortest distance.
