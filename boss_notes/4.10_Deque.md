# 4.10 Deque (Double Ended Queue)

> **Exam Frequency:** 🟢 Moderate Frequency | **Typical Marks:** 02–04 | **Section:** A  
> **Course Outcome:** Maps to CO1 / CO2

---

## 1. Core Definition
A **Deque** (short for Double-Ended Queue, pronounced "deck") is a linear data structure in which **insertion and deletion operations can be performed at BOTH ends** (Front and Rear). It does not strictly follow the FIFO rule.

---

## 2. Types of Deque
To maintain some control, Deques are often restricted to two main variations:

### 1. Input-Restricted Deque
- **Insertion** is restricted to only **ONE** end (Rear).
- **Deletion** can be done from **BOTH** ends (Front and Rear).

### 2. Output-Restricted Deque
- **Deletion** is restricted to only **ONE** end (Front).
- **Insertion** can be done at **BOTH** ends (Front and Rear).

---

## 3. Array Representation & Index Calculations
A Deque is usually implemented as a **circular array** to maximize space efficiency.

### Key Operations & Modulo Math:
1. **Insert at Rear:** `rear = (rear + 1) % MAX`
2. **Insert at Front:** `front = (front - 1 + MAX) % MAX` *(Because if front is 0, we need to wrap around to MAX-1)*
3. **Delete from Front:** `front = (front + 1) % MAX`
4. **Delete from Rear:** `rear = (rear - 1 + MAX) % MAX` *(Wrap around to MAX-1 if rear is 0)*

### Important Conditions:
- **Empty Check:** `front == -1`
- **Full Check:** `front == (rear + 1) % MAX`

---

## 4. Algorithms (Input-Restricted Example)

Here is how you handle an Input-Restricted Deque (Insert at Rear, Delete at Both).

### Enqueue at Rear (Standard Circular Insert)
```
Procedure INSERTREAR(value)
    If (rear + 1) % MAX = front Then
        Print "Overflow"
        Return
    If front = -1 Then
        Set front = 0; rear = 0
    Else
        Set rear = (rear + 1) % MAX
    Set deque[rear] = value

```

### Dequeue at Front (Standard Circular Delete)
```
void deleteFront() {
    If front = -1 Then
        Print "Underflow"
        Return
    if (front == rear) { // Last element
        Set front = -1; rear = -1
    Else
        Set front = (front + 1) % MAX

```

### Dequeue at Rear (Special Deque Delete)
```
void deleteRear() {
    If front = -1 Then
        Print "Underflow"
        Return
    if (front == rear) { // Last element
        Set front = -1; rear = -1
    Else
        rear = (rear - 1 + MAX) % MAX; // Circular decrement

```

---

## 5. Time & Space Complexity
- **Time Complexity:** O(1) for all insertion and deletion operations.
- **Space Complexity:** O(N) where N is the MAX capacity of the array.

---

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"What is a Deque? Explain its types." (04 marks)**
   - *Strategy:* Provide the definition. Clearly define Input-Restricted and Output-Restricted variants with bullet points.
2. **"Write the formula to insert at the front of a Deque implemented using a circular array." (02 marks)**
   - *Strategy:* `front = (front - 1 + MAX) % MAX`. Explain that the `+ MAX` ensures the result is never negative in C before applying the modulo.

### Practice Questions:
1. If `MAX = 5`, `front = 0`. What is the new value of `front` if an element is inserted at the front? *(Answer: (0 - 1 + 5) % 5 = 4 % 5 = 4)*.
