# 12.1 Hashing & Collision Resolution

> **Exam Frequency:** 🟡 High Frequency (Supplementary) | **Typical Marks:** 04–06 | **Section:** B  
> **Course Outcome:** Maps to CO3

---

## 1. Core Definition
**Hashing** is a technique used to uniquely identify a specific object from a group of similar objects. It is used to implement a dictionary or a set where operations like Search, Insert, and Delete happen in **O(1) average time complexity**.

- **Hash Function:** A mathematical formula that maps a large data key (like a name or large ID number) to a small integer index.
- **Hash Table:** An array where the data is stored based on the index generated by the Hash Function.

### Example:
Hash Function: `H(k) = k % 10` (where 10 is the table size).
If Key = 45, `H(45) = 45 % 10 = 5`. The value is stored at `Table[5]`.

---

## 2. The Problem of Collisions
A **Collision** occurs when the Hash Function generates the **exact same index for two different keys**.
- *Example:* If we insert 45, it goes to index 5. If we then insert 85, `H(85) = 85 % 10 = 5`. But index 5 is already occupied!

Since hash tables have a finite size, collisions are mathematically inevitable (Pigeonhole Principle). We must use a **Collision Resolution Technique** to handle this.

---

## 3. Resolution 1: Separate Chaining (Linked Lists)

### Concept
Instead of storing the actual data element in the array index, the Hash Table stores **pointers to Linked Lists**. If multiple keys hash to the same index, they are simply appended as new nodes to the linked list at that index.

### Trace Example
Insert: `[15, 25, 12, 22]` using `H(k) = k % 10`

1. Insert 15: `H(15) = 5`. Table[5] $\rightarrow$ 15
2. Insert 25: `H(25) = 5`. Table[5] $\rightarrow$ 15 $\rightarrow$ 25 *(Collision handled by chaining)*
3. Insert 12: `H(12) = 2`. Table[2] $\rightarrow$ 12
4. Insert 22: `H(22) = 2`. Table[2] $\rightarrow$ 12 $\rightarrow$ 22

**Final Hash Table:**
```text
Table[0] -> NULL
Table[1] -> NULL
Table[2] -> 12 -> 22 -> NULL
Table[3] -> NULL
Table[4] -> NULL
Table[5] -> 15 -> 25 -> NULL
...
```

### Pros & Cons
- **Pros:** Hash table never "fills up" (lists can grow infinitely). Easy to implement.
- **Cons:** Cache performance is poor due to linked lists. Extra memory required for pointers.

---

## 4. Resolution 2: Linear Probing (Open Addressing)

### Concept
In Open Addressing, all elements are stored directly inside the hash table array (no linked lists). 
If a collision occurs at index `i`, **Linear Probing** checks the very next index `i+1`, then `i+2`, wrapping around the array until an empty slot is found.

**Probing Formula:** `Index = (H(k) + i) % Size` (where `i` is the attempt number: 0, 1, 2...).

### Trace Example
Insert: `[15, 25, 35]` into a table of size 10 using `H(k) = k % 10`

1. Insert 15: `H(15) = 5`. `Table[5]` is empty. Store 15 at index 5.
2. Insert 25: `H(25) = 5`. `Table[5]` is full! 
   - Probe 1: `(5 + 1) % 10 = 6`. `Table[6]` is empty. Store 25 at index 6.
3. Insert 35: `H(35) = 5`. `Table[5]` is full!
   - Probe 1: `(5 + 1) % 10 = 6`. `Table[6]` is full! (Has 25).
   - Probe 2: `(5 + 2) % 10 = 7`. `Table[7]` is empty. Store 35 at index 7.

**Final Hash Table:**
```text
Table[5] = 15
Table[6] = 25
Table[7] = 35
```

### Primary Clustering Issue
Linear probing suffers from **Primary Clustering**. Contiguous blocks of occupied slots build up, causing long probe sequences and degrading search performance from $O(1)$ to $O(N)$.

## 5. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Define collision in hashing. How is it resolved using Separate Chaining?" (05 marks)**
   - *Strategy:* Define Hash Function and Collision. Draw a small vertical array representing the Hash Table, with arrows pointing to Linked List nodes (like the diagram in Section 3) to physically show how chaining works.
2. **"Insert the following keys into a hash table of size 10 using linear probing: 43, 23, 11, 33" (05 marks)**
   - *Strategy:* Trace step-by-step. 
     - 43 $\rightarrow$ Index 3. 
     - 23 $\rightarrow$ Index 3 (Collision!) $\rightarrow$ Index 4. 
     - 11 $\rightarrow$ Index 1. 
     - 33 $\rightarrow$ Index 3 (Collision!) $\rightarrow$ Index 4 (Collision!) $\rightarrow$ Index 5. 
     Draw the final array clearly showing indices 0 to 9.

### Practice Questions:
1. Why is deletion difficult in Open Addressing (Linear Probing)? *(Answer: If you just delete an element, it leaves a "hole". Future searches might hit this hole and falsely assume the element they are probing for doesn't exist. You must replace deleted items with a special 'Tombstone' marker).*
