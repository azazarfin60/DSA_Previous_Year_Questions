# 10.8 Backtracking Concept & State Space Tree

> **Exam Frequency:** 🟢 Moderate Frequency | **Typical Marks:** 03–05 | **Section:** B  
> **Course Outcome:** Maps to CO4 / CO5

---

## 1. Core Definition
**Backtracking** is an algorithmic technique for solving problems recursively by trying to build a solution incrementally, one piece at a time.
- If at any point the algorithm realizes that the current partial solution cannot possibly lead to a valid complete solution, it **Abandons (Prunes)** that path.
- It then **Backtracks** to the previous step to try a different path.

**Real-World Analogy:** Solving a Sudoku puzzle. You place numbers until you hit a contradiction. Once a contradiction is found, you erase the last number (backtrack) and try the next possible number.

---

## 2. State Space Tree
To visualize how a backtracking algorithm explores all possible choices, we use a **State Space Tree**.

- **Root Node:** The initial state of the problem.
- **Internal Nodes:** Partial solutions representing choices made so far.
- **Branches:** The decisions/choices made.
- **Leaf Nodes:** Complete solutions (either valid or invalid).

### The Pruning Process:
When the algorithm is at a node and checks a **bounding function** (a rule) that determines the current path is invalid, it "prunes" or cuts off that entire branch of the tree. This saves massive amounts of time compared to Brute Force (which explores every single leaf node regardless).

---

## 3. Example Trace: Subset Sum Problem
**Problem:** Given a set of numbers `{1, 2, 3}`, find all subsets that sum exactly to `3`.

**State Space Tree Generation:**
At each step for a number, we make a binary choice: **Include it (1)** or **Exclude it (0)**.

```text
                           Start (Sum=0)
                         /               \
              [Include 1]                 [Exclude 1]
             Sum=1                           Sum=0
            /     \                         /     \
       [Inc 2]   [Exc 2]               [Inc 2]   [Exc 2]
       Sum=3      Sum=1                 Sum=2      Sum=0
       /   \      /   \                 /   \      /   \
  [Inc3] [Exc3] [Inc3][Exc3]       [Inc3] [Exc3] [Inc3][Exc3]
  Sum=6   Sum=3 Sum=4 Sum=1         Sum=5  Sum=2  Sum=3 Sum=0
```

**Pruning:**
Notice that on the far left, after including 1 and 2, `Sum = 3`. If we try to include 3, `Sum = 6`. Since 6 > 3, we immediately know this path is invalid. In a backtracking algorithm, we wouldn't even evaluate deeper nodes on this path; we would prune it.

**Valid Solutions Found:**
- Include 1, Include 2, Exclude 3 (Subset: `{1, 2}`)
- Exclude 1, Exclude 2, Include 3 (Subset: `{3}`)

## 4. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Define Backtracking and State Space Tree." (04 marks)**
   - *Strategy:* Give the definition of backtracking emphasizing "pruning" and "abandoning invalid paths". Draw a small generic tree showing a branch being cut off with an 'X' to visually represent pruning.
2. **"Draw the State Space Tree for finding the subset sum of 5 from the set {2, 3, 4}." (05 marks)**
   - *Strategy:* Draw the binary tree just like Section 3. At every level, add one number from the set. Mark paths that exceed 5 as "Pruned" and paths that exactly equal 5 as "Valid Solution".

### Practice Questions:
1. What is the fundamental difference between Backtracking and Brute Force? *(Answer: Brute force evaluates every possible complete solution. Backtracking evaluates partial solutions and stops early if the path is invalid, saving time).*
