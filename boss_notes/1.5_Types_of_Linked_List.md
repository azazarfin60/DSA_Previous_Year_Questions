# 1.5 Types of Linked List

> **Exam Frequency:** 🟡 High Frequency | **Typical Marks:** 02–05 | **Section:** A  
> **Course Outcome:** Maps to CO1 / CO2

---

## 1. Prerequisite Knowledge
Before studying the types of linked lists, you must understand:
- **Pointers:** Variables that store memory addresses (`int *p`).
- **Dynamic Memory Allocation:** Functions like `malloc()` used to allocate memory during runtime in C.
- **Structures (`struct`):** Used to group different data types (e.g., an integer for data and a pointer for the next node) into a single unit.

---

## 2. Singly Linked List (SLL)

### Definition
A Singly Linked List is a collection of nodes where each node contains two fields: a **data field** to store the value, and a **next pointer field** that stores the memory address of the next node in the sequence.

### Key Properties
- Navigation is **forward only** (unidirectional).
- The `next` pointer of the last node points to `NULL`.
- Requires less memory per node compared to a doubly linked list.

### Node Structure
```
Node:
    DATA                     // the actual value stored
    NEXT (pointer)           // address of the next node
```

### Diagram
```text
  Head
   │
   ▼
 ┌──────┬──────┐    ┌──────┬──────┐    ┌──────┬──────┐
 │ Data │ Next ├───►│ Data │ Next ├───►│ Data │ NULL │
 └──────┴──────┘    └──────┴──────┘    └──────┴──────┘
```

### When to Use
Use SLL when you only need to traverse the list in one direction and want to save memory overhead.

---

## 3. Doubly Linked List (DLL)

### Definition
A Doubly Linked List is a linked list where each node contains **three fields**: a data field, a **prev pointer** pointing to the previous node, and a **next pointer** pointing to the next node.

### Key Properties
- Navigation is **bidirectional** (forward and backward).
- The `prev` pointer of the first node and the `next` pointer of the last node point to `NULL`.
- Deletion is easier and faster if a pointer to the node to be deleted is given (no need to traverse to find the previous node).

### Node Structure
```
Node:
    PREV (pointer)           // address of the previous node
    DATA                     // the actual value stored
    NEXT (pointer)           // address of the next node
```

### Diagram
```text
  Head
   │
   ▼
 ┌──────┬──────┬──────┐    ┌──────┬──────┬──────┐    ┌──────┬──────┬──────┐
 │ NULL │ Data │ Next ├───►│ Prev │ Data │ Next ├───►│ Prev │ Data │ NULL │
 └──────┴──────┴──────┘◄───┴──────┴──────┴──────┘◄───┴──────┴──────┴──────┘
```

### When to Use
Use DLL when frequent backward traversal or bidirectional searching is required, such as implementing browser history (Back/Forward buttons).

---

## 4. Circular Linked List (CLL)

### Definition
A Circular Linked List is a variation of a linked list where the **last node points back to the first node**, forming a continuous circle. It can be implemented as either a Singly Circular or Doubly Circular linked list.

### Key Properties
- **No `NULL` pointers** at the end of the list.
- Any node can be a starting point for traversal.
- To avoid infinite loops during traversal, we must stop when the current pointer reaches the starting node again.

### Diagram (Singly Circular)
```text
  Head
   │
   ▼
 ┌──────┬──────┐    ┌──────┬──────┐    ┌──────┬──────┐
 │ Data │ Next ├───►│ Data │ Next ├───►│ Data │ Next │
 └──────┴──┬───┘    └──────┴──────┘    └──────┴───┬──┘
   ▲       │                                      │
   │       └──────────────────────────────────────┘
   │                                              │
   └──────────────────────────────────────────────┘
```

### When to Use
Use CLL for applications requiring continuous looping, such as CPU scheduling algorithms (Round Robin) or multiplayer board games (passing turns).

---

## 5. Header Linked List

### Definition
A Header Linked List is a linked list that contains a special node at the beginning called the **Header Node**. This node does not represent an actual data item in the sequence; instead, it stores metadata about the list.

### Key Properties
- The header node may store information like the **number of nodes** in the list, maximum/minimum values, or a pointer to the last node.
- It simplifies operations like insertion and deletion at the beginning since the "actual" first node is always the second node in the chain, preventing the Head pointer itself from changing constantly.

### Diagram
```text
  Head
   │
   ▼
 ┌───────┬──────┐    ┌──────┬──────┐    ┌──────┬──────┐
 │ Count │ Next ├───►│ Data │ Next ├───►│ Data │ NULL │
 └───────┴──────┘    └──────┴──────┘    └──────┴──────┘
  (Header Node)        (Node 1)           (Node 2)
```

## 6. Comparison Table (Important for Exam)

| Feature | Singly Linked List | Doubly Linked List | Circular Linked List |
|---|---|---|---|
| **Traversal Direction** | Forward only | Forward and Backward | Forward (can loop continuously) |
| **Memory per Node** | Lower (1 pointer) | Higher (2 pointers) | Lower (1 pointer) |
| **Null Termination** | Last node points to `NULL` | First prev & Last next are `NULL` | No `NULL` pointers |
| **Deletion of Given Node** | O(N) - must find previous node | O(1) - prev pointer available | O(N) |
| **Complexity of finding end**| O(N) | O(N) | O(N) |

**Summary:** 
- Prefer **Singly Linked List** for memory efficiency and simple forward operations.
- Prefer **Doubly Linked List** when fast deletion and bidirectional traversal outweigh the memory cost.
- Prefer **Circular Linked List** when data must be treated as a continuous loop.

---

## 7. Time & Space Complexity

| List Type | Access (i-th index) | Search | Space Complexity |
|---|---|---|---|
| **Singly** | O(N) | O(N) | O(N) |
| **Doubly** | O(N) | O(N) | O(N) |
| **Circular** | O(N) | O(N) | O(N) |

*Note: All linked lists require O(N) for access and search because they do not support random access (unlike arrays).*

## 8. Exam Strategy & Past Questions

### Typical Exam Questions:
1. **"Differentiate between singly and doubly linked list." (04 marks)**
   - *Strategy:* Draw the comparison table with at least 4 points. Add small text diagrams of node structures.
2. **"What is a circular linked list? Mention its applications." (03 marks)**
   - *Strategy:* Provide the definition, draw the diagram showing the last node pointing to the first, and list 2 applications (Round Robin scheduling, multiplayer games).
3. **"Explain the concept of a header linked list." (02 marks)**
   - *Strategy:* Define it, explain that the first node stores metadata (like length) instead of element data, and provide a quick diagram.

### Practice Questions:
1. Write the C `struct` definition for a Doubly Circular Linked List node.
2. Why does a doubly linked list require more memory than a singly linked list? State one advantage that justifies this extra memory.
