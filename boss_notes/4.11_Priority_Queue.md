# 4.11 Priority Queue

> **Exam Frequency:** 🟢 Moderate Frequency | **Typical Marks:** 03–05 | **Section:** A  
> **Course Outcome:** Maps to CO1 / CO2

---

## 1. Core Definition
A **Priority Queue** is a special type of queue in which each element is associated with a **priority value**. 
- Elements are dequeued based on their priority, **not** their arrival time (violates strict FIFO).
- If two elements have the same priority, they are served according to their order in the queue (FIFO applies as a tie-breaker).

### Real-world Analogy:
A hospital emergency room. Patients arrive sequentially, but a patient with a critical injury (high priority) will be treated before someone who arrived earlier with a minor cut.

---

## 2. Types of Priority Queues
1. **Ascending Priority Queue:** The element with the **lowest numerical value** has the highest priority (e.g., 1 is higher priority than 5).
2. **Descending Priority Queue:** The element with the **highest numerical value** has the highest priority (e.g., 100 is higher priority than 10).

---

## 3. Implementations of Priority Queue
There are three main ways to implement it: Array, Linked List, and Heap. The class notes specifically focus on **Using Multiple Queues**.

### Representation using Multiple Queues (Array of Queues)
Instead of shifting elements in a single array to maintain order, we maintain a separate physical queue for each priority level.

- Let there be 3 priority levels: P1 (Highest), P2, P3 (Lowest).
- We create an array of 3 queue structures: `Queue Q[3]`.
- `Q[0]` holds P1 items, `Q[1]` holds P2 items, etc.

#### Insertion (Enqueue)
- If an element has priority $K$, simply perform a normal enqueue operation on `Q[K]`.
- Time Complexity: O(1)

#### Deletion (Dequeue)
- Always check `Q[0]` first. If it is not empty, dequeue from it.
- If `Q[0]` is empty, check `Q[1]`.
- Proceed sequentially.
- Time Complexity: O(P) where P is the number of priority levels.

---

## 4. Alternate Implementation: Linked List
In a Linked List, we can keep the list sorted by priority.

- **Node Structure:**
  ```
  Node:
      DATA                     // the actual value stored
      PRIORITY                 // integer priority level
      NEXT (pointer)           // address of the next node
  ```
- **Insert:** Traverse the list to find the correct spot based on priority value and insert. (O(N) time).
- **Delete:** Simply remove the head node. (O(1) time).

*(Note: The most optimal way to implement a priority queue is using a **Heap** data structure, which provides O(log N) for both insert and delete. This is covered in chapter 5.13).*

---

## 5. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Define Priority Queue. How can it be implemented using multiple queues?" (05 marks)**
   - *Strategy:* Define the priority rule and the FIFO tie-breaker. Explain the "Array of Queues" concept where each index `i` is a separate queue for priority `i`. Explain the O(1) insert and O(P) delete mechanism.
2. **"Differentiate between a simple queue and a priority queue." (03 marks)**
   - *Strategy:* Use a comparison table: Simple Queue processes strictly by arrival time (FIFO); Priority Queue processes by assigned value. Simple Queue is easier to implement; Priority Queue often requires sorting or heaps.

### Practice Questions:
1. Why is a standard Array a poor choice for a single-queue implementation of a Priority Queue? *(Answer: Because inserting an element in the middle to maintain sorted priority order requires shifting O(N) elements).*
