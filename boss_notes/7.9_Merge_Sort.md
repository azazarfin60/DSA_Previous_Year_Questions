# 7.9 Merge Sort

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 05–08 | **Section:** A  
> **Course Outcome:** Maps to CO2

---

## 1. Prerequisite Knowledge
- **Recursion:** A function calling itself.
- **Divide and Conquer Strategy:** 
  1. **Divide:** Break the problem into smaller subproblems.
  2. **Conquer:** Solve the subproblems recursively.
  3. **Combine:** Merge the solutions to get the final answer.

---

## 2. Core Definition
**Merge Sort** is a highly efficient, comparison-based sorting algorithm that strictly follows the Divide and Conquer paradigm. 
- It divides the input array into two halves.
- It recursively calls itself for the two halves until the sub-arrays are of size 1 (which are inherently sorted).
- It then **merges** the two sorted halves back into a single sorted array.

---

## 3. Algorithm (Pseudocode)

Merge Sort requires two functions: `mergeSort` (for dividing) and `merge` (for combining).

### Step 1: The Divide Function (`mergeSort`)
```
Procedure MERGESORT(arr, left, right)
    If left < right Then
        // Find the middle point
        Set mid = (left + right) / 2

        // Recursively sort first and second halves
        mergeSort(arr, left, mid)
        mergeSort(arr, mid + 1, right)

        // Merge the sorted halves
        merge(arr, left, mid, right)

```

### Step 2: The Conquer/Combine Function (`merge`)
```
Procedure MERGE(arr, left, mid, right)
    Declare i, j, k
    Set n1 = mid - left + 1; // Size of Left Subarray
    Set n2 = right - mid;    // Size of Right Subarray

    // Create temporary arrays
    Declare L[n1], R[n2]

    // Copy data to temp arrays L[] and R[]
    Set for (i = 0; i < n1; i++) L[i] = arr[left + i]
    Set for (j = 0; j < n2; j++) R[j] = arr[mid + 1 + j]

    // Merge the temp arrays back into arr[left..right]
    i = 0; // Initial index of first subarray
    j = 0; // Initial index of second subarray
    k = left; // Initial index of merged subarray

    While i < n1 AND j < n2 do
        If L[i] <= R[j] Then
            Set arr[k] = L[i]
            Set i = i + 1
        Else
            Set arr[k] = R[j]
            Set j = j + 1
        Set k = k + 1

    // Copy the remaining elements of L[], if any
    While i < n1 do
        Set arr[k] = L[i]
        Set i = i + 1
        Set k = k + 1

    // Copy the remaining elements of R[], if any
    While j < n2 do
        Set arr[k] = R[j]
        Set j = j + 1
        Set k = k + 1

```

This trace is specifically asked for in exams. You must show the **Divide** phase going down, and the **Merge** phase going up.

**Array:** `[38, 27, 43, 3, 9, 82, 10]`

### Phase 1: Divide (Splitting)
```text
                   [38, 27, 43, 3, 9, 82, 10]
                             /        \
              [38, 27, 43, 3]          [9, 82, 10]
                 /        \               /      \
            [38, 27]     [43, 3]        [9, 82]   [10]
             /   \        /   \          /   \      |
          [38]   [27]  [43]   [3]      [9]   [82]  [10]
```
*(Base case reached: all arrays are size 1 and sorted).*

### Phase 2: Conquer (Merging)
```text
          [38]   [27]  [43]   [3]      [9]   [82]  [10]
             \   /        \   /          \   /      |

                 \        /               \      /
              [3, 27, 38, 43]          [9, 10, 82]
                             \        /
                   [3, 9, 10, 27, 38, 43, 82]
```

## 5. Time & Space Complexity

### Time Complexity
- **Best Case:** O(N log N)
- **Worst Case:** O(N log N)
- **Average Case:** O(N log N)
*Justification:* The array is always divided into two halves, taking exactly $\log_2 N$ levels. At each level, merging takes O(N) operations. Therefore, total time is always exactly $O(N \log N)$.

### Space Complexity
- **Worst Case:** O(N)
*Justification:* The `merge` function requires temporary arrays `L` and `R` which collectively take O(N) extra space.

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Explain the Merge Sort algorithm using the divide and conquer method. Trace it for: 12, 11, 13, 5, 6, 7." (06 marks)**
   - *Strategy:* Give the brief definition of D&C. You do NOT need to write the massive C-code unless specifically asked. Instead, draw the massive **Partition Tree** exactly as shown in Section 4. Ensure you clearly label the "Divide Phase" and the "Merge Phase".
2. **"What is the time and space complexity of merge sort? Why?" (03 marks)**
   - *Strategy:* Time: O(N log N), Space: O(N). Write the short 2-line justification provided in Section 5.

### Practice Questions:
1. Why is Merge Sort not considered an "in-place" sorting algorithm? *(Answer: Because the `merge` step requires O(N) extra memory allocation to temporarily hold the separated arrays before writing them back).*
