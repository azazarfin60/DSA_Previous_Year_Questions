# 6.6 Binary Search

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 04–06 | **Section:** A  
> **Course Outcome:** Maps to CO2

---

## 1. Prerequisite Knowledge
- **Crucial Condition:** Binary Search **ONLY works on SORTED arrays**. If the array is unsorted, it must be sorted first.
- **Divide and Conquer Approach:** The algorithm works by repeatedly dividing the search interval in half.

---

## 2. Working Principle
1. Define two pointers: `low` (start of array) and `high` (end of array).
2. Calculate the `mid` index: `mid = (low + high) / 2`.
3. Compare the `key` with the element at `arr[mid]`:
   - If `key == arr[mid]`, search is successful. Return `mid`.
   - If `key < arr[mid]`, the key must lie in the left half. Update `high = mid - 1`.
   - If `key > arr[mid]`, the key must lie in the right half. Update `low = mid + 1`.
4. Repeat steps 2-3 while `low <= high`.
5. If `low > high`, the element does not exist. Return `-1`.

---

## 3. Algorithms (Iterative and Recursive)

### 3.1 Iterative Binary Search

**C++ Style:**
```cpp
int binarySearchIterative(int arr[], int n, int key) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = (low + high) / 2;
        if (arr[mid] == key)
            return mid;             // Found
        else if (key < arr[mid])
            high = mid - 1;         // Search left half
        else
            low = mid + 1;          // Search right half
    }
    return -1; // Not found
}
```

**OR, Textbook Style:**
```
Declare binarySearchIterative(int arr[], int n, int key) {
    Set low = 0
    Set high = n - 1

    While low <= high do
        Set mid = (low + high) / 2

        If arr[mid] = key Then
            Return mid;             // Found
        else if (key < arr[mid])
            high = mid - 1;         // Search left half
        else
            low = mid + 1;          // Search right half
    Return -1; // Not found

```

### 3.2 Recursive Binary Search

**C++ Style:**
```cpp
int binarySearchRecursive(int arr[], int low, int high, int key) {
    if (low <= high) {
        int mid = (low + high) / 2;
        if (arr[mid] == key)
            return mid;
        else if (key < arr[mid])
            return binarySearchRecursive(arr, low, mid - 1, key);
        else
            return binarySearchRecursive(arr, mid + 1, high, key);
    }
    return -1; // Base case: not found
}
```

**OR, Textbook Style:**
```
Declare binarySearchRecursive(int arr[], int low, int high, int key) {
    If low <= high Then
        Set mid = (low + high) / 2

        If arr[mid] = key Then
            Return mid
        else if (key < arr[mid])
            Return binarySearchRecursive(arr, low, mid - 1, key)
        else
            Return binarySearchRecursive(arr, mid + 1, high, key)
    Return -1; // Base case: not found

```

## 4. Trace Example

**Array:** `[2, 5, 8, 12, 16, 23, 38, 56, 72, 91]` (Size $N=10$, Sorted ✓)
**Search Key:** `23`

| Pass | `low` | `high` | `mid = (low+high)/2` | `arr[mid]` | Comparison (`key=23`) | Action |
|---|---|---|---|---|---|---|
| 1 | 0 | 9 | `(0+9)/2 = 4` | `arr[4] = 16` | `23 > 16` | `low = mid + 1 = 5` |
| 2 | 5 | 9 | `(5+9)/2 = 7` | `arr[7] = 56` | `23 < 56` | `high = mid - 1 = 6` |
| 3 | 5 | 6 | `(5+6)/2 = 5` | `arr[5] = 23` | `23 == 23` | **Match! Return index 5** |

## 5. Time & Space Complexity

### Time Complexity
- **Best Case:** O(1) - The key is exactly at the middle index on the first check.
- **Worst Case:** O(log N) - The key is at the ends of the array or not present. The array size is halved at every step ($N \rightarrow N/2 \rightarrow N/4 \rightarrow \dots \rightarrow 1$). The number of steps is $\log_2 N$.
- **Average Case:** O(log N).

### Space Complexity
- **Iterative:** O(1) - Only `low`, `high`, and `mid` variables are used.
- **Recursive:** O(log N) - Space is consumed on the Call Stack due to recursive function calls.

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Write the iterative algorithm for Binary Search. What is its time complexity?" (05 marks)**
   - *Strategy:* Write the `while (low <= high)` loop. State O(log N). Make sure to mention the prerequisite that the array must be sorted.
2. **"Trace Binary Search for the key 72 in the array: 2, 5, 8, 12, 16, 23, 38, 56, 72, 91" (04 marks)**
   - *Strategy:* Use the tabular trace format shown in Section 4. Do not just write the answer. The professor wants to see the values of `low`, `high`, and `mid` updating iteratively.
3. **"Compare Linear Search and Binary Search." (03 marks)**
   - *Strategy:* Table format. Linear: Unsorted, O(N) time, Sequential approach. Binary: Sorted only, O(log N) time, Divide & Conquer approach.
