# 7.6 Insertion Sort

> **Exam Frequency:** 🟢 Moderate Frequency | **Typical Marks:** 04–05 | **Section:** A  
> **Course Outcome:** Maps to CO2

---

## 1. Core Definition
**Insertion Sort** is a simple sorting algorithm that builds the final sorted array one item at a time. It works exactly like sorting playing cards in your hands.
- The array is virtually split into a **sorted** and an **unsorted** part.
- Values from the unsorted part are picked and placed at the correct position in the sorted part.

---

## 2. Working Principle
1. Assume the first element (index 0) is already sorted.
2. Take the next element (index 1) and store it in a temporary variable `key`.
3. Compare `key` with the elements in the sorted part (moving right to left).
4. If a sorted element is greater than the `key`, shift that sorted element one position to the right.
5. Continue shifting until you find an element smaller than the `key` or reach the beginning of the array.
6. Insert the `key` into this found empty spot.
7. Repeat for all elements in the unsorted part.

---

## 3. Algorithm (C-Style)

```
Procedure INSERTIONSORT(arr, n)
    Declare i, key, j
    For i = 1 to n - 1 do
        key = arr[i]; // The element to be placed
        j = i - 1;    // The last index of the sorted part

        // Move elements of arr[0..i-1] that are greater than key
        // to one position ahead of their current position
        While j >= 0 AND arr[j] > key do
            Set arr[j + 1] = arr[j]
            Set j = j - 1
        // Place the key in its correct position
        Set arr[j + 1] = key

```

---

## 4. Trace Example

**Array:** `[4, 3, 2, 10, 12, 1, 5, 6]` (Trace of first 3 passes)
We are sorting in **Ascending** order.

*Initial State:* `[4 | 3, 2, 10, 12, 1, 5, 6]` (4 is sorted)

### Pass 1 (`i = 1`):
- `key = 3`. Compare with 4. `4 > 3`, so shift 4 right.
- `[4 | 4, ...]`
- Reached start. Insert 3.
- *State:* `[3, 4 | 2, 10, 12, 1, 5, 6]`

### Pass 2 (`i = 2`):
- `key = 2`. Compare with 4. `4 > 2`, shift 4 right.
- Compare with 3. `3 > 2`, shift 3 right.
- Reached start. Insert 2.
- *State:* `[2, 3, 4 | 10, 12, 1, 5, 6]`

### Pass 3 (`i = 3`):
- `key = 10`. Compare with 4. `4 < 10`, NO SHIFT.
- Insert 10.
- *State:* `[2, 3, 4, 10 | 12, 1, 5, 6]`

*(Algorithm continues in this manner until all elements are in the sorted subset).*

---

## 5. Time & Space Complexity

### Time Complexity
- **Best Case:** O(N) - When the array is already sorted. The outer loop runs $N$ times, but the inner `while` loop never executes because `arr[j] > key` is false immediately.
- **Worst Case:** O(N²) - When the array is reverse sorted. Every new element must be compared to all previously sorted elements, causing maximum shifting.
- **Average Case:** O(N²).

### Space Complexity
- **O(1)** - It is an **in-place** sorting algorithm.

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Explain the working principle of Insertion Sort and provide its pseudocode." (05 marks)**
   - *Strategy:* Use the "playing cards" analogy. Write the algorithm clearly, making sure to highlight the `while (j >= 0 && arr[j] > key)` condition.
2. **"Trace insertion sort for the elements: 12, 11, 13, 5, 6" (04 marks)**
   - *Strategy:* Show the state of the array after every pass. Draw a vertical line `|` or use brackets to clearly separate the sorted left half from the unsorted right half.

### Practice Questions:
1. What is the main advantage of Insertion Sort over Bubble Sort? *(Answer: Insertion Sort does fewer swaps (it shifts instead of swapping continuously) and performs exceptionally well on "nearly sorted" arrays, acting almost in O(N) time).*
