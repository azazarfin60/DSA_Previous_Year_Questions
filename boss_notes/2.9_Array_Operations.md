# 2.9 Array Operations

> **Exam Frequency:** 🟡 High Frequency | **Typical Marks:** 03–05 | **Section:** A  
> **Course Outcome:** Maps to CO1

---

## 1. Prerequisite Knowledge
- **Array Definition:** A contiguous block of memory storing elements of the same type.
- **Indices:** 0-indexed in C (first element is `arr[0]`).
- **Size vs Capacity:** Capacity is the maximum memory allocated (e.g., `int arr[100]`), while size ($N$) is the number of elements currently stored.

---

## 2. Traversal
**Concept:** Visiting every element in the array exactly once, usually to print, update, or sum the elements.

### Pseudocode
```
Procedure TRAVERSE(arr, n)
    For i = 0 to n - 1 do
        Print arr[i]
    Print output

```
**Time Complexity:** O(N) | **Space Complexity:** O(1)

---

## 3. Insertion

### Concept
Inserting an element means adding a new value to the array. 
- If inserting at the **end**, it takes O(1) time.
- If inserting at the **beginning or a specific index**, we must **shift** all subsequent elements one position to the right to make space. This takes O(N) time.

### Algorithm: Insert at Position `pos`
1. Check if array is full (`n >= capacity`). If yes, return Overflow.
2. Check if `pos` is valid (`pos < 0` or `pos > n`). If invalid, return Error.
3. Loop `i` from `n-1` down to `pos`:
   a. `arr[i+1] = arr[i]` (Shift elements right)
4. `arr[pos] = new_value`
5. Increment size `n = n + 1`.

### Trace Example: Insert 99 at index 2
- Initial Array: `[10, 20, 30, 40]`, `n=4`
- Shift `arr[3]` to `arr[4]`: `[10, 20, 30, 40, 40]`
- Shift `arr[2]` to `arr[3]`: `[10, 20, 30, 30, 40]`
- Insert 99 at `arr[2]`: `[10, 20, 99, 30, 40]`, `n=5`

### Pseudocode
```
Declare insert(int arr[], int n, int capacity, int value, int pos) {
    If n >= capacity Then
        Print "Overflow"
        Return n
    // Shift elements to the right
    For i = n - 1, i >= pos, i-- do
        Set arr[i + 1] = arr[i]
    Set arr[pos] = value
    Return n + 1; // Return new size

```
**Time Complexity:** O(N) worst-case.

---

## 4. Deletion

### Concept
Removing an element from the array.
- Deleting from the **end** is O(1) (just decrement $N$).
- Deleting from the **beginning or middle** requires **shifting** all subsequent elements one position to the left to fill the gap. This takes O(N) time.

### Algorithm: Delete from Position `pos`
1. Loop `i` from `pos` to `n-2`:
   a. `arr[i] = arr[i+1]` (Shift elements left)
2. Decrement size `n = n - 1`.

### Trace Example: Delete index 1
- Initial Array: `[10, 20, 30, 40]`, `n=4`
- Shift `arr[2]` to `arr[1]`: `[10, 30, 30, 40]`
- Shift `arr[3]` to `arr[2]`: `[10, 30, 40, 40]`
- Decrement `n`: `n=3`. Logical Array is `[10, 30, 40]`.

### Pseudocode
```
Declare delete(int arr[], int n, int pos) {
    If pos < 0 OR pos >= n Then
        Print "Invalid position"
        Return n
    // Shift elements to the left
    For i = pos to n - 1 - 1 do
        Set arr[i] = arr[i + 1]
    Return n - 1; // Return new size

```
**Time Complexity:** O(N) worst-case.

---

## 5. Linear Search

### Concept
Iterate through the array sequentially until the target element is found or the end of the array is reached. Works on both sorted and unsorted arrays.

### Pseudocode
```
Declare linearSearch(int arr[], int n, int key) {
    For i = 0 to n - 1 do
        If arr[i] = key Then
            Return i; // Return index where found
    Return -1; // Not found

```
**Time Complexity:** Best O(1), Worst/Avg O(N).

---

## 6. Merging Arrays

### Concept
Combining two arrays into a single array. If both arrays are sorted, they can be merged into a third sorted array efficiently in O(N+M) time using two pointers.

### Algorithm (Merging Two Sorted Arrays)
1. Initialize pointers `i=0` (for arr1), `j=0` (for arr2), `k=0` (for merged).
2. While `i < size1` AND `j < size2`:
   a. If `arr1[i] < arr2[j]`, `merged[k++] = arr1[i++]`.
   b. Else, `merged[k++] = arr2[j++]`.
3. Copy any remaining elements from `arr1` (if `i < size1`).
4. Copy any remaining elements from `arr2` (if `j < size2`).

**Time Complexity:** O(N + M) | **Space Complexity:** O(N + M) for the new array.

## 7. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Write an algorithm to insert an element into the $k$-th position of an array." (04 marks)**
   - *Strategy:* Write the shifting loop clearly. Explicitly mention `for i = n-1 down to k`. Note that insertion requires shifting elements to the right. State O(N) complexity.
2. **"Explain deletion in a linear array with an example trace." (03 marks)**
   - *Strategy:* Show an array `[A, B, C, D]`. Delete index 1 (B). Show the left-shift of C and D. Give the 3-line pseudocode.

### Practice Questions:
1. Trace the process of merging `[1, 3, 5]` and `[2, 4, 6]`.
2. Why is insertion in an array computationally more expensive than insertion in a Linked List?
