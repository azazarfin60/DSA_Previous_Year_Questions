# 8.15 Kruskal's Algorithm

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 06–10 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Prerequisite Knowledge
- **Minimum Spanning Tree (MST):** Connects all $V$ vertices with $V-1$ edges and minimum total cost, without cycles.
- **Disjoint Set (Union-Find):** A data structure specifically used to detect if adding an edge between two nodes will create a cycle.
- **Unlike Prim's**, which grows a single connected tree from a starting node, Kruskal's algorithm can build a "forest" of disconnected edges that eventually merge into a single tree.

---

## 2. Working Principle
Kruskal's algorithm is a greedy algorithm based purely on edge weights. 
1. It looks at ALL edges in the graph.
2. It sorts them from cheapest to most expensive.
3. It picks the cheapest edge and adds it to the MST, **provided it does not form a cycle**.
4. It stops when exactly $V - 1$ edges have been added.

---

## 3. Algorithm

1. Extract all edges from the graph.
2. **Sort** all edges in non-decreasing order of their weights.
3. Initialize an empty MST.
4. Iterate through the sorted edge list:
   a. Pick the smallest edge `(u, v)`.
   b. Check if adding this edge forms a cycle (using Union-Find).
   c. If NO cycle is formed, include this edge in the MST.
   d. If a cycle is formed, discard the edge.
5. Stop when the MST contains exactly $V - 1$ edges.

---

## 4. Trace Example (Exam Essential)

**Given Graph (V=4):**
```text
      (2)
   A ----- B
   | \     |
(3)|  (1)  |(4)
   |     \ |
   C ----- D
      (5)
```

**Step 1: Sort Edges**
1. A-D (1)
2. A-B (2)
3. A-C (3)
4. B-D (4)
5. C-D (5)

**Step 2: Trace Table (Adding Edges)**

| Edge Picked | Weight | Action | MST Edges | MST Cost | Note |
|---|---|---|---|---|---|
| A-D | 1 | **Accept** | A-D | 1 | No cycle |
| A-B | 2 | **Accept** | A-D, A-B | 3 | No cycle |
| A-C | 3 | **Accept** | A-D, A-B, A-C | 6 | No cycle |
| B-D | 4 | *Reject* | A-D, A-B, A-C | 6 | Forms cycle A-B-D-A |
| C-D | 5 | *Reject* | A-D, A-B, A-C | 6 | Forms cycle A-C-D-A |

**Condition met:** Total vertices $V = 4$. Total edges accepted = 3 ($V-1$). Algorithm terminates.
**Total Minimum Cost:** 6

---

## 5. Time Complexity
- **Sorting Edges:** O(E log E)
- **Cycle Detection (Union-Find):** O(E $\alpha$(V)) where $\alpha$ is the inverse Ackermann function (nearly O(1)).
- **Total Time Complexity:** **O(E log E)** or **O(E log V)**.

---

## 6. Prim's vs Kruskal's Comparison

| Feature | Prim's Algorithm | Kruskal's Algorithm |
|---|---|---|
| **Strategy** | Node-based (grows one tree from a start node) | Edge-based (picks cheapest edges globally) |
| **Intermediate State**| Always a single connected tree | A disconnected forest that eventually merges |
| **Data Structure** | Priority Queue (Min-Heap) | Disjoint Set (Union-Find) |
| **Best Suited For** | **Dense graphs** (lots of edges) | **Sparse graphs** (few edges) |

---

## 7. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Find the Minimum Spanning Tree of the given graph using Kruskal's Algorithm. State the total cost." (08 marks)**
   - *Strategy:* List all edges in sorted order first! This is mandatory. Then draw the trace table showing which edges are Accepted or Rejected (with the reason "Forms Cycle"). Draw the final MST.
2. **"Compare Prim's and Kruskal's algorithms." (04 marks)**
   - *Strategy:* Draw the table in Section 6. The most critical point is "Node-based vs Edge-based" and "Dense vs Sparse graphs".

### Practice Questions:
1. In Kruskal's algorithm, how do we efficiently check if adding an edge `(u, v)` forms a cycle? *(Answer: By using a Disjoint Set (Union-Find) data structure. If `Find(u) == Find(v)`, they are already in the same set, so adding an edge will create a cycle).*
