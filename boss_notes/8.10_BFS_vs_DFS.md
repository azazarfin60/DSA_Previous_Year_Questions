# 8.10 BFS vs DFS

> **Exam Frequency:** 🟡 High Frequency | **Typical Marks:** 03–05 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Core Distinction

The primary difference between Breadth-First Search (BFS) and Depth-First Search (DFS) lies in their traversal strategy and the underlying data structure they use to manage the nodes being explored.

- **BFS** spreads out horizontally, level-by-level, making it ideal for finding the shortest path or exploring neighboring nodes quickly.
- **DFS** dives down vertically, reaching the deepest leaf nodes before backtracking, making it ideal for pathfinding, maze solving, or topological sorting.

---

## 2. Comparison Table (Crucial for Exams)

When asked to compare BFS and DFS, recreate this exact table.

| Feature | BFS (Breadth-First Search) | DFS (Depth-First Search) |
|---|---|---|
| **Data Structure Used** | **Queue** (FIFO) | **Stack** (LIFO) or Recursion |
| **Traversal Approach** | Level-by-level (Horizontally) | Deep down branches (Vertically) |
| **Shortest Path** | Yes, finds the shortest path in unweighted graphs. | No, does not guarantee the shortest path. |
| **Memory/Space Efficiency** | Takes more memory (queue holds entire level). Space is $O(W)$ where $W$ is max width. | Takes less memory (stack holds current path). Space is $O(H)$ where $H$ is max height. |
| **Speed / Goal Finding** | Slower if the target is far away from the root. | Faster if the target is deep in the graph/tree. |
| **Application** | Finding shortest path, web crawlers, GPS navigation systems. | Solving puzzles (mazes, Sudoku), detecting cycles. |
| **Backtracking** | No backtracking involved. | Heavy use of backtracking when hitting a dead end. |

---

## 3. Time Complexity Equality

It is important to note that **both algorithms have the exact same Time Complexity:**
- **O(V + E)** (Using Adjacency List)
- **O(V²)** (Using Adjacency Matrix)

*Why?* Because regardless of the order they visit the nodes, both algorithms must eventually visit every single vertex `V` and inspect every single edge `E` to complete the traversal.

---

## 4. Exam Strategy

### Typical Exam Questions:
1. **"Differentiate between BFS and DFS." (04 marks)**
   - *Strategy:* Draw the table with at least 5 points. You must mention Queue vs Stack, Level vs Depth, and Shortest Path.
2. **"Which traversal would you use to find the shortest path out of a maze? Why?" (02 marks)**
   - *Strategy:* **BFS**. Because BFS explores all paths simultaneously level-by-level, the first time it reaches the exit, it is mathematically guaranteed to be the shortest path (fewest number of steps). DFS might find a path, but it could be a very long, winding one.

*(Note: If the exam asks to compare tree traversals, remember that Preorder, Inorder, and Postorder are all types of DFS. Level Order is BFS).*
