# 8.14 Prim's Algorithm

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 06–10 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Prerequisite Knowledge
- **Minimum Spanning Tree (MST):** Connects all vertices with $V-1$ edges and minimum total cost, without cycles.
- **Greedy Approach:** Prim's algorithm makes the locally optimal choice at each step (picking the cheapest connected edge).

---

## 2. Working Principle
Prim's algorithm builds the MST step-by-step. It starts from an arbitrary node and continuously adds the **cheapest edge** that connects a *visited* node to an *unvisited* node.

**The "Cut" Concept:**
At any step, the graph is divided into two sets:
- Set 1: Visited nodes (already in the MST).
- Set 2: Unvisited nodes.
Prim's looks at all edges crossing the "cut" between Set 1 and Set 2, picks the one with minimum weight, and moves the connected unvisited node into Set 1.

---

## 3. Algorithm

1. Initialize an empty set `MST` to store edges.
2. Keep track of two sets of vertices: `Visited` and `Unvisited`.
3. Pick any arbitrary node as the starting node. Move it from `Unvisited` to `Visited`.
4. While `Unvisited` is not empty (i.e., we haven't selected $V-1$ edges yet):
   a. Look at ALL edges that connect a node in `Visited` to a node in `Unvisited`.
   b. Select the edge with the **minimum weight**.
   c. Add this edge to `MST`.
   d. Move the newly connected node from `Unvisited` to `Visited`.
5. The algorithm finishes when all nodes are in `Visited`.

---

## 4. Trace Example (Exam Essential)

**Given Graph:**
```text
      (2)
   A ----- B
   | \     |
(3)|  (1)  |(4)
   |     \ |
   C ----- D
      (5)
```
*Vertices:* A, B, C, D
*Start Node:* **A**

### Trace Table:

| Step | Visited Set | Unvisited Set | Edges crossing the "Cut" | Min Edge Selected | MST Total Cost |
|---|---|---|---|---|---|
| Init | `{A}` | `{B, C, D}` | A-B(2), A-C(3), A-D(1) | **A-D (1)** | 1 |
| 1 | `{A, D}` | `{B, C}` | A-B(2), A-C(3), D-B(4), D-C(5) | **A-B (2)** | 1+2 = 3 |
| 2 | `{A, D, B}` | `{C}` | A-C(3), B(none to C), D-C(5) | **A-C (3)** | 3+3 = 6 |
| 3 | `{A, D, B, C}`| `{}` | All nodes visited. | - | **6** |

**Final MST Edges:** (A,D), (A,B), (A,C)
**Total Minimum Cost:** 6

*(Note: We did NOT select D-B(4) or D-C(5) because cheaper paths to B and C already existed from A).*

---

## 5. Time Complexity
- **Using Adjacency Matrix:** O(V²)
- **Using Adjacency List + Min-Priority Queue (Heap):** O(E log V)
*(This is because finding the minimum edge takes O(log V) time using a Min-Heap).*

---

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Find the Minimum Spanning Tree of the given graph using Prim's Algorithm. State the total cost." (08 marks)**
   - *Strategy:* DO NOT just draw the final tree. You MUST draw the trace table shown in Section 4. Showing the Visited/Unvisited sets and listing the available edges at each step is where the marks are awarded. Finally, draw the resulting MST and boldly box the total cost.
2. **"Explain the working principle of Prim's algorithm." (04 marks)**
   - *Strategy:* Write the 5-step algorithm in Section 3. Clearly explain the concept of splitting nodes into "Visited" and "Unvisited" sets and picking the minimum crossing edge.

### Practice Questions:
1. If you start Prim's algorithm from node `D` instead of `A` in the example above, will the final MST be different? *(Answer: No. The algorithm will pick D-A(1), then A-B(2), then A-C(3). The final tree and cost are identical).*
