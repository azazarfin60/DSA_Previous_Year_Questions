# 8.18 Bellman-Ford Algorithm

> **Exam Frequency:** 🟡 High Frequency | **Typical Marks:** 06–08 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Prerequisite Knowledge
- **Dijkstra's Limitation:** Dijkstra fails when a graph has **negative edge weights**.
- **Negative Weight Cycle:** A loop in the graph where the sum of the edge weights is negative. If you keep looping through it, your total distance drops to $-\infty$. No shortest path can exist in a graph with a negative cycle.

---

## 2. Core Definition
The **Bellman-Ford Algorithm** is a Single-Source Shortest Path (SSSP) algorithm that correctly handles graphs with **negative edge weights**.
- It does this by using a Dynamic Programming approach rather than a Greedy one.
- **Key Feature:** It can detect if a graph contains a **negative weight cycle** (and will report an error if it does).

---

## 3. Working Principle (Relaxation)
Instead of greedily picking nodes, Bellman-Ford simply takes **all edges** and relaxes them sequentially.
- It repeats this entire edge-relaxation process **$V - 1$ times** (where $V$ is the number of vertices).
- *Why $V-1$ times?* Because the maximum possible length of the shortest path in a graph with $V$ vertices is $V-1$ edges.

### Edge Relaxation Formula:
If `Distance[u] + weight(u, v) < Distance[v]`:
   `Distance[v] = Distance[u] + weight(u, v)`

---

## 4. Algorithm

1. **Initialization:**
   - Initialize `Distance` array to $\infty$ for all vertices.
   - Set `Distance[Start] = 0`.
2. **Relaxation (The Core Loop):**
   - Loop `i` from $1$ to $V - 1$:
     - For *every edge* `(u, v)` with weight `w` in the graph:
       - If `Distance[u] + w < Distance[v]`, update `Distance[v]`.
3. **Negative Cycle Detection:**
   - Loop one more time (the $V$-th time) through all edges.
   - If ANY edge can *still* be relaxed (`Distance[u] + w < Distance[v]`), then the graph contains a **Negative Weight Cycle**.
   - If so, return an error.

---

## 5. Trace Example

**Vertices ($V=3$):** 1, 2, 3
**Edges:** 
- $1 \rightarrow 2$ (weight 4)
- $1 \rightarrow 3$ (weight 5)
- $3 \rightarrow 2$ (weight -2)
*Start Node:* **1**

### Iteration Trace:
*Init Array:* `[0, ∞, ∞]`

**Pass 1 (Relax all edges):**
- Edge $1 \rightarrow 2$ (4): `D[1]+4 < D[2]` $\rightarrow 0+4 < \infty$. Update `D[2] = 4`.
- Edge $1 \rightarrow 3$ (5): `D[1]+5 < D[3]` $\rightarrow 0+5 < \infty$. Update `D[3] = 5`.
- Edge $3 \rightarrow 2$ (-2): `D[3]-2 < D[2]` $\rightarrow 5-2 < 4$. Update `D[2] = 3`.
*Array after Pass 1:* `[0, 3, 5]`

**Pass 2 (Relax all edges):**
*(Since $V=3$, we do $V-1 = 2$ passes)*
- Check all edges again. None can be relaxed further.
*Array after Pass 2:* `[0, 3, 5]`

**Pass 3 (Negative Cycle Check):**
- Check all edges. None can be relaxed. No negative cycle.

**Final Shortest Distances from Node 1:**
- Node 1: 0
- Node 2: 3
- Node 3: 5

---

## 6. Time Complexity
- **Time Complexity:** **O(V × E)**
  *(We loop $V-1$ times, and in each loop, we process all $E$ edges).*
- **Comparison:** Much slower than Dijkstra's O((V+E)logV), which is why Bellman-Ford is only used when negative weights are present.

---

## 7. Exam Strategy & Past Questions

### Typical Exam Questions:
1. **"Write the Bellman-Ford algorithm and state its time complexity." (06 marks)**
   - *Strategy:* Write the 3-step algorithm clearly (Init, Relax $V-1$ times, Check for Negative Cycle). State O(V × E).
2. **"How does Bellman-Ford detect a negative weight cycle?" (02 marks)**
   - *Strategy:* Explain that after $V-1$ relaxations, the shortest paths are mathematically guaranteed to be found. If a $V$-th relaxation pass manages to decrease any distance further, it proves the existence of a negative cycle endlessly pulling the sum down.
3. **"In a graph with negative cycles, what does Bellman-Ford output?" (02 marks)**
   - *Strategy:* It does NOT output a distance array. It must output an error/failure message because shortest paths are undefined in the presence of negative cycles (as highlighted in class notes).
