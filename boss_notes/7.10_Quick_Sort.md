# 7.10 Quick Sort

> **Exam Frequency:** 🟡 High Frequency (Supplementary) | **Typical Marks:** 06–08 | **Section:** A  
> **Course Outcome:** Maps to CO2

---

## 1. Core Definition
**Quick Sort** is a highly efficient, comparison-based sorting algorithm that uses the **Divide and Conquer** strategy. 
Unlike Merge Sort, which does all the heavy lifting during the *merge* step, Quick Sort does the heavy lifting during the *divide* step (called **Partitioning**).

---

## 2. Working Principle
1. **Choose a Pivot:** Select an element from the array to act as the pivot (usually the first, last, or middle element).
2. **Partitioning (The core of Quick Sort):** Rearrange the array so that:
   - All elements *smaller* than the pivot are moved to the left of the pivot.
   - All elements *greater* than the pivot are moved to the right of the pivot.
   - *(The pivot is now in its final sorted position).*
3. **Recursion:** Recursively apply the above steps to the sub-array of elements with smaller values and the sub-array of elements with greater values.

---

## 3. Algorithm (Pseudocode)

### The Main Recursive Function
```
Procedure QUICKSORT(arr, low, high)
    If low < high Then
        // pi is partitioning index, arr[pi] is now at right place
        Set pi = partition(arr, low, high)

        // Separately sort elements before partition and after partition
        quickSort(arr, low, pi - 1)
        quickSort(arr, pi + 1, high)

```

### The Partition Function (Taking Last Element as Pivot)
```
Declare partition(int arr[], int low, int high) {
    Set pivot = arr[high]; // Pivot element
    Set i = (low - 1);     // Index of smaller element

    For j = low to = high - 1 - 1 do
        // If current element is smaller than the pivot
        If arr[j] < pivot Then
            i++; // increment index of smaller element
            // Swap arr[i] and arr[j]
            Set temp = arr[i]
            Set arr[i] = arr[j]
            Set arr[j] = temp
    // Swap arr[i+1] and arr[high] (put pivot in its final place)
    Set temp = arr[i + 1]
    Set arr[i + 1] = arr[high]
    Set arr[high] = temp

    Return (i + 1); // Return pivot's new index

```

## 4. Partition Trace (Exam Essential)

Professors almost always ask to trace the **Partition** step for a given array. Let's trace it using the algorithm above (Last element = Pivot).

**Array:** `[10, 80, 30, 90, 40, 50, 70]`
**`low` = 0, `high` = 6**
**Pivot = `arr[6]` = 70**
**`i` = -1**

| Loop `j` | `arr[j]` | Compare (`arr[j] < 70`) | Action | Array State |
|---|---|---|---|---|
| `j=0` | 10 | 10 < 70 (True) | `i++` (i=0). Swap `arr[0]` & `arr[0]` | `[10, 80, 30, 90, 40, 50, 70]` |
| `j=1` | 80 | 80 < 70 (False) | Do nothing | `[10, 80, 30, 90, 40, 50, 70]` |
| `j=2` | 30 | 30 < 70 (True) | `i++` (i=1). Swap `arr[1]` & `arr[2]` | `[10, 30, 80, 90, 40, 50, 70]` |
| `j=3` | 90 | 90 < 70 (False) | Do nothing | `[10, 30, 80, 90, 40, 50, 70]` |
| `j=4` | 40 | 40 < 70 (True) | `i++` (i=2). Swap `arr[2]` & `arr[4]` | `[10, 30, 40, 90, 80, 50, 70]` |
| `j=5` | 50 | 50 < 70 (True) | `i++` (i=3). Swap `arr[3]` & `arr[5]` | `[10, 30, 40, 50, 80, 90, 70]` |
| *End Loop*| - | - | Swap `arr[i+1]` (arr[4]=80) & `Pivot` (70) | `[10, 30, 40, 50, 70, 90, 80]` |

**Result after 1st Partition:** `[10, 30, 40, 50, 70, 90, 80]`
- Notice that `70` is exactly in its correct sorted position (index 4).
- Everything to the left is `< 70`.
- Everything to the right is `> 70`.

---

## 5. Time & Space Complexity

### Time Complexity
- **Best Case:** O(N log N) — The pivot always divides the array into two exactly equal halves.
- **Average Case:** O(N log N)
- **Worst Case:** **O(N²)** — Occurs when the array is already sorted (or reverse sorted) and the smallest/largest element is always picked as the pivot. This causes one sub-array to have $N-1$ elements and the other 0 elements.

### Space Complexity
- **O(log N)** for the recursive call stack (O(N) in worst case). It is an **in-place** sorting algorithm (unlike Merge Sort).

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Explain the Quick Sort algorithm and trace its partition phase for the array [35, 50, 15, 25, 80, 20, 90, 45]." (08 marks)**
   - *Strategy:* Give the 3-step definition (Choose pivot, Partition, Recurse). Use the exact Trace Table format from Section 4. Clearly define which element you are using as the pivot (last element is standard).
2. **"Why is the worst-case complexity of Quick Sort O(N²)? How can it be avoided?" (03 marks)**
   - *Strategy:* Mention the already-sorted array scenario causing extremely unbalanced partitions ($N-1$ and $0$). It can be avoided by picking a random pivot or using the "Median-of-three" pivot selection method.
