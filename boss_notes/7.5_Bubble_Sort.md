# 7.5 Bubble Sort

> **Exam Frequency:** рџџЎ High Frequency | **Typical Marks:** 04вЂ“06 | **Section:** A  
> **Course Outcome:** Maps to CO2

---

## 1. Core Definition
**Bubble Sort** is one of the simplest sorting algorithms. It works by repeatedly stepping through the list, comparing adjacent elements, and **swapping them if they are in the wrong order**. 
- In each pass, the largest remaining element "bubbles up" to its correct position at the end of the array.

---

## 2. Working Principle
1. Loop through the array from `i = 0` to `n-1`. This outer loop controls the number of passes.
2. For each pass, loop through the unsorted portion of the array from `j = 0` to `n-i-1`.
3. Compare `arr[j]` and `arr[j+1]`.
4. If `arr[j] > arr[j+1]` (for ascending order), swap them.
5. After one complete inner loop, the largest element is placed at the end.
6. Repeat until the array is fully sorted.

---

## 3. Basic Algorithm (C-Style)

```
Procedure BUBBLESORT(arr, n)
    for (int i = 0; i < n - 1; i++) {           // Number of passes
        for (int j = 0; j < n - i - 1; j++) {   // Compare adjacent elements
            If arr[j] > arr[j+1] Then
                // Swap
                Set temp = arr[j]
                Set arr[j] = arr[j+1]
                Set arr[j+1] = temp

```

---

## 4. Optimized Bubble Sort
The basic algorithm always performs O(NВІ) comparisons even if the array becomes sorted early. 
**Optimization:** We introduce a `flag` (or `swapped` boolean). If a full pass occurs without ANY swaps, the array is already sorted, and we can immediately break out of the loop.

### Optimized Algorithm
```
Procedure OPTIMIZEDBUBBLESORT(arr, n)
    For i = 0 to n - 1 - 1 do
        Set swapped = 0; // False

        For j = 0 to n - i - 1 - 1 do
            If arr[j] > arr[j+1] Then
                Set temp = arr[j]
                Set arr[j] = arr[j+1]
                Set arr[j+1] = temp
                swapped = 1; // True (a swap happened)

        // If no two elements were swapped by inner loop, array is sorted
        If swapped = 0 Then
            break

```

---

## 5. Trace Example

**Array:** `[5, 1, 4, 2, 8]`
We are sorting in **Ascending** order.

### Pass 1 (`i = 0`):
- `[5, 1, 4, 2, 8]` $\rightarrow$ Compare 5 & 1 $\rightarrow$ Swap $\rightarrow$ `[1, 5, 4, 2, 8]`
- `[1, 5, 4, 2, 8]` $\rightarrow$ Compare 5 & 4 $\rightarrow$ Swap $\rightarrow$ `[1, 4, 5, 2, 8]`
- `[1, 4, 5, 2, 8]` $\rightarrow$ Compare 5 & 2 $\rightarrow$ Swap $\rightarrow$ `[1, 4, 2, 5, 8]`
- `[1, 4, 2, 5, 8]` $\rightarrow$ Compare 5 & 8 $\rightarrow$ No Swap
*(8 is now in its correct final position)*

### Pass 2 (`i = 1`):
- `[1, 4, 2, 5, 8]` $\rightarrow$ Compare 1 & 4 $\rightarrow$ No Swap
- `[1, 4, 2, 5, 8]` $\rightarrow$ Compare 4 & 2 $\rightarrow$ Swap $\rightarrow$ `[1, 2, 4, 5, 8]`
- `[1, 2, 4, 5, 8]` $\rightarrow$ Compare 4 & 5 $\rightarrow$ No Swap
*(5 is now in its correct final position)*

### Pass 3 (`i = 2`):
- `[1, 2, 4, 5, 8]` $\rightarrow$ Compare 1 & 2 $\rightarrow$ No Swap
- `[1, 2, 4, 5, 8]` $\rightarrow$ Compare 2 & 4 $\rightarrow$ No Swap
*(Optimized version stops here because `swapped == 0` during Pass 3).*

**Final Sorted Array:** `[1, 2, 4, 5, 8]`

---

## 6. Time & Space Complexity

### Time Complexity
- **Best Case:** O(N) - Only occurs when using the **Optimized** version on an already sorted array.
- **Worst Case:** O(NВІ) - When the array is reverse sorted.
- **Average Case:** O(NВІ) - Since it uses nested loops iterating roughly $N \times N/2$ times.

### Space Complexity
- **O(1)** - It is an **in-place** sorting algorithm (requires no extra array).
