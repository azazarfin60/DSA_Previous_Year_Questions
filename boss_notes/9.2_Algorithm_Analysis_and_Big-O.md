# 9.2 Algorithm Analysis and Big-O Notation

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 02–05 | **Section:** A  
> **Course Outcome:** Maps to CO1 / CO5

---

## 1. Core Definition
**Algorithm Analysis** is the process of evaluating the performance of an algorithm, primarily focusing on two factors:
1. **Time Complexity:** How the runtime increases as the size of the input ($N$) increases.
2. **Space Complexity:** How the memory requirement increases as the size of the input ($N$) increases.

Because different computers run at different speeds, we do not measure time in seconds. Instead, we count the **number of basic operations** performed.

---

## 2. Asymptotic Notations
Asymptotic notations are mathematical tools used to describe the limiting behavior of a function when the input tends towards a particular value or infinity.

There are three primary notations:

### 1. Big-O Notation ($O$) - Worst Case (Upper Bound)
- **Definition:** Represents the maximum time an algorithm can take. It guarantees that the function will never grow faster than a certain rate.
- **Math:** $f(n) = O(g(n))$ if there exists positive constants $c$ and $n_0$ such that $0 \le f(n) \le c \cdot g(n)$ for all $n \ge n_0$.
- *Example:* Linear Search takes $O(N)$ because, in the worst case, the key is at the very end of the array.

### 2. Omega Notation ($\Omega$) - Best Case (Lower Bound)
- **Definition:** Represents the minimum time an algorithm will take. It guarantees that the function will take at least this much time.
- **Math:** $f(n) = \Omega(g(n))$ if there exists positive constants $c$ and $n_0$ such that $0 \le c \cdot g(n) \le f(n)$ for all $n \ge n_0$.
- *Example:* Linear Search takes $\Omega(1)$ because, in the best case, the key is at the very first index.

### 3. Theta Notation ($\Theta$) - Average/Exact Case (Tight Bound)
- **Definition:** Represents the exact bound of an algorithm. Used when the best-case and worst-case complexities are identical.
- **Math:** $f(n) = \Theta(g(n))$ if there exists positive constants $c_1, c_2$, and $n_0$ such that $0 \le c_1 \cdot g(n) \le f(n) \le c_2 \cdot g(n)$ for all $n \ge n_0$.
- *Example:* Initializing an array of size $N$ always takes exactly $\Theta(N)$ time.

---

## 3. Order of Growth (Hierarchy)
You must memorize the following complexities from best (fastest) to worst (slowest):

1. **$O(1)$**: Constant Time (e.g., Array index access)
2. **$O(\log N)$**: Logarithmic Time (e.g., Binary Search)
3. **$O(N)$**: Linear Time (e.g., Linear Search, Traversal)
4. **$O(N \log N)$**: Linearithmic Time (e.g., Merge Sort, Quick Sort avg)
5. **$O(N^2)$**: Quadratic Time (e.g., Bubble Sort, 2D Array traversal)
6. **$O(2^N)$**: Exponential Time (e.g., Recursive Fibonacci)
7. **$O(N!)$**: Factorial Time (e.g., Traveling Salesperson brute force)

---

## 4. How to Calculate Big-O
**Rule 1: Drop Constants.**
- $O(5N) \rightarrow O(N)$
- $O(N/2) \rightarrow O(N)$

**Rule 2: Drop Non-Dominant Terms.**
- If $f(N) = N^2 + 5N + 100$, as $N$ becomes huge, $N^2$ dominates the growth.
- $O(N^2 + 5N + 100) \rightarrow O(N^2)$

---

## 5. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Define Big-O, Omega, and Theta notations." (06 marks)**
   - *Strategy:* Provide the English definition (Upper, Lower, Tight bound) and the exact mathematical inequality definitions shown in Section 2.
2. **"Arrange the following time complexities in ascending order of their growth rate: $O(N^2), O(1), O(N!), O(N \log N), O(2^N)$" (03 marks)**
   - *Strategy:* Use the hierarchy list: $O(1) < O(N \log N) < O(N^2) < O(2^N) < O(N!)$.

### Practice Questions:
1. What is the Big-O time complexity of a function with two sequential (non-nested) `for` loops, each running $N$ times? *(Answer: $O(N) + O(N) = O(2N) \rightarrow O(N)$).*
