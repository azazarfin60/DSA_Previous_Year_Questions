# 10.4 Tower of Hanoi

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 05–08 | **Section:** B  
> **Course Outcome:** Maps to CO2

---

## 1. Core Definition
The **Tower of Hanoi** is a classic mathematical puzzle that serves as the quintessential example of **Recursion**. 
- **The Setup:** There are 3 pegs (Source, Auxiliary/Helper, Destination) and $N$ disks of varying sizes stacked on the Source peg in order of size (largest at the bottom).
- **The Goal:** Move the entire stack to the Destination peg.

### The Rules:
1. Only **one disk** can be moved at a time.
2. Every move involves taking the **top disk** from one peg and placing it onto another.
3. A larger disk **cannot** be placed on top of a smaller disk.

---

## 2. Recursive Logic
To move $N$ disks from Source (A) to Destination (C) using Aux (B):
1. **Base Case:** If there is only 1 disk, simply move it from Source to Destination.
2. **Recursive Step:**
   - Move top $N-1$ disks from Source (A) to Aux (B) using Dest (C) as the helper.
   - Move the $N^{th}$ (largest) disk directly from Source (A) to Dest (C).
   - Move the $N-1$ disks from Aux (B) to Dest (C) using Source (A) as the helper.

---

## 3. Algorithm (Pseudocode)

```
// n = number of disks
// src = Source peg ('A')
// aux = Auxiliary peg ('B')
// dest = Destination peg ('C')

Procedure TOWEROFHANOI(n, src, aux, dest)
    If n = 1 Then
        Print "Move disk 1 from to", src, dest
        Return
    // Step 1: Move n-1 disks from Source to Aux
    towerOfHanoi(n - 1, src, dest, aux)

    // Step 2: Move the nth disk from Source to Dest
    Print "Move disk from to", n, src, dest

    // Step 3: Move n-1 disks from Aux to Dest
    towerOfHanoi(n - 1, aux, src, dest)

```

---

## 4. Trace Example (Exam Essential)

**Problem:** Trace the algorithm for $N = 3$ disks. Pegs: A (Source), B (Aux), C (Dest).

*Initial Call:* `towerOfHanoi(3, A, B, C)`

**Step-by-Step Trace (7 steps total):**
1. Move disk 1 from A to C
2. Move disk 2 from A to B
3. Move disk 1 from C to B
*(At this point, disks 1 & 2 are on B. Disk 3 is free).*
4. Move disk 3 from A to C
*(Largest disk is now safely on the destination).*
5. Move disk 1 from B to A
6. Move disk 2 from B to C
7. Move disk 1 from A to C

**Total Moves:** 7

---

## 5. Mathematical Analysis
Let $T(n)$ be the number of moves required for $n$ disks.
Looking at the algorithm, the relation is:
> $T(n) = 2T(n-1) + 1$

Solving this via substitution yields the formula:
> **Total Moves = $2^n - 1$**

### Time and Space Complexity
- **Time Complexity:** **O($2^n$)** — Because the number of moves doubles for every new disk added. This is exponential time.
- **Space Complexity:** **O(n)** — The maximum depth of the recursive call stack is exactly $n$.

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Write a recursive C function for the Tower of Hanoi problem." (04 marks)**
   - *Strategy:* Write the exact code from Section 3. Ensure the base case (`if n==1`) is present to stop infinite recursion. Pay very close attention to the order of the `src, aux, dest` arguments in the two recursive calls.
2. **"Trace the output of Tower of Hanoi for N=3." (05 marks)**
   - *Strategy:* Write out the 7 text steps as shown in Section 4. You do not need to draw physical pictures of the pegs unless specifically asked, but the text "Move disk X from Y to Z" is mandatory.
3. **"What is the mathematical relation for the number of moves required for N disks?" (02 marks)**
   - *Strategy:* State $2^n - 1$. Calculate it for a sample to prove it (e.g., $N=3 \rightarrow 2^3 - 1 = 7$). State the time complexity is $O(2^n)$.
