# 6.4 Linear Search

> **Exam Frequency:** 🟢 Moderate Frequency | **Typical Marks:** 02–04 | **Section:** A  
> **Course Outcome:** Maps to CO1 / CO2

---

## 1. Core Definition
**Linear Search** (or Sequential Search) is the simplest searching algorithm. It works by checking every element in the array one by one from the beginning until the desired element (the **key**) is found, or the end of the array is reached.

---

## 2. Working Principle
1. Start from the first element (index 0).
2. Compare the current element with the `key`.
3. If it matches, return the index.
4. If it does not match, move to the next element.
5. If the end of the array is reached and no match is found, return `-1` (indicating not found).

**Important Note:** Linear Search does **not** require the array to be sorted.

---

## 3. Algorithm & Pseudocode (C)

```
Declare linearSearch(int arr[], int n, int key) {
    For i = 0 to n - 1 do
        If arr[i] = key Then
            Return i; // Element found at index i
    Return -1; // Element not found

```

---

## 4. Trace Example

**Array:** `[15, 5, 20, 35, 2, 42]`
**Search Key:** `35`

| Step | Index (`i`) | Value `arr[i]` | Condition `arr[i] == 35` | Action |
|---|---|---|---|---|
| 1 | 0 | 15 | `15 == 35` (False) | Move to next |
| 2 | 1 | 5 | `5 == 35` (False) | Move to next |
| 3 | 2 | 20 | `20 == 35` (False) | Move to next |
| 4 | 3 | 35 | `35 == 35` (True) | **Match! Return index 3.** |

---

## 5. Time & Space Complexity

### Time Complexity
- **Best Case:** O(1) - The key is found at the very first index (`arr[0]`).
- **Worst Case:** O(N) - The key is at the very last index, or not in the array at all. We must scan all N elements.
- **Average Case:** O(N) - On average, the key is somewhere in the middle, requiring N/2 comparisons. In Big-O, constants are dropped, so O(N).

### Space Complexity
- **O(1)** - Only a few variables (`i`, `n`, `key`) are used regardless of array size.

---

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Write the C function for Linear Search and state its best and worst-case time complexity." (04 marks)**
   - *Strategy:* Write the `for` loop pseudocode. Explicitly state Best Case = O(1) and Worst Case = O(N) with a 1-sentence justification.
2. **"When would you prefer Linear Search over Binary Search?" (02 marks)**
   - *Strategy:* Mention two scenarios: 1) When the array is **unsorted** (Binary Search requires a sorted array). 2) When the dataset is extremely small (the overhead of sorting isn't worth it).

### Practice Questions:
1. Modify the `linearSearch` function to return the total *count* of occurrences of the `key` if it appears multiple times in the array.
