# 5.16 B-Tree

> **Exam Frequency:** 🟢 Moderate Frequency | **Typical Marks:** 04–06 | **Section:** B  
> **Course Outcome:** Maps to CO3

---

## 1. Prerequisite Knowledge
- **Binary Search Tree (BST):** A node can have at most 1 key and 2 children.
- **Disk Storage:** Binary trees are kept in RAM. But databases store millions of records on the hard disk. Reading from a disk is slow. If a BST has a height of 20, it requires 20 disk reads. We need a tree that is much "fatter" and "shorter" to minimize disk access.

---

## 2. Core Definition
A **B-Tree** is a self-balancing, generalized search tree (also called an **m-way search tree**) where a single node can contain *multiple keys* and have *multiple children*. It is optimized for systems that read and write large blocks of data (like databases and file systems).

### Order of a B-Tree
The **Order (m)** of a B-Tree dictates the maximum number of children a node can have.

---

## 3. Strict Properties of a B-Tree of Order 'm'

You **must** memorize these properties for the exam:

1. **Maximum Children:** Every node has at most `m` children.
2. **Maximum Keys:** Every node has at most `m - 1` keys.
3. **Minimum Children:** Every internal node (except root) has at least $\lceil m/2 \rceil$ children.
4. **Minimum Keys:** Every node (except root) has at least $\lceil m/2 \rceil - 1$ keys.
5. **Root Constraints:** The root has at least 2 children if it is not a leaf node.
6. **Leaf Level:** ALL leaf nodes appear at the **exact same level** (perfectly balanced from bottom up).
7. **Sorted Keys:** Within a single node, the keys are sorted in ascending order.

*Example: B-Tree of Order 4:*
- Max children = 4
- Max keys per node = 3
- Min children (non-root) = 2
- Min keys (non-root) = 1

---

## 4. Insertion Principle (The Split)

Unlike a BST which grows from the top-down (adding leaves at the bottom), **a B-Tree grows from the bottom-up**.

1. Always insert the new key into the appropriate **leaf node**.
2. **Overflow Condition:** If a node exceeds `m - 1` keys after insertion, it must be **split**.
3. **How to Split:**
   - Sort the keys in the overflowing node.
   - Find the **median (middle)** key.
   - Push the median key **UP** to the parent node.
   - Split the remaining keys into two new nodes: one containing keys less than the median, and one containing keys greater than the median.
4. If the parent overflows due to the promoted median, the parent is also split. This can propagate all the way to the root. If the root splits, a new root is created, and the tree height increases by 1.

---

## 5. Trace Example: Insert into a B-Tree of Order 3

Order $m = 3$. 
- Max keys = 2. Max children = 3. 
- Splitting happens when a node gets 3 keys.

**Step 1: Insert 10, 20**
```text
  [ 10 | 20 ]
```

**Step 2: Insert 30**
Node becomes `[ 10 | 20 | 30 ]` -> OVERFLOW! (Max keys is 2)
- Median is 20. Push 20 up.
- Left child gets `[10]`, Right child gets `[30]`.
```text
       [ 20 ]
      /      \
  [ 10 ]    [ 30 ]
```

**Step 3: Insert 40, 50**
- 40 goes right of 20: `[ 30 | 40 ]`. No overflow.
- 50 goes right of 20: `[ 30 | 40 | 50 ]` -> OVERFLOW!
- Median is 40. Push 40 up to parent node `[20]`.
- Parent becomes `[ 20 | 40 ]`.
- Left of 40 is `[30]`, Right of 40 is `[50]`.
```text
       [ 20 | 40 ]
      /     |     \
  [ 10 ] [ 30 ] [ 50 ]
```

*(Notice how all leaf nodes remained at the same level!)*

---

## 6. Time Complexity
| Operation | Time Complexity |
|---|---|
| Search | O(log N) |
| Insert | O(log N) |
| Delete | O(log N) |

*(Note: The base of the logarithm is the branching factor `m`, which makes it extremely fast for large datasets compared to base-2 binary trees).*

## 7. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"State the properties of a B-Tree of order m." (04 marks)**
   - *Strategy:* List the 7 properties mentioned in Section 3 exactly as written. This is a purely theoretical question that frequently appears.
2. **"Construct a B-Tree of order 3 with the following data: 10, 20, 30, 40, 50." (05 marks)**
   - *Strategy:* Show the tree trace exactly like Section 5. Show the node right before it splits, then draw an arrow indicating "Split at median", and draw the new state. Mention that B-trees grow bottom-up.
3. **"Why is B-Tree preferred over AVL tree for database indexing?" (02 marks)**
   - *Strategy:* Explain the disk access concept. AVL trees are deep and narrow, causing many expensive disk reads. B-Trees are shallow and wide (store many keys per node), pulling massive chunks of data in a single disk read.

### Practice Questions:
1. Construct a B-Tree of order 4 inserting: 5, 3, 21, 9, 1, 13, 2, 7, 10. Show step-by-step splits.
