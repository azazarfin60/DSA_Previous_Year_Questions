# 3.8 Infix, Prefix, and Postfix Expressions

> **Exam Frequency:** рЯФі Very High Frequency | **Typical Marks:** 04вАУ08 | **Section:** A/B  
> **Course Outcome:** Maps to CO2

---

## 1. Core Definitions
Mathematical expressions consist of **operands** (variables like A, B, numbers like 5, 10) and **operators** (+, -, *, /, ^).

There are three ways to write these expressions based on the position of the operator:

1. **Infix Notation:** Operator is *between* the operands.
   - Format: `Operand1 Operator Operand2`
   - Example: `A + B`
   - *Note: This is how humans write math, but it requires rules of precedence and brackets for computers to evaluate.*

2. **Prefix Notation (Polish Notation):** Operator is *before* the operands.
   - Format: `Operator Operand1 Operand2`
   - Example: `+ A B`

3. **Postfix Notation (Reverse Polish Notation):** Operator is *after* the operands.
   - Format: `Operand1 Operand2 Operator`
   - Example: `A B +`
   - *Note: This is the most efficient format for computers to evaluate using a Stack, as it requires NO brackets and NO precedence rules.*

---

## 2. Operator Precedence and Associativity
Before converting expressions, you must memorize these rules:

| Operator | Symbol | Precedence Level | Associativity (Tie-breaker) |
|---|---|---|---|
| **Exponentiation** | `^` or `$` or `вЖС` | 1 (Highest) | Right-to-Left |
| **Multiplication / Division** | `*` , `/` | 2 | Left-to-Right |
| **Addition / Subtraction** | `+` , `-` | 3 (Lowest) | Left-to-Right |

*Important:* Because exponentiation is Right-to-Left, `A^B^C` is evaluated as `A^(B^C)`. If you see a `^` on the stack and another `^` is scanned, **do not pop** the stack!

---

## 3. Infix to Postfix Conversion (Crucial Exam Topic)

This is one of the most frequently asked questions in the exam. You **must** trace this using a table.

### Algorithm (Using a Stack)
1. Add a closing parenthesis `)` to the end of the Infix expression.
2. Push an opening parenthesis `(` onto the Stack.
3. **Scan** the Infix expression from **left to right**.
4. For each scanned symbol:
   - **If Operand (A, B, 1, 2):** Add it directly to the Output string.
   - **If `(`:** Push it onto the Stack.
   - **If Operator (`+`, `-`, etc.):**
     - Repeatedly **POP** operators from the Stack and add to Output **IF** the popped operator has **greater than or equal precedence** to the scanned operator. *(Exception: Do not pop `^` if the scanned operator is also `^`).*
     - **PUSH** the scanned operator onto the Stack.
   - **If `)`:**
     - Repeatedly POP from the Stack and add to Output until a `(` is encountered.
     - POP and discard the `(`.
5. Repeat until the Stack is empty.

### Exam Trace Example: `A+(B*C-(D/E^F)*G)*H` (Appeared 2021 & 2024)

|---|---|---|---|

## 4. Evaluation of Postfix Expression

Once an expression is in Postfix, a computer can evaluate it in O(N) time using a stack.

### Algorithm
1. Create an empty Stack.
2. **Scan** the Postfix expression from **left to right**.
3. For each symbol:
   - **If Operand (Number):** PUSH it onto the Stack.
   - **If Operator (`+`, `-`, etc.):**
     - POP the top two operands from the Stack.
     - Let the first popped value be **A** (right operand).
     - Let the second popped value be **B** (left operand).
     - Calculate **Result = B (Operator) A**.
     - PUSH the **Result** back onto the Stack.
4. When the expression ends, the final answer is the only value left in the Stack.

### Exam Trace Example: Evaluate `2 3 4 * + 5 -`

| Scanned | Stack | Action |
|---|---|---|
| 2 | `2` | Push 2 |
| 3 | `2, 3` | Push 3 |
| 4 | `2, 3, 4` | Push 4 |
| * | `2, 12` | Pop 4(A), Pop 3(B). Calc 3*4=12. Push 12. |
| + | `14` | Pop 12(A), Pop 2(B). Calc 2+12=14. Push 14. |
| 5 | `14, 5` | Push 5 |
| - | `9` | Pop 5(A), Pop 14(B). Calc 14-5=9. Push 9. |

> **Final Result:** 9

## 5. Infix to Prefix Conversion

### Algorithm
1. **Reverse** the Infix string.
2. Replace all `(` with `)` and all `)` with `(`.
3. Apply the **Infix to Postfix** algorithm to this modified string.
4. **Reverse** the resulting Postfix string to get the final Prefix expression.

### Class Trace Example: Convert `A * B + (C - D)` to Prefix
- **Step 1:** Reverse string $\rightarrow$ `) D - C ( + B * A`
- **Step 2:** Swap brackets $\rightarrow$ `( D - C ) + B * A`
- **Step 3:** Apply Infix $\rightarrow$ Postfix on `( D - C ) + B * A`:
  - Result of postfix conversion: `D C - B A * +`
- **Step 4:** Reverse the result $\rightarrow$ `+ * A B - C D`

> **Final Prefix:** `+ * A B - C D`

## 6. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Convert the following infix expression into postfix using stack: `X + Y * Z - W / U`" (05 marks)**
   - *Strategy:* Draw the 4-column table (Scanned Symbol, Stack, Output, Action). Show every single step explicitly. Do not skip steps to save space!
2. **"Evaluate the postfix expression: `5 6 2 + * 12 4 / -`" (04 marks)**
   - *Strategy:* Draw a 3-column table (Scanned, Stack, Action). Remember: The first popped element is the *right* side of the operator (Operand 2). For division `12 4 /`, you pop 4, then 12. You calculate `12 / 4 = 3`, NOT `4 / 12`.
3. **"Why is postfix notation preferred over infix in compilers?" (02 marks)**
   - *Strategy:* Explain that postfix doesn't require parentheses to dictate precedence, and it can be evaluated linearly in a single left-to-right pass O(N) using a simple Stack.

### Practice Questions:
1. Convert `(A + B) * C - (D - E) ^ F` to Postfix.
2. Evaluate Postfix: `10 2 8 * + 3 -`
