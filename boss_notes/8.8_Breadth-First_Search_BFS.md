# 8.8 Breadth-First Search (BFS)

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 04–06 | **Section:** B  
> **Course Outcome:** Maps to CO4

---

## 1. Prerequisite Knowledge
- **Queue Data Structure:** BFS strictly relies on a Queue (FIFO) to keep track of nodes to visit next.
- **Visited Array:** To prevent infinite loops (since graphs can have cycles), we must keep an array marking nodes as visited.

---

## 2. Core Definition
**Breadth-First Search (BFS)** is an algorithm for traversing or searching tree or graph data structures. It starts at the tree root (or some arbitrary node of a graph, sometimes referred to as a 'search key') and explores all of the neighbor nodes at the present depth prior to moving on to the nodes at the next depth level.
- **Analogy:** Dropping a stone in a pond. The ripples spread outward in concentric circles.

---

## 3. Algorithm

1. Create a `Queue`.
2. Create a `Visited` array/boolean map and initialize all nodes as `False`.
3. Push the **Starting Node** into the Queue and mark it as `Visited = True`.
4. While the Queue is not empty:
   a. `Dequeue` a node from the front. Print/Process it.
   b. For every adjacent neighbor of this dequeued node:
      - If the neighbor is `False` in the Visited array:
         - Mark it as `Visited = True`.
         - `Enqueue` it into the Queue.

---

## 4. Pseudocode (C-Style)

```
Procedure BFS(graph[V][V], startNode)
    Set visited[V] = {0}; // 0 means unvisited
    Declare queue[MAX]
    Set front = 0, rear = 0

    // Start node
    Set visited[startNode] = 1
    queue[rear++] = startNode; // Enqueue

    Print "BFS Traversal:"

    while (front < rear) { // While queue is not empty
        Set current = queue[front++]; // Dequeue
        Print current

        // Check all adjacent nodes
        For i = 0 to V - 1 do
            If graph[current][i] = 1 AND visited[i] = 0 Then
                Set visited[i] = 1
                queue[rear++] = i; // Enqueue neighbor

```

## 5. Trace Example

**Graph:**
```text
      A
    / | \
   B  C  D
  / \    |
 E   F   G
```
*Start Node: A*

### Trace Table (Crucial for Exams):

| Step | Queue (Front $\rightarrow$ Rear) | Visited Array (`True` elements) | Dequeued & Output | Neighbors to Queue |
|---|---|---|---|---|
| Init | `[A]` | A | - | - |
| 1 | `[]` | A, B, C, D | **A** | B, C, D |
| 2 | `[C, D]` | A, B, C, D, E, F | **B** | E, F |
| 3 | `[D, E, F]` | A, B, C, D, E, F | **C** | None |
| 4 | `[E, F]` | A, B, C, D, E, F, G | **D** | G |
| 5 | `[F, G]` | A, B, C, D, E, F, G | **E** | None |
| 6 | `[G]` | A, B, C, D, E, F, G | **F** | None |
| 7 | `[]` | A, B, C, D, E, F, G | **G** | None |

> **Final BFS Output:** `A B C D E F G`

*(Note: Depending on the order you check neighbors, `A C B D...` is also valid. Alphabetical order is standard).*

## 6. Time & Space Complexity

### Time Complexity
- **O(V + E)** when using an Adjacency List.
- **O(V²)** when using an Adjacency Matrix (because scanning the row for neighbors takes O(V)).
*Justification:* Every vertex is enqueued exactly once, and every edge is examined exactly twice (in an undirected graph).

### Space Complexity
- **O(V)** — Worst case, the Queue will hold all nodes in the widest level of the graph.

### Typical Exam Questions:
1. **"Write the algorithm for Breadth-First Search (BFS)." (04 marks)**
   - *Strategy:* Write the 4-step algorithm clearly. Mention the Queue and the Visited array explicitly.

   - *Strategy:* Always use the Trace Table format. It clearly demonstrates you understand how the Queue drives the traversal level-by-level.
