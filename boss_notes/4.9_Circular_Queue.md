# 4.9 Circular Queue

> **Exam Frequency:** 🔴 Very High Frequency | **Typical Marks:** 04–06 | **Section:** A  
> **Course Outcome:** Maps to CO1 / CO2

---

## 1. Prerequisite Knowledge
- A **Linear Queue** suffers from a major drawback: once `rear` reaches `MAX-1`, we cannot enqueue new elements even if there is empty space at the front (due to previous dequeues).
- To solve this, we use the **Modulo arithmetic (`%`)** operator to wrap the `rear` and `front` pointers back to 0.

---

## 2. Core Definitions
A **Circular Queue** is a linear data structure where the operations are performed based on FIFO, but the last position is connected back to the first position to make a circle. It is also called a **Ring Buffer**.

### The Golden Formulas:
- Instead of `rear = rear + 1`, we use: **`rear = (rear + 1) % MAX`**
- Instead of `front = front + 1`, we use: **`front = (front + 1) % MAX`**

### Crucial Conditions:
1. **Empty Condition:** `front == -1`
2. **Full Condition:** `(rear + 1) % MAX == front` 
   *(This means the next position of `rear` overlaps with `front`, indicating no empty slots left).*

---

## 3. Enqueue Operation

### Algorithm
1. Check if the queue is full: `If (rear + 1) % MAX == front`, return Overflow.
2. Check if the queue is empty: `If front == -1`, set `front = 0` and `rear = 0`.
3. Else, update rear circularly: `rear = (rear + 1) % MAX`.
4. Insert element: `queue[rear] = value`.

### Pseudocode
```
Declare queue[MAX]
Set front = -1
Set rear = -1

Procedure ENQUEUE(value)
    If (rear + 1) % MAX = front Then
        Print "Circular Queue Overflow!"
        Return
    if (front == -1) { // First element insertion
        Set front = 0
        Set rear = 0
    Else
        rear = (rear + 1) % MAX; // Circular increment
    Set queue[rear] = value

```

---

## 4. Dequeue Operation

### Algorithm
1. Check if the queue is empty: `If front == -1`, return Underflow.
2. Store the value at `queue[front]`.
3. Check if it was the last element: `If front == rear`, set `front = -1` and `rear = -1` (Queue is now empty).
4. Else, update front circularly: `front = (front + 1) % MAX`.
5. Return the value.

### Pseudocode
```
Declare dequeue() {
    If front = -1 Then
        Print "Circular Queue Underflow!"
        Return -1
    Set value = queue[front]

    if (front == rear) { // Only one element was present
        Set front = -1
        Set rear = -1
    Else
        front = (front + 1) % MAX; // Circular increment
    Return value

```

---

## 5. Execution Trace Example
**Assume MAX = 3**

| Action | `front` | `rear` | Queue Array `[0,1,2]` | Note |
|---|---|---|---|---|
| Init | -1 | -1 | `[ , , ]` | Empty |
| Enqueue 10 | 0 | 0 | `[10, , ]` | First insertion |
| Enqueue 20 | 0 | 1 | `[10, 20, ]` | |
| Enqueue 30 | 0 | 2 | `[10, 20, 30]` | Queue is full |
| Enqueue 40 | 0 | 2 | `[10, 20, 30]` | OVERFLOW! `(2+1)%3 == 0 (front)` |
| Dequeue | 1 | 2 | `[ , 20, 30]` | 10 removed. Space freed at index 0. |
| Dequeue | 2 | 2 | `[ , , 30]` | 20 removed. |
| Enqueue 50 | 2 | 0 | `[50, , 30]` | **WRAPS AROUND!** `rear = (2+1)%3 = 0` |

---

## 6. Time & Space Complexity
- **Time Complexity:** O(1) for both Enqueue and Dequeue.
- **Space Complexity:** O(N) where N is the maximum capacity of the array.

---

## 7. Exam Strategy & Practice Questions

### Typical Exam Questions:
1. **"Write the pseudocode for insert and delete operations in a Circular Queue." (06 marks)**
   - *Strategy:* Provide the codes exactly as written above. The examiner specifically looks for the `(rear + 1) % MAX` modulo logic. If you just write `+ 1`, you will lose marks.
2. **"How does a Circular Queue solve the problem of a Linear Queue?" (03 marks)**
   - *Strategy:* Give a brief trace (like the one in section 5) showing that when `rear` reaches the end of the array, the modulo operator wraps it back to index 0 to utilize space freed by previous dequeues.

### Practice Questions:
1. For `MAX=5`, `front=3`, `rear=4`. Is the queue full? Prove mathematically. *(Answer: `(4+1)%5 = 0`. But `front=3`. So $0 \neq 3$. Queue is NOT full).*
