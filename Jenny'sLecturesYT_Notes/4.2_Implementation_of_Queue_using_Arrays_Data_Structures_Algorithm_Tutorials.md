## 📌 Executive Summary
Implementing a Queue using static arrays requires managing two pointer variables: `front` and `rear`. 
- `rear` tracks where to insert new data (`enqueue`).
- `front` tracks from where to remove data (`dequeue`).
- Globally initializing both variables to `-1` perfectly represents an empty state.

---

## 🛠️ Global Setup

We declare the array and the pointers globally so they can be accessed directly by all our fundamental functions without needing to pass them as pointers.

```c
#include <stdio.h>
#define N 5 // Maximum capacity of the queue

int queue[N];
int front = -1;
int rear = -1;
```

---

## 1️⃣ Enqueue Operation (Insertion)

The `enqueue` operation places a new element at the `rear`. It must handle three specific states:
1. **Overflow:** Queue is full.
2. **First Element:** Queue is completely empty.
3. **Normal Insertion:** Queue has space.

**Time Complexity:** $O(1)$

### C Code Snippet
```c
void enqueue(int x) {
    // 1. Check for Overflow
    if (rear == N - 1) {
        printf("Overflow: Queue is full\n");
    } 
    // 2. Check if this is the first element being inserted
    else if (front == -1 && rear == -1) {
        front = rear = 0; // Both point to the 0th index
        queue[rear] = x;
    } 
    // 3. Normal insertion
    else {
        rear++;           // Shift rear forward
        queue[rear] = x;  // Insert at new rear
    }
}
```

---

## 2️⃣ Dequeue Operation (Deletion)

The `dequeue` operation removes the element currently at the `front`. 
*Note: We don't actually erase the data in memory; we simply shift the `front` pointer forward, making the old data "garbage" outside the logical bounds of the queue.*

It must handle three states:
1. **Underflow:** Queue is already empty.
2. **Last Element:** Queue has exactly one element. Deleting it should reset the queue to the empty `-1` state.
3. **Normal Deletion:** Queue has multiple elements.

**Time Complexity:** $O(1)$

### C Code Snippet
```c
void dequeue() {
    // 1. Check for Underflow
    if (front == -1 && rear == -1) {
        printf("Underflow: Queue is empty\n");
    } 
    // 2. Check if only ONE element is left
    else if (front == rear) {
        printf("Dequeued: %d\n", queue[front]);
        front = rear = -1; // Reset to empty state
    } 
    // 3. Normal deletion
    else {
        printf("Dequeued: %d\n", queue[front]);
        front++; // Shift front forward, abandoning the old data
    }
}
```

---

## 3️⃣ Peek and Display Operations

### Peek ($O(1)$)
Returns the element at the front without modifying the queue.
```c
void peek() {
    if (front == -1 && rear == -1) {
        printf("Queue is empty\n");
    } else {
        printf("Front data: %d\n", queue[front]);
    }
}
```

### Display ($O(n)$)
Traverses the logical bounds of the queue from `front` to `rear`.
```c
void display() {
    if (front == -1 && rear == -1) {
        printf("Queue is empty\n");
    } else {
        printf("Queue elements: ");
        // Loop specifically from front to rear
        for (int i = front; i < rear + 1; i++) {
            printf("%d ", queue[i]);
        }
        printf("\n");
    }
}
```

## ⚠️ The "Wasted Space" Drawback

If you perform the following sequence on a queue of size 5:
1. Enqueue 5 items (`rear` becomes 4).
2. Dequeue 3 items (`front` becomes 3).

Indices 0, 1, and 2 are now technically empty. However, if you try to enqueue a new item, the code evaluates `if (rear == N - 1)` (which is `4 == 4`), and it prints **Overflow**, denying the insertion.

> [!TIP]
> **Exam Connection:**
> This exact limitation of the standard linear array implementation is what necessitates the invention of the **Circular Queue**, where we use the modulo operator `(rear + 1) % N` to wrap the rear pointer back to the beginning of the array.
