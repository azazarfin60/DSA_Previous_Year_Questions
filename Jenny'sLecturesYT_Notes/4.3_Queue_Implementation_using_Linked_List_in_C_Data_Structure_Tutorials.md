# 4.3 Queue Implementation using Linked List | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
Implementing a Queue using a Linked List solves the static size limitations of arrays. The queue can grow dynamically without needing to copy elements into a new, larger array (which would cost $O(n)$ time).

To satisfy the Queue ADT requirement that `enqueue` and `dequeue` operate in **$O(1)$ constant time**, we **must maintain two pointers**:
1. `front`: Points to the first node (for $O(1)$ deletions).
2. `rear`: Points to the last node (for $O(1)$ insertions).

If we only maintained a `front` pointer, inserting at the end would require an $O(n)$ traversal, violating the core efficiency rule of a queue.

---

## 🛠️ Global Setup

We define a standard Singly Linked List node and declare `front` and `rear` globally.

```c
#include <stdio.h>
#include <stdlib.h>

struct node {
    int data;
    struct node *next;
};

// Initialize empty queue
struct node *front = NULL;
struct node *rear = NULL;
```

---

## 1️⃣ Enqueue Operation (Insertion)

The `enqueue` operation always happens at the `rear`. We allocate memory dynamically and update the `rear` pointer.
**Time Complexity:** $O(1)$

### C Code Snippet
```c
void enqueue(int x) {
    struct node *newnode;
    newnode = (struct node*) malloc(sizeof(struct node));
    newnode->data = x;
    newnode->next = NULL;

    // 1. If queue is empty
    if (front == NULL && rear == NULL) {
        front = rear = newnode;
    } 
    // 2. Normal insertion at the rear
    else {
        rear->next = newnode; // Connect old rear to newnode
        rear = newnode;       // Shift rear pointer to newnode
    }
}
```

---

## 2️⃣ Dequeue Operation (Deletion)

The `dequeue` operation always happens at the `front`. We must anchor the node to be deleted using a `temp` pointer so we can safely `free()` it from memory after shifting the `front` pointer.
**Time Complexity:** $O(1)$

### C Code Snippet
```c
void dequeue() {
    struct node *temp;

    // 1. Check for Underflow
    if (front == NULL && rear == NULL) {
        printf("Underflow: Queue is empty\n");
    } 
    // 2. Normal deletion from the front
    else {
        temp = front; // Anchor the node to delete
        printf("Dequeued: %d\n", front->data);
        
        front = front->next; // Shift front forward
        free(temp);          // Release memory
        
        // Safety check: If we deleted the last node, reset rear as well
        if (front == NULL) {
            rear = NULL;
        }
    }
}
```

> [!WARNING]
> **Memory Leak Trap:**
> In array implementations, we just shift the `front` index forward and ignore the old data. In a Linked List, you **must** use the `free()` function to release the dynamically allocated memory back to the heap. Failure to do so results in a memory leak.

---

## 3️⃣ Peek and Display Operations

### Peek ($O(1)$)
Returns the data at the `front` without modifying the list structure.
```c
void peek() {
    if (front == NULL && rear == NULL) {
        printf("Queue is empty\n");
    } else {
        printf("Front data: %d\n", front->data);
    }
}
```

### Display ($O(n)$)
Traverses the linked list from `front` to `NULL`.
```c
void display() {
    struct node *temp;

    if (front == NULL && rear == NULL) {
        printf("Queue is empty\n");
    } else {
        temp = front;
        printf("Queue elements: ");
        while (temp != NULL) {
            printf("%d ", temp->data);
            temp = temp->next;
        }
        printf("\n");
    }
}
```
