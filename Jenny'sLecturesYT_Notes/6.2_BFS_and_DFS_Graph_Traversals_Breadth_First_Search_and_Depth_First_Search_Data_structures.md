# 6.2 BFS and DFS Graph Traversals | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=pcKY4hjDrxk)

---

## 📌 Executive Summary
Graph Traversal refers to the systematic process of visiting all the nodes (vertices) of a graph. The two fundamental algorithms for traversing a graph are **BFS (Breadth-First Search)** and **DFS (Depth-First Search)**. Both algorithms can start from any arbitrary node as the "root", and depending on the order in which adjacent nodes are processed, a single graph can have multiple valid BFS or DFS traversals.

---

## 🌊 Breadth-First Search (BFS)
**Concept:** BFS is a **Level-Order Traversal**. It explores the graph broadly. Starting from a root node, it visits all immediate neighbors (adjacent nodes) first, before moving on to the neighbors of those neighbors.

**Data Structure Used:** **Queue (FIFO - First In, First Out)**

### Algorithm / Working Principle
1. Choose a starting node (root) and enqueue it into the Queue.
2. Mark the node as visited and print it.
3. While the Queue is not empty:
   - Dequeue the front node.
   - Find all of its **unvisited** adjacent vertices.
   - Enqueue all these unvisited adjacent vertices into the Queue.
   - Mark them as visited and print them.
4. Repeat until the Queue is empty.

> **Key Characteristic:** BFS explores outward in concentric circles or levels. It ensures that all nodes at distance $d$ from the root are visited before any node at distance $d+1$.

---

## 🕳️ Depth-First Search (DFS)
**Concept:** DFS goes as deep as possible down one path before retreating. Starting from the root, it picks *one* unvisited adjacent node, explores it, picks one of *its* unvisited adjacent nodes, and continues until it hits a **dead end** (a node with no unvisited adjacent nodes). Once it hits a dead end, it **backtracks**.

**Data Structure Used:** **Stack (LIFO - Last In, First Out)**

### Algorithm / Working Principle
1. Choose a starting node (root), push it onto the Stack, and print it (mark as visited).
2. While the Stack is not empty:
   - Look at the top node of the Stack.
   - If it has any **unvisited** adjacent vertex, pick *one* of them, push it onto the Stack, and print it (mark as visited).
   - If it does **not** have any unvisited adjacent vertices (it is a dead end), **pop** the node off the Stack. This is the **Backtracking** step.
3. Repeat until the Stack is completely empty.

> **Key Characteristic:** Backtracking is the defining feature of DFS. The Stack explicitly remembers the path taken so the algorithm can retrace its steps when a dead end is reached.

---

## ⚖️ BFS vs. DFS Comparison

| Feature | Breadth-First Search (BFS) | Depth-First Search (DFS) |
| :--- | :--- | :--- |
| **Data Structure** | **Queue (FIFO)** | **Stack (LIFO)** |
| **Traversal Style** | Level by Level (Broad) | Path by Path (Deep) |
| **Backtracking?** | No | Yes |
| **Multiple Answers?**| Yes, depends on queue insertion order | Yes, depends on stack insertion order |
| **Primary Use Case** | Finding Shortest Path in unweighted graphs | Detecting cycles, solving mazes, topological sorting |

> **Exam Tip:** If a question simply asks for "the BFS/DFS traversal" without specifying alphabetical or numerical priority for breaking ties, clearly state: *"Note: Multiple valid traversals exist based on the order adjacent nodes are selected. I have selected nodes in ascending numerical order for this trace."* Always mention the underlying Data Structure (Queue for BFS, Stack for DFS) to secure full marks.
