# 6.5 Prim's Algorithm for Minimum Spanning Tree | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=ZtZaR7EcI5Y)

---

## 📌 Executive Summary

Prim's algorithm finds the **Minimum Spanning Tree (MST)** of a weighted, connected, undirected graph using a **greedy approach** — always picking the minimum weight edge from the explored set.

---

## 🔑 MST Properties
- Spans **all vertices** (V vertices → V-1 edges)
- **Minimum total weight**
- **No cycles**
- May not be unique if equal weights exist

---

## 📐 Prim's Algorithm

1. Start from any vertex, add it to MST set
2. Find the **minimum weight edge** connecting MST set to a non-MST vertex
3. Add that edge and vertex to MST
4. Repeat until all vertices included (V-1 edges)

---

## 📐 Example

```
Graph:
   A --4-- B
   |     / |
   8   2   6
   | /     |
   C --3-- D
   
Start from A:

Step 1: MST = {A}
        Edges from A: A-B(4), A-C(8)
        Min = A-B(4) → MST = {A, B}

Step 2: MST = {A, B}
        Edges: A-C(8), B-C(2), B-D(6)
        Min = B-C(2) → MST = {A, B, C}

Step 3: MST = {A, B, C}
        Edges: A-C(8)skip, C-D(3), B-D(6)
        Min = C-D(3) → MST = {A, B, C, D}

MST edges: A-B(4), B-C(2), C-D(3)
Total weight = 4 + 2 + 3 = 9
```

```
MST:
   A --4-- B
          /
        2
       /
      C --3-- D

Total = 9
```

## 📐 Table Format (Exam-friendly)

| Step | MST Set | Edge Selected | Weight |
|------|---------|--------------|--------|
| 1 | {A} | A—B | 4 |
| 2 | {A, B} | B—C | 2 |
| 3 | {A, B, C} | C—D | 3 |

**Total MST weight = 9**

---

## ⏱️ Complexity

| Implementation | Time |
|---------------|------|
| Adjacency Matrix | **O(V²)** |
| Binary Heap + Adj List | **O(E log V)** |
| Fibonacci Heap | **O(E + V log V)** |

---

## 📊 Prim's vs Kruskal's

| Feature | Prim's | Kruskal's |
|---------|--------|-----------|
| Approach | Vertex-based (grow MST) | Edge-based (sort edges) |
| Best for | **Dense** graphs | **Sparse** graphs |
| Data structure | Priority queue | Union-Find |
| Time (matrix) | O(V²) | O(E log E) |

---

## 💡 Key Takeaways

- **Greedy**: always pick minimum edge from MST to non-MST
- Start from **any vertex** — result is same
- V-1 edges in final MST
- Dense graphs → Prim's, Sparse → Kruskal's
- Draw the MST table in exams for full marks
