# 6.5 Prim's Algorithm for Minimum Spanning Tree | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=M-b5h0Bw090)

---

## 📌 Executive Summary
**Prim's Algorithm** is a Greedy Algorithm used to find the Minimum Spanning Tree (MST) for a weighted undirected graph. Unlike Kruskal's Algorithm (which sorts all edges and connects the smallest ones globally), Prim's Algorithm starts from an arbitrary root node and "grows" the spanning tree one node at a time by always selecting the cheapest outgoing edge from the currently built tree.

---

## ⚙️ Step-by-Step Algorithm

### Step 1: Preprocessing
Before applying Prim's logic, you must simplify the graph:
1. **Remove all loops:** If there is an edge starting and ending at the same vertex (e.g., $A \rightarrow A$), remove it entirely, regardless of its weight.
2. **Remove parallel edges:** If there are multiple edges between the same two vertices (e.g., $A \rightarrow B$ with weights 6 and 8), **keep only the edge with the minimum weight** (keep 6, remove 8).

### Step 2: Initialization
- Select **any arbitrary vertex** to be the root node. (It does not matter which node you pick; the total cost of the final MST will remain exactly the same).
- Mark this node as part of the MST.

### Step 3: Growing the Tree
- Look at **ALL** vertices currently included in the MST.
- Identify all outgoing edges from these vertices that connect to an *unvisited* vertex (a vertex not yet in the MST).
- Out of all these candidate edges, pick the one with the **minimum absolute weight**.
- Add this edge and the newly reached vertex to the MST.

### Step 4: Termination
- Repeat Step 3 until all vertices have been visited.
- The MST will officially be complete when it contains exactly $V - 1$ edges (where $V$ is the total number of vertices).

---

## ⚠️ Critical Exam Warning: The Most Common Mistake
> **Common Pitfall:** When a student adds a new vertex $X$ to the MST, they often only look at the outgoing edges from $X$ to pick the next edge. **This is incorrect!**
> 
> **The Right Way:** You must evaluate the outgoing edges from $X$ **AND** the remaining outgoing edges from every other vertex you have already visited. The "minimum edge" is chosen from the combined pool of all edges extending outward from the *entire* growing tree, not just the latest leaf node.

---

## 📐 Algorithm Trace Example
*(Based on a 6-Vertex Graph from the transcript)*

1. **Start:** Choose node `E` arbitrarily.
2. **Pass 1:** Edges going out of `E` have weights `7` and `8`. 
   - *Action:* Pick `7` (connects `E` to `A`).
   - *MST contains:* `{E, A}`.
3. **Pass 2:** Now look at edges going out of **BOTH** `E` and `A`. 
   - `E` has an edge of weight `8`. `A` has edges of weights `3` and `6`.
   - *Action:* Pick `3` (connects `A` to `B`).
   - *MST contains:* `{E, A, B}`.
4. **Pass 3:** Look at edges from `E`, `A`, and `B`.
   - The available edges have weights `8`, `6`, and a new edge from `B` of weight `3`.
   - *Action:* Pick `3` (connects `B` to `C`).
   - *MST contains:* `{E, A, B, C}`.
5. **Pass 4 & 5:** Continue this logic, always scanning the entire "frontier" of the MST for the absolute smallest available edge that doesn't create a cycle, until exactly 5 edges have been selected.

---

## ⏱️ Complexity Analysis
| Implementation | Time Complexity |
| :--- | :--- |
| **Using Adjacency Matrix & Array** | **O(V²)** |
| **Using Adjacency List & Min-Heap** | **O(E log V)** |

> **Exam Tip:** Prim's algorithm is best suited for **dense graphs** (where the number of edges $E$ is close to $V^2$), whereas Kruskal's algorithm is preferred for **sparse graphs**. Always ensure your final drawing contains exactly $V-1$ edges and no closed loops.
