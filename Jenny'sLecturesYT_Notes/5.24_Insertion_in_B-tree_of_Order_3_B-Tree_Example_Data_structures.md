# 5.24 Insertion in B-Tree of Order 3 | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
When constructing a B-Tree, all new keys are inserted into the **leaf nodes**. If inserting a key causes the leaf node to exceed its maximum allowed capacity, the node **overflows** and must be split.

The core splitting algorithm involves finding the **median** element of the overfilled node and pushing it **one level up** to the parent, effectively splitting the remaining elements into left and right children.

---

## ⚙️ The Splitting Algorithm

**The Overflow Condition:**
If a key is inserted and the node exceeds $M - 1$ keys, it overflows.

**The Splitting Rule:**
1. Temporarily insert the new key into the node to find the true sorted sequence.
2. Identify the **Median (Middle)** element.
3. Push the Median element **one level UP** to the parent node.
4. Split the remaining elements into two new nodes:
   - Elements smaller than the median become the left child.
   - Elements greater than the median become the right child.
5. *Note: If the parent node also overflows after receiving the median, recursively apply the splitting rule to the parent until the root splits. If the root splits, the height of the tree increases by 1.*

## 📝 Exam Walkthrough: Order 3 Insertion

> **Task:** Construct a B-Tree of Order 3 by inserting the values `1 to 10`.

**Parameters:** 
- Order $M = 3$. 
- Max Keys per node = $M - 1 = 2$.

### Step-by-Step Construction

**1. Insert 1, 2:**
The root node has space for 2 keys.
```text
[ 1 | 2 ]
```

**2. Insert 3 (Overflow):**
The node becomes `[ 1 | 2 | 3 ]`. This exceeds the max keys (2).
- The median element is **2**.
- Push **2** up to become the new root.
- Left child is `[ 1 ]`, right child is `[ 3 ]`.
```text
      [ 2 ]
     /     \
   [1]    [3]
```

**3. Insert 4:**
Compare with root (2). 4 is greater, so go right. Insert into leaf `[3]`.
```text
      [ 2 ]
     /     \
   [1]    [3 | 4]
```

**4. Insert 5 (Overflow):**
Go right. Leaf becomes `[ 3 | 4 | 5 ]`. Overflow!
- The median is **4**.
- Push **4** up to the parent `[ 2 ]`.
- Left child is `[ 3 ]`, right child is `[ 5 ]`.
```text
      [ 2 | 4 ]
     /    |    \
   [1]   [3]   [5]
```

**5. Insert 6:**
Go right of 4. Insert into leaf `[5]`.
```text
      [ 2 | 4 ]
     /    |    \
   [1]   [3]   [5 | 6]
```

**6. Insert 7 (Cascading Overflow):**
Go right of 4. Leaf becomes `[ 5 | 6 | 7 ]`. Overflow!
- Median is **6**. Push 6 up.
- Parent becomes `[ 2 | 4 | 6 ]`. Overflow!
- *Recursive Split:* Parent median is **4**. Push 4 up to become the new root.
- The tree grows in height.
```text
           [ 4 ]
          /     \
      [ 2 ]     [ 6 ]
     /    |     |    \
   [1]   [3]   [5]  [7]
```

**7. Insert 8, 9, 10:**
Continue the same logic, splitting leaves as they hit 3 elements, and pushing medians up to the internal nodes.
*Final State after 10:*
```text
           [ 4 ]
          /     \
      [ 2 ]     [ 6 | 8 ]
     /    |     |    |    \
   [1]   [3]   [5]  [7]  [9 | 10]
```
