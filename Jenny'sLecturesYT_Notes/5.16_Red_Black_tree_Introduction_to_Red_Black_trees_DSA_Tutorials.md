## 📌 Executive Summary

Red-Black Tree is a **self-balancing BST** with an extra color bit (Red/Black) per node. It ensures the tree remains approximately balanced, guaranteeing O(log n) operations.

---

## 🔑 Properties (All 5 Must Hold)

1. Every node is either **Red** or **Black**
2. **Root** is always **Black**
3. Every **leaf (NULL)** is **Black** (NIL nodes)
4. If a node is **Red**, both children must be **Black** (no two consecutive reds)
5. Every path from a node to its descendant NULL nodes has the **same number of Black nodes** (Black Height)

---

## 📐 Black Height
> Number of **black nodes** on any path from a node to a NULL leaf (not counting the node itself).

```
Example:
         [B:13]
        /      \
     [R:8]     [R:17]
     /   \       \
  [B:1] [B:11]  [B:25]
                /
             [R:22]
```

Black height of root = 2 (each path has 2 black nodes to NULL)

---

## 📐 Properties Derived

| Property | Value |
|----------|-------|
| Max height | **2 × log₂(n+1)** |
| Search time | **O(log n)** |
| Insert time | **O(log n)** |
| Delete time | **O(log n)** |

---

## 📊 AVL vs Red-Black Tree

| Feature | AVL | Red-Black |
|---------|-----|-----------|
| Balance | Strict (BF ∈ {-1,0,1}) | Relaxed (color rules) |
| Height | ≤ 1.44 log n | ≤ 2 log n |
| Search | **Faster** (shorter) | Slightly slower |
| Insert/Delete | More rotations | **Fewer rotations** |
| Use case | Read-heavy | **Write-heavy** |
| Used in | Databases | Java TreeMap, Linux kernel |

---

## 💡 Key Takeaways

- 5 properties define a valid Red-Black tree
- **No two consecutive red nodes** — most tested property
- **Equal black height** on all paths — ensures balance
- Less strict than AVL → fewer rotations on insert/delete
- Root is ALWAYS black
- NULL nodes are considered black
