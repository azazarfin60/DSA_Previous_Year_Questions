## 📌 Executive Summary
Constructing an expression tree directly from an Infix expression requires complex precedence and associativity checks. In computer science, a much more efficient programmatic approach is to first convert the Infix expression to **Postfix**, and then use a **Stack** to construct the tree in a single pass. 

---

## 🔑 The Stack of Pointers Algorithm
Unlike mathematical evaluation where the stack holds integer values, when constructing a tree, the **stack holds pointers to Tree Nodes**.

### Algorithm Steps
1. Scan the Postfix expression from **Left to Right**.
2. **If you encounter an Operand (e.g., A, B, 5):**
   - Create a new Leaf Node containing the operand.
   - Push the **pointer** to this node onto the Stack.
3. **If you encounter an Operator (e.g., +, -, *, /):**
   - Create a new Internal Node containing the operator.
   - Pop two pointers from the top of the stack.
     - **CRITICAL RULE:** The *first* popped pointer becomes the **Right Child** of the operator node.
     - The *second* popped pointer becomes the **Left Child** of the operator node.
   - Push the pointer to this newly created subtree back onto the Stack.
4. **End of Expression:**
   - When the string is fully scanned, the Stack will contain exactly one pointer. This pointer points to the **Root** of the fully constructed Expression Tree.

## 📐 Trace Example
**Postfix Expression:** `A B * C / E F / G * + K + X Y * -`

*Note: This is the exact postfix equivalent of the complex tree built in the previous note (3.12).*

| Token Scanned | Action Performed | Stack State (Holding Pointers) |
| :--- | :--- | :--- |
| `A` | Create Node `A`. | `[Ptr(A)]` |
| `B` | Create Node `B`. | `[Ptr(A), Ptr(B)]` |
| `*` | Pop `B` (Right child), Pop `A` (Left child). Create Node `*`. | `[Ptr(*)]` |
| `C` | Create Node `C`. | `[Ptr(*), Ptr(C)]` |
| `/` | Pop `C` (Right), Pop `*` (Left). Create Node `/`. | `[Ptr(/)]` |
| `E` | Create Node `E`. | `[Ptr(/), Ptr(E)]` |
| `F` | Create Node `F`. | `[Ptr(/), Ptr(E), Ptr(F)]` |
| `/` | Pop `F` (Right), Pop `E` (Left). Create Node `/`. | `[Ptr(/), Ptr(/)]` |
| `G` | Create Node `G`. | `[Ptr(/), Ptr(/), Ptr(G)]` |
| `*` | Pop `G` (Right), Pop `/` (Left). Create Node `*`. | `[Ptr(/), Ptr(*)]` |
| `+` | Pop `*` (Right), Pop `/` (Left). Create Node `+`. | `[Ptr(+)]` |
| `K` | Create Node `K`. | `[Ptr(+), Ptr(K)]` |
| `+` | Pop `K` (Right), Pop `+` (Left). Create Node `+`. | `[Ptr(+)]` |
| `X` | Create Node `X`. | `[Ptr(+), Ptr(X)]` |
| `Y` | Create Node `Y`. | `[Ptr(+), Ptr(X), Ptr(Y)]` |
| `*` | Pop `Y` (Right), Pop `X` (Left). Create Node `*`. | `[Ptr(+), Ptr(*)]` |
| `-` | Pop `*` (Right), Pop `+` (Left). Create Node `-`. | `[Ptr(-)]` |
| *End* | Return remaining pointer in stack. | **Root = Node `-`** |

### Step-by-Step Visual of Subtree Creation
When the first `*` operator is scanned:
```text
Stack had: [Ptr(A), Ptr(B)]
Pop B -> Set as Right Child
Pop A -> Set as Left Child

Creates Subtree:
    [*]
   /   \
 [A]   [B]

Stack now holds: [Ptr(*)]
```

## 🚨 Exam Warning
If a question asks you to write the C/C++ pseudo-code for this tree construction, you must emphasize that the stack is of type `struct Node*` (Pointer to Node), not `int` or `char`.

```c
// Correct Stack Type Definition
struct Node* stack[MAX];

// Incorrect
int stack[MAX];
char stack[MAX];
```
