## 📌 Executive Summary
Implementing a Circular Queue using a Linked List combines the dynamic resizing advantages of a linked list with the continuous loop structure of a circular queue.

Because we are using dynamic memory allocation (`malloc`), **there is no maximum size limit**, which means we never have to write an Overflow condition. 

To maintain the circular property, the most critical rule is:
**`rear->next` must ALWAYS point to `front`**.

---

## 🛠️ Global Setup

We maintain our standard `front` and `rear` pointers to ensure $O(1)$ enqueue and dequeue times.

```c
#include <stdio.h>
#include <stdlib.h>

struct node {
    int data;
    struct node *next;
};

struct node *front = NULL;
struct node *rear = NULL;
```

---

## 1️⃣ Enqueue Operation (Insertion)

When inserting a new node at the `rear`, we must ensure that the new `rear` reaches back and connects to the `front`.
**Time Complexity:** $O(1)$

### C Code Snippet
```c
void enqueue(int x) {
    struct node *newnode;
    newnode = (struct node*) malloc(sizeof(struct node));
    newnode->data = x;
    newnode->next = NULL;

    // 1. Check if queue is empty
    if (front == NULL && rear == NULL) {
        front = rear = newnode;
        rear->next = front; // Circular connection to itself
    } 
    // 2. Normal insertion
    else {
        rear->next = newnode; // Connect old rear to newnode
        rear = newnode;       // Shift rear to newnode
        rear->next = front;   // 🌟 SECURE THE CIRCULAR LINK 🌟
    }
}
```

---

## 2️⃣ Dequeue Operation (Deletion)

When deleting the `front` node, we shift the `front` pointer forward. Because the `front` has changed, the `rear` must be updated to point to the *new* `front` before we release the old memory.
**Time Complexity:** $O(1)$

### C Code Snippet
```c
void dequeue() {
    struct node *temp;

    // 1. Check for Underflow
    if (front == NULL && rear == NULL) {
        printf("Underflow: Queue is empty\n");
    } 
    // 2. Check if only ONE element is left
    else if (front == rear) {
        temp = front;
        front = rear = NULL; // Queue is now empty
        free(temp);
    } 
    // 3. Normal deletion
    else {
        temp = front;            // Anchor the node to delete
        front = front->next;     // Shift front forward
        rear->next = front;      // 🌟 SECURE THE CIRCULAR LINK 🌟
        free(temp);              // Release memory
    }
}
```

> [!CAUTION]
> **Exam Trap (Forgetting the Rear Update):**
> If you write `front = front->next;` and immediately `free(temp);`, your list is broken. The `rear->next` pointer is still pointing at the memory address you just freed! You **must** write `rear->next = front;` prior to freeing.

---

## 3️⃣ Display Operation ($O(n)$)

Because there are no `NULL` pointers in a circular linked list, a standard `while(temp != NULL)` loop will result in an infinite loop. We must stop traversing when we reach the node that points back to `front`.

> [!WARNING]
> **The "Missing Last Node" Bug:**
> If we use `while (temp->next != front)`, the loop halts the moment it lands on the `rear` node. Therefore, the data inside the `rear` node is **never printed** by the loop. You must explicitly print it one last time outside the loop.

### C Code Snippet
```c
void display() {
    struct node *temp;

    if (front == NULL && rear == NULL) {
        printf("Queue is empty\n");
        return;
    }

    temp = front;
    printf("Queue elements: ");
    
    // Stop exactly when temp reaches the rear node
    while (temp->next != front) {
        printf("%d ", temp->data);
        temp = temp->next;
    }
    
    // TRAP FIX: Print the final node (rear)
    printf("%d\n", temp->data); 
}
```

*Note: The `peek()` operation remains simply `return front->data;` ($O(1)$).*
