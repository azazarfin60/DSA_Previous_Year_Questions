## 📌 Executive Summary

This lecture covers the **four types of linked lists**: Singly, Doubly, Circular, and Doubly Circular Linked List — with node structure, memory representation, and C struct definition for each.

---

## 🔑 Four Types

### 1. Singly Linked List
- **Node:** 2 parts — Data + Next pointer
- Single link, forward traversal only
- Last node's `next` = NULL

```
Head → [3|200] → [7|300] → [8|NULL]
```

```c
struct node {
    int data;
    struct node *next;
};
```

### 2. Doubly Linked List
- **Node:** 3 parts — Prev + Data + Next
- Bidirectional traversal
- First node's `prev` = NULL, Last node's `next` = NULL

```
NULL ← [NULL|1|200] ⇄ [100|2|300] ⇄ [200|3|NULL]
```

```c
struct node {
    struct node *prev;
    int data;
    struct node *next;
};
```

### 3. Circular Linked List
- Same as singly, but **last node → first node**
- No NULL in the list

```
Head → [3|200] → [7|300] → [8|100] → (back to head)
```

### 4. Doubly Circular Linked List
- Doubly + Circular combined
- Last node's next → first, First node's prev → last

```
[300|1|200] ⇄ [100|2|300] ⇄ [200|3|100]
   ↑________________________↓ (circular both ways)
```

---

## 📊 Comparison

| Feature | Singly | Doubly | Circular | Doubly Circular |
|---------|--------|--------|----------|-----------------|
| Parts/node | 2 | 3 | 2 | 3 |
| Traversal | Forward | Both | Forward (circular) | Both (circular) |
| Last→next | NULL | NULL | First node | First node |
| Memory | Least | More | Same as singly | Most |

---

## 💡 Key Takeaways

- **Singly** = simplest, one-direction
- **Doubly** = bidirectional, extra memory per node
- **Circular** = no end, useful for round-robin scheduling
- **Doubly Circular** = most flexible, bidirectional + circular
