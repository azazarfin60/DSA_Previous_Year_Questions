# 2.2 Types of Linked Lists | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
There are four primary variations of linked lists. They differ based on two key properties: **directionality** (can you traverse backwards?) and **circularity** (does the end connect back to the beginning?). 

---

## 1️⃣ Singly Linked List (SLL)
This is the most fundamental type of linked list. It allows traversal in only one direction (forward).

- **Node Structure:** 2 parts (Data, Next Pointer).
- **First Node:** Pointed to by `head`.
- **Last Node:** The `next` pointer contains `NULL` (indicating the end).
- **Direction:** Unidirectional.

### Logical Representation
```text
Head -> [ Data | Next ] -> [ Data | Next ] -> [ Data | NULL ]
```

### C Implementation of Node
```c
struct node {
    int data;
    struct node *next;
};
```

---

## 2️⃣ Doubly Linked List (DLL)
A doubly linked list allows traversal in both directions (forward and backward) at the cost of extra memory per node for an additional pointer.

- **Node Structure:** 3 parts (Previous Pointer, Data, Next Pointer).
- **First Node:** The `prev` pointer contains `NULL`.
- **Last Node:** The `next` pointer contains `NULL`.
- **Direction:** Bidirectional.

### Logical Representation
```text
NULL <- [ Prev | Data | Next ] <-> [ Prev | Data | Next ] -> NULL
  ^                                                           ^
 Head                                                        End
```

### C Implementation of Node
```c
struct node {
    int data;
    struct node *prev;
    struct node *next;
};
```

---

## 3️⃣ Circular (Singly) Linked List (CLL)
A variation of the singly linked list where the last node does not point to `NULL`. Instead, it connects back to the first node, forming a circle.

- **Node Structure:** 2 parts (Data, Next Pointer) - *Same as SLL*.
- **First Node:** Pointed to by `head`.
- **Last Node:** The `next` pointer contains the address of the **First Node** (`head`).
- **Direction:** Unidirectional (but continuous in a loop).

### Logical Representation
```text
      -----------------------------------------
      |                                       |
      v                                       |
Head -> [ Data | Next ] -> [ Data | Next ] ----
```

---

## 4️⃣ Doubly Circular Linked List (DCLL)
This is the most complex variation, combining the properties of both Doubly and Circular linked lists.

- **Node Structure:** 3 parts (Previous Pointer, Data, Next Pointer) - *Same as DLL*.
- **Last Node:** The `next` pointer contains the address of the **First Node** (`head`).
- **First Node:** The `prev` pointer contains the address of the **Last Node**.
- **Direction:** Bidirectional and continuous.

### Logical Representation
```text
      -------------------------------------------------------
      |                                                     |
      v                                                     |
Head -> [ Prev | Data | Next ] <-> [ Prev | Data | Next ] <-|
      |                                                     |
      -------------------------------------------------------
```

## ⚖️ Comparison Table

| Feature | Singly | Doubly | Circular (Singly) | Doubly Circular |
| :--- | :--- | :--- | :--- | :--- |
| **Pointers per node** | 1 | 2 | 1 | 2 |
| **Null pointers exist?** | Yes (1) | Yes (2) | No | No |
| **Backward Traversal?** | No | Yes | No | Yes |
| **Last node points to** | `NULL` | `NULL` | First Node | First Node |
| **First node's prev points to**| N/A | `NULL` | N/A | Last Node |
| **Memory Overhead** | Lowest | High | Lowest | High |
