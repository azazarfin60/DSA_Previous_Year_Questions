## 📌 Executive Summary

Merge Sort uses **Divide and Conquer** — splits the array into halves recursively, then **merges** sorted halves. Always O(n log n) but needs O(n) extra space.

---

## 📐 Algorithm

1. **Divide**: Split array into two halves
2. **Conquer**: Recursively sort each half
3. **Merge**: Combine two sorted halves into one sorted array

---

## 💻 C Code

```c
void merge(int arr[], int l, int mid, int r) {
    int n1 = mid - l + 1;
    int n2 = r - mid;
    int L[n1], R[n2];    // Temp arrays

    for (int i = 0; i < n1; i++) L[i] = arr[l + i];
    for (int j = 0; j < n2; j++) R[j] = arr[mid + 1 + j];

    int i = 0, j = 0, k = l;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j])
            arr[k++] = L[i++];
        else
            arr[k++] = R[j++];
    }
    while (i < n1) arr[k++] = L[i++];    // Copy remaining
    while (j < n2) arr[k++] = R[j++];
}

void mergeSort(int arr[], int l, int r) {
    if (l < r) {
        int mid = (l + r) / 2;
        mergeSort(arr, l, mid);        // Sort left half
        mergeSort(arr, mid + 1, r);    // Sort right half
        merge(arr, l, mid, r);         // Merge sorted halves
    }
}
```

## ⏱️ Complexity

| Case | Time |
|------|------|
| **Best** | **O(n log n)** |
| **Average** | **O(n log n)** |
| **Worst** | **O(n log n)** |

**Space: O(n)** — needs auxiliary array
**Stable: ✅ Yes**

### Why O(n log n)?
```
Divide: log n levels of splitting
Merge: O(n) work at each level
Total: n × log n
```

## 📊 Recurrence Relation
```
T(n) = 2T(n/2) + O(n)
By Master Theorem: T(n) = O(n log n)
```

## 💡 Key Takeaways

- **Always O(n log n)** — consistent performance
- Needs **O(n) extra space** — main disadvantage
- **Stable** sort — preserves order of equal elements
- Best for **linked lists** (no random access needed)
- Divide & Conquer paradigm
