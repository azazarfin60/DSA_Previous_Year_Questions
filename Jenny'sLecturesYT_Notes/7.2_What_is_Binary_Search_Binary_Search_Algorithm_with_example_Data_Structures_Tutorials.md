## 📌 Executive Summary

Binary Search works on a **sorted array** by repeatedly dividing the search interval in half. Much faster than linear search — O(log n).

---

## 📐 Algorithm

1. Set `low = 0`, `high = n-1`
2. Calculate `mid = (low + high) / 2`
3. If `arr[mid] == key` → found!
4. If `key < arr[mid]` → search left: `high = mid - 1`
5. If `key > arr[mid]` → search right: `low = mid + 1`
6. Repeat until `low > high` (not found)

---

## 💻 C Code

```c
int binarySearch(int arr[], int n, int key) {
    int low = 0, high = n - 1;

    while (low <= high) {
        int mid = (low + high) / 2;

        if (arr[mid] == key)
            return mid;              // Found
        else if (key < arr[mid])
            high = mid - 1;          // Search left
        else
            low = mid + 1;           // Search right
    }
    return -1;                       // Not found
}
```

## 📐 Trace Example

```
Array: [2, 5, 8, 12, 16, 23, 38, 56, 72, 91]   Key = 23

Pass 1: low=0, high=9, mid=4 → arr[4]=16 < 23 → low=5
Pass 2: low=5, high=9, mid=7 → arr[7]=56 > 23 → high=6
Pass 3: low=5, high=6, mid=5 → arr[5]=23 = 23 → Found at index 5! ✅
```

## ⏱️ Time Complexity

| Case | Time |
|------|------|
| **Best** | **O(1)** (mid element) |
| **Worst** | **O(log₂ n)** |
| **Average** | **O(log₂ n)** |

**Space: O(1)** iterative, O(log n) recursive

### Why O(log n)?
```
n elements → n/2 → n/4 → n/8 → ... → 1
Number of steps = log₂(n)

For n = 1024: log₂(1024) = 10 comparisons only!
vs Linear: up to 1024 comparisons
```

## 📐 Recursive Version

```c
int binarySearchRec(int arr[], int low, int high, int key) {
    if (low > high) return -1;

    int mid = (low + high) / 2;

    if (arr[mid] == key) return mid;
    else if (key < arr[mid])
        return binarySearchRec(arr, low, mid - 1, key);
    else
        return binarySearchRec(arr, mid + 1, high, key);
}
```

---

## 📊 Linear vs Binary Search

| Feature | Linear | Binary |
|---------|--------|--------|
| Array | Sorted or unsorted | **Sorted only** |
| Time | O(n) | **O(log n)** |
| Approach | Sequential | Divide & Conquer |

---

## 💡 Key Takeaways

- **Prerequisite**: array must be **sorted**
- Halves search space each step → O(log n)
- Much faster than linear for large arrays
- Can be iterative or recursive
