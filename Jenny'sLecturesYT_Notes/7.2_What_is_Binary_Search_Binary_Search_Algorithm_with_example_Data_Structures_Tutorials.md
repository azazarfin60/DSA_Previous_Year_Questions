# 7.2 Binary Search Algorithm | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=RWeYvE77yEE)

---

## 📌 Executive Summary
**Binary Search** is an extremely fast searching algorithm based on the **Divide and Conquer** technique. Unlike Linear Search which checks every element, Binary Search repeatedly divides the search space in half. 
> **CRITICAL REQUIREMENT:** Binary Search can **ONLY** be applied to arrays that are already **sorted** (either ascending or descending).

---

## 🔑 Core Concept
1. **Initialize Pointers:** Set two pointers: `L` (Left) at the 0th index, and `R` (Right) at the last index ($N - 1$).
2. **Find Middle:** Calculate the middle index: `mid = (L + R) / 2` (taking the floor value).
3. **Compare and Conquer:** There are three possible cases:
   - **Case 1 (Found):** If `A[mid] == data`, the search stops. Return `mid`.
   - **Case 2 (Left Sub-array):** If `data < A[mid]`, the element must be to the left. Update `R = mid - 1`.
   - **Case 3 (Right Sub-array):** If `data > A[mid]`, the element must be to the right. Update `L = mid + 1`.
4. **Stopping Condition:** Repeat steps 2 and 3 until `L > R`. If `L` crosses `R`, it means the search space is exhausted and the element is **not present**.

---

## 📐 Step-by-Step Trace

**Given Sorted Array:** `A = [5, 9, 17, 23, 25, 45, 59, 63, 71, 89]` *(N = 10)*
**Target Data:** `59`

### Pass 1
- `L = 0`, `R = 9`
- `mid = (0 + 9) / 2 = 4`
- `A[4] = 25`. Compare with target: $59 > 25$.
- *Action:* Target is greater, so search the right half. Update `L = mid + 1 = 5`.

### Pass 2
- `L = 5`, `R = 9`
- `mid = (5 + 9) / 2 = 7`
- `A[7] = 63`. Compare with target: $59 < 63$.
- *Action:* Target is smaller, so search the left half. Update `R = mid - 1 = 6`.

### Pass 3
- `L = 5`, `R = 6`
- `mid = (5 + 6) / 2 = 5`
- `A[5] = 45`. Compare with target: $59 > 45$.
- *Action:* Target is greater. Update `L = mid + 1 = 6`.

### Pass 4
- `L = 6`, `R = 6`
- `mid = (6 + 6) / 2 = 6`
- `A[6] = 59`. Compare with target: $59 == 59$.
- *Action:* **Element Found!** Return index `6`.

*(If we were searching for `60`, the next step would make `L = 7`, `R = 6`. Since `L > R`, the loop would break, printing "Not Found").*

---

## 💻 C Code Implementation

```c
int binarySearch(int a[], int n, int data) {
    int l = 0;
    int r = n - 1;
    int mid;
    
    // Continue searching as long as the search space is valid
    while (l <= r) {
        mid = (l + r) / 2; // In C, integer division automatically gives the floor value
        
        if (data == a[mid]) {
            return mid; // Element found, return the index
        }
        else if (data < a[mid]) {
            r = mid - 1; // Discard right half
        }
        else {
            l = mid + 1; // Discard left half
        }
    }
    
    return -1; // Element not found
}
```

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Best Case Time** | **O(1)** | The target element happens to be exactly at the middle index on the very first try. |
| **Worst Case Time** | **O(log N)** | The target is at the extreme ends or not present at all. The array is halved repeatedly: $N \rightarrow N/2 \rightarrow N/4 \rightarrow ... \rightarrow 1$. Mathematically, $N / 2^k = 1 \implies k = \log_2(N)$ iterations. |
| **Average Case Time**| **O(log N)** | |
| **Space Complexity** | **O(1)** | The iterative implementation operates in-place using only three variables (`l`, `r`, `mid`). |

> **Exam Tip:** If asked "Why is Binary Search $O(\log n)$?", quickly write down the halving progression: $N, N/2, N/4 ... 1$. Let $k$ be the number of steps. Set $\frac{N}{2^k} = 1$, which solves to $2^k = N$, therefore $k = \log_2(N)$. Examiners heavily reward this mathematical justification!
