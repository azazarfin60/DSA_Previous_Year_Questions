# 3.8 Infix to Prefix Conversion (Using Stack) | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=0kE2mEDT4yA) *(Note: Placeholder link)*

---

## 📌 Executive Summary
Converting an Infix expression to **Prefix** (Polish Notation) using a stack is highly similar to the Postfix conversion process. However, because Prefix notation places operators *before* operands, we must conceptually process the expression backwards. To achieve this scanning from left to right, we use a **Reverse $\rightarrow$ Process $\rightarrow$ Reverse** technique.

---

## 🔑 The 3-Step Master Algorithm

### Step 1: Reverse the Infix Expression
Reverse the original expression strictly character by character. 
*Note: Because you reversed it, the right parenthesis `)` now acts as the start of a block, and the left parenthesis `(` acts as the end!*

### Step 2: Process using Modified Stack Rules
Scan the reversed string from Left to Right. The rules are almost identical to Postfix, with a few critical flips:
1. **Operands:** Print directly.
2. **Right Parenthesis `)`:** Push directly onto the stack. *(Flipped from Postfix)*
3. **Left Parenthesis `(`:** Pop and print operators until `)` is found. Discard both. *(Flipped from Postfix)*
4. **Operator (Incoming > Top):** Push directly.
5. **Operator (Incoming < Top):** Pop Top, re-test Incoming against new Top.
6. **Operator (Incoming == Top):** Check original associativity:
   - **Left-to-Right (`+`, `-`, `*`, `/`):** **PUSH** incoming directly. *(Flipped!)*
   - **Right-to-Left (`^`):** **POP** Top, re-test incoming. *(Flipped!)*

### Step 3: Reverse the Final Output
The output generated by Step 2 is not the final answer. You must reverse it one last time to get the correct Prefix expression.

---

## 📐 Trace Example

Let's convert a simplified version of the standard complex expression: `K + L - M * N`

### 1. Reverse the Input
Original: `K + L - M * N`
Reversed: `N * M - L + K`

### 2. Process Reversed String
| Token Scanned | Stack State (Top at right) | Output String | Rule Applied |
| :--- | :--- | :--- | :--- |
| `N` | | `N` | Operand $\rightarrow$ Print |
| `*` | `*` | `N` | Empty stack $\rightarrow$ Push |
| `M` | `*` | `N M` | Operand $\rightarrow$ Print |
| `-` | `-` | `N M *` | `-` < `*`. Pop `*`. Push `-`. |
| `L` | `-` | `N M * L` | Operand $\rightarrow$ Print |
| `+` | `- +` | `N M * L` | `+` == `-`. Associativity is L-to-R. **Rule says PUSH!** |
| `K` | `- +` | `N M * L K` | Operand $\rightarrow$ Print |
| *End* | *Empty* | `N M * L K + -` | Pop remaining operators |

### 3. Reverse the Output
Result from Step 2: `N M * L K + -`
Final Prefix Expression: `- + K L * M N`

---

## 🚨 Exam Warning: The Associativity Trap
The most critical place students lose marks in Prefix conversion is handling equal precedence operators. 

**Why does Left-to-Right push instead of pop?**
Because we reversed the string! In the original string `A + B - C`, the `+` comes first. In the reversed string `C - B + A`, the `-` comes first. If we pop the `-` when the `+` arrives, they will end up evaluated in the wrong order. By *pushing* the `+` on top of the `-`, it gets popped *first* at the end of the expression, guaranteeing the Left-to-Right operators are evaluated correctly when the final string is reversed.

> **Exam Tip:**
> - Infix to **Postfix**: Equal Precedence (Left-to-Right) = **POP**.
> - Infix to **Prefix**: Equal Precedence (Left-to-Right) = **PUSH**.
