# 6.10 Topological Sorting (with Examples) | How to find all Topological Orderings of a Graph

📺 [Watch on YouTube](https://www.youtube.com/watch?v=dis_c84ejhQ)

---

## 📌 Executive Summary

Topological Sort produces a **linear ordering** of vertices in a DAG such that for every edge (u,v), u appears before v. Only possible in **Directed Acyclic Graphs (DAG)**.

---

## 🔑 Definition
> A linear ordering of vertices where for every directed edge u→v, vertex u comes **before** v in the ordering.

### Prerequisites
- Graph must be **directed** and **acyclic** (DAG)
- If graph has a cycle → topological sort is **impossible**

---

## 📐 Algorithm 1: Kahn's Algorithm (BFS-based)

1. Calculate **in-degree** of all vertices
2. Enqueue all vertices with **in-degree 0**
3. While queue not empty:
   - Dequeue vertex u, add to result
   - For each neighbor v of u: reduce in-degree by 1
   - If in-degree of v becomes 0 → enqueue v
4. If result size ≠ V → **cycle exists!**

---

## 📐 Algorithm 2: DFS-based

1. Do DFS from each unvisited vertex
2. After processing all neighbors, push vertex to **stack**
3. Pop all from stack → topological order

---

## 📐 Example

```
Graph:
  5 → 0 ← 4
  5 → 2    4 → 1
  2 → 3
  3 → 1

In-degrees: 0:1, 1:2, 2:1, 3:1, 4:0, 5:0
```

### Kahn's Algorithm Trace

| Step | Queue | Remove | Result | Update In-degrees |
|------|-------|--------|--------|--------------------|
| Init | [4, 5] | — | — | — |
| 1 | [5] | 4 | [4] | 0→0, 1→1 |
| 2 | [0] | 5 | [4,5] | 0→0, 2→0 |
| 3 | [2] | 0 | [4,5,0] | — |
| 4 | [3] | 2 | [4,5,0,2] | 3→0 |
| 5 | [1] | 3 | [4,5,0,2,3] | 1→0 |
| 6 | [] | 1 | [4,5,0,2,3,1] | — |

**Topological Order: 4, 5, 0, 2, 3, 1** ✅

---

## ⚠️ Multiple Valid Orderings

> Topological sort is **not unique**. Another valid ordering: `5, 4, 0, 2, 3, 1`

When multiple vertices have in-degree 0 simultaneously, any can be chosen.

---

## 🎯 Applications
- **Task scheduling** (prerequisites)
- **Build systems** (compile order)
- **Course prerequisite** ordering

---

## ⏱️ Complexity
- **Time: O(V + E)**
- **Space: O(V)**

---

## 💡 Key Takeaways

- Only works on **DAG** (Directed Acyclic Graph)
- Multiple valid orderings may exist
- Kahn's = BFS with in-degree; DFS = post-order + reverse
- If result size < V → cycle detected
- Very common exam question!
