# 6.10 Topological Sorting | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=Q9PIxaNGnig) *(Note: Placeholder link)*

---

## 📌 Executive Summary
**Topological Sorting** (or Topological Ordering) is a linear ordering of the vertices in a graph such that for every directed edge $U \rightarrow V$, vertex $U$ strictly appears *before* vertex $V$ in the sequence. It is widely used for scheduling tasks with dependencies (e.g., University course prerequisites, build systems).

---

## 🔑 Crucial Prerequisites
You cannot perform a Topological Sort on just any graph. The graph **MUST** be a **DAG** (Directed Acyclic Graph).
1. **Directed:** Edges must have a one-way direction. It cannot be applied to undirected graphs.
2. **Acyclic:** The graph must NOT contain any cycles. If a cycle exists, a linear dependency ordering is mathematically impossible.

> **Note:** A single Directed Acyclic Graph (DAG) can have **multiple valid** topological orderings.

---

## ⚙️ Step-by-Step Algorithm
To perform topological sorting manually, we use the **In-Degree Deletion Method** (often known as Kahn's Algorithm conceptually):

### Step 1: Calculate Initial In-Degrees
- Find the **in-degree** (the number of incoming edges) for every vertex in the graph.

### Step 2: Select a Vertex
- Look for a vertex that has an **in-degree of 0** (meaning no other tasks need to be completed before it).
- If there are multiple vertices with an in-degree of 0, you can choose *any* of them. (This choice is what leads to multiple valid topological sorts).
- Write this vertex down as the next element in your Topological Order.

### Step 3: Delete and Update
- **Delete** the chosen vertex from the graph.
- **Delete all outgoing edges** originating from this vertex.
- Because edges were deleted, **update (decrement)** the in-degrees of the neighboring vertices that those edges pointed to.

### Step 4: Repeat
- Repeat Steps 2 and 3 until all vertices have been deleted and added to the topological order.

---

## 🚨 The Cycle Detection Rule
What happens if you try to perform this algorithm on a graph that *does* contain a cycle?
- You will follow the steps normally until you reach a point where vertices are still left in the graph, but **none of them have an in-degree of 0**.
- Because every remaining vertex has an in-degree of at least 1, you are stuck. This deadlock mathematically proves that a **Cycle exists** in the graph, making topological sorting impossible.

---

## 📐 Algorithm Trace Example
*(Based on a 4-vertex graph from the transcript: A, B, C, D)*

1. **Initial State:**
   - `In-Degree(A) = 0`, `In-Degree(C) = 0`
   - `In-Degree(B) = 2`, `In-Degree(D) = 2`
2. **Pass 1:**
   - Choose `A` (in-degree 0). Output: `[A]`
   - Delete `A` and its outgoing edges.
   - Update neighbors: `B` becomes 1, `C` remains 0.
3. **Pass 2:**
   - Choose `C` (in-degree 0). Output: `[A, C]`
   - Delete `C` and its outgoing edges.
   - Update neighbors: `D` becomes 0, `B` becomes 0.
4. **Pass 3:**
   - Choose `D` (in-degree 0). Output: `[A, C, D]`
   - Delete `D`. Update `B` (remains 0).
5. **Pass 4:**
   - Choose `B` (in-degree 0). Output: `[A, C, D, B]`

*Alternative Valid Ordering (if we picked C first): `[C, A, B, D]`*

> **Exam Tip:** In an exam, always draw a small table tracking the In-Degree of every vertex. Cross out and update the numbers as you "delete" nodes. This proves your algorithmic working to the examiner and prevents careless calculation errors. Always explicitly state: *"Multiple valid topological orderings exist because multiple nodes had an in-degree of 0 at step X."*
