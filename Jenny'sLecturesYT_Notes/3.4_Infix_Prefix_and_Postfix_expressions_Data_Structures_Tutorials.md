## 📌 Executive Summary

This lecture explains the **three expression notations** — Infix, Prefix, and Postfix — and the **precedence/associativity rules** needed to evaluate infix expressions. Prefix and Postfix don't need these rules, making them easier for computers.

---

## 🔑 Three Notations

| Type | Format | Example | Also Called |
|------|--------|---------|------------|
| **Infix** | Operand **Op** Operand | `A + B` | — |
| **Prefix** | **Op** Operand Operand | `+ A B` | Polish Notation |
| **Postfix** | Operand Operand **Op** | `A B +` | Reverse Polish Notation |

---

## 📐 Operator Precedence (High → Low)

| Priority | Operator | Associativity |
|----------|----------|---------------|
| 1 | `( )` Parentheses | — |
| 2 | `^` Exponent | **Right to Left** |
| 3 | `*` `/` Multiply, Divide | Left to Right |
| 4 | `+` `-` Add, Subtract | Left to Right |

### When to use Associativity?
When two operators have **same precedence**, check associativity to decide evaluation order.

---

## 📐 Conversion Examples

### Infix → Prefix
```
Infix:   A * B + C
Step 1:  *AB + C       (multiply first — higher precedence)
Prefix:  +*ABC
```

### Infix → Postfix
```
Infix:   A * B + C
Step 1:  AB* + C       (multiply first)
Postfix: AB*C+
```

---

## 📐 Right-to-Left Associativity Example
```
Infix: 2 ^ 2 ^ 3
Same precedence → check associativity → Right to Left

Evaluate right first: 2^3 = 8
Then: 2^8 = 256  ✅

NOT: (2^2)^3 = 4^3 = 64  ❌
```

## 🎯 Why Prefix/Postfix?

| Feature | Infix | Prefix/Postfix |
|---------|-------|----------------|
| Needs precedence rules? | ✅ Yes | ❌ No |
| Needs parentheses? | ✅ Yes | ❌ No |
| Easy for humans? | ✅ Yes | ❌ No |
| Easy for computers? | ❌ No | ✅ Yes |
| Parsing time | More | Less |
| Memory usage | More | Less |

---

## 💡 Key Takeaways

- **Infix** = human-readable but needs extra rules
- **Prefix/Postfix** = computer-friendly, no rules needed
- Precedence: `() > ^ > *, / > +, -`
- `^` is **right-to-left** associative (unique!)
- All three are **equivalent** representations of the same expression
