# 7.9 Heap Sort and Heapify Method | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=HqPJF2L5h9U)

---

## 📌 Executive Summary
**Heap Sort** is an efficient, comparison-based sorting algorithm that utilizes the Max Heap data structure. It operates in two major phases: first, building a Max Heap from the unsorted array, and second, repeatedly extracting the maximum element (the root) and moving it to the end of the array. 

While inserting elements one-by-one into a heap takes $O(N \log N)$ time, the highly optimized **Heapify (Build-Heap)** method can construct a Max Heap from an arbitrary array in **$O(N)$ linear time**.

---

## 🔑 Core Concept: The Heapify Method
The genius of the Heapify method relies on the mathematical properties of a Complete Binary Tree.
1. **Leaf Nodes are Heaps:** In an array of size $N$, all nodes from index $\lfloor N/2 \rfloor + 1$ up to $N$ are **leaf nodes**. A single leaf node, by definition, is already a valid max heap.
2. **Bottom-Up Construction:** Instead of starting from an empty heap and inserting elements (top-down), the Heapify method starts from the **last non-leaf node** (index $\lfloor N/2 \rfloor$) and works its way backwards to the root (index 1).
3. **The Subroutine (`maxHeapify`):** For a given node `i`, `maxHeapify` assumes that its left and right subtrees are already valid max heaps. It compares `i` with its children. If a child is larger, it swaps them and recursively calls itself on the affected child subtree to fix any cascading violations.

---

## 📐 The Two Phases of Heap Sort

### Phase 1: Build Max Heap (using Heapify)
Convert the unsorted array into a valid Max Heap.
- Loop `i` from $\lfloor N / 2 \rfloor$ down to $1$.
- Call `maxHeapify(A, N, i)`.

### Phase 2: Deletion (Sorting)
Once the Max Heap is built, the largest element is at the root `A[1]`.
1. **Swap:** Swap the root `A[1]` with the last element of the heap `A[i]`.
2. **Shrink:** The largest element is now at its correct sorted position at the end of the array. Conceptually reduce the heap size by 1.
3. **Restore:** The new root may violate the max heap property. Call `maxHeapify(A, i-1, 1)` to restore the heap.
4. Repeat this until the heap size is 1. The array is now sorted in **ascending order**.

## 💻 C Code Implementation

### 1. The Max Heapify Function
```c
void maxHeapify(int A[], int n, int i) {
    int largest = i;        // Initialize largest as root
    int left = 2 * i;       // Left child
    int right = 2 * i + 1;  // Right child
    
    // If left child is larger than root
    if (left <= n && A[left] > A[largest]) {
        largest = left;
    }
    
    // If right child is larger than largest so far
    if (right <= n && A[right] > A[largest]) {
        largest = right;
    }
    
    // If largest is not root, swap and continue heapifying
    if (largest != i) {
        int temp = A[i];
        A[i] = A[largest];
        A[largest] = temp;
        
        // Recursively heapify the affected sub-tree
        maxHeapify(A, n, largest);
    }
}
```

### 2. The Heap Sort Function
```c
void heapSort(int A[], int n) {
    
    // Phase 1: Build Max Heap using Heapify (Bottom-Up)
    // Start from the last non-leaf node
    for (int i = n / 2; i >= 1; i--) {
        maxHeapify(A, n, i);
    }
    
    // Phase 2: One by one extract an element from heap
    for (int i = n; i >= 2; i--) {
        // Swap root (max element) with the last element
        int temp = A[1];
        A[1] = A[i];
        A[i] = temp;
        
        // Call maxHeapify on the reduced heap
        // Note the size is now i-1, and we heapify from root (1)
        maxHeapify(A, i - 1, 1);
    }
}
```
*(Note: This code assumes the array is 1-indexed. If using 0-indexed arrays in C, left = `2i + 1`, right = `2i + 2`, and loop bounds must be adjusted).*

## ⏱️ Complexity Analysis

| Operation | Complexity | Reason |
| :--- | :--- | :--- |
| **Building Max Heap** | **O(N)** | Using the `maxHeapify` method bottom-up takes linear time, which is mathematically proven using the sum of geometric series (nodes at the bottom move very little, only the root moves the full height). |
| **Deletion Phase** | **O(N log N)** | We delete $N$ elements, and each deletion triggers a `maxHeapify` which takes $O(\log N)$ time. |
| **Total Heap Sort Time** | **O(N log N)** | Guaranteed $O(N \log N)$ in Best, Worst, and Average cases. |
| **Space Complexity** | **O(1)** | Heap sort is an **in-place** sorting algorithm (excluding recursive stack overhead for heapify, which can be done iteratively to guarantee $O(1)$). |

> **Exam Tip:** A very common trick question is the time complexity of *building* a heap vs. *heap sort*. **Building a heap** using heapify is **$O(N)$**. The entire **Heap Sort algorithm** is **$O(N \log N)$**. Also, remember: Max Heap is used to sort in *Ascending* order, and Min Heap is used to sort in *Descending* order.
