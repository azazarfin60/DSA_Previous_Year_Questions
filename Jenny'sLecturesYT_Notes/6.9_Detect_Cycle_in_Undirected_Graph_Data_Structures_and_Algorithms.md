## 📌 Executive Summary
Detecting a cycle in an **Undirected Graph** can be accomplished using several methods (BFS, DFS, Disjoint Sets, etc.). This note focuses on using **Breadth-First Search (BFS)**. The core logic relies on using a Queue data structure and a 3-state tracking mechanism to identify if an unvisited node is reached from two different paths.

---

## 🔑 Core Concept: The 3-State Flag Array
Just like in directed graphs, a simple boolean `visited` array is not descriptive enough to robustly detect cycles. We use a `flag` variable with three states:

| Flag Value | State | Meaning in BFS |
| :---: | :--- | :--- |
| **-1** | **Unvisited** | The vertex has never been encountered. |
| **0** | **In Queue** | The vertex has been reached and placed in the Queue, but its neighbors have not been explored yet. |
| **1** | **Traversed** | The vertex has been popped from the Queue and fully explored. |

---

## ⚙️ Step-by-Step Algorithm
1. Initialize the `flag` of all vertices to `-1` (unvisited).
2. Choose an arbitrary starting node. Put it in the **Queue** and set its flag to `0`.
3. **While the Queue is not empty:**
   - **Dequeue** the front node (let's call it $U$) and change its flag to **1** (Traversed).
   - Check all **adjacent vertices** of $U$ (let's call an adjacent vertex $V$):
     - **Case A (Flag is -1):** $V$ is unvisited. Enqueue $V$ and change its flag to `0`.
     - **Case B (Flag is 1):** $V$ is completely traversed. Ignore it.
     - **Case C (Flag is 0): 🚨 CYCLE DETECTED!** 
       - *Why?* If $V$ has a flag of `0`, it means $V$ is *already* sitting in the Queue waiting to be processed. This proves that some *other* node already discovered $V$ before $U$ did. Two distinct paths reaching the same unvisited node guarantees a cycle! Return `TRUE`.
4. If the Queue becomes completely empty without ever triggering Case C, the graph contains no cycles. Return `FALSE`.

---

## 📐 Tracing the Cycle Detection

Let's assume an undirected graph with vertices A, B, C, D, E containing a cycle (B-D-E-B).

1. **Start at A:**
   - Enqueue A. `Flag[A] = 0`.
2. **Dequeue A:**
   - `Flag[A] = 1`.
   - Adjacent to A: B and C. Both are `-1`.
   - Enqueue B and C. `Flag[B] = 0`, `Flag[C] = 0`.
   - *Queue State:* `[B, C]`
3. **Dequeue B:**
   - `Flag[B] = 1`.
   - Adjacent to B: A (flag is 1 $\rightarrow$ ignore), D and E (flags are `-1`).
   - Enqueue D and E. `Flag[D] = 0`, `Flag[E] = 0`.
   - *Queue State:* `[C, D, E]`
4. **Dequeue C:**
   - `Flag[C] = 1`.
   - Adjacent to C: A (flag is 1 $\rightarrow$ ignore).
   - *Queue State:* `[D, E]`
5. **Dequeue D:**
   - `Flag[D] = 1`.
   - Adjacent to D: B (flag is 1 $\rightarrow$ ignore), **E**.
   - Check flag of E: **`Flag[E] == 0`**.
   - **🚨 CYCLE DETECTED!** Node E was put in the Queue by Node B, but Node D is also connected to E. This confirms the cycle B-D-E.

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Time Complexity** | **O(V + E)** | Worst-case scenario (no cycles), we visit every vertex ($V$) and check every edge ($E$) twice (since undirected). |
| **Space Complexity** | **O(V)** | To maintain the Queue and the `flag` array for $V$ vertices. |

> **Exam Tip:** When tracing BFS cycle detection in exams, clearly draw the Queue state at each step. Examiners specifically look for the exact moment you evaluate a node whose flag is `0`. Mentioning the explicit rule — *"If any vertex finds its adjacent vertex with flag 0, it means the graph contains a cycle"* — is a guaranteed mark scorer.
