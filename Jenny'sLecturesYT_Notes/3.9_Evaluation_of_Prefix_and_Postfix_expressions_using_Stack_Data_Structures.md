# 3.9 Evaluation of Prefix and Postfix expressions using Stack | Data Structures

## 📌 Executive Summary

This lecture explains how to **evaluate Prefix and Postfix** expressions using a stack. Postfix scans left-to-right; Prefix scans right-to-left.

---

## 📐 Postfix Evaluation

### Algorithm
1. Scan **left to right**
2. **Operand** → push to stack
3. **Operator** → pop two operands, evaluate, push result
   - **Second popped = first operand**, First popped = second operand

### Example: `5 3 + 8 2 - *`

| Symbol | Action | Stack |
|--------|--------|-------|
| 5 | push | [5] |
| 3 | push | [5, 3] |
| + | pop 3,5 → 5+3=8 | [8] |
| 8 | push | [8, 8] |
| 2 | push | [8, 8, 2] |
| - | pop 2,8 → 8-2=6 | [8, 6] |
| * | pop 6,8 → 8×6=48 | [48] |

> **Result = 48** ✅

---

## 📐 Prefix Evaluation

### Algorithm
1. Scan **right to left**
2. **Operand** → push to stack
3. **Operator** → pop two operands, evaluate, push result
   - **First popped = first operand**, Second popped = second operand

### Example: `* + 5 3 - 8 2`

| Symbol (R→L) | Action | Stack |
|-------------|--------|-------|
| 2 | push | [2] |
| 8 | push | [2, 8] |
| - | pop 8,2 → 8-2=6 | [6] |
| 3 | push | [6, 3] |
| 5 | push | [6, 3, 5] |
| + | pop 5,3 → 5+3=8 | [6, 8] |
| * | pop 8,6 → 8×6=48 | [48] |

> **Result = 48** ✅

---

## ⚠️ Operand Order Matters!

| Evaluation | First Popped | Second Popped | Operation |
|-----------|-------------|---------------|-----------|
| **Postfix** | op2 | op1 | op1 ○ op2 |
| **Prefix** | op1 | op2 | op1 ○ op2 |

For `-` and `/`, order matters! `8-2 ≠ 2-8`

---

## 💡 Key Takeaways

- Postfix: scan **L→R**, second popped is first operand
- Prefix: scan **R→L**, first popped is first operand
- Both use stack for operands only
- No precedence/associativity rules needed!
- Very common exam question — practice with examples
