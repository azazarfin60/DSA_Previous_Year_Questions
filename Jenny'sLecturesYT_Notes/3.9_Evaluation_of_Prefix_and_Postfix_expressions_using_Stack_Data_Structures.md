# 3.9 Evaluation of Prefix and Postfix Expressions | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
Once an expression is converted into Prefix or Postfix format, computers use a **Stack** to evaluate the mathematical result. Both formats are computationally efficient $O(n)$ because they do not require precedence or associativity rules. However, the algorithms differ in **scanning direction** and the **order of operands** when popping.

---

## 1️⃣ Evaluating a Prefix Expression
Prefix expressions evaluate operators *before* operands (`+ A B`).

### The Prefix Algorithm
1. **Scan Direction:** Right to Left.
2. **If Operand:** Push it onto the stack.
3. **If Operator:** Pop two elements from the stack.
   - **Crucial Rule:** 
     - The first popped element is **Operand 1** (`op1`).
     - The second popped element is **Operand 2** (`op2`).
   - Perform: `op1 <operator> op2`.
   - Push the computed result back onto the stack.
4. **End:** The final remaining element in the stack is the answer.

### Trace Example: Prefix
**Expression:** `- + 2 * 3 4 / 16 ^ 2 3`
*(Scan from Right $\leftarrow$ Left)*

| Token | Action | Stack State (Top at right) | Calculation / Logic |
| :--- | :--- | :--- | :--- |
| `3` | Push | `3` | Operand |
| `2` | Push | `3, 2` | Operand |
| `^` | Pop two | ` ` | `op1=2`, `op2=3` $\rightarrow$ `2 ^ 3 = 8` |
| | Push res | `8` | Push 8 |
| `16`| Push | `8, 16` | Operand |
| `/` | Pop two | ` ` | `op1=16`, `op2=8` $\rightarrow$ `16 / 8 = 2` |
| | Push res | `2` | Push 2 |
| `4` | Push | `2, 4` | Operand |
| `3` | Push | `2, 4, 3` | Operand |
| `*` | Pop two | `2` | `op1=3`, `op2=4` $\rightarrow$ `3 * 4 = 12` |
| | Push res | `2, 12` | Push 12 |
| `2` | Push | `2, 12, 2` | Operand |
| `+` | Pop two | `2` | `op1=2`, `op2=12` $\rightarrow$ `2 + 12 = 14` |
| | Push res | `2, 14` | Push 14 |
| `-` | Pop two | ` ` | `op1=14`, `op2=2` $\rightarrow$ `14 - 2 = 12` |
| | Push res | `12` | Final Answer: **12** |

## 2️⃣ Evaluating a Postfix Expression
Postfix expressions evaluate operators *after* operands (`A B +`).

### The Postfix Algorithm
1. **Scan Direction:** Left to Right.
2. **If Operand:** Push it onto the stack.
3. **If Operator:** Pop two elements from the stack.
   - **Crucial Rule:** 
     - The first popped element is **Operand 2** (`op2`).
     - The second popped element is **Operand 1** (`op1`).
   - Perform: `op1 <operator> op2`.
   - Push the computed result back onto the stack.
4. **End:** The final remaining element in the stack is the answer.

### Trace Example: Postfix
**Expression:** `2 3 4 * + 16 2 3 ^ / -`
*(Scan from Left $\rightarrow$ Right)*

| Token | Action | Stack State (Top at right) | Calculation / Logic |
| :--- | :--- | :--- | :--- |
| `2` | Push | `2` | Operand |
| `3` | Push | `2, 3` | Operand |
| `4` | Push | `2, 3, 4` | Operand |
| `*` | Pop two | `2` | `op2=4`, `op1=3` $\rightarrow$ `3 * 4 = 12` |
| | Push res | `2, 12` | Push 12 |
| `+` | Pop two | ` ` | `op2=12`, `op1=2` $\rightarrow$ `2 + 12 = 14` |
| | Push res | `14` | Push 14 |
| `16`| Push | `14, 16` | Operand |
| `2` | Push | `14, 16, 2` | Operand |
| `3` | Push | `14, 16, 2, 3` | Operand |
| `^` | Pop two | `14, 16` | `op2=3`, `op1=2` $\rightarrow$ `2 ^ 3 = 8` |
| | Push res | `14, 16, 8` | Push 8 |
| `/` | Pop two | `14` | `op2=8`, `op1=16` $\rightarrow$ `16 / 8 = 2` |
| | Push res | `14, 2` | Push 2 |
| `-` | Pop two | ` ` | `op2=2`, `op1=14` $\rightarrow$ `14 - 2 = 12` |
| | Push res | `12` | Final Answer: **12** |

## 🚨 Exam Warning: The Subtraction / Division Trap
If you mix up `op1` and `op2` during an exam, addition and multiplication will still give you the correct answer ($A+B = B+A$). 

However, **Subtraction and Division will fail** ($A-B \neq B-A$).

> **Memorize This Difference:**
> - **Prefix Evaluation:** First popped = `op1`. Second popped = `op2`. ($op1 - op2$)
> - **Postfix Evaluation:** First popped = `op2`. Second popped = `op1`. ($op1 - op2$)
