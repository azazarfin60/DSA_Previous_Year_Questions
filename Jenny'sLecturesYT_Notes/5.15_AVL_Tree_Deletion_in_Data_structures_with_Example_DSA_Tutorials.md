# 5.15 AVL Tree Deletion with Example | DSA Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=1s9K0X9ApeI)

---

## 📌 Executive Summary
This lecture explains how to delete nodes from an AVL tree. The core concept is that you perform a standard Binary Search Tree (BST) deletion, and then recalculate Balance Factors (BF). If the tree becomes unbalanced, you perform standard AVL rotations to restore it. 

---

## 🔑 Rules for AVL Deletion
1. **Delete as in BST:**
   - **Leaf Node (0 children):** Just delete it.
   - **1 Child:** Replace the node with its only child.
   - **2 Children:** Replace the node with its **Inorder Predecessor** (largest in left subtree) OR **Inorder Successor** (smallest in right subtree), then delete that predecessor/successor node.
2. **Check Balance Factors:** After deletion, calculate the BF of every node moving up to the root.
3. **Rotate if necessary:** If any BF is out of the `[-1, 0, 1]` range, apply the appropriate rotation (LL, RR, LR, RL) to balance it.

---

## 📐 Step-by-Step Deletion Example

We begin with the balanced AVL tree constructed in the previous lecture (5.14).

**Starting AVL Tree:**
```text
          14 
       /      \
      11       19 
     /  \     /  \
    7   12   17   53 
   / \   \   /   /  \
  4   8  13 16  20  60
```
**Deletions to perform:** `8, 7, 11, 14, 17`

### 1. Delete `8`
- `8` is a leaf node. Simply delete it.
- **Check BFs:** The tree remains balanced.
```text
          14 
       /      \
      11       19 
     /  \     /  \
    7   12   17   53 
   /     \   /   /  \
  4      13 16  20  60
```

### 2. Delete `7`
- `7` has one child (`4`).
- Replace `7` with `4`.
- **Check BFs:** The tree remains balanced.
```text
          14 
       /      \
      11       19 
     /  \     /  \
    4   12   17   53 
         \   /   /  \
        13  16  20  60
```

### 3. Delete `11` (Triggers RR Rotation)
- `11` has two children.
- **Action:** Replace with Inorder Predecessor. The left subtree of `11` only contains `4`. So, replace `11` with `4`.
```text
          14 
       /      \
     [4]       19 
        \     /  \
        12   17   53 
         \   /   /  \
        13  16  20  60
```
- **Check BFs:** Node `4` has BF = 0 - 2 = **-2**. (Unbalanced!)
- **Path:** The unbalance is caused by the right subtree `12` and its right child `13`. This is an **RR (Right-Right)** case.
- **Fix:** Perform a Left Rotation on nodes `{4, 12, 13}`. The median is `12`. `12` goes up, `4` goes left, `13` stays right.
```text
Balanced Tree:
          14 
       /      \
      12       19 
     /  \     /  \
    4   13   17   53 
             /   /  \
            16  20  60
```

### 4. Delete `14`
- `14` (the root) has two children.
- **Action:** Replace with Inorder Predecessor (largest in left subtree). The left subtree is rooted at `12`. The largest element is `13`. Replace `14` with `13`.
- **Check BFs:** The tree remains balanced.
```text
          13 
       /      \
      12       19 
     /        /  \
    4        17   53 
             /   /  \
            16  20  60
```

### 5. Delete `17`
- `17` has one child (`16`).
- Replace `17` with `16`.
- **Check BFs:** The tree remains balanced.
```text
          13 
       /      \
      12       19 
     /        /  \
    4        16   53 
                 /  \
                20  60
```

---

## 💡 Key Takeaway
Deletions in an AVL tree can cause unbalance just like insertions. Always re-check the Balance Factors going up the tree. If you replace a node with its predecessor/successor, make sure to check the BFs starting from where that predecessor/successor was *originally* removed!
