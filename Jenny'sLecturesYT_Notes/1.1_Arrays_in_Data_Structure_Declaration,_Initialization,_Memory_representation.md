## 📌 Executive Summary

This lecture introduces 1D arrays — their need, declaration syntax, initialization (compile-time vs runtime), memory representation, address calculation formula, and how elements are accessed in constant time. It also highlights array drawbacks like fixed size and same-data-type restriction.

---

## 🔑 Core Concepts

### Why Arrays?
- A single variable stores only **one value** at a time
- To store marks of 60 students → need 60 variables? Impractical!
- **Array** = store multiple values under **one variable name**

### Definition
> An array is a **fixed-size**, sequential collection of data items of the **same data type**, stored in **consecutive memory locations**.

### Types of Arrays
1. **1D Array** — single row with multiple columns
2. **2D Array** — rows × columns (matrix)
3. **Multi-dimensional Array** — 3D and above

---

## 💻 Declaration (C Language)

```c
// General Syntax
data_type array_name[size];    // size must be a CONSTANT

// Examples
int a[60];      // integer array of size 60
char b[10];     // character array of size 10
float c[5];     // float array of size 5

// INVALID declarations:
int a[];        // ❌ no size specified
int a[n];       // ❌ n is a variable (use #define N 100 instead)
```

### Using Macro for Size
```c
#define N 100
int main() {
    int a[N];   // ✅ Valid — N is a compile-time constant
}
```

---

## 💻 Initialization

### Compile-Time (Static)
```c
int a[5] = {6, 2, 4, 3, 0};   // all 5 elements specified
int a[5] = {6, 2};             // remaining filled with 0 → {6, 2, 0, 0, 0}
int a[5] = {6, 2, 4, 3, 0, 7}; // ❌ ERROR — exceeds size
```

### Runtime (Dynamic — using loops)
```c
int a[5];
printf("Enter elements of array:\n");
for (int i = 0; i < 5; i++) {
    scanf("%d", &a[i]);
}
```

## 🧠 Memory Representation

### How an `int` is Stored
- `int a = 5;` → allocates **4 bytes** (32 bits)
- 5 in binary: `00000000 00000000 00000000 00000101`
- Stored at consecutive byte addresses (e.g., 100, 101, 102, 103)

### Array Memory Layout
For `int a[5] = {6, 2, 4, 3, 0}`:

```
Index:    [0]    [1]    [2]    [3]    [4]
Value:     6      2      4      3      0
Address: 100    104    108    112    116
         ←4B→  ←4B→  ←4B→  ←4B→  ←4B→

Total memory = 5 × 4 = 20 bytes (consecutive)
```

For `char b[10] = {'H','E','L','L','O'}`:
```
Total memory = 10 × 1 = 10 bytes
Remaining 5 positions filled with '\0' (null)
```

---

## 📐 Address Calculation Formula

```
Address(a[i]) = Base_Address + i × sizeof(data_type)
```

**Example:** Find address of `a[2]` where base address = 100, int = 4 bytes:
```
Address(a[2]) = 100 + 2 × 4 = 108  ✅
```

> **This enables Random Access** — any element accessed in **O(1)** time using just one addition and one multiplication.

## ⚠️ Drawbacks of Arrays

| Drawback | Explanation |
|----------|-------------|
| **Fixed size** | Size declared at compile time cannot change at runtime |
| **Same data type only** | Cannot mix int and char in same array |
| **Wastage of space** | Declaring size 100 but using only 10 → 90 wasted |
| **Costly resizing** | To expand: allocate new larger block → copy old data → expensive |

---

## 💡 Key Takeaways

- Array index starts from **0** (goes 0 to n-1)
- Elements stored in **contiguous memory locations**
- **Random access** in O(1) time using address formula
- Fixed size is the main limitation → leads to **Linked Lists** (covered later)
- **Time Complexity of access: O(1)**
