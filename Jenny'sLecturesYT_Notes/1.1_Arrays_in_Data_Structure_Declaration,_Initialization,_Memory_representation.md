## 📌 Executive Summary
An Array is a linear data structure consisting of a collection of elements of the **same data type**, stored in **contiguous (continuous) memory locations**. 

Because memory is allocated contiguously, arrays allow for **Random Access** in $O(1)$ time, provided you know the base address and the index of the element.

**Key Properties:**
- **Fixed Size:** The size must be declared at compile time and cannot be changed at runtime (in C).
- **Homogeneous:** All elements must be of the identical data type.
- **No Bound Checking:** In C, there is no automatic upper-bound checking. The programmer must write code to prevent overflow.

---

## 📝 Declaration & Initialization

### Declaration
You must specify the data type, the name, and the size (which must be a constant).
```c
int A[5]; // Declares an integer array of size 5
```

### Initialization
Arrays can be initialized at **Compile Time** or **Run Time**.
- **Compile Time:**
  ```c
  int A[5] = {6, 2, 4, 3, 0};
  ```
- **Run Time:** Populated using a loop and `scanf`.
  ```c
  for(int i = 0; i < 5; i++) {
      scanf("%d", &A[i]);
  }
  ```

## 🧮 Memory Representation & Address Calculation

To access an element $A[i]$, the compiler calculates its exact memory address using a simple mathematical formula.

**Formula:**
$$ \text{Address of } A[i] = \text{Base\_Address} + (i \times \text{Size\_of\_Data\_Type}) $$

**Example:**
Suppose an integer array `int A[5]` has a Base Address of `100`. Assuming an integer takes `4 bytes`.
- Address of `A[0]` = $100 + (0 \times 4) = 100$
- Address of `A[2]` = $100 + (2 \times 4) = 108$

> [!TIP]
> This formula is why array indexing usually starts at `0`. If indexing started at `1`, the formula would require an extra subtraction operation $(i - 1)$, slightly slowing down calculation.
