# 4.7 Introduction to DEQUE (Double Ended Queue) | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
A **DEQUE** (pronounced "deck"), or Double Ended Queue, is a generalized version of a queue. 
Unlike a standard queue that strictly enforces FIFO (insert at rear, delete at front), a Deque allows **both insertion and deletion at BOTH ends** (Front and Rear).

Because of this flexibility, a Deque can emulate both a Stack (LIFO) and a Queue (FIFO):
- **To use as a Stack:** Restrict insertion and deletion to the *same* end.
- **To use as a Queue:** Restrict insertion to one end and deletion to the *opposite* end.

---

## 🗂️ Types of DEQUE

There are two primary variations of a Deque based on applied restrictions:

### 1. Input Restricted Deque
- **Rule:** Insertion is restricted to **only one end**, but deletion is allowed from **both ends**.
- *Example:* You can only insert at the Rear, but you can delete from both the Front and the Rear.

### 2. Output Restricted Deque
- **Rule:** Deletion is restricted to **only one end**, but insertion is allowed at **both ends**.
- *Example:* You can only delete from the Front, but you can insert at both the Front and the Rear.

---

## ⚙️ Standard Operations

A fully functional Deque supports the following core operations, all of which must execute in **$O(1)$ time complexity**:

1. `insertFront()`: Add an element to the front.
2. `insertRear()`: Add an element to the rear (Standard Enqueue).
3. `deleteFront()`: Remove an element from the front (Standard Dequeue).
4. `deleteRear()`: Remove an element from the rear.

*Auxiliary Operations:* `getFront()`, `getRear()`, `isFull()`, `isEmpty()`.

---

## 💾 Memory Representation

To achieve $O(1)$ time complexity for all four core operations, a Deque is practically implemented using either:
1. **Circular Arrays:** (Using modulo arithmetic to wrap pointers around the array bounds).
2. **Doubly Linked Lists:** (Allowing bidirectional $O(1)$ pointer traversal at the tail).

---

## 🚀 Real-World Applications

> [!TIP]
> **Exam Advantage:** 
> Memorizing these applications, particularly the scheduling algorithm, makes for a high-scoring theory answer.

1. **Undo / Redo Operations:** Software applications use Deques to store recent actions.
2. **Palindrome Checker:** A word can be read from both the front and the rear simultaneously to check for symmetry (e.g., "radar").
3. **Multiprocessor Scheduling (A-Steal Algorithm):** 
   - Each processor in a multi-core system maintains its own Deque of threads (jobs).
   - A processor normally executes threads from the **front** of its own Deque.
   - If a processor finishes all its threads (its Deque becomes empty), it will look at another busy processor's Deque and "steal" a thread from the **rear** to execute. This balances the load efficiently.
