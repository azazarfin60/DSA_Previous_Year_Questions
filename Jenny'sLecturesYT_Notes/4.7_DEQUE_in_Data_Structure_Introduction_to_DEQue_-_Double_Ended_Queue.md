## 📌 Executive Summary

This lecture introduces the **Deque (Double-Ended Queue)** — a generalization of both Stack and Queue that allows insertion and deletion from **both front and rear**.

---

## 🔑 Definition
> A Deque allows insert and delete at **both ends** — front and rear.

```
Front ← [Insert/Delete] ←→ [Insert/Delete] → Rear
```

---

## 📊 Types of Deque

| Type | Front | Rear |
|------|-------|------|
| **Input-Restricted** | Delete only | Insert + Delete |
| **Output-Restricted** | Insert + Delete | Insert only |
| **Unrestricted** | Insert + Delete | Insert + Delete |

---

## 💻 Operations
```c
void insertFront(int data);    // Insert at front
void insertRear(int data);     // Insert at rear
int deleteFront();              // Remove from front
int deleteRear();               // Remove from rear
```

---

## 🎯 Deque as Stack and Queue

| Behavior | Operations Used |
|----------|----------------|
| **Stack** (LIFO) | insertFront + deleteFront |
| **Queue** (FIFO) | insertRear + deleteFront |

> Deque is more **general** — can behave as both Stack and Queue!

---

## 💻 Circular Array Implementation (Key Formulas)
```
Next position = (current + 1) % MAX
Prev position = (current - 1 + MAX) % MAX
```

---

## 💡 Key Takeaways

- Deque = **Double-Ended Queue** — insert/delete at both ends
- Can simulate both Stack and Queue
- Two restricted types: input-restricted and output-restricted
- Implemented using **circular array** for space efficiency
- All operations: **O(1)**
