## 📐 B-Tree Deletion — 3 Cases

### Case 1: Key in Leaf Node
- If leaf has **more than minimum** keys → simply delete
- If leaf has **exactly minimum** → borrow from sibling or merge

### Case 2: Key in Internal Node
- Replace with **inorder predecessor** (max of left subtree) or **inorder successor** (min of right subtree)
- Then delete the replacement from its leaf position

### Case 3: Underflow After Deletion
- **3a:** Sibling has extra key → **borrow** through parent (rotation)
- **3b:** Sibling also has minimum → **merge** with sibling through parent key

## 📐 Borrow (Rotation):
```
Before:            After (borrow from left sibling):
  [... P ...]       [... R ...]
   /     \           /     \
 [Q,R]   [S]      [Q]    [P,S]
```
Parent key comes down, sibling key goes up.

## 📐 Merge:
```
Before:                After:
  [... P ...]          [...]
   /     \               |
 [Q]     [R]          [Q,P,R]
```
Merge siblings + parent key → may cause parent underflow → propagate up.

## ⏱️ Time: O(log n). Height may decrease if root underflows.
