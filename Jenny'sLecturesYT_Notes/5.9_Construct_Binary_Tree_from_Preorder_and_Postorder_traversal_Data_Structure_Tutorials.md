# 5.9 Construct Binary Tree from Preorder and Postorder traversal | Data Structure Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=5lWJpTEnyow)

---

## 📌 Executive Summary

This lecture shows constructing a binary tree from **Preorder and Postorder** — possible **only for Full Binary Trees** (every node has 0 or 2 children). Without inorder, the tree may not be unique.

---

## 📐 Algorithm (Full Binary Tree Only)

1. First of Preorder = **Root**
2. **Second** of Preorder = Root of **left subtree**
3. Find that element in Postorder → all elements **up to and including** it = left subtree
4. Remaining elements = right subtree
5. Recurse

---

## 📐 Example

```
Preorder:  A B D E C F G
Postorder: D E B F G C A
```

### Step 1: Root = **A** (first of preorder)

### Step 2: Left root = **B** (second of preorder)
Find B in postorder at index 2 → left = {D, E, B}, right = {F, G, C}

### Step 3: Recurse on left {Preorder: B D E, Postorder: D E B}
Root = B, left root = D → D found at index 0 → left={D}, right={E}

### Step 4: Recurse on right {Preorder: C F G, Postorder: F G C}
Root = C, left root = F → F found at index 0 → left={F}, right={G}

### Result:
```
       A
      / \
     B   C
    / \ / \
   D  E F  G
```

---

## ⚠️ Limitation

| Condition | Can construct unique tree? |
|-----------|---------------------------|
| Full Binary Tree | ✅ Yes |
| Non-full tree | ❌ No — multiple trees possible |
| Inorder + Preorder | ✅ Always unique |
| Inorder + Postorder | ✅ Always unique |

---

## 💡 Key Takeaways

- Without Inorder, need **Full Binary Tree** guarantee
- Second element of Preorder = left subtree root
- Find it in Postorder to split left/right
- If not full binary tree → ambiguous result
