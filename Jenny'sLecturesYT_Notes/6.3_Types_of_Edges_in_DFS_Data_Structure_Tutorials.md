## 📌 Executive Summary
When a **Depth First Search (DFS)** is performed on a **Directed Graph**, every edge in the graph can be uniquely classified into one of four categories based on the discovery/finish timestamps of the nodes. This classification is powerful for analyzing graph properties, such as detecting cycles.

---

## ⏱️ The Concept of Time in DFS
To classify edges, DFS assigns two timestamps to every node:
1. **Starting Time (Discovery Time):** The time when the node is first visited/discovered.
2. **Finishing Time:** The time when the algorithm has completely explored all adjacent nodes of that vertex and is forced to backtrack.

*Note: In the transcript trace, time simply increments by 1 every time a node is discovered or finished.*

---

## 🗂️ The Four Types of Edges

Let us analyze an edge **$E = X \rightarrow Y$** (an edge from node $X$ to node $Y$).

### 1. Tree Edge
- **Definition:** These are the edges that are actively traversed during the DFS to discover *unvisited* nodes. 
- **Property:** If node $Y$ was unvisited when the edge $X \rightarrow Y$ was explored, it is a Tree Edge. These edges form the "DFS Tree" (or DFS Forest).

### 2. Forward Edge
- **Definition:** An edge $X \rightarrow Y$ that connects a node $X$ to one of its **descendants** $Y$ in the DFS tree, but is **NOT** a Tree Edge itself.
- **Timestamp Check:** `start_time(X) < start_time(Y)`
  - *Logical Meaning:* $X$ was discovered *before* $Y$.
- **Path Check:** There exists a path from $X$ to $Y$ consisting exclusively of Tree Edges.

### 3. Back Edge (Backward Edge)
- **Definition:** An edge $X \rightarrow Y$ that connects a node $X$ to one of its **ancestors** $Y$ in the DFS tree.
- **Timestamp Check:** `start_time(Y) < start_time(X)`
  - *Logical Meaning:* $Y$ was discovered *before* $X$ (and $Y$ has not finished yet).
- **Path Check:** There exists a path from $Y$ to $X$ consisting exclusively of Tree Edges.
- **🔥 Critical Application:** A directed graph contains a **Cycle** if and only if it contains at least one Back Edge.

### 4. Cross Edge
- **Definition:** An edge $X \rightarrow Y$ that connects two nodes where **neither is an ancestor or descendant** of the other. It can connect nodes in the same DFS tree or across different DFS trees in a forest.
- **Timestamp Check:** `start_time(Y) < start_time(X)`
  - *Logical Meaning:* $Y$ was discovered *before* $X$.
- **Path Check:** There is **NO path** from $Y$ to $X$ via Tree Edges.

---

## 🔍 Step-by-Step Edge Classification Logic

If you are given an edge $X \rightarrow Y$ and asked to classify it (assuming it's not a Tree Edge):

1. **Check Discovery Times:**
   - If $X$ was discovered *before* $Y$ $\implies$ **Forward Edge**.
   - If $Y$ was discovered *before* $X$, move to Step 2.
2. **Check for Ancestry (Path):**
   - Is there a path of Tree Edges from $Y$ to $X$?
   - **YES:** It is a **Back Edge**.
   - **NO:** It is a **Cross Edge**.

---

## 🧮 The Union Property
An important mathematical property of DFS Edge Classification is that every single edge in the directed graph belongs to exactly one of these four sets. Therefore:
> **$\text{Tree Edges} \cup \text{Forward Edges} \cup \text{Back Edges} \cup \text{Cross Edges} = \text{Total Edges in the Graph}$**

> **Exam Tip:** If a question asks you to classify edges, start by drawing the DFS tree with solid lines (these are the Tree Edges). Write down the sequence in which nodes were visited. Then, map the remaining edges as dotted lines and apply the Ancestor/Descendant logic. Always mention that **Back Edges = Cycles**!
