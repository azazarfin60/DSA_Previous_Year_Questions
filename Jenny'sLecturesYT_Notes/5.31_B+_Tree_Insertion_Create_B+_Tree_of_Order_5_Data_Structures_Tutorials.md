## 📌 Executive Summary

This lecture provides a **detailed worked example** of constructing a B+ Tree of **Order 5** — reinforcing the copy-up (leaf) vs move-up (internal) split rules.

---

## 📐 Order 5 Constraints

| Node Type | Max Keys | Min Keys | Max Children |
|-----------|---------|---------|-------------|
| Internal | 4 | 2 | 5 |
| Leaf | 4 | 2 | — |
| Root | 4 | 1 | 5 |

---

## 📐 Example: Insert 5, 8, 1, 7, 3, 12, 9, 6, 20, 15, 25

```
Insert 5, 8, 1, 7: [1, 5, 7, 8]  (4 keys — full)

Insert 3: Overflow → Split leaf
          Middle = 5 (3rd element for order 5)
          Copy 5 up

              [5]
             /   \
        [1, 3]   [5, 7, 8]

Insert 12:    [5]
             /   \
        [1, 3]   [5, 7, 8, 12]  (4 keys — full)

Insert 9: Overflow! Split right leaf
          Sorted: [5, 7, 8, 9, 12] → middle = 8, copy up

              [5, 8]
             /  |   \
        [1,3] [5,7] [8, 9, 12]

Insert 6:     [5, 8]
             /  |   \
        [1,3] [5,6,7] [8, 9, 12]

Insert 20:    [5, 8]
             /  |   \
        [1,3] [5,6,7] [8, 9, 12, 20]

Insert 15:  Right leaf overflow! [8,9,12,15,20]
            Middle=12, copy up

              [5, 8, 12]
             /  |   |    \
        [1,3] [5,6,7] [8,9] [12,15,20]

Insert 25:    [5, 8, 12]
             /  |   |    \
        [1,3] [5,6,7] [8,9] [12,15,20,25]
```

All leaf nodes linked: `[1,3] → [5,6,7] → [8,9] → [12,15,20,25]`

---

## 📐 Split Rules Recap

```
Leaf Split (Order 5):
  [a, b, c, d, e] → Left: [a, b]  Right: [c, d, e]
  Copy c to parent

Internal Split (Order 5):
  [a, b, c, d, e] → Left: [a, b]  Right: [d, e]
  Move c to parent (c NOT in either child)
```

---

## 💡 Key Takeaways

- Order 5: max 4 keys per node, split at 3rd element
- Leaf split → **COPY** middle; Internal split → **MOVE** middle
- Practice with different orders (3, 4, 5) — exam may use any
- All leaves at same depth, linked for range queries
