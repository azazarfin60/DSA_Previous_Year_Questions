# 2.14 Circular Linked List: Creation and Display | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
A **Circular Linked List (CLL)** is a variation of a singly linked list where the `next` pointer of the last node points back to the `head` node rather than `NULL`. This creates a closed loop. The node structure is exactly the same as a Singly Linked List; the only difference is how the final link is managed and how the list is traversed without falling into an infinite loop.

---

## 🛠️ Phase 1: Creation

Creating a Circular Linked List is 99% identical to creating a Singly Linked List. You dynamically allocate memory using `malloc()` in a loop and attach nodes. The crucial difference happens *after* the loop ends.

### The Circular Link
In a Singly Linked List, the last node's `next` remains `NULL`. In a Circular Linked List, you must connect the last node back to the `head`.

### C Code Snippet
```c
struct node {
    int data;
    struct node *next;
};
struct node *head = NULL;

void createCLL() {
    struct node *newnode, *temp;
    int choice = 1;

    while (choice) {
        newnode = (struct node*) malloc(sizeof(struct node));
        printf("Enter data: ");
        scanf("%d", &newnode->data);
        newnode->next = NULL;

        if (head == NULL) {
            head = temp = newnode;
        } else {
            temp->next = newnode;
            temp = newnode;
        }
        printf("Continue? (0,1): ");
        scanf("%d", &choice);
    }

    // 🌟 THE CRITICAL CIRCULAR LINK 🌟
    // After the loop, 'temp' points to the very last node created.
    // Point its 'next' back to 'head' to form the circle.
    if (head != NULL) {
        temp->next = head; 
    }
}
```

---

## 🛠️ Phase 2: Display / Traversal

Because there is no `NULL` in a Circular Linked List, you cannot use the standard `while (temp != NULL)` traversal loop. If you do, your program will print the list endlessly in an infinite loop.

Instead, we must stop traversing when we recognize we are looking at the `head` again. The correct condition is `while (temp->next != head)`.

### C Code Snippet

```c
void displayCLL() {
    struct node *temp;
    
    if (head == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    temp = head;
    
    // Traverse until the NEXT node is the head
    while (temp->next != head) {
        printf("%d -> ", temp->data);
        temp = temp->next;
    }
    
    // Print the very last node's data
    printf("%d\n", temp->data); 
}
```

> [!CAUTION]
> **Exam Trap (The "Off-By-One" Printing Error):**
> Students often write the `while(temp->next != head)` loop perfectly, but fail to realize what happens when the loop terminates. 
> When `temp` reaches the last node, `temp->next == head` evaluates to true, so the loop exits **BEFORE** printing the data of the last node! 
> You **MUST** write one extra `printf("%d", temp->data);` immediately *outside* and *after* the `while` loop to print the final element. Failing to do so will cost you marks for incomplete output!
