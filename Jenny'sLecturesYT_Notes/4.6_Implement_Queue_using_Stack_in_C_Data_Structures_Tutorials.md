## 📌 Executive Summary
A Queue follows the **FIFO** (First In First Out) principle, while a Stack follows the **LIFO** (Last In First Out) principle. 

To simulate a Queue, we require **two Stacks (`S1` and `S2`)**.

There are two primary approaches to implementing this:
| Approach | Enqueue Time Complexity | Dequeue Time Complexity |
| :--- | :--- | :--- |
| **Method 1 (Enqueue is Costly)** | $O(n)$ | $O(1)$ |
| **Method 2 (Dequeue is Costly)** | **$O(1)$** | **$O(n)$** |

*In this tutorial, we focus on **Method 2: Making the Dequeue operation costly.***

---

## 🛠️ The Strategy (Method 2)

We will use `S1` as our primary storage. `S2` will act purely as a temporary, supportive stack during the dequeue operation to help us reverse the order of elements. We also maintain a `count` variable to track how many elements are currently in our simulated queue.

### Enqueue Operation ($O(1)$)
Simply push the new element onto `S1` and increment `count`.

### Dequeue Operation ($O(n)$)
Because we want to remove the *first* element inserted (which is currently sitting at the very bottom of `S1`), we must:
1. Pop all elements from `S1` and push them onto `S2`. (The order is now reversed, and the element we want is at the top of `S2`).
2. Pop the top element from `S2`. This is our dequeued data.
3. Decrement `count`.
4. Pop all remaining elements from `S2` and push them back onto `S1`.

## 💻 C Code Implementation

### 1. Global Setup
```c
#define N 5
int S1[N], S2[N];
int top1 = -1, top2 = -1;
int count = 0; // Tracks total elements in the queue

// Assume standard stack operations exist: 
// push1(x), push2(x), pop1(), pop2()
```

### 2. Enqueue Function
```c
void enqueue(int x) {
    push1(x); // Push data directly into S1
    count++;  // Increment the total number of elements
}
```

### 3. Dequeue Function
```c
void dequeue() {
    int i, a, b;
    
    if (top1 == -1 && top2 == -1) {
        printf("Queue is empty\n");
    } else {
        // Step 1: Transfer all elements from S1 to S2
        for (i = 0; i < count; i++) {
            a = pop1();
            push2(a);
        }
        
        // Step 2: Pop the target element from S2
        b = pop2();
        printf("Dequeued element: %d\n", b);
        
        // Step 3: Decrement count
        count--;
        
        // Step 4: Transfer remaining elements back to S1
        for (i = 0; i < count; i++) {
            a = pop2();
            push1(a);
        }
    }
}
```

> [!TIP]
> **Exam Advantage (Understanding the Code vs Theory):**
> You can write the transfer logic in a single line: `push2(pop1());`. This pops the value from `S1` and immediately passes it as the argument to the `push2()` function.

### 4. Display Function ($O(n)$)

Because we always push elements back to `S1` after a dequeue operation, `S1` is our definitive storage. To display the queue, we simply print the elements of `S1` starting from index 0 up to `top1`.

```c
void display() {
    if (top1 == -1) {
        printf("Queue is empty\n");
    } else {
        printf("Queue elements: ");
        for (int i = 0; i <= top1; i++) {
            printf("%d ", S1[i]);
        }
        printf("\n");
    }
}
```

> [!WARNING]
> **Exam Trap (The `count` loop):**
> Notice that the display loop condition is `i <= top1`, not `i < count`. If you use `count` while printing an array-based stack directly, and your indices are not perfectly aligned with the count (e.g., due to array structuring or specific implementations), you risk an out-of-bounds error. Always iterate based on the structural boundary (`top1`).
