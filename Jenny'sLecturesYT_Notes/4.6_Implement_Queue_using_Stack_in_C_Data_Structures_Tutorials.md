## 📌 Executive Summary

This lecture shows how to implement a **Queue using two Stacks**. Two methods: costly enqueue or costly dequeue. The costly dequeue method is more common and gives **amortized O(1)** dequeue.

---

## 📐 Method: Costly Dequeue

### Idea
- **s1** for enqueue (just push)
- **s2** for dequeue (transfer from s1 when s2 is empty)

```c
// Enqueue: O(1)
void enqueue(int data) {
    push(s1, data);
}

// Dequeue: Amortized O(1)
int dequeue() {
    if (isEmpty(s2)) {
        while (!isEmpty(s1))
            push(s2, pop(s1));     // Transfer all
    }
    return pop(s2);
}
```

---

## 🔄 Trace
```
enqueue(1), enqueue(2), enqueue(3):
  s1: [1, 2, 3]   s2: []

dequeue():
  s2 empty → transfer: s1→s2 → s2: [3, 2, 1]   s1: []
  pop(s2) → returns 1 ✅ (FIFO!)

dequeue():
  s2 not empty → pop(s2) → returns 2 ✅

enqueue(4):
  push(s1, 4) → s1: [4]   s2: [3]

dequeue():
  s2 not empty → pop(s2) → returns 3 ✅
```

---

## ⏱️ Complexity

| Operation | Time |
|-----------|------|
| Enqueue | O(1) |
| Dequeue | **Amortized O(1)**, Worst case O(n) |

---

## 💡 Key Takeaways

- Use **two stacks**: s1 for push, s2 for pop
- Transfer s1→s2 only when s2 is **empty**
- LIFO (stack) → FIFO (queue) by **reversing** through transfer
- Each element is moved at most **twice** → amortized O(1)
