## 📌 Executive Summary
Selection Sort is a highly intuitive, comparison-based sorting algorithm. Its fundamental logic is simple: repeatedly *select* the minimum element from the unsorted portion of the list and place it at the beginning. While simple to write, it is inefficient for large datasets, performing at $O(n^2)$ regardless of the input's initial order.

---

## 🔑 Core Concept
1. **Divide the Array:** Conceptually divide the array into two subarrays:
   - A **Sorted Subarray** (initially empty).
   - An **Unsorted Subarray** (initially the entire array).
2. **The Algorithm (Passes):**
   - Trace the *unsorted* subarray to find the **minimum element**.
   - **Swap** this minimum element with the element located at the *beginning* of the unsorted subarray.
   - The boundary of the sorted subarray extends one element to the right.
3. **Number of Passes:** To sort an array of `n` elements, you need exactly `n - 1` passes. (Once `n-1` elements are sorted, the final element is naturally in its correct place).

---

## 📐 Step-by-Step Trace

**Given Array:** `A = [7, 4, 10, 8, 3, 1]` *(n = 6)*
- Initial Sorted: `[]`
- Initial Unsorted: `[7, 4, 10, 8, 3, 1]`

### Pass 1
- **Unsorted subarray:** `[7, 4, 10, 8, 3, 1]`
- **Find Min:** The minimum element is `1`.
- **Swap:** Swap `1` with the first element of the unsorted subarray (`7`).
- **Array:** `[1 | 4, 10, 8, 3, 7]` *(Sorted | Unsorted)*

### Pass 2
- **Unsorted subarray:** `[4, 10, 8, 3, 7]`
- **Find Min:** The minimum element is `3`.
- **Swap:** Swap `3` with the first unsorted element (`4`).
- **Array:** `[1, 3 | 10, 8, 4, 7]`

### Pass 3
- **Unsorted subarray:** `[10, 8, 4, 7]`
- **Find Min:** The minimum element is `4`.
- **Swap:** Swap `4` with the first unsorted element (`10`).
- **Array:** `[1, 3, 4 | 8, 10, 7]`

### Pass 4
- **Unsorted subarray:** `[8, 10, 7]`
- **Find Min:** The minimum element is `7`.
- **Swap:** Swap `7` with the first unsorted element (`8`).
- **Array:** `[1, 3, 4, 7 | 10, 8]`

### Pass 5
- **Unsorted subarray:** `[10, 8]`
- **Find Min:** The minimum element is `8`.
- **Swap:** Swap `8` with the first unsorted element (`10`).
- **Final Array:** `[1, 3, 4, 7, 8, 10]`
*(The 6th element, 10, is now automatically in its correct position).*

## 💻 C Code Implementation

```c
void selectionSort(int a[], int n) {
    int i, j, min, temp;
    
    // Outer loop for the passes (n-1 passes)
    for(i = 0; i < n - 1; i++) {
        
        min = i; // Assume the first element of the unsorted array is the minimum
        
        // Inner loop to find the actual minimum element in the unsorted array
        for(j = i + 1; j < n; j++) {
            if(a[j] < a[min]) {
                min = j; // Update the index of the minimum element
            }
        }
        
        // If the minimum element is not the one we initially assumed, swap them
        if(min != i) {
            temp = a[i];
            a[i] = a[min];
            a[min] = temp;
        }
    }
}
```

## ⏱️ Complexity Analysis

| Case | Complexity | Reason |
| :--- | :--- | :--- |
| **Worst Case** | **O(n²)** | The inner loop must always scan the entire remaining unsorted subarray to find the minimum element, resulting in $\approx \frac{n(n-1)}{2}$ comparisons. |
| **Best Case** | **O(n²)** | Even if the array is already perfectly sorted, the algorithm doesn't know it. It will *still* blindly scan the entire unsorted subarray every pass to verify the minimum. |
| **Average Case** | **O(n²)** | General unsorted array. |
| **Space Complexity**| **O(1)** | Sorts in-place; only requires variables for index tracking and swapping. |

> **Exam Tip:** The main defining characteristic of Selection Sort is that its Best Case time complexity is **$O(n^2)$**, unlike Bubble or Insertion sort which can be optimized to $O(n)$ for sorted arrays. However, Selection Sort makes the **minimum possible number of swaps** ($O(n)$ swaps), making it useful when memory writing operations are highly expensive.
