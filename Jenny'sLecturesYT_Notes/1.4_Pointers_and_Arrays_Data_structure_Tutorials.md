# 1.4 Pointers and Arrays | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
Understanding how arrays interact with pointers is critical for memory management and parameter passing in C. Internally, the compiler treats array operations as pointer arithmetic.

An array name by itself (e.g., `A`) acts as a **constant pointer** to the first element of the array. It inherently holds the **Base Address** of the memory block.

---

## 🧠 Array & Pointer Equivalencies

Because array indexing is just pointer arithmetic under the hood, the following syntax variations are mathematically identical and will yield the exact same value.

If you have `int A[5];` and a pointer `int *P = A;`:

**To get the Address of the $i$-th element:**
- `&A[i]`
- `A + i`
- `P + i`

**To get the Value of the $i$-th element:**
- `A[i]`
- `*(A + i)`
- `*(i + A)`
- `i[A]` *(Yes, `2[A]` is valid C syntax and perfectly returns `A[2]`!)*
- `P[i]`
- `*(P + i)`

---

## ⚠️ Key Differences: Array Name vs. Pointer Variable

While `A` and `P` act very similarly, there is a fundamental difference in memory.

1. **Reassignment / Incrementing:**
   - A pointer variable is a true variable. You **can** do `P++` or `P = &another_var`.
   - An array name is a constant pointer. It is bound to its allocated memory block. You **CANNOT** do `A++` or reassign `A`. This will throw a compile-time error.

2. **The `&A` Trap (Exam Trap!):**
   Both `A` and `&A` will print the exact same base memory address (e.g., `100`). However, pointer arithmetic treats them differently based on their *type scale*:
   - `A + 1`: Points to the **next element**. Adds `sizeof(int)`. (Address becomes `104`).
   - `&A + 1`: Points to the memory immediately following the **entire array block**. If the array has 5 integers ($5 \times 4 = 20$ bytes), `&A + 1` skips the whole array. (Address jumps to `120`).
