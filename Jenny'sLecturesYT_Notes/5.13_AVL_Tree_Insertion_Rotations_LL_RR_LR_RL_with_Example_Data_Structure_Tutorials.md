# 5.13 AVL Tree - Insertion, Rotations (LL, RR, LR, RL) | Data Structure Tutorials

ЁЯУ║ [Watch on YouTube](https://www.youtube.com/watch?v=YWqla0UX-38)

---

## ЁЯУМ Executive Summary
This lecture introduces AVL trees, which are self-balancing Binary Search Trees. It defines the Balance Factor and explains the four types of rotations (Left, Right, Right-Left, Left-Right) required to maintain balance during insertion, using a simple alphabetic example (`x, y, z`).

---

## ЁЯФС Core Concepts
1. **AVL Tree is a BST:** All left elements are smaller, right elements are greater. Duplicate elements are not allowed.
2. **Balance Factor (BF):** 
   > `BF = Height(Left Subtree) - Height(Right Subtree)`
3. **Balancing Condition:** For *every* node, the BF must be **`-1`, `0`, or `1`**. If any node hits `-2` or `2`, the tree is unbalanced and must be fixed immediately via rotations.

---

## ЁЯУР The Four Rotation Cases

Jenny uses an alphabetic example where `x < y < z`.

### Case 1: Right-Right (RR) Insertion тЖТ Left Rotation
**Insert Order:** `x`, `y`, `z`

1. Insert `x` (BF=0).
2. Insert `y` (right of `x`). BF(x) = 0 - 1 = -1. OK.
3. Insert `z` (right of `y`). 
   - BF(y) = 0 - 1 = -1.
   - BF(x) = 0 - 2 = **-2** (Unbalanced!)
```text
Unbalanced Tree (RR):
    x  (-2)
     \
      y  (-1)
       \
        z  (0)
```
**Fix:** The unbalance is due to two Right insertions. Perform a **Left Rotation** on the median element (`y`). `y` goes up, `x` is pulled down to the left.
```text
Balanced Tree:
      y  (0)
     / \
   x(0) z(0)
```

### Case 2: Left-Left (LL) Insertion тЖТ Right Rotation
**Insert Order:** `z`, `y`, `x`

1. Insert `z` (BF=0).
2. Insert `y` (left of `z`). BF(z) = 1 - 0 = 1. OK.
3. Insert `x` (left of `y`). 
   - BF(z) = 2 - 0 = **2** (Unbalanced!)
```text
Unbalanced Tree (LL):
        z  (2)
       /
      y  (1)
     /
    x  (0)
```
**Fix:** Unbalance due to two Left insertions. Perform a **Right Rotation**. `y` goes up, `z` is pulled down to the right.
```text
Balanced Tree:
      y  (0)
     / \
   x(0) z(0)
```

### Case 3: Right-Left (RL) Insertion тЖТ Right-Left Rotation
**Insert Order:** `x`, `z`, `y`

1. Insert `x` (BF=0).
2. Insert `z` (right of `x`). BF(x) = -1.
3. Insert `y` (left of `z`, since `y < z` and `y > x`).
   - BF(x) = 0 - 2 = **-2** (Unbalanced!)
```text
Unbalanced Tree (RL):
    x  (-2)
     \
      z  (1)
     /
    y  (0)
```
**Fix:** Unbalance due to Right then Left. Requires **Two Rotations**.
1. **Right Rotation** on the bottom two nodes (`z` and `y`). Pull `z` down, `y` goes up.
```text
Intermediate (RR):
    x  (-2)
     \
      y  (-1)
       \
        z  (0)
```
2. **Left Rotation** on the whole structure (just like Case 1).
```text
Balanced Tree:
      y  (0)
     / \
   x(0) z(0)
```

### Case 4: Left-Right (LR) Insertion тЖТ Left-Right Rotation
**Insert Order:** `z`, `x`, `y`

1. Insert `z` (BF=0).
2. Insert `x` (left of `z`). BF(z) = 1.
3. Insert `y` (right of `x`).
   - BF(z) = 2 - 0 = **2** (Unbalanced!)
```text
Unbalanced Tree (LR):
        z  (2)
       /
      x  (-1)
       \
        y  (0)
```
**Fix:** Unbalance due to Left then Right. Requires **Two Rotations**.
1. **Left Rotation** on the bottom two nodes (`x` and `y`). Pull `x` down, `y` goes up.
```text
Intermediate (LL):
        z  (2)
       /
      y  (1)
     /
    x  (0)
```
2. **Right Rotation** on the whole structure (just like Case 2).
```text
Balanced Tree:
      y  (0)
     / \
   x(0) z(0)
```

## ЁЯТб Key Takeaways
- The median element always becomes the local root after balancing.
- **LL** unbalance requires a single **Right** rotation.
- **RR** unbalance requires a single **Left** rotation.
- **LR** requires **Left** then **Right** rotation.
- **RL** requires **Right** then **Left** rotation.
