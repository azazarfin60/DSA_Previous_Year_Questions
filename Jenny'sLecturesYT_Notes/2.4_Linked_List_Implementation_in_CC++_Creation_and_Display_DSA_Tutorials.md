# 2.4 Linked List Implementation (Creation & Display) | Data Structures

---

## 📌 Executive Summary
Implementing a Linked List in C requires dynamic memory allocation using `malloc()`. We must define our own node structure and carefully maintain a minimum of two pointers: `head` (which strictly anchors the start of the list) and `temp` (which is used to traverse the list or keep track of the last node during creation).

---

## 🛠️ The 3 Essential Components

### 1. Defining the Node Structure
Because a linked list node contains two different data types, we use a C `struct`.

```c
struct node {
    int data;               // Stores the actual value
    struct node *next;      // Pointer to the next node
};
```

### 2. The Required Pointers
To manage the list without losing memory references, we need three pointers of type `struct node*`:
- `head`: Always points to the very first node. **Never move this pointer** during traversal, or you will lose access to your list!
- `newnode`: Temporarily holds the address of the newly allocated node.
- `temp`: A multi-purpose pointer used to hold the address of the *last* node during list creation, or used to step through the list during traversal.

### 3. Dynamic Memory Allocation
We use `malloc()` to allocate exact memory for one node at runtime.
```c
// malloc returns a void pointer, so we typecast it to (struct node*)
newnode = (struct node*) malloc(sizeof(struct node));
```

---

## 💻 Full C Implementation: Create and Display

Below is the standard, exam-ready implementation for creating a linked list dynamically based on user input, and then printing it.

```c
#include <stdio.h>
#include <stdlib.h>

// 1. Define the structure
struct node {
    int data;
    struct node *next;
};

int main() {
    struct node *head = NULL, *newnode, *temp;
    int choice = 1, count = 0;

    // ----------------------------------------
    // Phase 1: CREATION
    // ----------------------------------------
    while (choice) {
        // Allocate memory for the new node
        newnode = (struct node*) malloc(sizeof(struct node));
        
        printf("Enter data: ");
        scanf("%d", &newnode->data);
        newnode->next = NULL; // Always initialize next to NULL

        // If it's the very first node
        if (head == NULL) {
            head = temp = newnode;
        } 
        // If the list already has nodes
        else {
            temp->next = newnode; // Link the previous node to the new node
            temp = newnode;       // Move temp to the new last node
        }

        printf("Do you want to continue? (0,1): ");
        scanf("%d", &choice);
    }

    // ----------------------------------------
    // Phase 2: DISPLAY (Traversal)
    // ----------------------------------------
    temp = head; // Reset temp to the start of the list
    printf("\nLinked List Data: ");
    
    while (temp != NULL) {
        printf("%d -> ", temp->data);
        temp = temp->next; // Move temp to the next node
        count++;           // Optional: count the nodes
    }
    printf("NULL\n");
    
    printf("Total nodes: %d\n", count);

    return 0;
}
```

## 🚨 Exam Warning: The Dangling `head` Trap
A very common mistake students make when writing traversal code is modifying the `head` pointer directly:

```c
// ❌ FATAL ERROR: Doing this destroys the list
while (head != NULL) {
    printf("%d", head->data);
    head = head->next; 
}
```
If you do this, by the end of the loop, `head` will equal `NULL`. You have permanently lost the starting address of your linked list, causing a massive memory leak and making the list inaccessible for future operations.

**Always use a `temp` pointer to traverse!**
```c
// ✅ CORRECT
temp = head;
while (temp != NULL) { ... }
```
