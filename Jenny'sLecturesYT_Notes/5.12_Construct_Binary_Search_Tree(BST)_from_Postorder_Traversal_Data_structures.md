# 5.12 Construct Binary Search Tree(BST) from Postorder Traversal | Data structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=RttBwUVWfV8)

---

## 📌 Executive Summary

Same concept as preorder construction — BST can be uniquely built from **Postorder alone**. Root is the **last** element, then split by value comparison.

---

## 📐 Algorithm

1. **Last element** = Root
2. Elements **< root** = left subtree
3. Elements **> root** = right subtree
4. Recursively construct

---

## 📐 Example: Postorder = `[1, 7, 5, 50, 40, 10]`

```
Step 1: Root = 10 (last)
        < 10: [1, 7, 5]  → left
        > 10: [50, 40]   → right

Step 2: Left — Postorder [1, 7, 5]
        Root = 5, < 5: [1], > 5: [7]

Step 3: Right — Postorder [50, 40]
        Root = 40, > 40: [50]

Result:
       10
      /  \
     5    40
    / \     \
   1   7    50

Same tree as preorder! ✅
```

---

## 📊 Summary: BST from Single Traversal

| Given | Root | Split Method |
|-------|------|-------------|
| **Preorder** | First element | values < root → left, > root → right |
| **Postorder** | Last element | values < root → left, > root → right |
| **Inorder** | ❌ Can't determine | Always sorted — ambiguous tree |

---

## 💡 Key Takeaways

- Postorder: root = **last** element
- Same splitting logic as preorder
- BST property makes inorder redundant
- Verify: inorder of result should be **sorted**
