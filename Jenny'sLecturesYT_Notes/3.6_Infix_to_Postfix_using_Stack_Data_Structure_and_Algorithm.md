# 3.6 Infix to Postfix using Stack (Tabular Method) | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=k1tC5PjC0o8) *(Note: Placeholder link)*

---

## 📌 Executive Summary
Converting an Infix expression to Postfix without a stack is possible, but it requires **multiple scans** of the expression, making it computationally inefficient ($O(n^2)$ or worse). Using a **Stack** guarantees the conversion is completed in exactly **one scan** from left to right ($O(n)$ time complexity). This note demonstrates the formal tabular method required in university exams.

---

## 🔑 Crucial Precedence & Associativity Reminder

| Priority | Operator | Associativity (Tie-Breaker) | Stack Action on Equal Precedence |
| :---: | :---: | :--- | :--- |
| **1 (Highest)** | `( )` | N/A | Push |
| **2** | `^` | **Right to Left** | **Push incoming `^`** (Do not pop) |
| **3** | `*`, `/` | **Left to Right** | **Pop top operator**, then Push incoming |
| **4 (Lowest)** | `+`, `-` | **Left to Right** | **Pop top operator**, then Push incoming |

---

## 📐 Mega Trace Example
**Expression:** `K + L - M * N + ( O ^ P ) * W / U / V * T + Q`

| Token Scanned | Stack State (Top at right) | Postfix Output | Rule Applied / Logic |
| :--- | :--- | :--- | :--- |
| `K` | | `K` | Operand $\rightarrow$ Print |
| `+` | `+` | `K` | Empty stack $\rightarrow$ Push |
| `L` | `+` | `K L` | Operand $\rightarrow$ Print |
| `-` | `-` | `K L +` | `-` == `+` (L-to-R). Pop `+`, Push `-`. |
| `M` | `-` | `K L + M` | Operand $\rightarrow$ Print |
| `*` | `- *` | `K L + M` | `*` > `-`. Push `*`. |
| `N` | `- *` | `K L + M N` | Operand $\rightarrow$ Print |
| `+` | `+` | `K L + M N * -` | `+` < `*`. Pop `*`. New top is `-`. `+` == `-` (L-to-R). Pop `-`. Push `+`. |
| `(` | `+ (` | `K L + M N * -` | `(` always pushed. |
| `O` | `+ (` | `K L + M N * - O` | Operand $\rightarrow$ Print |
| `^` | `+ ( ^` | `K L + M N * - O` | Top is `(`. Push `^`. |
| `P` | `+ ( ^` | `K L + M N * - O P` | Operand $\rightarrow$ Print |
| `)` | `+` | `K L + M N * - O P ^` | Pop until `(`. Pop `^`. Discard `(`, `)`. |
| `*` | `+ *` | `K L + M N * - O P ^` | `*` > `+`. Push `*`. |
| `W` | `+ *` | `K L + M N * - O P ^ W` | Operand $\rightarrow$ Print |
| `/` | `+ /` | `... W *` | `/` == `*` (L-to-R). Pop `*`. Push `/`. |
| `U` | `+ /` | `... W * U` | Operand $\rightarrow$ Print |
| `/` | `+ /` | `... W * U /` | `/` == `/` (L-to-R). Pop `/`. Push `/`. |
| `V` | `+ /` | `... W * U / V` | Operand $\rightarrow$ Print |
| `*` | `+ *` | `... W * U / V /` | `*` == `/` (L-to-R). Pop `/`. Push `*`. |
| `T` | `+ *` | `... W * U / V / T` | Operand $\rightarrow$ Print |
| `+` | `+` | `... W * U / V / T * +`| `+` < `*`. Pop `*`. New top `+`. `+` == `+` (L-to-R). Pop `+`. Push `+`. |
| `Q` | `+` | `... W * U / V / T * + Q`| Operand $\rightarrow$ Print |
| *End* | *Empty* | `... W * U / V / T * + Q +`| Pop remaining `+`. |

**Final Postfix Output:** `K L + M N * - O P ^ W * U / V / T * + Q +`

---

## 🚨 The Right-to-Left Associativity Trap (Exponents)
Consider the sub-expression: `Q ^ J ^ A`

1. `Q` is printed.
2. `^` is pushed to stack. Stack: `[^]`
3. `J` is printed.
4. Incoming `^` meets Top `^`. 
   - **Crucial Difference:** Because exponentiation is **Right-to-Left** associative, you **DO NOT POP**. You simply push the incoming `^` onto the stack.
   - Stack becomes: `[^, ^]`
5. `A` is printed.
6. End of expression: Pop both `^`.
7. **Final Postfix:** `Q J A ^ ^`

> **Exam Tip:** Examiners deliberately put sequences like `A / B * C` or `X ^ Y ^ Z` into expressions to test your knowledge of associativity. 
> - For `/` and `*`, you must POP the first one out before pushing the second.
> - For `^` and `^`, you MUST stack them on top of each other.
