## 📌 Executive Summary
A **Heap** is a specialized tree-based data structure that satisfies two main properties:
1. **Shape Property:** It is an **almost complete binary tree** (all levels are completely filled except possibly the last level, which is filled from left to right).
2. **Order Property (Max Heap):** The value of every parent node is **greater than or equal to** ($\ge$) the values of its child nodes. The root node always contains the maximum value.
*(Conversely, in a Min Heap, parent $\le$ children, and the root is the minimum value).*

Heaps are typically represented using an **Array** rather than pointers to save space. For a node at index `i` (starting at 1):
- **Parent:** $\lfloor i / 2 \rfloor$
- **Left Child:** $2 \times i$
- **Right Child:** $2 \times i + 1$

---

## 📥 Insertion in Max Heap
**Rule:** You must always insert new elements as the **last leaf node** (to maintain the complete binary tree property), and then adjust the tree **upwards** (Bottom-Up approach).

### Step-by-Step Procedure
1. Increase the size of the array $N = N + 1$.
2. Insert the new data at the end of the array (the next available leaf node).
3. **Heapify Up:** Compare the newly inserted node with its parent (index $\lfloor i / 2 \rfloor$).
4. If the new node is greater than its parent, **swap** them.
5. Repeat the comparison and swapping upwards until you reach the root, or until the parent is greater than the node.

### 💻 C Pseudocode for Insertion
```c
void insertHeap(int A[], int n, int value) {
    n = n + 1;         // Increase heap size
    A[n] = value;      // Insert at the last free position
    int i = n;
    
    // Bubble up until we reach root or parent is greater
    while (i > 1) {
        int parent = i / 2;
        if (A[parent] < A[i]) {
            // Swap if parent is smaller (violates Max Heap)
            int temp = A[parent];
            A[parent] = A[i];
            A[i] = temp;
            i = parent; // Move index pointer up to parent
        } else {
            break; // Max heap property satisfied
        }
    }
}
```

## 📤 Deletion in Max Heap
**Rule:** In a standard Heap, you can **ONLY delete the Root Node** (the maximum element). You cannot randomly delete intermediate nodes. After deletion, the tree is adjusted **downwards** (Top-Down approach).

### Step-by-Step Procedure
1. **Remove Root:** Delete the root element (index 1).
2. **Replace:** Take the **last element** of the heap (the rightmost leaf node) and place it at the root.
3. Decrease the heap size $N = N - 1$.
4. **Heapify Down:** Compare the new root with its left child ($2i$) and right child ($2i+1$).
5. Find the **largest** of the two children.
6. If the largest child is greater than the parent, **swap** them.
7. Repeat this downwards until the node is greater than both children, or it becomes a leaf.

### 📝 Trace Example
1. Assume Root is `50` and the last leaf element is `10`.
2. Delete `50`. Move `10` to the root.
3. `10`'s children are `45` and `35`. `45` is the larger child.
4. Compare `10` with `45`. Since `10 < 45`, swap them.
5. Now `10` is at the position where `45` was. Check `10`'s new children (`33` and `16`). 
6. `33` is the larger child. Swap `10` and `33`.
7. Continue until `10` reaches its valid position.

> **Crucial Observation:** When you delete the root (e.g., `50`), the last space in the array (`A[n]`) becomes free. You can store the deleted `50` in that exact free space. If you keep deleting the root and placing it at the end of the array, the array will eventually become **sorted in ascending order**. This is the exact mechanism behind **Heap Sort**.

## ⏱️ Complexity Analysis

| Operation | Time Complexity | Reason |
| :--- | :--- | :--- |
| **Insertion** | **O(log N)** | The new node might need to bubble all the way up to the root. Max comparisons = height of the tree, which is $\log_2(N)$. |
| **Deletion** | **O(log N)** | The swapped root node might need to bubble all the way down to the leaves. Max comparisons = height of the tree. |
| **Find Max** | **O(1)** | The maximum element is always instantly accessible at the root `A[1]`. |

> **Exam Tip:** Always draw the array representation alongside the tree diagram. When asked to insert an element, **do not** place it directly where it logically belongs (like in a BST). You **must** place it at the very end of the array/tree first, and then swap it upwards. Examiners look for this specific step!
