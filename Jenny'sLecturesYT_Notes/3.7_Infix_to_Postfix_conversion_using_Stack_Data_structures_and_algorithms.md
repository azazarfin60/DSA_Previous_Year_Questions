# 3.7 Infix to Postfix Conversion (Advanced Tracing Example) | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=J_pqWzN48C4) *(Note: Placeholder link)*

---

## 📌 Executive Summary
This note provides an advanced, complete walkthrough of converting a complex Infix expression to Postfix using a Stack. It reinforces the critical rules surrounding **Left-to-Right Associativity** (which causes operators of equal precedence to pop) and **Parentheses handling**.

---

## 🔑 Quick Rule Recap
- **Operand:** Print directly.
- **`(` :** Push directly.
- **`)` :** Pop and print until `(`, then discard both.
- **Operator (Incoming > Top):** Push directly.
- **Operator (Incoming < Top):** Pop Top, re-test Incoming against new Top.
- **Operator (Incoming == Top):** Check associativity.
  - Left-to-Right (`+`, `-`, `*`, `/`): Pop Top, then Push Incoming.
  - Right-to-Left (`^`): Push Incoming directly.

---

## 📐 Advanced Trace Example

**Expression:** `A - B + ( M ^ N ) * ( O + P ) - Q / R ^ S * U + Z`

| Token Scanned | Stack State (Top at right) | Postfix Output | Rule Applied / Logic |
| :--- | :--- | :--- | :--- |
| `A` | | `A` | Operand $\rightarrow$ Print |
| `-` | `-` | `A` | Empty stack $\rightarrow$ Push |
| `B` | `-` | `A B` | Operand $\rightarrow$ Print |
| `+` | `+` | `A B -` | `+` == `-` (L-to-R). Pop `-`. Push `+`. |
| `(` | `+ (` | `A B -` | `(` always pushed. |
| `M` | `+ (` | `A B - M` | Operand $\rightarrow$ Print |
| `^` | `+ ( ^` | `A B - M` | Top is `(`. Push `^`. |
| `N` | `+ ( ^` | `A B - M N` | Operand $\rightarrow$ Print |
| `)` | `+` | `A B - M N ^` | Pop until `(`. Pop `^`. Discard `(`, `)`. |
| `*` | `+ *` | `A B - M N ^` | `*` > `+`. Push `*`. |
| `(` | `+ * (` | `A B - M N ^` | `(` always pushed. |
| `O` | `+ * (` | `A B - M N ^ O` | Operand $\rightarrow$ Print |
| `+` | `+ * ( +` | `A B - M N ^ O` | Top is `(`. Push `+`. |
| `P` | `+ * ( +` | `A B - M N ^ O P` | Operand $\rightarrow$ Print |
| `)` | `+ *` | `A B - M N ^ O P +` | Pop until `(`. Pop `+`. Discard `(`, `)`. |
| `-` | `-` | `... O P + * +` | `-` < `*`. Pop `*`. New top `+`. `-` == `+` (L-to-R). Pop `+`. Push `-`. |
| `Q` | `-` | `... O P + * + Q` | Operand $\rightarrow$ Print |
| `/` | `- /` | `... O P + * + Q` | `/` > `-`. Push `/`. |
| `R` | `- /` | `... O P + * + Q R` | Operand $\rightarrow$ Print |
| `^` | `- / ^` | `... O P + * + Q R` | `^` > `/`. Push `^`. |
| `S` | `- / ^` | `... O P + * + Q R S` | Operand $\rightarrow$ Print |
| `*` | `- *` | `... Q R S ^ /` | `*` < `^`. Pop `^`. New top `/`. `*` == `/` (L-to-R). Pop `/`. Push `*`. |
| `U` | `- *` | `... Q R S ^ / U` | Operand $\rightarrow$ Print |
| `+` | `+` | `... Q R S ^ / U * -`| `+` < `*`. Pop `*`. New top `-`. `+` == `-` (L-to-R). Pop `-`. Push `+`. |
| `Z` | `+` | `... Q R S ^ / U * - Z`| Operand $\rightarrow$ Print |
| *End* | *Empty* | `... Q R S ^ / U * - Z +`| Pop remaining `+`. |

**Final Postfix Output:** `A B - M N ^ O P + * + Q R S ^ / U * - Z +`

---

> **Exam Tip:** Notice what happened at the sequence `- Q / R ^ S * U`. 
> 1. The `/` was pushed above the `-`.
> 2. The `^` was pushed above the `/`.
> 3. When `*` arrived, it was smaller than `^`, triggering a pop of `^`. Then `*` was equal to `/`, triggering a pop of `/`. The `*` was finally pushed above the `-`. 
> 
> This "cascade popping" is exactly what examiners test for in long expressions. Always re-test the incoming operator against the *new* top of the stack after every pop!
