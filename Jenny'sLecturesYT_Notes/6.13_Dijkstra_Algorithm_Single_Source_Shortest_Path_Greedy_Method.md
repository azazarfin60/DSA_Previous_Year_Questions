# 6.13 Dijkstra's Algorithm (Single Source Shortest Path) | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=XB4MIexjvY0)

---

## 📌 Executive Summary
**Dijkstra's Algorithm** is a **Greedy Algorithm** designed to solve the **Single Source Shortest Path** problem. Given a graph and a single starting vertex (the source), the algorithm calculates the absolute shortest path distance from that source to every other reachable vertex in the graph. It works for both directed and undirected graphs.

---

## 🔑 Core Concepts & Initialization
The algorithm maintains a "Distance Array" that stores the shortest known distance from the source to each vertex.

1. **Initialization:**
   - Set the distance of the **Source Vertex** to `0`.
   - Set the distance of **all other vertices** to $\infty$ (Infinity).
2. **The Relaxation Formula:**
   For any selected node $U$ pointing to an adjacent unvisited node $V$:
   ```c
   if (Distance[U] + Cost(U, V) < Distance[V]) {
       Distance[V] = Distance[U] + Cost(U, V);
       Parent[V] = U; // Used for path reconstruction
   }
   ```

---

## ⚙️ Step-by-Step Algorithm
1. **Select the Minimum:** Scan the list of all *unvisited* vertices and select the one with the absolute **minimum distance**. (In the first step, this will always be the source vertex because its distance is 0 and others are $\infty$).
2. **Mark as Visited:** Mark this chosen vertex $U$ as "visited" or "selected".
   > **🚨 Crucial Rule:** Once a vertex is marked as visited, its shortest distance is **finalized**. You can *never* update or change the distance of a visited vertex again.
3. **Relax Adjacent Nodes:** Look at all *unvisited* adjacent vertices $V$ connected to $U$. Apply the Relaxation Formula. If going through $U$ provides a strictly shorter path to $V$ than $V$'s current known distance, update $V$'s distance.
4. **Repeat:** Repeat Steps 1-3 until all vertices have been visited.

---

## 🛤️ Path Reconstruction
Finding the shortest *distance* (e.g., 9) is easy, but how do you find the actual *path* (e.g., A $\rightarrow$ C $\rightarrow$ B $\rightarrow$ D)?
- Whenever you successfully "relax" an edge and update `Distance[V]`, you must store the node that provided this better path in a Parent array: `Parent[V] = U`.
- **To trace the path:** Start at the destination node and look up its Parent. Then look up the Parent of that Parent, continuing backwards until you reach the Source. Reverse this sequence to get the final path from Source to Destination.

---

## ⚠️ Drawbacks of Dijkstra's Algorithm (Exam Essential)
Dijkstra's greedy approach comes with a fatal flaw regarding negative numbers:

1. **Negative Edge Weights:** 
   - Dijkstra **may or may not work** if the graph contains negative edge weights. 
   - *Why?* Because Dijkstra assumes that adding more edges to a path can only increase its total cost (which is true for positive weights). Therefore, it "locks" the distance of a node once visited. A negative edge discovered later might provide a cheaper route to an already-locked node, but Dijkstra will incorrectly ignore it.
2. **Negative Weight Cycles:**
   - Dijkstra will **surely fail (infinite loop/incorrect results)** if the graph contains a Negative Weight Cycle (a cycle where the sum of edge weights is negative).

---

## ⏱️ Complexity Analysis

| Implementation | Time Complexity |
| :--- | :--- |
| **Using Array / Matrix** | **O(V²)** |
| **Using Min-Heap / Priority Queue** | **O((V + E) log V)** or roughly **O(E log V)** |

> **Exam Tip:** When tracing Dijkstra in an exam, draw a table with columns for every vertex. In each row, circle the minimum value you select, and update the values in the remaining unvisited columns. Cross out columns once they are selected so you don't accidentally update them again! Also, if asked "Which algorithm handles negative edges?", the answer is **Bellman-Ford**, NOT Dijkstra.
