## 📌 Executive Summary
A **Circular Queue** solves the major drawback of a linear queue implemented via an array: **wasted memory space**. 

In a linear queue, if `rear` reaches the end of the array, the queue is considered "Full", even if earlier indices have been freed by dequeue operations. A Circular Queue connects the end of the array back to the beginning, creating a continuous loop. 

This is achieved mathematically using the **Modulo Operator (`%`)**.

---

## 📐 The Magic Formula: Modulo Arithmetic

Instead of simply incrementing our pointers (`++`), we use the modulo operator against the maximum size of the queue (`N`).
- **Incrementing Rear:** `rear = (rear + 1) % N`
- **Incrementing Front:** `front = (front + 1) % N`

*Example (N=5): If `rear` is 4, `(4 + 1) % 5 = 0`. The pointer gracefully wraps around to the beginning of the array.*

---

## 1️⃣ Enqueue Operation (Insertion)

The most critical change is the **Overflow (Full) Condition**. The queue is full if the *next* position after `rear` is currently occupied by the `front` pointer.
**Time Complexity:** $O(1)$

### C Code Snippet
```c
#define N 5
int queue[N];
int front = -1, rear = -1;

void enqueue(int x) {
    // 1. Check for Overflow (The Circular Condition)
    if ((rear + 1) % N == front) {
        printf("Overflow: Queue is full\n");
    } 
    // 2. Check for empty queue
    else if (front == -1 && rear == -1) {
        front = rear = 0;
        queue[rear] = x;
    } 
    // 3. Normal insertion with Wrap-Around
    else {
        rear = (rear + 1) % N;
        queue[rear] = x;
    }
}
```

---

## 2️⃣ Dequeue Operation (Deletion)

The underflow and single-element logic remains identical to a linear queue. We only change how the `front` pointer steps forward.
**Time Complexity:** $O(1)$

### C Code Snippet
```c
void dequeue() {
    // 1. Check for Underflow
    if (front == -1 && rear == -1) {
        printf("Underflow: Queue is empty\n");
    } 
    // 2. Check if only ONE element is left
    else if (front == rear) {
        printf("Dequeued: %d\n", queue[front]);
        front = rear = -1; // Reset to empty
    } 
    // 3. Normal deletion with Wrap-Around
    else {
        printf("Dequeued: %d\n", queue[front]);
        front = (front + 1) % N; // Shift front circularly
    }
}
```

---

## 3️⃣ Display Operation ($O(n)$)

Because the queue wraps around, we can no longer use a simple `for(i = front; i <= rear; i++)` loop. If `front` is 3 and `rear` is 1, that loop would fail immediately.

We must traverse using a `while` loop, stepping circularly.

> [!WARNING]
> **Exam Trap (The "Missing Last Element" Display Bug):**
> If you write `while (i != rear)`, the loop will terminate the exact moment `i` lands on the last element, meaning the data at the `rear` index is **never printed**! You must manually print the final element *after* the loop closes.

### C Code Snippet
```c
void display() {
    int i = front;
    
    if (front == -1 && rear == -1) {
        printf("Queue is empty\n");
        return;
    }
    
    printf("Queue elements: ");
    
    // Stop exactly when i reaches rear
    while (i != rear) {
        printf("%d ", queue[i]);
        i = (i + 1) % N; // Circular increment
    }
    
    // TRAP FIX: Print the final element sitting at the rear index
    printf("%d\n", queue[rear]); 
}
```

*Note: The `peek()` operation is identical to the linear queue (`return queue[front]`), as it does not involve moving pointers.*
