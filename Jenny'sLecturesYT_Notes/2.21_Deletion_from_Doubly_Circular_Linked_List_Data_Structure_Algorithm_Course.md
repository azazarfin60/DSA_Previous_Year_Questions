# 2.21 Deletion from a Doubly Circular Linked List | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
Deleting from a Doubly Circular Linked List (DCLL) involves safely bypassing the node to be deleted and freeing its memory. Because we maintain both `head` and `tail` pointers, and because we have backward (`prev`) links, **deletion from the end is an $O(1)$ operation**, giving the DCLL a massive performance advantage over the Singly Circular Linked List (which requires $O(n)$ to delete the tail).

---

## 1️⃣ Case 1: Deletion from the Beginning

To delete the first node, we shift the `head` pointer forward and then repair the circular loop between the new `head` and the `tail`.
**Time Complexity:** $O(1)$

### C Code Snippet
```c
void deleteFromBeginning() {
    struct node *temp;
    
    if (head == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    temp = head; // Anchor the node to be deleted
    
    // Edge Case: Only 1 node in the list
    if (head->next == head) {
        head = tail = NULL;
        free(temp);
    } 
    // Normal Case: >1 node
    else {
        head = head->next;    // 1. Shift head to the second node
        head->prev = tail;    // 2. Repair circular backward link
        tail->next = head;    // 3. Repair circular forward link
        free(temp);           // 4. Release memory
    }
}
```

---

## 2️⃣ Case 2: Deletion from the End

> [!TIP]
> **Exam Advantage ($O(1)$ Tail Deletion):** 
> If an exam asks to compare Circular Linked Lists and Doubly Circular Linked Lists, highlight this exact scenario. In a Singly CLL, deleting the last node requires an $O(n)$ traversal to find the second-to-last node. In a Doubly CLL, `tail->prev` gives us the second-to-last node instantly.
> **Time Complexity:** $O(1)$

### C Code Snippet
```c
void deleteFromEnd() {
    struct node *temp;
    
    if (head == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    temp = tail; // Anchor the node to be deleted
    
    if (head->next == head) {
        head = tail = NULL;
        free(temp);
    } else {
        tail = tail->prev;    // 1. Shift tail BACKWARDS to the 2nd-to-last node
        tail->next = head;    // 2. Repair circular forward link
        head->prev = tail;    // 3. Repair circular backward link
        free(temp);           // 4. Release memory
    }
}
```

## 3️⃣ Case 3: Deletion from a Specific Position

To delete a node from the middle, we traverse directly to the target node. Because each node has bidirectional links, we can easily access its left and right neighbors to bypass it.
**Time Complexity:** $O(n)$

### Algorithm Steps:
1. Traverse to the target `pos` using a `temp` pointer.
2. Connect `temp`'s left neighbor directly to `temp`'s right neighbor.
3. Handle the edge case where the deleted node happens to be the last node (must update `tail`).
4. Free `temp`.

### C Code Snippet
```c
void deleteFromPosition() {
    struct node *temp;
    int pos, i = 1;
    
    printf("Enter position: ");
    scanf("%d", &pos);
    
    // (Assume validation: if pos == 1, call deleteFromBeginning)
    
    temp = head;
    
    // Traverse directly to the node to be deleted
    while (i < pos) {
        temp = temp->next;
        i++;
    }
    
    // 1. Bypass the node (Left neighbor points to Right neighbor)
    temp->prev->next = temp->next;
    
    // 2. Bypass the node (Right neighbor points to Left neighbor)
    temp->next->prev = temp->prev;
    
    // 3. 🌟 CORNER CASE: If we deleted the last node, shift tail backwards 🌟
    if (temp->next == head) {
        tail = temp->prev;
    }
    
    // 4. Release memory
    free(temp);
}
```
