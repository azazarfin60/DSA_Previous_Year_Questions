## 📐 B+ Tree Deletion Rules

### If key is in leaf:
1. Delete the key
2. If leaf still has ≥ minimum keys → done
3. If underflow:
   - **Borrow** from sibling if sibling has extra
   - Otherwise **merge** with sibling

### If key is also in internal node:
- Delete from leaf
- **Update** internal node with the new separator (smallest of right subtree)

### Key Difference from B-Tree:
> B+ Tree: data only in leaves, so internal keys are just **separators** — they may need updating after deletion but actual data removal only happens at leaf level.

## 📐 Merge in B+ Tree:
```
Before:           After merge:
  [... 15 ...]     [...]
   /     \            |
 [10]   [20,25]    [10,20,25]  ← leaf merged, 15 removed from parent
```
Note: In B+ Tree leaf merge, the parent key is **discarded** (not merged in), since leaves hold their own keys.

## 💡 B+ Tree deletion is simpler at leaf level but needs careful separator updates.
