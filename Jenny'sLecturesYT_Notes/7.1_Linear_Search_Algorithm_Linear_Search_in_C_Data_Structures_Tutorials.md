## 📌 Executive Summary
**Linear Search** (also known as Sequential Search) is the simplest searching algorithm. It works by sequentially checking each element of the array from the beginning until a match is found or the whole array has been searched. While intuitive, it is inefficient for large datasets compared to algorithms like Binary Search.

---

## 🔑 Core Concept
1. **Iterative Checking:** Start at the first element (index `0`).
2. **Comparison:** Compare the current element `A[i]` with the target `data`.
3. **Stopping Conditions:** The search stops under two conditions:
   - **Condition 1 (Found):** The target `data` matches `A[i]`. We output the index and instantly **break** out of the loop.
   - **Condition 2 (Not Found):** We have traversed the entire array (`i == n`) and no match was found.
4. **The `break` Keyword:** It is critical to use a `break` statement as soon as the element is found. Without `break`, the loop will unnecessarily continue checking the rest of the array, wasting processing time.

---

## 📐 Step-by-Step Trace

**Given Array:** `A = [15, 5, 20, 35, 2, 42, 67, 17]` *(N = 8)*
**Target Data:** `42`

1. `i = 0`: `A[0] = 15`. Is $15 == 42$? No. `i++`
2. `i = 1`: `A[1] = 5`. Is $5 == 42$? No. `i++`
3. `i = 2`: `A[2] = 20`. Is $20 == 42$? No. `i++`
4. `i = 3`: `A[3] = 35`. Is $35 == 42$? No. `i++`
5. `i = 4`: `A[4] = 2`. Is $2 == 42$? No. `i++`
6. `i = 5`: `A[5] = 42`. Is $42 == 42$? **YES.**
   - Action: Print "Element found at index 5".
   - Action: Execute `break`. The search terminates immediately without checking indices 6 and 7.

---

## 💻 C Code Implementation

There are two common ways to handle the "Not Found" condition. 

### Method 1: Checking the Index Limit
```c
void linearSearch(int a[], int n, int data) {
    int i;
    for(i = 0; i < n; i++) {
        if(a[i] == data) {
            printf("Element found at index %d\n", i);
            // Element found at position i + 1
            break; // Crucial: Stop searching once found
        }
    }
    
    // If the loop finished without breaking, 'i' will equal 'n'
    if(i == n) {
        printf("Element not found\n");
    }
}
```

### Method 2: Using a Flag Variable
```c
void linearSearchFlag(int a[], int n, int data) {
    int i;
    int found = 0; // 0 means false, 1 means true
    
    for(i = 0; i < n; i++) {
        if(a[i] == data) {
            printf("Element found at index %d\n", i);
            found = 1;
            break;
        }
    }
    
    if(found == 0) {
        printf("Element not found\n");
    }
}
```

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Best Case Time** | **O(1)** | The target element is found at the very first index (`i = 0`). Only 1 comparison is made. |
| **Worst Case Time** | **O(N)** | The target element is at the very last index (`i = N-1`) or is **not present** in the array at all. The loop runs $N$ times. |
| **Average Case Time**| **O(N)** | Defined mathematically: $\frac{\text{Sum of all comparisons}}{\text{Number of cases}} = \frac{1 + 2 + 3 + ... + N}{N} = \frac{\frac{N(N+1)}{2}}{N} = \frac{N+1}{2}$. Ignoring constants, this is $O(N)$. |
| **Space Complexity** | **O(1)** | Operates in-place, requiring only a few extra variables (`i`, `found`). |

> **Exam Tip:** When writing the code snippet for Linear Search, examiners explicitly look for the `break` statement inside the `if` block. Forgetting it turns your best-case time complexity into $O(N)$, which is a major algorithmic flaw!
