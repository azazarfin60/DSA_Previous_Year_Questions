## 📌 Executive Summary

Linear Search (Sequential Search) checks **every element** from start to end until the target is found or the array is exhausted. Simplest search — works on unsorted arrays.

---

## 💻 C Code

```c
int linearSearch(int arr[], int n, int key) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == key)
            return i;           // Found at index i
    }
    return -1;                  // Not found
}
```

---

## 📐 Trace Example

```
Array: [5, 3, 8, 1, 9]    Key = 8

i=0: arr[0]=5 ≠ 8
i=1: arr[1]=3 ≠ 8
i=2: arr[2]=8 = 8 → Found at index 2! ✅
```

---

## ⏱️ Time Complexity

| Case | Comparisons | Time |
|------|------------|------|
| **Best** | 1 (first element) | **O(1)** |
| **Worst** | n (last or not found) | **O(n)** |
| **Average** | n/2 | **O(n)** |

**Space: O(1)** — no extra space

---

## 📊 When to Use

| Use Linear Search When | Don't Use When |
|----------------------|---------------|
| Array is **unsorted** | Array is sorted (use Binary Search) |
| Array is **small** | Array is large |
| One-time search | Repeated searches |

---

## 💡 Key Takeaways

- Simplest search algorithm — check each element sequentially
- Works on **both sorted and unsorted** arrays
- Time: O(n) — not efficient for large datasets
- No preprocessing required
