# 8.3 Double Hashing | Collision Resolution Technique | Data Structures and algorithms

## 📌 Executive Summary

Double Hashing uses **two hash functions** to resolve collisions. The second hash determines the **step size** for probing, eliminating the clustering problem of linear probing.

---

## 📐 Formula

```
h(key, i) = (h1(key) + i × h2(key)) % table_size
```

Where:
- `h1(key) = key % table_size` — primary hash
- `h2(key) = PRIME - (key % PRIME)` — step size
- PRIME = a prime number smaller than table_size
- i = probe number (0, 1, 2, ...)

---

## 📐 Example: Table size=7, PRIME=5

Insert keys: 27, 43, 692, 72

```
h1(key) = key % 7
h2(key) = 5 - (key % 5)

h(27): h1=27%7=6, slot 6 empty → place at 6
h(43): h1=43%7=1, slot 1 empty → place at 1
h(692): h1=692%7=6, COLLISION!
        h2=5-(692%5)=5-2=3
        i=1: (6 + 1×3) % 7 = 2 → slot 2 empty → place at 2
h(72):  h1=72%7=2, COLLISION!
        h2=5-(72%5)=5-2=3
        i=1: (2 + 1×3) % 7 = 5 → slot 5 empty → place at 5

Table: [_, 43, 692, _, _, 72, 27]
Index:  0   1    2   3  4   5   6
```

---

## 📊 Linear vs Quadratic vs Double Hashing

| Method | Probe Sequence | Clustering |
|--------|---------------|-----------|
| **Linear** | h+1, h+2, h+3... | Primary clustering ⚠️ |
| **Quadratic** | h+1², h+2², h+3²... | Secondary clustering ⚠️ |
| **Double** | h+h2, h+2h2, h+3h2... | **No clustering** ✅ |

---

## ⚠️ Requirements for Double Hashing
1. `h2(key)` must **never return 0** (otherwise no probing)
2. Table size should be **prime** (ensures all slots are probed)
3. `h2` and table size should be **coprime**

---

## ⏱️ Complexity
- Average: **O(1/(1-α))** where α = load factor
- Best among open addressing methods

---

## 💡 Key Takeaways

- Two hash functions eliminate clustering
- `h2` determines **variable step size** — different for each key
- Better distribution than linear/quadratic probing
- Table size should be **prime** for best results
- Most efficient open addressing technique
