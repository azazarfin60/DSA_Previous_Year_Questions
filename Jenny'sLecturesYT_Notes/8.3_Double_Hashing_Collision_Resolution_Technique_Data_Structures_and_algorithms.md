# 8.3 Hashing - Double Hashing | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
**Double Hashing** is a closed hashing (open addressing) collision resolution technique. 

Unlike linear or quadratic probing which use a fixed mathematical sequence ($i$ or $i^2$), Double Hashing uses a **second hash function** to determine the step size for probing. This creates a highly pseudo-random probe sequence that virtually eliminates both primary and secondary clustering.

---

## 📐 The Double Hashing Formula

When a collision occurs at the ideal hash index, the next location is determined by:

**$H_i(k) = (U + V \cdot i) \pmod{M}$**

*Where:*
- **$U$ (Primary Hash):** The original hash index calculated using the first hash function. $U = H_1(k)$
- **$V$ (Secondary Hash / Step Size):** Calculated using the second hash function. $V = H_2(k) \pmod{M}$
- $i$ = The probe attempt number ($0, 1, 2, ..., M-1$).
- $M$ = The size of the hash table.

---

## 📝 Exam Walkthrough: Double Hashing

> **Problem Statement:**
> Store the keys `[3, 2, 9, 6, 11, 13, 7, 12]` into a Hash Table of size $M = 10$.
> Use Hash Function 1: $H_1(k) = (2k + 3) \pmod{10}$.
> Use Hash Function 2: $H_2(k) = (3k + 1) \pmod{10}$.
> Resolve collisions using **Double Hashing**.

### Step 1: Initial Insertions (No Collisions)
We calculate $U = H_1(k)$:
- $k=3 \rightarrow \text{Index } 9$ (Free)
- $k=2 \rightarrow \text{Index } 7$ (Free)
- $k=9 \rightarrow \text{Index } 1$ (Free)
- $k=6 \rightarrow \text{Index } 5$ (Free)

---

### Step 2: Resolving Collisions (Calculating $V$)

**Inserting 11:** Ideal index $U = 5$ (Collision with 6)
- **Calculate $V$:** $H_2(11) = (3(11) + 1) \pmod{10} = 34 \pmod{10} = \mathbf{4}$
- Formula is now: $(5 + 4 \cdot i) \pmod{10}$
- $i=1$: $(5 + 4) \pmod{10} = 9$ (Full)
- $i=2$: $(5 + 8) \pmod{10} = 13 \pmod{10} = \mathbf{3}$ (Free!). Insert 11 at **Index 3**.

> [!CAUTION]
> **Exam Trap 1 (The Zero Step Size):** Look at what happens with key 13.

**Inserting 13:** Ideal index $U = 9$ (Collision with 3)
- **Calculate $V$:** $H_2(13) = (3(13) + 1) \pmod{10} = 40 \pmod{10} = \mathbf{0}$
- Formula is now: $(9 + 0 \cdot i) \pmod{10}$
- Because $V=0$, no matter what $i$ becomes, the formula will always evaluate to 9. We are stuck in an infinite loop. 
- **Result:** Key 13 **cannot be inserted** with these specific hash functions. A good $H_2(k)$ must never evaluate to 0.

> [!CAUTION]
> **Exam Trap 2 (The Cycle Trap):** Look at what happens with key 7.

**Inserting 7:** Ideal index $U = 7$ (Collision with 2)
- **Calculate $V$:** $H_2(7) = (3(7) + 1) \pmod{10} = 22 \pmod{10} = \mathbf{2}$
- Formula is now: $(7 + 2 \cdot i) \pmod{10}$
- Let's probe:
  - $i=1$: $(7+2)=9$ (Full)
  - $i=2$: $(7+4) \pmod{10}=1$ (Full)
  - $i=3$: $(7+6) \pmod{10}=3$ (Full)
  - $i=4$: $(7+8) \pmod{10}=5$ (Full)
  - $i=5$: $(7+10) \pmod{10}=7$ (Full) ... we have looped back to the start!
- **Result:** Key 7 **cannot be inserted**. The step size (2) and the table size (10) are not co-prime, creating a closed loop that misses the empty slots.

**Inserting 12:** Ideal index $U = 7$ (Collision)
- **Calculate $V$:** $H_2(12) = (3(12) + 1) \pmod{10} = 37 \pmod{10} = \mathbf{7}$
- Formula is now: $(7 + 7 \cdot i) \pmod{10}$
- $i=1$: $(7 + 7) \pmod{10} = 14 \pmod{10} = \mathbf{4}$ (Free!). Insert 12 at **Index 4**.
