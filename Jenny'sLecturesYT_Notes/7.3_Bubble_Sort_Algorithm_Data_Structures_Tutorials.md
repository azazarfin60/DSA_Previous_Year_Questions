## 📌 Executive Summary

Bubble Sort repeatedly **swaps adjacent elements** if they are in the wrong order, "bubbling" the largest element to the end in each pass. Simple but inefficient.

---

## 📐 Algorithm

```c
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {          // n-1 passes
        int swapped = 0;
        for (int j = 0; j < n - 1 - i; j++) {  // Reduce range each pass
            if (arr[j] > arr[j + 1]) {
                // Swap
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = 1;
            }
        }
        if (!swapped) break;  // Optimization: already sorted
    }
}
```

---

## 📐 Trace: Sort [5, 3, 8, 1, 2]

### Pass 1 (i=0): Compare adjacent, bubble largest to end
```
[5, 3, 8, 1, 2] → swap 5,3 → [3, 5, 8, 1, 2]
[3, 5, 8, 1, 2] → 5<8 no swap
[3, 5, 8, 1, 2] → swap 8,1 → [3, 5, 1, 8, 2]
[3, 5, 1, 8, 2] → swap 8,2 → [3, 5, 1, 2, 8]  ← 8 in place
```

### Pass 2 (i=1):
```
[3, 5, 1, 2, 8] → 3<5 no swap
[3, 5, 1, 2, 8] → swap 5,1 → [3, 1, 5, 2, 8]
[3, 1, 5, 2, 8] → swap 5,2 → [3, 1, 2, 5, 8]  ← 5 in place
```

### Pass 3 (i=2):
```
[3, 1, 2, 5, 8] → swap 3,1 → [1, 3, 2, 5, 8]
[1, 3, 2, 5, 8] → swap 3,2 → [1, 2, 3, 5, 8]  ← 3 in place
```

### Pass 4 (i=3):
```
[1, 2, 3, 5, 8] → no swaps → Already sorted! ✅
```

**Result: [1, 2, 3, 5, 8]**

## ⏱️ Complexity

| Case | Time | When |
|------|------|------|
| **Best** | **O(n)** | Already sorted (with optimization) |
| **Average** | **O(n²)** | Random order |
| **Worst** | **O(n²)** | Reverse sorted |

**Space: O(1)** — in-place
**Stable: ✅ Yes** (equal elements maintain relative order)

---

## 💡 Key Takeaways

- Each pass pushes **largest unsorted** to correct position
- `n-1-i` comparisons in pass i (skip already sorted end)
- **Swapped flag** optimization: stop early if no swaps in a pass
- Simple but O(n²) — only use for small or nearly sorted data
