# 7.3 Bubble Sort Algorithm | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=Jdtq5uKz-w4)

## 📌 Executive Summary
This lecture introduces Bubble Sort, a simple comparison-based sorting algorithm. It explains the basic working principle (adjacent swapping), provides a step-by-step dry run, constructs the C code, and then introduces an essential optimization using a `flag` variable to improve the best-case time complexity.

---

## 🔑 Core Concept
- **Working Principle:** Compare adjacent elements (`A[j]` and `A[j+1]`). If they are in the wrong order, swap them.
- **The "Bubble" Effect:** After the 1st pass, the largest element "bubbles up" to the very end of the array. After the 2nd pass, the second largest element bubbles up, and so on.
- **Number of Passes:** To sort an array of `n` elements, you need maximum `n - 1` passes.

---

## 📐 Step-by-Step Trace

**Given Array:** `A = [15, 16, 6, 8, 5]`  *(n = 5)*
We need `n-1 = 4` passes.

### Pass 1
- Compare 15, 16 → No swap. Array: `[15, 16, 6, 8, 5]`
- Compare 16, 6 → Swap. Array: `[15, 6, 16, 8, 5]`
- Compare 16, 8 → Swap. Array: `[15, 6, 8, 16, 5]`
- Compare 16, 5 → Swap. Array: `[15, 6, 8, 5, 16]`
> *Result:* `16` (the largest element) is at its correct sorted position.

### Pass 2
*Note: We don't need to compare the last element anymore.*
- Compare 15, 6 → Swap. Array: `[6, 15, 8, 5, 16]`
- Compare 15, 8 → Swap. Array: `[6, 8, 15, 5, 16]`
- Compare 15, 5 → Swap. Array: `[6, 8, 5, 15, 16]`
> *Result:* `15` is at its correct position.

### Pass 3
- Compare 6, 8 → No swap. Array: `[6, 8, 5, 15, 16]`
- Compare 8, 5 → Swap. Array: `[6, 5, 8, 15, 16]`
> *Result:* `8` is at its correct position.

### Pass 4
- Compare 6, 5 → Swap. Array: `[5, 6, 8, 15, 16]`
> *Result:* `6` is at its correct position. Since 4 elements are sorted, the 5th element (`5`) is automatically at its correct position. The array is fully sorted!

## 💻 C Code Implementation

### Basic Bubble Sort
```c
void bubbleSort(int a[], int n) {
    int i, j, temp;
    // Outer loop for passes: (n-1) passes
    for(i = 0; i < n - 1; i++) {
        // Inner loop for comparisons. 
        // Notice `n - 1 - i` to ignore elements that are already sorted at the end.
        for(j = 0; j < n - 1 - i; j++) {
            if(a[j] > a[j+1]) {
                // Swap
                temp = a[j];
                a[j] = a[j+1];
                a[j+1] = temp;
            }
        }
    }
}
```

## 🚀 Optimized Bubble Sort

**The Problem:** What if the array is already sorted (e.g., `[2, 3, 4, 6, 10]`)? The basic algorithm will still execute all `n-1` passes blindly, wasting time.

**The Solution:** Use a `flag` variable. 
If during any complete pass, **zero swaps** occur, it means the array is already sorted, and we can immediately `break` out of the outer loop.

### Optimized Code
```c
void optimizedBubbleSort(int a[], int n) {
    int i, j, temp, flag;
    for(i = 0; i < n - 1; i++) {
        flag = 0; // Reset flag at the start of each pass
        for(j = 0; j < n - 1 - i; j++) {
            if(a[j] > a[j+1]) {
                // Swap
                temp = a[j];
                a[j] = a[j+1];
                a[j+1] = temp;
                flag = 1; // Mark that a swap occurred
            }
        }
        // If no swaps occurred in this pass, the array is sorted
        if(flag == 0) {
            break; 
        }
    }
}
```

## ⏱️ Complexity Analysis

| Case | Complexity | Reason |
| :--- | :--- | :--- |
| **Worst Case** | **O(n²)** | Array is in descending order. Outer loop runs `n-1` times, inner loop runs `n-1` times. |
| **Best Case** | **O(n)** | Array is already sorted. With the *optimized* version, the inner loop runs once, `flag` remains 0, and the outer loop breaks. |
| **Average Case** | **O(n²)** | General unsorted array. |
| **Space Complexity**| **O(1)** | Sorts in-place; only requires a few integer variables (`i, j, temp, flag`). |
