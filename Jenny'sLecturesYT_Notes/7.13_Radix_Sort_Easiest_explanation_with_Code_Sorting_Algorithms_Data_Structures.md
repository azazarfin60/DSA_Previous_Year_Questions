## 📌 Executive Summary
While the conceptual overview of Radix Sort (sorting digit-by-digit into buckets) is straightforward, its actual programmatic implementation relies heavily on using a modified **Counting Sort** as a subroutine. This note focuses entirely on the C code implementation of Radix Sort, demonstrating how to mathematically extract digits using integer division and modulo operators, and how to feed those digits into a stable Counting Sort algorithm.

---

## 🔑 Core Logic: Digit Extraction
To perform Counting Sort on specific digits without actually converting integers into arrays of characters, we use math:
`Digit = (Number / Position) % 10`

Where `Position` starts at 1 (for the 1s place) and multiplies by 10 for each subsequent pass.
- **1s Place:** `(432 / 1) % 10 = 432 % 10 = 2`
- **10s Place:** `(432 / 10) % 10 = 43 % 10 = 3`
- **100s Place:** `(432 / 100) % 10 = 4 % 10 = 4`

We loop this process as long as `Max_Element / Position > 0`.

## 💻 C Code Implementation

### 1. The Modified Counting Sort Subroutine
This is identical to the standard Counting Sort, but the index we use for the `count` array is mathematically extracted using the `pos` multiplier.

```c
void countSort(int a[], int n, int pos) {
    int count[10] = {0}; // Base 10 means we always need exactly 10 buckets (0-9)
    int b[n];            // Output array
    int i;
    
    // 1. Store the frequency of the specific digit
    for(i = 0; i < n; i++) {
        int digit = (a[i] / pos) % 10;
        count[digit]++;
    }
    
    // 2. Modify Count array to store actual positions (Cumulative sum)
    for(i = 1; i < 10; i++) {
        count[i] = count[i] + count[i - 1];
    }
    
    // 3. Build the output array B by scanning A from right to left (for stability)
    for(i = n - 1; i >= 0; i--) {
        int digit = (a[i] / pos) % 10;
        b[--count[digit]] = a[i]; // Pre-decrement gives the exact 0-based index
    }
    
    // 4. Copy the sorted elements back to original array A
    for(i = 0; i < n; i++) {
        a[i] = b[i];
    }
}
```

### 2. The Main Radix Sort Function
This function coordinates finding the maximum number to determine the number of digits, and driving the loop that calls `countSort`.

```c
// Helper function to find the maximum element in the array
int getMax(int a[], int n) {
    int max = a[0];
    for(int i = 1; i < n; i++) {
        if(a[i] > max) {
            max = a[i];
        }
    }
    return max;
}

void radixSort(int a[], int n) {
    // 1. Find the maximum element to determine the maximum number of digits
    int max = getMax(a, n);
    
    // 2. Call counting sort for every digit, starting from 1s place (pos = 1)
    // pos will be 1, 10, 100, 1000, etc.
    for(int pos = 1; max / pos > 0; pos = pos * 10) {
        countSort(a, n, pos);
    }
}
```

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Time Complexity** | **O(d * (n + k))** | - $d$ = Number of digits in the maximum element.<br>- $n$ = Total number of elements in the array.<br>- $k$ = The base of the number system (10 for decimal numbers).<br><br>The `radixSort` loop runs $d$ times. Inside, `countSort` takes $O(n + k)$ time. Total time is $O(d \times (n + k))$. |
| **Space Complexity** | **O(n + k)** | Space is required for the output array `b` of size $n$, and the `count` array of size $k$ (which is a constant 10). |

> **Exam Tip:** Ensure you can write the mathematical digit extraction logic `(a[i] / pos) % 10` flawlessly. If asked to write Radix Sort in an exam, breaking it into these three functions (`getMax`, `countSort`, `radixSort`) makes the code extremely clean and easy for the examiner to follow.
