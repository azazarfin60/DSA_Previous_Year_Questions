## 📌 Executive Summary
Quick Sort is a highly efficient, Divide-and-Conquer sorting algorithm. Its backbone is the **Partition** operation, which selects a "pivot" element and places it in its correct sorted position while simultaneously ensuring that all smaller elements are to its left and all larger elements are to its right.

---

## 🔑 Core Concept: Divide and Conquer
1. **Choose a Pivot:** Pick an element to serve as the pivot. This could be the first, last, median, or a random element. *(This tutorial uses the **first element** as the pivot).*
2. **Partition the Array:**
   - Rearrange the array so that all elements $\le$ the pivot are moved to its left.
   - All elements $>$ the pivot are moved to its right.
   - *Result:* The pivot is now at its final, correct sorted position.
3. **Recursion:** Recursively apply the same Quick Sort logic to the left sub-array and the right sub-array.

---

## 📐 The Partition Algorithm (Two Pointers Technique)

Let's understand how to partition an array using `start` and `end` pointers.

**Goal:** Move `start` forward to find a value *greater* than the pivot, and move `end` backward to find a value *smaller* than the pivot. When found, swap them.

**Rules:**
1. Let `pivot = A[lower_bound]`.
2. Let `start = lower_bound` and `end = upper_bound`.
3. **Increment `start`** as long as `A[start] <= pivot`. (Stop when you find a larger element).
4. **Decrement `end`** as long as `A[end] > pivot`. (Stop when you find a smaller/equal element).
5. If `start < end`, **swap `A[start]` and `A[end]`**.
6. Repeat steps 3-5 until `start > end`.
7. When `start > end`, the partitioning is complete. **Swap the `pivot` (at `lower_bound`) with `A[end]`**.
8. Return the `end` index (the pivot's final position).

### Step-by-Step Trace Example
*Imagine tracing an array where Pivot = 7. `start` moves right, `end` moves left.*
- `start` stops at 10 ($10 > 7$).
- `end` stops at 5 ($5 \le 7$).
- Swap 10 and 5.
- Continue moving pointers.
- When `start` crosses `end`, you swap the Pivot (7) with the element at the `end` pointer. 7 is now fixed in place!

## 💻 C Code Implementation

### 1. The Partition Function (The Backbone)
```c
int partition(int a[], int lb, int ub) {
    int pivot = a[lb];
    int start = lb;
    int end = ub;
    int temp;

    while (start < end) {
        // Move start right while elements are <= pivot
        while (a[start] <= pivot) {
            start++;
        }
        // Move end left while elements are > pivot
        while (a[end] > pivot) {
            end--;
        }
        // If pointers haven't crossed, swap elements
        if (start < end) {
            temp = a[start];
            a[start] = a[end];
            a[end] = temp;
        }
    }
    
    // When start > end, swap pivot with a[end]
    temp = a[lb];
    a[lb] = a[end];
    a[end] = temp;
    
    // Return the final position of the pivot
    return end;
}
```

### 2. The Recursive Quick Sort Function
```c
void quickSort(int a[], int lb, int ub) {
    if (lb < ub) {
        // 1. Get the partition index (pivot's correct position)
        int loc = partition(a, lb, ub);
        
        // 2. Recursively sort the left sub-array
        quickSort(a, lb, loc - 1);
        
        // 3. Recursively sort the right sub-array
        quickSort(a, loc + 1, ub);
    }
}
```

## ⏱️ Complexity Analysis

| Case | Complexity | Reason |
| :--- | :--- | :--- |
| **Best / Average Case** | **O(n log n)** | The pivot perfectly divides the array into two equal halves every time. The recursion tree has a depth of $\log n$, and at each level, partitioning takes $O(n)$ time. |
| **Worst Case** | **O(n²)** | Occurs when the array is **already sorted** (or strictly descending) and we pick the *first* (or last) element as the pivot. The partition process fails to divide the array evenly (resulting in sub-arrays of size $0$ and $n-1$), creating a highly skewed recursion tree of depth $n$. |

> **Exam Tip:** Quick Sort is often called an "in-place" sort because it doesn't require massive auxiliary arrays like Merge Sort. However, its worst-case scenario ($O(n^2)$) makes it vulnerable if pivot selection isn't randomized or chosen via the median.
