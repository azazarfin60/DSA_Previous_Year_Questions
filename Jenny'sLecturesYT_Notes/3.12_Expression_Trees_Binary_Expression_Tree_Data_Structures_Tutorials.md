## 📌 Executive Summary
A **Binary Expression Tree** is a specific type of binary tree used to represent mathematical expressions in memory. Because it captures the strict hierarchy of operations, it eliminates the need for parentheses to dictate evaluation order. 

- **Internal Nodes:** Store the **Operators** (`+`, `-`, `*`, `/`, `^`).
- **Leaf Nodes (External Nodes):** Store the **Operands** (variables or constants like `A`, `B`, `5`).

---

## 🌳 Constructing from Infix (Top-Down Method)
While you *can* convert an Infix expression to Postfix and build a tree bottom-up, it is often faster to construct the tree directly from the Infix expression using a top-down splitting approach.

### The Core Logic
In an expression tree, the lowest leaf nodes are evaluated first, and the root node is evaluated last. Therefore, to find the root of the entire tree, we must find the operator in the Infix expression that is evaluated **LAST**.

### The 3-Step Algorithm
1. **Find the Least Precedence Operator:** Scan the entire expression and find the operator with the absolute lowest precedence.
2. **Resolve Ties (Associativity):** If multiple operators share the lowest precedence, use associativity to determine which is evaluated last:
   - For **Left-to-Right** (`+`, `-`, `*`, `/`), the **rightmost** operator is evaluated last. Make it the root.
   - For **Right-to-Left** (`^`), the **leftmost** operator is evaluated last. Make it the root.
3. **Split and Recurse:** Make this operator the Root node. Everything to its left becomes the left subtree, and everything to its right becomes the right subtree. Repeat steps 1-3 for each subtree until only operands are left.

## 📐 Trace Example: Top-Down Construction
**Expression:** `A * B / C + E / F * G + K - X * Y`

**Step 1: Find the Root**
- Lowest precedence operators are `+`, `+`, `-`.
- Because `+` and `-` are Left-to-Right associative, the one evaluated last is the **rightmost** one.
- The rightmost lowest operator is the `-`.
- **Root = `-`**
- Left Subtree: `A * B / C + E / F * G + K`
- Right Subtree: `X * Y`

**Step 2: Build Left Subtree** `(A * B / C + E / F * G + K)`
- Lowest precedence: `+`, `+`.
- Rightmost `+` is evaluated last.
- **Node = `+`**
- Left Sub-Subtree: `A * B / C + E / F * G`
- Right Sub-Subtree: `K` (Leaf Node! ✅)

**Step 3: Continue Left** `(A * B / C + E / F * G)`
- Lowest precedence: `+`.
- **Node = `+`**
- Left Part: `A * B / C`
- Right Part: `E / F * G`

**Step 4: Resolve Multiplication/Division Blocks**
- Take `A * B / C`:
  - `*` and `/` have same precedence. Associativity is Left-to-Right.
  - Rightmost operator is `/` (Evaluated last).
  - **Node = `/`** $\rightarrow$ Left child is `A * B` (which becomes node `*` with children `A`, `B`). Right child is `C`.
- Take `E / F * G`:
  - Rightmost operator is `*`.
  - **Node = `*`** $\rightarrow$ Left child is `E / F` (which becomes node `/` with children `E`, `F`). Right child is `G`.

**Step 5: Build Right Subtree** `(X * Y)`
- Only one operator.
- **Node = `*`** $\rightarrow$ Left child `X`, Right child `Y`.

### Text Representation of Final Tree
```text
                  [-]
                /     \
             [+]      [*]
            /   \     / \
         [+]    [K] [X] [Y]
        /   \
      [/]   [*]
      / \   / \
    [*] [C][/] [G]
    / \    / \
  [A] [B][E] [F]
```

## 🔄 Relationship to Traversals
Once the expression tree is built, traversing it yields the different expression notations:

| Traversal Type | Order | Output Result |
| :--- | :--- | :--- |
| **Preorder Traversal** | Root $\rightarrow$ Left $\rightarrow$ Right | **Prefix Expression** |
| **Postorder Traversal** | Left $\rightarrow$ Right $\rightarrow$ Root | **Postfix Expression** |
| **Inorder Traversal** | Left $\rightarrow$ Root $\rightarrow$ Right | **Infix Expression** *(May require manual parentheses insertion)* |

> **Exam Tip:** A very common 2-mark question asks: *"Which tree traversal of an expression tree yields a Postfix expression?"* Answer: **Postorder traversal**.
