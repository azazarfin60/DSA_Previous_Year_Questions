# 6.12 Bridges (Cut Edge) in a Graph | Graph Theory & Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=1eaJHTgOFOQ) *(Note: Placeholder link)*

---

## 📌 Executive Summary
In Graph Theory, a **Bridge** (also known as a **Cut Edge**) is an edge whose removal strictly increases the number of connected components in the graph. In simpler terms, if deleting an edge causes a connected graph to become disconnected into two separate pieces, that edge is a Bridge.

---

## 🔑 Core Concept & Logic
To efficiently find all bridges in a graph, we use **Depth First Search (DFS)** augmented with time-tracking variables. This is the foundation of *Tarjan's Bridge-Finding Algorithm*.

We must maintain two timestamps for every vertex during the DFS:
1. **Starting Time (`discovery_time`):** The sequence number when the vertex is first visited.
2. **Lowest Reachable Time (`lowest_time`):** The smallest starting time of any node that can be reached from this vertex (excluding its direct parent).

### The "I Know Someone Better" Intuition
The video lecturer uses an excellent analogy:
- When DFS is backtracking from a child node $U$ to its parent node $V$, the parent $V$ essentially threatens: *"If I remove the edge between us, you will be disconnected."*
- The child $U$ checks its `lowest_time`.
- If $U$ has a path to an ancestor of $V$ (meaning `lowest_time[U] <= discovery_time[V]`), $U$ replies: *"No, I won't be disconnected. I know someone discovered earlier than you, so I have an alternate path."* $\implies$ **NOT A BRIDGE**.
- If $U$ only connects to its own descendants (meaning `lowest_time[U] > discovery_time[V]`), $U$ has no alternate path back up. $\implies$ **🚨 IT IS A BRIDGE**.

---

## ⚙️ Step-by-Step Algorithm
1. Start DFS from any arbitrary root vertex. Set its `discovery_time` and `lowest_time` to 1.
2. As you visit each new vertex, increment a global timer and assign both `discovery_time` and `lowest_time` to the current timer value.
3. **During DFS Traversal of Node $U$:**
   - Look at all adjacent vertices $W$ of $U$.
   - **Ignore the Parent:** If $W$ is the parent of $U$, skip it.
   - **Forward Edge:** If $W$ is unvisited, call `DFS(W)`. When the recursive call returns, update $U$'s lowest time: 
     `lowest_time[U] = min(lowest_time[U], lowest_time[W])`
   - **Back Edge:** If $W$ is already visited (and is not the parent), it means we found an alternate path! Update $U$'s lowest time:
     `lowest_time[U] = min(lowest_time[U], discovery_time[W])`
4. **Bridge Condition Check:**
   - After the loop over all adjacent vertices finishes, and right before $U$ backtracks to its parent $V$, check the critical condition:
   - **If `lowest_time[U] > discovery_time[V]` $\implies$ The edge $V \rightarrow U$ is a Bridge!**

---

## 📐 Algorithm Trace Example
Let's trace a fragment of a graph: $V \rightarrow U \rightarrow W$
- $V$'s `discovery_time` = 10.
- $U$'s `discovery_time` = 11.
- $W$'s `discovery_time` = 12.

**Scenario A (No Alternate Path):**
- $W$ cannot reach anywhere else. Its `lowest_time` stays 12.
- $W$ backtracks to $U$. $U$ updates its `lowest_time` to `min(11, 12) = 11`.
- $U$ backtracks to $V$. Check condition: `lowest_time[U] (11) > discovery_time[V] (10)`.
- **Result:** $V \rightarrow U$ is a Bridge.

**Scenario B (Alternate Path exists):**
- $W$ has a back-edge pointing directly to $V$. 
- $W$ updates its `lowest_time` to $V$'s discovery time: `min(12, 10) = 10`.
- $W$ backtracks to $U$. $U$ takes the minimum from $W$: `lowest_time[U] = min(11, 10) = 10`.
- $U$ backtracks to $V$. Check condition: `lowest_time[U] (10) > discovery_time[V] (10)` is **FALSE**.
- **Result:** $V \rightarrow U$ is NOT a Bridge.

---

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Time Complexity** | **$\Theta(V + E)$** | The algorithm is a single slightly-modified DFS pass. In the worst case, every vertex and every edge is visited exactly once. |
| **Space Complexity** | **O(V)** | Needed for the recursion stack, `discovery_time` array, and `lowest_time` array. |

> **Exam Tip:** In university exams, you will likely be given a graph and asked to list the bridges. You can usually find them visually by looking for edges that don't belong to any cycle. However, if asked to trace the algorithm, you **must** draw a table showing the `discovery_time` and `lowest_time` for every node to prove the mathematical condition `lowest_time[Child] > discovery_time[Parent]`.
