## 📌 Executive Summary
When inserting data into a Circular Linked List (CLL), it is highly optimized to maintain **only a `tail` pointer**. Because `tail->next` always gives us the address of the first node (the `head`), we can perform insertions at both the beginning and the end of the list in **$O(1)$ time**.

This note assumes we are using the optimized single-pointer (`tail`) approach.

---

## 1️⃣ Case 1: Insertion at the Beginning

To insert a new node at the beginning, we must place it exactly between the `tail` and the current first node (`tail->next`). We do not move the `tail` pointer itself, we only update what it points to.
**Time Complexity:** $O(1)$

### Algorithm Steps:
1. Create `newnode` and set its `data`.
2. **Empty List Check:** If `tail == NULL`, set `tail = newnode` and `tail->next = newnode` (circle of 1).
3. **Connect right-side:** `newnode->next = tail->next;` (New node points to the old first node).
4. **Connect left-side:** `tail->next = newnode;` (Tail now points to the new first node).

### C Code Snippet
```c
void insertAtBeginning() {
    struct node *newnode;
    newnode = (struct node*) malloc(sizeof(struct node));
    printf("Enter data: ");
    scanf("%d", &newnode->data);
    newnode->next = NULL;

    if (tail == NULL) {
        tail = newnode;
        tail->next = newnode;
    } else {
        // 1. Secure link to the first node
        newnode->next = tail->next;
        // 2. Update tail to point to the new first node
        tail->next = newnode;
    }
}
```

## 2️⃣ Case 2: Insertion at the End

Inserting at the end is almost identical to inserting at the beginning. We wedge the node between the current `tail` and the first node. The *only* difference is that we must mathematically declare this new node as the end of the list by shifting the `tail` pointer to it.
**Time Complexity:** $O(1)$

### C Code Snippet
```c
void insertAtEnd() {
    struct node *newnode;
    newnode = (struct node*) malloc(sizeof(struct node));
    printf("Enter data: ");
    scanf("%d", &newnode->data);
    newnode->next = NULL;

    if (tail == NULL) {
        tail = newnode;
        tail->next = newnode;
    } else {
        newnode->next = tail->next; 
        tail->next = newnode;
        
        // 🌟 THE ONLY DIFFERENCE FROM INSERT_BEGINNING 🌟
        tail = newnode; // Shift tail to the newly inserted node
    }
}
```

> [!CAUTION]
> **Exam Trap (The Lost Circle):**
> Look closely at the `else` block above. What happens if you write `tail->next = newnode;` *before* `newnode->next = tail->next;`?
> If you overwrite `tail->next` first, you have instantly erased the address of the first node! `tail->next` was the *only* variable storing where the circle began. Always secure the right-side link (`newnode->next = tail->next;`) before overwriting `tail->next`.

## 3️⃣ Case 3: Insertion at a Specific Position

To insert between two intermediate nodes, we must traverse to the node *just before* the desired position. Because we don't have a `head` variable, we initialize our traversal pointer `temp` to `tail->next`.
**Time Complexity:** $O(n)$

### Algorithm Steps:
1. Initialize `temp = tail->next;` (Points to the first node).
2. Traverse to `pos - 1` using a `while` loop.
3. Update pointers (Right-side first, then left-side).

### C Code Snippet
```c
void insertAtPosition() {
    struct node *newnode, *temp;
    int pos, i = 1;
    
    printf("Enter position: ");
    scanf("%d", &pos);
    
    // (Assume length validation code is here...)

    newnode = (struct node*) malloc(sizeof(struct node));
    printf("Enter data: ");
    scanf("%d", &newnode->data);
    newnode->next = NULL;

    // 1. Initialize temp to the first node
    temp = tail->next;
    
    // 2. Traverse to the node immediately before the desired position
    while (i < pos - 1) {
        temp = temp->next;
        i++;
    }
    
    // 3. Update links (ALWAYS right-side first)
    newnode->next = temp->next; // Secure the right-side node
    temp->next = newnode;       // Connect the left-side node
}
```
