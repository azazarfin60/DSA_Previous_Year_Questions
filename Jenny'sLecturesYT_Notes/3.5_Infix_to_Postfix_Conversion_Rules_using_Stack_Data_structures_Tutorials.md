## ­¤ōī Executive Summary
Converting an Infix expression (human-readable) to a Postfix expression (machine-readable) relies heavily on the **Stack** data structure (LIFO - Last In First Out). The stack temporarily holds **operators** and **parentheses** while ensuring that mathematical precedence and associativity rules are strictly maintained.

---

## ­¤öæ The 6 Golden Conversion Rules
When scanning an Infix expression from **Left to Right**, process each symbol using the following rules:

### 1. Operands (A, B, 1, 2)
- **Rule:** If the incoming symbol is an operand, **do not put it in the stack**.
- **Action:** Immediately print it (append it to the postfix output string).

### 2. Left Parenthesis `(`
- **Rule:** If the incoming symbol is `(`, it has the highest priority to enter.
- **Action:** Push it directly onto the stack.

### 3. Right Parenthesis `)`
- **Rule:** A closing parenthesis triggers the evaluation of the sub-expression.
- **Action:** Repeatedly **Pop** operators from the stack and print them to the output **until** a Left Parenthesis `(` is found at the top of the stack. Finally, pop the `(` and discard it (do not print it).

### 4. Empty Stack or `(` on Top
- **Rule:** If the incoming symbol is an operator and the stack is empty, OR the top of the stack is `(`, there are no precedence conflicts.
- **Action:** Push the incoming operator onto the stack.

### 5. Incoming Operator (+, -, *, /, ^)
When an operator arrives and the stack already has an operator on top, you must compare their **Precedence**.

- **Case A: Higher Precedence**
  - If Incoming > Top of Stack (e.g., Incoming `*`, Top `+`)
  - **Action:** Push the incoming operator onto the stack.
- **Case B: Lower Precedence**
  - If Incoming < Top of Stack (e.g., Incoming `+`, Top `*`)
  - **Action:** Pop the top operator and print it. Then **re-test** the incoming operator against the *new* top of the stack. Repeat until you can push it.
- **Case C: Equal Precedence**
  - If Incoming == Top of Stack (e.g., Incoming `*`, Top `/`)
  - **Action:** Check **Associativity**.
    - If associativity is **Left-to-Right** (for `+`, `-`, `*`, `/`): Pop and print the top operator, then Push the incoming operator.
    - If associativity is **Right-to-Left** (for `^`): Push the incoming operator.

### 6. End of Expression
- **Rule:** When you finish reading the infix string, the stack might still hold operators.
- **Action:** Pop all remaining operators from the stack one by one and print them to the output until the stack is empty.

## ­¤ōÉ Tracing Examples

### Example 1: `A + B / C`
| Symbol Scanned | Stack State (Top at right) | Postfix Output | Rule Applied |
| :--- | :--- | :--- | :--- |
| `A` | | `A` | Operand (Rule 1) |
| `+` | `+` | `A` | Empty Stack (Rule 4) |
| `B` | `+` | `A B` | Operand (Rule 1) |
| `/` | `+ /` | `A B` | `/` > `+` (Rule 5A) |
| `C` | `+ /` | `A B C` | Operand (Rule 1) |
| *End* | *Empty* | `A B C / +` | Pop Remaining (Rule 6) |

### Example 2: `A - B / C * D + E`
1. `A` $\rightarrow$ Print. Output: `A`
2. `-` $\rightarrow$ Stack Empty. Push. Stack: `[-]`
3. `B` $\rightarrow$ Print. Output: `A B`
4. `/` $\rightarrow$ `/` > `-`. Push. Stack: `[-, /]`
5. `C` $\rightarrow$ Print. Output: `A B C`
6. `*` $\rightarrow$ Incoming `*` == Top `/`. Left-to-Right associativity! **Pop `/`**, then push `*`. 
   - Output: `A B C /`. Stack: `[-, *]`
7. `D` $\rightarrow$ Print. Output: `A B C / D`
8. `+` $\rightarrow$ Incoming `+` < Top `*`. **Pop `*`**. 
   - Output: `A B C / D *`. Stack: `[-]`
   - Re-test: Incoming `+` == Top `-`. Left-to-Right associativity! **Pop `-`**, then push `+`. 
   - Output: `A B C / D * -`. Stack: `[+]`
9. `E` $\rightarrow$ Print. Output: `A B C / D * - E`
10. *End* $\rightarrow$ Pop remaining `+`. 
    - **Final Output:** `A B C / D * - E +`

> **Exam Tip:** The most common mistake students make is handling operators of **Equal Precedence**. Memorize this rule: For standard math operators (+, -, *, /), equal precedence means the top of the stack gets popped because of Left-to-Right associativity! Do not just stack them on top of each other.
