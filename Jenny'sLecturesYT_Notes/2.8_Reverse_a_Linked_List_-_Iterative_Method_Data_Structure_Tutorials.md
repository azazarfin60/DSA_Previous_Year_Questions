## 📌 Executive Summary

This lecture explains reversing a singly linked list using the **iterative 3-pointer technique** (`prevNode`, `currentNode`, `nextNode`). All links are reversed in a single traversal — O(n) time, O(1) space.

---

## 🔑 Concept

Reversing = **reversing the direction of all links** (not swapping data).

```
Before:  Head → [5|200] → [6|150] → [1|300] → [7|NULL]
After:   Head → [7|150] → [1|200] → [6|100] → [5|NULL]
```

---

## 📐 Algorithm

| Pointer | Purpose | Initial Value |
|---------|---------|---------------|
| `prevNode` | Tracks previous node | NULL |
| `currentNode` | Current node being processed | head |
| `nextNode` | Saves next before link breaks | head |

### Steps per iteration:
```
1. nextNode = nextNode->next       // Save next
2. currentNode->next = prevNode    // Reverse link
3. prevNode = currentNode          // Advance prev
4. currentNode = nextNode          // Advance current
```
After loop: `head = prevNode`

---

## 💻 C Code

```c
void reverse() {
    struct node *prevNode = NULL;
    struct node *currentNode = head;
    struct node *nextNode = head;

    while (nextNode != NULL) {
        nextNode = nextNode->next;      // 1. Save next
        currentNode->next = prevNode;   // 2. Reverse link
        prevNode = currentNode;         // 3. Move prev
        currentNode = nextNode;         // 4. Move current
    }
    head = prevNode;  // Update head to last node
}
```

## 🔄 Step-by-Step Trace

```
Initial: [5|200] → [6|150] → [1|300] → [7|NULL]
         prev=NULL, curr=100, next=100

Iter 1: next=200, [5|NULL], prev=100, curr=200
Iter 2: next=150, [6|100], prev=200, curr=150
Iter 3: next=300, [1|200], prev=150, curr=300
Iter 4: next=NULL,[7|150], prev=300, curr=NULL → STOP

head = prev = 300 → [7|150] → [1|200] → [6|100] → [5|NULL] ✅
```

---

## ⏱️ Complexity

| Metric | Value |
|--------|-------|
| **Time** | **O(n)** — single traversal |
| **Space** | **O(1)** — only 3 pointers |

---

## 💡 Key Takeaways

- **3 pointers**: previous, current, next
- Save next **before** breaking the link
- After loop, **update head** to `prevNode`
- Works correctly even with a single node
- Order: save → reverse → advance
