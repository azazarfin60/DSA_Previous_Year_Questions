# 2.8 Reverse a Linked List (Iterative Method) | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
Reversing a linked list does **not** mean swapping the data values between nodes. Instead, it means reversing the direction of the `next` pointers so that the last node becomes the first node, and every node points to its previous node. Iteratively reversing a list requires a precise "3-Pointer Sliding Window" technique to avoid disconnecting and losing the list.

---

## 🧠 The 3-Pointer Concept

To safely reverse a link, we need to know three things simultaneously:
1. **The node we are currently reversing:** (`currentnode`)
2. **Where it should point to now:** (`prevnode` - the node behind it)
3. **Where to go next:** (`nextnode` - the rest of the unreversed list)

If we only use two pointers and say `currentnode->next = prevnode`, the link to the rest of the list is instantly severed, and the remainder of the list is lost in memory. The `nextnode` pointer acts as a temporary anchor to prevent this.

### Initializing the Pointers
```c
struct node *prevnode = NULL;   // The first node's new 'next' will be NULL
struct node *currentnode = head;
struct node *nextnode = head;
```

---

## 🛠️ The Iterative Algorithm

### The 4-Step Loop (`while nextnode != NULL`):

1. **Advance `nextnode`:** First, secure the rest of the list by moving `nextnode` forward.
   `nextnode = nextnode->next;`
2. **Reverse the Link:** Now it's safe to break the current node's forward link and point it backward.
   `currentnode->next = prevnode;`
3. **Slide `prevnode` forward:** Move `prevnode` up to where we are currently standing.
   `prevnode = currentnode;`
4. **Slide `currentnode` forward:** Move `currentnode` up to the rest of the list (where `nextnode` is waiting).
   `currentnode = nextnode;`

### Final Step:
When the loop terminates, `currentnode` and `nextnode` will both be `NULL` (they fell off the end). But `prevnode` will be pointing exactly at the *last* node of the original list. 
**You must update the `head` pointer to this node to finalize the reversal.**

---

## 💻 C Code Snippet

```c
void reverseList() {
    struct node *prevnode = NULL;
    struct node *currentnode = head;
    struct node *nextnode = head;

    // Slide the 3 pointers down the list
    while (nextnode != NULL) {
        nextnode = nextnode->next;     // 1. Secure the remainder
        currentnode->next = prevnode;  // 2. Reverse the actual link
        
        // 3 & 4. Slide current and prev forward for the next iteration
        prevnode = currentnode;
        currentnode = nextnode;
    }

    // Assign head to the new starting node
    head = prevnode;
}
```

> [!CAUTION]
> **Exam Trap (Forgetting the Head):** The most common mistake in this algorithm is completing the while loop perfectly, but forgetting the final `head = prevnode;` line outside the loop. If you forget this, `head` will still point to the original first node (which now has `next = NULL`), and your entire reversed list will be completely inaccessible, resulting in an output of just one element!

---

## ⏱️ Complexity Analysis
- **Time Complexity:** $O(n)$. We pass through the linked list exactly once.
- **Space Complexity:** $O(1)$. We only allocate memory for three temporary pointer variables, regardless of the length of the list.
