# 1.2 Array Operations: Traversal & Insertion | Data Structures

## 📌 Executive Summary
The primary operations performed on a 1D Array include Traversal and Insertion. Because an array occupies contiguous memory and has a fixed capacity, inserting a new element often requires shifting existing elements to avoid overwriting data, resulting in a linear time complexity $O(n)$ in the worst case.

---

## 🚶‍♂️ Traversal
Traversal means visiting every element of the array exactly once.
- **Process:** Use a simple loop from index `0` to `size - 1`.
- **Time Complexity:** $O(n)$.

**C Code Snippet:**
```c
for(int i = 0; i < size; i++) {
    printf("%d ", a[i]);
}
```

---

## 📥 Array Insertion Algorithms

When inserting an element at a specific position `pos` (where `index = pos - 1`), you must consider whether the array is sorted or unsorted.

### 1. Insertion in a SORTED Array (or Preserving Order)
If the array is sorted (or relative order matters), you **must shift elements** to the right to make space for the new element.

- **Process:** Start a loop from the *end* of the array (`size - 1`) down to the target `index`. Shift each element one position to the right.
- **Time Complexity:** 
  - Best Case (insert at end): $O(1)$
  - Worst Case (insert at beginning): $O(n)$
  - Average Case: $O(n)$

**C Code Snippet:**
```c
// Shift elements to the right
for(int i = size - 1; i >= pos - 1; i--) {
    a[i+1] = a[i]; 
}
a[pos-1] = num; // Insert new number
size++;         // Increase array size
```

### 2. Insertion in an UNSORTED Array (Exam Hack!)
If the array is unsorted and order does not matter, **do not shift elements**. Shifting is computationally expensive.
- **Process:** Take the element currently sitting at the target `pos-1`, move it to the very end of the array (at index `size`), and then insert the new element at `pos-1`.
- **Time Complexity:** $O(1)$ constant time!

**C Code Snippet:**
```c
a[size] = a[pos-1]; // Move existing element to the end
a[pos-1] = num;     // Insert new number at target
size++;
```

> [!WARNING]
> **Exam Trap:**
> Arrays do not have automatic upper-bound checking. Before any insertion, you must explicitly check if the array is full (`size >= MAX_CAPACITY`) to prevent an **Overflow Condition**.
