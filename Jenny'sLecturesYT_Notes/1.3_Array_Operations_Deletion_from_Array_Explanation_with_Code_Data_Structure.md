# 1.3 Array Operations: Deletion | Data Structures

## 📌 Executive Summary
Deleting an element from a 1D Array involves removing the data at a specific position. Because arrays must maintain contiguous memory without "empty holes", deletion usually requires shifting elements, leading to an $O(n)$ time complexity. However, a constant time $O(1)$ trick exists for unsorted arrays.

---

## 🗑️ Array Deletion Algorithms

When deleting an element from a specific position `pos` (where `index = pos - 1`), the procedure depends entirely on whether the array is sorted.

### 1. Deletion in a SORTED Array (or Preserving Order)
To prevent "empty holes" in contiguous memory, you must **shift elements to the left** to fill the gap left by the deleted element.

- **Process:** Start a loop from the target `index` up to `size - 1`. Shift each element one position to the left.
- **Time Complexity:** Worst Case $O(n)$.

**C Code Snippet:**
```c
// Shift elements to the left
for(int i = pos - 1; i < size - 1; i++) {
    a[i] = a[i+1];
}
size--; // Decrease array size
```

### 2. Deletion in an UNSORTED Array (Exam Hack!)
Just like insertion, if order doesn't matter, avoid shifting entirely.
- **Process:** Take the very last element in the array (`a[size-1]`), and use it to overwrite the element you want to delete at `pos-1`. Then, reduce the size of the array.
- **Time Complexity:** $O(1)$ constant time!

**C Code Snippet:**
```c
a[pos-1] = a[size-1]; // Overwrite target with the last element
size--;               // Decrease size (effectively deleting the old last element)
```

> [!IMPORTANT]
> **Exam Tip:**
> If a question asks for the "best algorithm" to insert/delete in an **unsorted** array, always provide the $O(1)$ swap-with-last method, not the $O(n)$ shifting method. Evaluators look for this specific optimization.
