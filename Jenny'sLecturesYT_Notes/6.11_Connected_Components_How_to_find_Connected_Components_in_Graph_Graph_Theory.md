## 📌 Executive Summary

This lecture explains **Connected Components** — maximal sets of vertices where every pair is connected by a path. Finding them uses BFS/DFS on each unvisited vertex.

---

## 🔑 Definition
> A **connected component** is a maximal subgraph where every pair of vertices is connected by a path, and no vertex outside has a path to any vertex inside.

---

## 📐 Algorithm
```c
int visited[MAX] = {0};
int components = 0;

for (int i = 0; i < V; i++) {
    if (!visited[i]) {
        BFS(i) or DFS(i);   // Marks all reachable vertices
        components++;
    }
}
```

---

## 📐 Example
```
Graph:
  0 — 1    3 — 4    6
  |   |    |
  2   5    7

Component 1: {0, 1, 2, 5}
Component 2: {3, 4, 7}
Component 3: {6}

Total connected components = 3
```

---

## 🎯 Applications
- Determine if graph is **connected** (1 component = connected)
- Network analysis (isolated groups)
- Island counting problems

---

## ⏱️ Complexity
- **Time: O(V + E)** — each vertex and edge visited once
- **Space: O(V)** — visited array

---

## 💡 Key Takeaways

- Run BFS/DFS from each unvisited vertex → each run = one component
- Connected graph = exactly **1 component**
- For directed graphs → **Strongly Connected Components** (Kosaraju's/Tarjan's)
