# 6.11 Connected Components in a Graph | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=1eaJHTgOFOQ) *(Note: Placeholder link)*

---

## 📌 Executive Summary
In Graph Theory, a **Connected Component** (in an undirected graph) is a maximum subgraph in which any two vertices are connected to each other by paths. Furthermore, no vertex in this subgraph is connected to any vertex outside of it in the original super-graph. Finding connected components is essential for network analysis, cluster detection, and image processing.

---

## 🔑 Core Concept
- If you can reach every node from every other node, the graph has exactly **1 Connected Component** (the graph is fully connected).
- If the graph is broken into isolated "islands" of nodes, each isolated island is exactly **1 Connected Component**.
- **Algorithm Choice:** Both Depth-First Search (DFS) and Breadth-First Search (BFS) can be used to find connected components. The transcript demonstrates the **DFS approach**.
- **The Golden Rule:** The total number of connected components in a graph is equal to **the number of times the DFS function is explicitly called from the main loop** (ignoring the recursive calls within the DFS function itself).

---

## ⚙️ The Algorithm (Using DFS)

### Data Structures Needed
1. A **Graph** represented as an Adjacency Matrix or Adjacency List.
2. A `flag` array (or `visited` array) of size $V$ (number of vertices), initialized to `-1` (unvisited).
3. A `count` variable, initialized to `0`, to track the number of components.

### Step-by-Step Logic
1. **Initialize:** 
   ```c
   int count = 0;
   for (int i = 0; i < N; i++) {
       flag[i] = -1; // -1 means unvisited
   }
   ```
2. **Iterate through all vertices:**
   Loop through every vertex $V$ in the graph (from $0$ to $N-1$).
   - Check if $V$ is unvisited (`flag[V] == -1`).
   - If it is unvisited, it means we have discovered a **new disconnected island**.
   - Call the `DFS(V)` function.
   - After the `DFS` function fully returns, increment `count++` and optionally print a newline to separate components.
3. **The DFS Function (`DFS(V)`):**
   - Mark the current vertex as visited: `flag[V] = 1`.
   - Print the vertex $V$ (this groups it visually into the current component).
   - Look at all adjacent vertices $U$ of $V$.
   - If an adjacent vertex $U$ is unvisited (`flag[U] == -1`), recursively call `DFS(U)`.

### Pseudocode Core
```c
// Main Loop
for(int v = 0; v < N; v++) {
    if(flag[v] == -1) {
        DFS(v);
        count++;
        printf("\n"); // Move to next line for the next component
    }
}
printf("Total Connected Components: %d", count);
```

## 📐 Algorithm Trace Example
Imagine a graph with 9 vertices (0 to 8) containing three isolated islands:
- **Island 1:** 0, 1, 2, 3
- **Island 2:** 4, 5
- **Island 3:** 6, 7, 8

**Execution Trace:**
1. Loop starts at $V = 0$. `flag[0]` is `-1`.
2. Calls `DFS(0)`.
   - DFS recursively visits and prints `0, 1, 2, 3`, marking their flags as `1`.
3. DFS returns to the main loop. `count` becomes `1`.
4. Loop continues for $V = 1, 2, 3$. Their flags are already `1`, so the `if` condition is false. DFS is NOT called.
5. Loop reaches $V = 4$. `flag[4]` is `-1`.
6. Calls `DFS(4)`.
   - DFS recursively visits and prints `4, 5`, marking their flags as `1`.
7. DFS returns. `count` becomes `2`.
8. Loop reaches $V = 5$. Flag is `1`. Ignored.
9. Loop reaches $V = 6$. `flag[6]` is `-1`.
10. Calls `DFS(6)`.
    - DFS recursively visits and prints `6, 7, 8`, marking their flags as `1`.
11. DFS returns. `count` becomes `3`.
12. Loop finishes. Total connected components = 3.

---

> **Exam Tip:** If an exam asks you to find connected components programmatically, the simplest and most mark-friendly way to write it is using a `for` loop that wraps a standard DFS function. Always emphasize that a single DFS call recursively swallows an entire connected subgraph, which is why counting the number of outer-loop triggers gives the exact number of components.
