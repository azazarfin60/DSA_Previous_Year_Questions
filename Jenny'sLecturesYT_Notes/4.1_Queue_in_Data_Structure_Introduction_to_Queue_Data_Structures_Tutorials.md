## 📌 Executive Summary

This lecture introduces the **Queue** — a FIFO (First In, First Out) linear data structure. It covers enqueue, dequeue operations, array implementation, and the space wastage problem solved by circular queues.

---

## 🔑 Definition
> A Queue follows **FIFO** — the first element enqueued is the first one dequeued.

### Real-Life Analogies
- People in a line (first come, first served)
- Printer queue, CPU scheduling

---

## 💻 Operations

| Operation | Description | Time |
|-----------|-------------|------|
| `enqueue(x)` | Insert at rear | O(1) |
| `dequeue()` | Remove from front | O(1) |
| `peek()` | View front element | O(1) |
| `isEmpty()` | Check if empty | O(1) |

### Conditions
- **Overflow** → enqueue when full
- **Underflow** → dequeue when empty

---

## 💻 Array Implementation
```c
#define MAX 100
int queue[MAX];
int front = -1, rear = -1;

void enqueue(int data) {
    if (rear == MAX - 1) { printf("Overflow!\n"); return; }
    if (front == -1) front = 0;
    queue[++rear] = data;
}

int dequeue() {
    if (front == -1 || front > rear) { printf("Underflow!\n"); return -1; }
    return queue[front++];
}
```

### ⚠️ Space Wastage Problem
```
After several dequeue operations:
[_, _, 3, 4, 5]  front=2, rear=4
Positions 0,1 wasted — can't reuse!
```

**Solution: Circular Queue** using `(index + 1) % MAX`

---

## 📊 Stack vs Queue

| Feature | Stack | Queue |
|---------|-------|-------|
| Principle | LIFO | **FIFO** |
| Insert | push (top) | enqueue (rear) |
| Delete | pop (top) | dequeue (front) |
| Pointers | 1 (top) | 2 (front, rear) |

---

## 💡 Key Takeaways

- Queue = **FIFO**, uses **two pointers** (front and rear)
- Linear array queue has space wastage → use **circular queue**
- Applications: CPU scheduling, BFS, print queue, buffers
