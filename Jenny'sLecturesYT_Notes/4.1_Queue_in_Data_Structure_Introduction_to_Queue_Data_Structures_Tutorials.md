# 4.1 Queue Data Structure: Introduction and ADT | Data Structures

---

## 📌 Executive Summary
A **Queue** is a linear data structure (or Abstract Data Type) that stores a collection of elements. Unlike a Stack which follows LIFO (Last In First Out), a Queue strictly follows the **FIFO (First In First Out)** or **LILO (Last In Last Out)** principle.

The person who stands in line first gets the movie ticket first. The data inserted first is the data deleted first.

---

## 🛠️ The Two Ends of a Queue

To enforce the FIFO rule, a Queue operates using two distinct ends:
1. **Rear (or Tail):** All **insertions** happen at this end.
2. **Front (or Head):** All **deletions** happen at this end.

---

## ⚙️ Fundamental Operations (Queue ADT)

When defining a Queue as an Abstract Data Type (ADT), we define the following core operations. All of these typically execute in **$O(1)$ constant time**.

| Operation | Description |
| :--- | :--- |
| **`enqueue(data)`** | Adds an element to the `rear` of the queue. |
| **`dequeue()`** | Removes and returns the element at the `front` of the queue. |
| **`peek()` / `front()`** | Returns the element at the `front` *without* removing it. |
| **`isFull()`** | Checks if the queue has reached its maximum capacity. |
| **`isEmpty()`** | Checks if the queue has no elements. |

---

## 📉 Logical Representation & Status Conditions

When implemented using an array, we maintain two integer variables: `front` and `rear`, representing indices.

### The Empty State
- By default, an empty queue is represented by: `front = -1` and `rear = -1`.
- **Underflow Condition:** If you attempt to `dequeue()` when the queue is empty, an underflow error occurs.

### The Full State
- As we `enqueue()`, `rear` increments (`rear++`).
- **Overflow Condition:** If `rear == MAX_SIZE - 1`, the queue is full. You cannot `enqueue()` any more data.

> [!WARNING]
> **Exam Concept (The Linear Queue Drawback):**
> A major disadvantage of a standard linear queue implemented via arrays is **memory wastage**. 
> Suppose `MAX_SIZE` is 5. We enqueue 5 elements. `rear` is now 4. The queue is FULL. 
> If we then dequeue 3 elements, `front` shifts to 3. Indices 0, 1, and 2 are now completely empty and unused! However, if we try to enqueue a new element, the program will throw an Overflow error because `rear == 4` still evaluates to true. 
> *Solution:* This drawback is solved by using a **Circular Queue**.

---

## 🌍 Real-World Applications of Queues

Queues are utilized whenever data or requests must be processed in the exact order they arrive:

1. **Shared Resource Management:** When a single resource is shared among multiple consumers.
   - *Example:* A shared network **Printer**. If 5 computers send print commands simultaneously, the printer stores the commands in a queue and prints them First Come, First Served.
2. **Customer Service Systems:** Call center systems placing callers on hold in a queue until a representative becomes available.
3. **Operating Systems (CPU Scheduling):** 
   - When multiple processes need to execute, the OS places them in a **Ready Queue**.
   - Processes waiting for I/O operations are placed in a **Waiting Queue**. The CPU fetches them sequentially.
