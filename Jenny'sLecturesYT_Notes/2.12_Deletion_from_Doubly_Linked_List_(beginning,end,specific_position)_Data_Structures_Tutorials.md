# 2.12 Deletion from a Doubly Linked List (Beginning, End, Specific Position) | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
Deleting a node from a Doubly Linked List (DLL) is generally more efficient and straightforward than in a Singly Linked List (SLL). Because every node has a `prev` pointer, we only need to maintain **one pointer (`temp`)** to perform deletions at any position. Furthermore, if a `tail` pointer is maintained, deleting from the end becomes an $O(1)$ operation. 

As always, never forget to use the `free()` function to release the memory back to the system.

---

## 1️⃣ Case 1: Deletion from the Beginning

Deleting the first node requires moving the `head` pointer to the second node and completely severing the backward link.
**Time Complexity:** $O(1)$

### Algorithm Steps:
1. Check if the list is empty (`head == NULL`).
2. Point a `temp` pointer to `head`.
3. Shift the `head` forward: `head = head->next;`
4. Set the new first node's previous pointer to NULL: `head->prev = NULL;`
5. Free the memory: `free(temp);`

### C Code Snippet
```c
void deleteFromBeginning() {
    struct node *temp;
    
    if (head == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    temp = head;            // 1. Anchor the node to be deleted
    head = head->next;      // 2. Move head to the next node
    
    // 3. Sever the backward link from the new head
    if (head != NULL) {     // Check in case it was the only node
        head->prev = NULL;
    }
    
    free(temp);             // 4. Release memory
}
```

---

## 2️⃣ Case 2: Deletion from the End (Using `tail`)

In an SLL, deleting the last node took $O(n)$ time because we had to traverse the list with two pointers. In a DLL with a `tail` pointer, we can instantly access the last node and look backward to the second-to-last node.
**Time Complexity:** $O(1)$

### Algorithm Steps:
1. Check if the list is empty (`tail == NULL`).
2. Point a `temp` pointer to `tail`.
3. Shift the `tail` backward: `tail = tail->prev;`
4. Set the new last node's next pointer to NULL: `tail->next = NULL;`
5. Free the memory: `free(temp);`

### C Code Snippet
```c
void deleteFromEnd() {
    struct node *temp;
    
    if (tail == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    temp = tail;            // 1. Anchor the last node
    tail = tail->prev;      // 2. Move tail backward to the second-to-last node
    
    // 3. Sever the forward link from the new tail
    if (tail != NULL) {     // Check in case it was the only node
        tail->next = NULL;
    }
    
    free(temp);             // 4. Release memory
}
```

---

## 3️⃣ Case 3: Deletion from a Specific Position

To delete the $k^{th}$ node, we traverse the list until `temp` is pointing exactly at the $k^{th}$ node. Because `temp` has bidirectional links, we can easily bypass it by connecting its left neighbor directly to its right neighbor.
**Time Complexity:** $O(n)$ (Average case)

### Algorithm Steps:
1. Traverse using a `while` loop to land `temp` exactly on the node to be deleted.
2. Update the left node's forward link: `temp->prev->next = temp->next;`
3. Update the right node's backward link: `temp->next->prev = temp->prev;`
4. Free the memory: `free(temp);`

### C Code Snippet
```c
void deleteFromPosition() {
    struct node *temp;
    int pos, i = 1;
    
    printf("Enter position to delete: ");
    scanf("%d", &pos);
    
    temp = head;
    
    // 1. Traverse UNTIL temp points EXACTLY to the node to be deleted
    while (i < pos) {
        temp = temp->next;
        i++;
    }
    
    // 2. Left neighbor bypasses temp, points to right neighbor
    if (temp->prev != NULL) {
        temp->prev->next = temp->next;
    }
    
    // 3. Right neighbor bypasses temp, points to left neighbor
    if (temp->next != NULL) {
        temp->next->prev = temp->prev;
    }
    
    // 4. Release memory
    free(temp);
}
```

> [!TIP]
> **Exam Concept (DLL Superiority):** The deletion from a specific position highlights why Doubly Linked Lists are powerful. Notice that we only declared `struct node *temp`. We did not need a `prevnode` pointer like we did in a Singly Linked List, because DLL nodes intrinsically know who is behind them (`temp->prev`).
