# 5.20 Splay Tree Insertion | Data structure

📺 [Watch on YouTube](https://www.youtube.com/watch?v=1HeIZNP3w4A)

---

## 📌 Executive Summary

Splay Tree is a **self-adjusting BST** that moves the most recently accessed node to the **root** using **splaying** rotations (Zig, Zig-Zig, Zig-Zag). No balance factor needed.

---

## 🔑 Splay Tree Concept
> After every access (insert, search, delete), the accessed node is moved to the root via **splaying**. Recently accessed nodes are always near the root.

### Splaying Operations

| Case | Condition | Action |
|------|-----------|--------|
| **Zig** | Node is child of root | Single rotation |
| **Zig-Zig** | Node & parent are both left (or both right) children | Two same-direction rotations |
| **Zig-Zag** | Node is left child of right child (or vice versa) | Two opposite-direction rotations |

---

## 📐 Zig (Single Rotation)
```
Node x is child of root p:

    p              x
   / \    →       / \
  x   C         A    p
 / \                / \
A   B              B   C
```

## 📐 Zig-Zig (Same direction — both left or both right)
```
    g              x
   / \            / \
  p   D   →     A    p
 / \                / \
x   C              B    g
/ \                   / \
A  B                 C   D
```

## 📐 Zig-Zag (Opposite directions)
```
    g              x
   / \            / \
  p   D   →     p    g
 / \           / \  / \
A   x         A  B C   D
   / \
  B   C
```

---

## 📐 Insertion Example: Insert 5, 10, 15

```
Insert 5: [5]  (root — no splay needed)

Insert 10: BST insert → 5→10
           Splay 10: Zig
           Result: [10] with 5 as left child

Insert 15: BST insert → 10→15
           Splay 15: Zig
           Result: [15] with 10 as left, 5 as left of 10
```

---

## ⏱️ Complexity

| Operation | Amortized | Worst Case |
|-----------|-----------|------------|
| Search | **O(log n)** | O(n) |
| Insert | **O(log n)** | O(n) |
| Delete | **O(log n)** | O(n) |

---

## 💡 Key Takeaways

- No balance factor — relies on **splaying** to root
- Three splay cases: Zig, Zig-Zig, Zig-Zag
- Recently accessed = near root = **fast future access**
- **Amortized O(log n)** — good for locality of reference
- Simpler to implement than AVL/RBT
