## 📌 Executive Summary

This lecture introduces the **Doubly Linked List (DLL)** — a linked list where each node has a pointer to both the **previous** and the **next** node, enabling bidirectional traversal.

---

## 🔑 Node Structure

```
┌──────┬──────┬──────┐
│ PREV │ DATA │ NEXT │
│  4B  │  4B  │  4B  │
└──────┴──────┴──────┘
    12 bytes per node (32-bit)
```

### C Definition
```c
struct node {
    struct node *prev;   // Pointer to previous node
    int data;
    struct node *next;   // Pointer to next node
};
struct node *head = NULL;
```

### Memory Representation
```
Head → [NULL|5|200] ⇄ [100|6|300] ⇄ [200|8|NULL]
         @100            @200            @300
```

---

## 📊 Singly vs Doubly Linked List

| Feature | SLL | DLL |
|---------|-----|-----|
| Pointers per node | 1 (next) | 2 (prev + next) |
| Memory per node | 8 bytes | 12 bytes |
| Traversal | Forward only | Forward + Backward |
| Delete given node ptr | Need prev → O(n) | Direct → **O(1)** |

---

## 🎯 Advantages of DLL
1. **Bidirectional traversal** — can go forward and backward
2. **Easier deletion** — given a pointer to a node, can delete in O(1) without needing previous node
3. **Can traverse backward** from any node

## ⚠️ Disadvantages
1. **More memory** per node (extra pointer)
2. **More pointer updates** during insertion/deletion

---

## 💡 Key Takeaways

- DLL node = **prev + data + next**
- First node's `prev = NULL`, last node's `next = NULL`
- Enables **bidirectional traversal** at the cost of extra memory
- Key advantage: deletion of a given node in **O(1)**
