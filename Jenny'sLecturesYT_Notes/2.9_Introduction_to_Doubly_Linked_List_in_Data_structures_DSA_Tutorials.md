# 2.9 Introduction to Doubly Linked Lists | Data Structures

## 📌 Executive Summary
A **Doubly Linked List (DLL)** is a variation of the linked list where each node contains *two* pointers: one pointing to the next node in the sequence, and one pointing to the previous node. This structural change allows for bidirectional traversal at the cost of increased memory overhead per node.

---

## 🔗 Structure of a Doubly Linked List Node

Unlike a Singly Linked List (which has two parts: Data, Next), a Doubly Linked List node has **three parts**:
1. **Prev Pointer:** Stores the memory address of the previous node.
2. **Data Part:** Stores the actual information.
3. **Next Pointer:** Stores the memory address of the next node.

### Logical Representation
```text
NULL <- [ Prev | Data | Next ] <-> [ Prev | Data | Next ] -> NULL
  ^                                                           ^
 Head                                                        End
```
- The `prev` pointer of the **first node** always contains `NULL`.
- The `next` pointer of the **last node** always contains `NULL`.

### C Implementation of Node
```c
struct node {
    int data;               // Data part
    struct node *prev;      // Pointer to previous node
    struct node *next;      // Pointer to next node
};

struct node *head = NULL;   // Global head pointer
```

---

## ⚖️ Singly vs Doubly Linked List (Advantages & Disadvantages)

### Advantages of Doubly Linked List (over SLL) ✅
1. **Bidirectional Traversal:** You can traverse the list both forwards and backwards. This makes reversing or printing the list in reverse order incredibly simple.
2. **$O(1)$ Backward Lookups:** If a pointer `temp` is at a node, you can instantly look at the previous node using `temp->prev`. In a Singly Linked List, you cannot look backward at all.
3. **Easier Deletion:** To delete a specific node in an SLL, you must maintain *two* pointers (one for the node, one for the previous node) to fix the broken links. In a DLL, you only need *one* pointer (`temp`) because you can access the previous node directly via `temp->prev`.

### Disadvantages of Doubly Linked List ❌
1. **Higher Memory Overhead:** Every node requires extra memory to store the `prev` pointer. 
   - *Example (32-bit compiler):* 
     - SLL Node: 4 bytes (data) + 4 bytes (next) = **8 bytes**.
     - DLL Node: 4 bytes (data) + 4 bytes (next) + 4 bytes (prev) = **12 bytes**.
2. **More Complex Pointer Manipulation:** Every insertion and deletion operation requires modifying *two* sets of links (both forward and backward) instead of just one, making the code slightly more complex and error-prone.
