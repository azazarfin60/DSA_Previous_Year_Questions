## 🔑 B-Tree: Balanced multi-way search tree (designed for disk)

### Properties (Order m):
1. Every node has **at most m children**
2. Non-root internal node: **at least ⌈m/2⌉ children**
3. Root has **at least 2 children** (if not leaf)
4. All **leaves at same level**
5. A node with k children has **k-1 keys**

### Why B-Trees?
- Minimize **disk I/O** (few levels, many keys per node)
- Used in **databases** and **file systems**

### B-Tree vs BST
| Feature | BST | B-Tree |
|---------|-----|--------|
| Keys per node | 1 | m-1 |
| Children | 2 | up to m |
| Height | O(log₂ n) | O(log_m n) |
| Use | In-memory | **Disk-based** |
