# 5.27 Insertion in B-Tree of Order 4 (Data Structure)

📺 [Watch on YouTube](https://www.youtube.com/watch?v=aNU9XYYCHu8)

---

## 📌 Executive Summary

This lecture demonstrates B-Tree insertion of order 4 (2-3-4 Tree). A B-Tree of order m allows each node to have **at most m children** and **at most m-1 keys**. When a node overflows, it **splits**.

---

## 🔑 B-Tree Properties (Order m)

| Property | Value |
|----------|-------|
| Max children per node | **m** |
| Max keys per node | **m − 1** |
| Min children (non-root internal) | **⌈m/2⌉** |
| Min keys (non-root) | **⌈m/2⌉ − 1** |
| All leaves | Same level |

For **Order 4**: max keys = 3, min keys = 1, max children = 4

---

## 📐 Insertion Rules

1. **Always insert at a leaf node**
2. Insert in **sorted order** within the node
3. If node has **m keys** (overflow) → **Split**:
   - Middle key (⌈m/2⌉th) moves **up** to parent
   - Node splits into two nodes
4. If parent overflows → split propagates upward
5. If root splits → **tree height increases by 1**

---

## 📐 Example: Insert 5, 3, 21, 9, 1, 13, 2, 7, 10, 12, 4, 8 (Order 4)

```
Insert 5:       [5]
Insert 3:       [3, 5]
Insert 21:      [3, 5, 21]    (full — 3 keys)

Insert 9:  Overflow! Split: middle = 5 promoted
              [5]
             /   \
          [3]    [9, 21]

Insert 1:        [5]
                /   \
           [1,3]   [9, 21]

Insert 13:       [5]
                /   \
           [1,3]   [9, 13, 21]  (full)

Insert 2:        [5]
                /   \
          [1,2,3]  [9, 13, 21]  (both full)

Insert 7: Must go in right node → Overflow!
          Split [9,13,21] with 7: sorted = [7,9,13,21]
          Middle(13) goes up

              [5, 13]
             /   |   \
        [1,2,3] [7,9] [21]

Insert 10:    [5, 13]
             /   |   \
        [1,2,3] [7,9,10] [21]

Insert 12: Right-middle child overflow → split [7,9,10,12]
           Middle(9) goes up
               [5, 9, 13]
              / |    |    \
         [1,2,3][7] [10,12] [21]

Insert 4: Left child overflow [1,2,3,4] → split, middle(2) up
              [2, 5, 9, 13]  → Root overflow!
              Split root: middle(5) becomes new root

                    [5]
                   /    \
              [2]        [9, 13]
             / \        /  |   \
          [1] [3,4]  [7] [10,12] [21]

Insert 8:          [5]
                  /    \
             [2]        [9, 13]
            / \        /  |   \
         [1] [3,4]  [7,8] [10,12] [21]
```

## ⏱️ Complexity

| Operation | Time |
|-----------|------|
| Search | **O(log n)** |
| Insert | **O(log n)** |
| Delete | **O(log n)** |
| All leaves at | Same depth |

---

## 💡 Key Takeaways

- B-Tree grows **upward** (not downward like BST)
- Always insert at **leaf** level
- Overflow → **split** and promote middle key
- Root split → tree height increases
- Used in **databases and file systems**
