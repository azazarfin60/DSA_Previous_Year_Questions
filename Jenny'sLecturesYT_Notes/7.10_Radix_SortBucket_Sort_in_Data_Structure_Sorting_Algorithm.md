# 7.10 Radix Sort / Bucket Sort | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=XiuSW_mEn7g)

---

## 📌 Executive Summary
Radix Sort is a **non-comparison-based** sorting algorithm. Instead of comparing elements against each other (like Quick or Merge sort), it sorts numbers digit-by-digit, starting from the Least Significant Digit (LSD) up to the Most Significant Digit (MSD). It uses "buckets" corresponding to the base of the numbering system (e.g., 10 buckets for decimal numbers).

---

## 🔑 Core Concept
1. **Find Maximum:** Find the maximum number in the array. Count its digits (let's call this `d`).
2. **Standardize Digits:** Pad all other numbers with leading zeros so that every number has exactly `d` digits.
3. **Buckets:** Create buckets based on the numbering system. For decimal (Base-10), create **10 buckets** (0 through 9).
4. **The Algorithm (Passes):**
   - **Pass 1:** Sort the elements into buckets based on their **Least Significant Digit** (the 1s place).
   - Empty the buckets from bucket `0` to `9` (maintaining the order they were inserted - FIFO).
   - **Pass 2:** Take the new list and sort into buckets based on the **next digit** (the 10s place).
   - Empty the buckets.
   - Continue this for `d` passes.
5. **Number of Passes:** The algorithm requires exactly `d` passes, where `d` is the number of digits in the maximum element.

---

## 📐 Step-by-Step Trace

**Given Array:** `[15, 1, 10, 90, 321, 11, 802, 2, 123, 109]`
- **Max element:** `802` (3 digits). So, $d = 3$. We need **3 passes**.
- **Pad with zeros:** `[015, 001, 010, 090, 321, 011, 802, 002, 123, 109]`

### Pass 1: Sort by 1s Place (Least Significant Digit)
Examine the last digit of each number.
- Bucket 0: `010, 090`
- Bucket 1: `001, 321, 011`
- Bucket 2: `802, 002`
- Bucket 3: `123`
- Bucket 5: `015`
- Bucket 9: `109`
> **List after Pass 1:** `[010, 090, 001, 321, 011, 802, 002, 123, 015, 109]`

### Pass 2: Sort by 10s Place
Examine the middle digit of each number from the *new* list.
- Bucket 0: `001, 802, 002, 109`
- Bucket 1: `010, 011, 015`
- Bucket 2: `321, 123`
- Bucket 9: `090`
> **List after Pass 2:** `[001, 802, 002, 109, 010, 011, 015, 321, 123, 090]`

### Pass 3: Sort by 100s Place (Most Significant Digit)
Examine the first digit of each number from the *new* list.
- Bucket 0: `001, 002, 109, 010, 011, 015, 090`
- Bucket 1: `123`
- Bucket 3: `321`
- Bucket 8: `802`
> **List after Pass 3:** `[001, 002, 109, 010, 011, 015, 090, 123, 321, 802]`

**Remove Leading Zeros for Final Sorted Array:**
`[1, 2, 10, 11, 15, 90, 109, 123, 321, 802]`

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Time Complexity** | **O(d * (n + b))** | $d$ = number of digits in the max number. $n$ = number of elements. $b$ = base (10 for decimal). For each of the $d$ digits, we iterate over $n$ elements to put them in buckets, and then collect them from $b$ buckets. |
| **Space Complexity** | **O(n + b)** | Requires extra space for the buckets (often implemented as queues or linked lists) to hold the $n$ elements. |

> **Exam Tip:** Remember that Radix Sort uses another stable sorting algorithm (usually **Counting Sort**) as a subroutine to actually move the elements into buckets. If an exam asks for a sorting algorithm used for sorting **strings or words**, Radix/Bucket sort is the correct answer (using 26 buckets for Base-26 alphabets).
