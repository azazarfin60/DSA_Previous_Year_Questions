# 7.10 Radix Sort/Bucket Sort in Data Structure | Sorting Algorithm

📺 [Watch on YouTube](https://www.youtube.com/watch?v=JMlYkE8hGJM)

---

## 📌 Executive Summary

Radix Sort is a **non-comparison** sorting algorithm that sorts integers digit by digit, from **least significant digit (LSD)** to most significant. Uses **Counting Sort** as a subroutine for each digit.

---

## 📐 Algorithm

1. Find the **maximum number** to know the number of digits
2. For each digit position (units, tens, hundreds, ...):
   - Sort all elements by that digit using a **stable sort** (counting sort)

---

## 📐 Trace: Sort [170, 45, 75, 90, 802, 24, 2, 66]

### Pass 1 — Sort by Units digit (1s place):
```
Buckets:
0: 170, 90
2: 802, 2
4: 24
5: 45, 75
6: 66

After: [170, 90, 802, 2, 24, 45, 75, 66]
```

### Pass 2 — Sort by Tens digit:
```
Buckets:
0: 802, 2
2: 24
4: 45
6: 66
7: 170, 75
9: 90

After: [802, 2, 24, 45, 66, 170, 75, 90]
```

### Pass 3 — Sort by Hundreds digit:
```
Buckets:
0: 2, 24, 45, 66, 75, 90
1: 170
8: 802

After: [2, 24, 45, 66, 75, 90, 170, 802] ✅
```

---

## ⏱️ Complexity

| Metric | Value |
|--------|-------|
| **Time** | **O(d × (n + k))** |
| **Space** | **O(n + k)** |

Where:
- d = number of digits in max element
- n = number of elements
- k = range of digits (0-9 = 10)

For integers up to b bits: **O(n × b/log n)**

---

## 📊 Properties

| Property | Value |
|----------|-------|
| Comparison-based | ❌ No |
| In-place | ❌ No |
| Stable | ✅ Yes (must use stable subroutine) |
| Works on | Integers, strings |

---

## ⚠️ Stability is Critical
> Each digit-pass **must use a stable sort** (counting sort). If unstable, previous digit ordering is lost!

---

## 💡 Key Takeaways

- **Non-comparison** sort → can beat O(n log n) bound
- Sort digit by digit: **LSD to MSD**
- Uses **counting sort** for each digit pass
- **Stable** sort required at each step
- Best for: fixed-length integers, strings
