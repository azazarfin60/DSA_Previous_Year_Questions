# 3.11 Postfix/Prefix to Infix Conversion | Data Structures

---

## ­şôî Executive Summary
Converting machine-readable notations (Postfix and Prefix) back into human-readable notation (Infix) requires a **Stack**. The process is straightforward: whenever an operator is encountered, you pop two operands, wrap them in parentheses with the operator in the middle, and push the combined string back onto the stack. The primary traps for students lie in the **scanning direction** and the **order of operands**.

---

## 1´©ÅÔâú Postfix to Infix Conversion
**Rule of Thumb:** Postfix evaluates operators *after* operands, so we scan left-to-right. Because the rightmost operand enters the stack last, it is at the top of the stack.

### Algorithm
1. **Scan Direction:** Left to Right.
2. **If Operand:** Push it onto the stack as a string.
3. **If Operator:** Pop two strings from the stack.
   - Top element = `A` (The **Right** Operand)
   - Next element = `B` (The **Left** Operand)
   - Combine them: `( B <operator> A )`
   - Push this new combined string back onto the stack.

### Trace Example: Postfix to Infix
**Expression:** `A B + E F / *`
*(Scan Left $\rightarrow$ Right)*

| Token | Action | Stack State (Top at right) | Logic / String Formation |
| :--- | :--- | :--- | :--- |
| `A` | Push | `"A"` | Operand |
| `B` | Push | `"A"`, `"B"` | Operand |
| `+` | Pop two | ` ` | `A = "B"`, `B = "A"` $\rightarrow$ Form: `(A + B)` |
| | Push res | `"(A + B)"` | Push combined string |
| `E` | Push | `"(A + B)"`, `"E"` | Operand |
| `F` | Push | `"(A + B)"`, `"E"`, `"F"` | Operand |
| `/` | Pop two | `"(A + B)"` | `A = "F"`, `B = "E"` $\rightarrow$ Form: `(E / F)` |
| | Push res | `"(A + B)"`, `"(E / F)"` | Push combined string |
| `*` | Pop two | ` ` | `A = "(E / F)"`, `B = "(A + B)"` |
| | | | Form: `((A + B) * (E / F))` |
| | Push res | `"((A + B) * (E / F))"`| Push combined string |

**Final Infix Output:** `( (A + B) * (E / F) )`

## 2´©ÅÔâú Prefix to Infix Conversion
**Rule of Thumb:** Prefix evaluates operators *before* operands, so we scan right-to-left. Because we scan backwards, the leftmost operand enters the stack last, placing it at the top of the stack.

### Algorithm
1. **Scan Direction:** Right to Left.
2. **If Operand:** Push it onto the stack as a string.
3. **If Operator:** Pop two strings from the stack.
   - Top element = `A` (The **Left** Operand)  *(CRITICAL DIFFERENCE)*
   - Next element = `B` (The **Right** Operand)
   - Combine them: `( A <operator> B )`
   - Push this new combined string back onto the stack.

### Trace Example: Prefix to Infix
**Expression:** `* + A B / E F`
*(Scan Right $\leftarrow$ Left)*

| Token | Action | Stack State (Top at right) | Logic / String Formation |
| :--- | :--- | :--- | :--- |
| `F` | Push | `"F"` | Operand |
| `E` | Push | `"F"`, `"E"` | Operand |
| `/` | Pop two | ` ` | `A = "E"`, `B = "F"` $\rightarrow$ Form: `(E / F)` |
| | Push res | `"(E / F)"` | Push combined string |
| `B` | Push | `"(E / F)"`, `"B"` | Operand |
| `A` | Push | `"(E / F)"`, `"B"`, `"A"` | Operand |
| `+` | Pop two | `"(E / F)"` | `A = "A"`, `B = "B"` $\rightarrow$ Form: `(A + B)` |
| | Push res | `"(E / F)"`, `"(A + B)"` | Push combined string |
| `*` | Pop two | ` ` | `A = "(A + B)"`, `B = "(E / F)"` |
| | | | Form: `((A + B) * (E / F))` |
| | Push res | `"((A + B) * (E / F))"`| Push combined string |

**Final Infix Output:** `( (A + B) * (E / F) )`

## ­şÜ¿ Exam Warning: The 2 Major Differences
Examiners frequently ask for the difference between evaluating/converting Prefix and Postfix. Ensure you state these two exact points:

| Feature | Postfix to Infix | Prefix to Infix |
| :--- | :--- | :--- |
| **Scan Direction** | Left $\rightarrow$ Right | Right $\leftarrow$ Left |
| **Combining Order** | `B operator A` *(Second popped first)* | `A operator B` *(First popped first)* |
