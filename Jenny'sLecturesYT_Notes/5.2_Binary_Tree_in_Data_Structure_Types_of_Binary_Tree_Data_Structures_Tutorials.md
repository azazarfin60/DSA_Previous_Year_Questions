# 5.2 Binary Tree in Data Structure | Types of Binary Tree | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=vvey2QCs98o)

---

## 📌 Executive Summary

This lecture covers Binary Tree properties (max/min nodes, height formulas) and the **four types**: Full (Strict), Complete, Perfect, and Degenerate (Skewed). These definitions are frequently tested in exams.

---

## 🔑 Binary Tree Definition
> A tree where each node can have **at most 2 children** (left and right).

---

## 📐 Properties

### Nodes at Level i
| Property | Formula |
|----------|---------|
| Max nodes at level i | **2^i** |
| Max nodes in tree of height h | **2^(h+1) − 1** |
| Min nodes in tree of height h | **h + 1** |
| Min height for n nodes | **⌈log₂(n+1)⌉ − 1** |
| Max height for n nodes | **n − 1** |

---

## 📊 Types of Binary Trees

### 1. Full (Strict/Proper) Binary Tree
> Each node has **0 or 2 children** — no node has exactly 1 child.

```
       1            ✅ Full
      / \
     2   3
    / \
   4   5
```

**Properties:**
- Leaf nodes = Internal nodes + 1 → **n₀ = n₂ + 1**
- Min nodes at height h = **2h + 1**
- Max nodes = 2^(h+1) − 1

### 2. Complete Binary Tree
> All levels filled **except possibly the last**, which fills **left to right**.

```
       1            ✅ Complete
      / \
     2   3
    / \  /
   4  5 6

       1            ❌ NOT Complete
      / \           (gap on left,
     2   3           node on right)
    / \    \
   4  5     7
```

**Properties:**
- Min nodes at height h = **2^h**
- Max height = **log₂(n)**
- **Ideal for array representation** (no wasted space)

### 3. Perfect Binary Tree
> All internal nodes have 2 children AND all leaves at the **same level**.

```
       1            ✅ Perfect
      / \
     2   3
    / \ / \
   4  5 6  7
```

**Properties:**
- Total nodes = 2^(h+1) − 1
- Leaf nodes = 2^h
- **Every perfect tree is both complete AND full** (but not vice versa)

### 4. Degenerate (Skewed) Binary Tree
> Each internal node has **only 1 child** — essentially a linked list.

```
   1          1          1
    \        /            \
     2      2              2
      \    /              /
       3  3              3
Right   Left        Mixed
Skewed  Skewed      Degenerate
```

**Properties:**
- Height = n − 1 (maximum possible)
- Performance degrades to O(n) for all operations

---

## 📊 Relationship Between Types

```
Perfect ⊂ Complete ⊂ Binary Tree
Perfect ⊂ Full ⊂ Binary Tree

Perfect → always Complete ✅
Perfect → always Full ✅
Complete → not always Perfect ❌
Full → not always Perfect ❌
```

---

## 🔑 Node Representation in C
```c
struct node {
    struct node *left;    // Left child pointer
    int data;
    struct node *right;   // Right child pointer
};
```

---

## 💡 Key Takeaways

- **Full**: 0 or 2 children, never 1
- **Complete**: filled left-to-right, ideal for arrays/heaps
- **Perfect**: all leaves same level — both full AND complete
- **Degenerate**: 1 child only — worst case height
- Memorize: n₀ = n₂ + 1 (for full binary tree)
- Max nodes formula: **2^(h+1) − 1**
