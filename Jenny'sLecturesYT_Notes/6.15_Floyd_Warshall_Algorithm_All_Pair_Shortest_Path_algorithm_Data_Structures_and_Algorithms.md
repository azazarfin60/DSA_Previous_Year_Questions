# 6.15 Floyd Warshall Algorithm | All Pair Shortest Path algorithm | Data Structures and Algorithms

📺 [Watch on YouTube](https://www.youtube.com/watch?v=Gc4mWrmJBsw)

---

## 📌 Executive Summary

Floyd-Warshall finds **shortest paths between ALL pairs** of vertices using **Dynamic Programming**. Works with **negative edges** (but not negative cycles). Uses a V×V distance matrix.

---

## 📐 Algorithm

```c
// Initialize D[i][j] = weight(i,j) or ∞ if no edge, D[i][i] = 0
for (int k = 0; k < V; k++)           // Intermediate vertex
    for (int i = 0; i < V; i++)       // Source
        for (int j = 0; j < V; j++)   // Destination
            if (D[i][k] + D[k][j] < D[i][j])
                D[i][j] = D[i][k] + D[k][j];
```

### Core Idea
> `D[i][j] = min(D[i][j], D[i][k] + D[k][j])`
> 
> "Is it shorter to go from i to j directly, or through vertex k?"

---

## 📐 Example

```
Graph:
  1 --3-- 2
  |       |
  8       2
  |       |
  3 --1-- 4
  
  Also: 1→4 = ∞, 2→3 = ∞ (no direct edge)
```

### Initial Matrix D⁰:
```
     1    2    3    4
1  [ 0,   3,   8,   ∞ ]
2  [ 3,   0,   ∞,   2 ]
3  [ 8,   ∞,   0,   1 ]
4  [ ∞,   2,   1,   0 ]
```

### k=1 (through vertex 1):
```
D[2][3] = min(∞, D[2][1]+D[1][3]) = min(∞, 3+8) = 11
D[3][2] = min(∞, D[3][1]+D[1][2]) = min(∞, 8+3) = 11
D[4][2] via 1 = no improvement
```

### k=2 (through vertex 2):
```
D[1][4] = min(∞, D[1][2]+D[2][4]) = min(∞, 3+2) = 5
D[3][4] via 2 = no change (already 1)
D[4][1] = min(∞, D[4][2]+D[2][1]) = min(∞, 2+3) = 5
```

### Continue for k=3, k=4...

### Final Matrix:
```
     1    2    3    4
1  [ 0,   3,   6,   5 ]
2  [ 3,   0,   3,   2 ]
3  [ 6,   3,   0,   1 ]
4  [ 5,   2,   1,   0 ]
```

---

## 🎯 Detecting Negative Cycle
> If `D[i][i] < 0` for any vertex i → **negative cycle exists**

---

## ⏱️ Complexity
- **Time: O(V³)** — three nested loops
- **Space: O(V²)** — distance matrix

---

## 📊 When to Use

| Scenario | Algorithm |
|----------|-----------|
| Single source, no negative | **Dijkstra** |
| Single source, negative edges | **Bellman-Ford** |
| **All pairs** | **Floyd-Warshall** |

---

## 💡 Key Takeaways

- **Dynamic Programming** on intermediate vertices
- 3 nested loops: k (intermediate), i (source), j (destination)
- **k must be the outermost loop** — critical!
- Handles **negative edges** but not negative cycles
- Check diagonal for negative cycle detection
- Time: O(V³), Space: O(V²)
