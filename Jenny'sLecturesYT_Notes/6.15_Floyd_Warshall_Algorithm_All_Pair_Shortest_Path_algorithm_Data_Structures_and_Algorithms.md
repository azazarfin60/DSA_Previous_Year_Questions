# 6.15 Floyd-Warshall Algorithm (All-Pair Shortest Path) | Data Structures

## 📌 Executive Summary
The **Floyd-Warshall Algorithm** is a **Dynamic Programming** algorithm designed to solve the **All-Pair Shortest Path** problem. Instead of finding the shortest path from just one source (like Dijkstra or Bellman-Ford), it simultaneously finds the absolute shortest path between *every single pair* of vertices in the graph in one execution.

---

## 🔑 Core Concepts & Advantages
1. **Dynamic Programming Approach:** It builds the solution bottom-up. It calculates the shortest path between $I$ and $J$ by systematically allowing more and more vertices to act as intermediate "stepping stones" between them.
2. **Negative Edge Weights:** Unlike Dijkstra's algorithm, Floyd-Warshall **handles negative edge weights** perfectly.
3. **Graph Suitability:** Because its time complexity is quite high, it is best suited for **Dense Graphs** (where most vertices are connected to each other). For sparse graphs, running Dijkstra $V$ times is often more efficient.

---

## ⚙️ The Distance Matrix & Recurrence Formula
The algorithm heavily relies on a set of $V \times V$ matrices: $D^0, D^1, D^2, \dots, D^V$.

- **$D^0$ (Initial Base Matrix):** Represents the initial direct edge weights.
  - `D[i][i] = 0` (Diagonal elements are always 0).
  - `D[i][j] = weight` (If a direct directed edge exists from $I \rightarrow J$).
  - `D[i][j] = \infty` (If no direct edge exists).

### The Golden Formula
To generate the next matrix $D^K$ from the previous matrix $D^{K-1}$, we use the intermediate vertex $K$. We check if routing the path *through* $K$ is cheaper than the old path.

> **$D^k[I][J] = \min \left( D^{k-1}[I][J], \quad D^{k-1}[I][K] + D^{k-1}[K][J] \right)$**

---

## 📐 Step-by-Step Algorithm (Manual Trace Logic)
If you have 4 vertices (1, 2, 3, 4), you will generate 5 matrices: $D^0, D^1, D^2, D^3, D^4$. The final answer is $D^4$.

**How to quickly draw $D^K$ from $D^{K-1}$ in an exam:**
1. **Diagonals:** The diagonal will *always* remain `0`.
2. **The "Cross" Trick:** If you are calculating matrix **$D^K$**, the **$K$-th Row** and the **$K$-th Column** will remain *exactly the same* as they were in $D^{K-1}$. Write them down immediately without calculating.
   - *Example: When computing $D^2$, just copy Row 2 and Column 2 straight from $D^1$.*
3. **Calculate Remaining Cells:** For any remaining empty cell $(I, J)$, use the golden formula:
   - "Is the old value at $(I, J)$ smaller, or is $(I, K) + (K, J)$ smaller?"

---

## 💻 Pseudocode Structure
The algorithm is notoriously simple to implement, consisting of exactly three nested loops.
```c
// Initialize D matrix with direct edge weights D0
for (int k = 1; k <= V; k++) {          // The intermediate vertex
    for (int i = 1; i <= V; i++) {      // The source vertex
        for (int j = 1; j <= V; j++) {  // The destination vertex
            // The Golden Formula
            if (D[i][k] + D[k][j] < D[i][j]) {
                D[i][j] = D[i][k] + D[k][j];
            }
        }
    }
}
```

---

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Time Complexity** | **$\Theta(V^3)$** | Caused directly by the three nested loops (`k`, `i`, `j`), each running $V$ times. |
| **Space Complexity** | **O(V²)** | Required to store the $V \times V$ distance matrix. You do not need to store $V$ separate matrices in code; you can continually overwrite a single matrix. |

> **Exam Tip:** When a question asks you to manually trace Floyd-Warshall, you MUST draw out the grids for $D^0$, $D^1, \dots, D^V$. Use the "Cross Trick" mentioned above to save a massive amount of time! Highlight the working row and working column in each step.
