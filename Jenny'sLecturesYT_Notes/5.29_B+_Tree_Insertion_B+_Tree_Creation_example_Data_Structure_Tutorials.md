## 📌 Executive Summary

B+ Tree is a variant of B-Tree where **all data resides in leaf nodes** and leaf nodes are **linked** for sequential access. Internal nodes only store keys as guides.

---

## 🔑 B+ Tree vs B-Tree

| Feature | B-Tree | B+ Tree |
|---------|--------|---------|
| Data location | All nodes | **Leaf nodes only** |
| Leaf linking | No | **Yes (linked list)** |
| Duplicate keys | No | **Yes (in internal nodes)** |
| Sequential access | Slow | **Fast** |
| Fan-out | Lower | **Higher** (internal nodes smaller) |

---

## 📐 B+ Tree Properties (Order m)
- Internal node: max **m children**, max **m-1 keys**
- Leaf node: max **m-1 data entries**
- Leaf nodes linked as **doubly/singly linked list**
- On split: copy of middle key goes up (key stays in leaf too!)

---

## 📐 Insertion Rules

1. Insert at **leaf node** (sorted order)
2. If leaf overflows:
   - Split into two leaves
   - **Copy** middle key up to parent (key remains in right leaf)
3. If internal node overflows:
   - Split and **push** middle key up (key removed from node — like B-Tree)
4. Root split → new root created

### ⚠️ Key Difference from B-Tree
```
B-Tree split:  middle key MOVES UP (removed from node)
B+ Tree leaf split: middle key COPIES UP (stays in right leaf)
B+ Tree internal split: middle key MOVES UP (same as B-Tree)
```

---

## 📐 Example: Insert 5, 8, 1, 7, 3, 12, 9, 6 (Order 3)

```
Insert 5:    [5]
Insert 8:    [5, 8]    (max 2 keys for order 3 leaf)

Insert 1:    Overflow → split, copy 5 up
                 [5]
                /   \
             [1]    [5, 8]     ← 5 stays in right leaf!

Insert 7:        [5]
                /   \
             [1]    [5, 7, 8]  → Overflow! Split, copy 7 up
                 [5, 7]
                / |   \
             [1] [5] [7, 8]

Insert 3:      [5, 7]
              / |   \
          [1,3] [5] [7, 8]

Insert 12:     [5, 7]
              / |   \
          [1,3] [5] [7, 8, 12] → Overflow!

Split leaf, copy 8 up → internal overflow!
            [5, 7, 8] → Internal overflow! Push 7 up

                  [7]
                 /   \
              [5]     [8]
             / \      / \
          [1,3][5]  [7,8][12]

Leaf linked list: [1,3] → [5] → [7,8] → [12]
```

## 🎯 Advantages of B+ Tree

1. **Sequential access** via linked leaves — ideal for range queries
2. **Higher fan-out** → shallower tree → fewer disk reads
3. All data at same level → **uniform access time**

---

## 💡 Key Takeaways

- All data in **leaf nodes**, internal nodes are index only
- Leaf split: **COPY** key up; Internal split: **MOVE** key up
- Leaves linked for efficient **range queries**
- Used in **database indexing** (MySQL, PostgreSQL)
- B+ Tree is the standard for disk-based data structures
