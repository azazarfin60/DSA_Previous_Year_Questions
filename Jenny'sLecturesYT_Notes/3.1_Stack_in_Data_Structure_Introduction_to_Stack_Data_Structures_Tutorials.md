## 📌 Executive Summary

This lecture introduces the **Stack** — a LIFO (Last In, First Out) linear data structure. It covers push, pop, peek operations, overflow/underflow conditions, and array-based implementation.

---

## 🔑 Definition
> A Stack follows **LIFO** — the last element pushed is the first one popped.

### Real-Life Analogies
- Stack of plates, stack of books
- Undo operation (most recent action undone first)
- Browser back button
- Function call stack in programs

---

## 💻 Operations

| Operation | Description | Time |
|-----------|-------------|------|
| `push(x)` | Insert x at top | O(1) |
| `pop()` | Remove and return top | O(1) |
| `peek()` | Return top without removing | O(1) |
| `isEmpty()` | Check if empty | O(1) |
| `isFull()` | Check if full (array) | O(1) |

### Conditions
- **Overflow** → push when stack is full
- **Underflow** → pop when stack is empty

---

## 💻 Array-Based Implementation
```c
#define MAX 100
int stack[MAX];
int top = -1;           // -1 means empty

void push(int data) {
    if (top == MAX - 1) {
        printf("Stack Overflow!\n");
        return;
    }
    stack[++top] = data;
}

int pop() {
    if (top == -1) {
        printf("Stack Underflow!\n");
        return -1;
    }
    return stack[top--];
}

int peek() {
    if (top == -1) return -1;
    return stack[top];
}

int isEmpty() {
    return top == -1;
}
```

## 🎯 Applications of Stack
1. Expression conversion (infix ↔ postfix ↔ prefix)
2. Expression evaluation
3. Function call management (recursion)
4. Parenthesis matching
5. Undo/Redo operations
6. DFS traversal

---

## 💡 Key Takeaways

- Stack = **LIFO** structure
- `top` pointer tracks current top
- All operations are **O(1)**
- Array implementation has fixed size (overflow possible)
- Linked list implementation has dynamic size (no overflow)
