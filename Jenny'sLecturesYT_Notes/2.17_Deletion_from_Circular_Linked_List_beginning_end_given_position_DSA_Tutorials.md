## 📌 Executive Summary
When deleting nodes from a Circular Linked List (CLL) that maintains only a `tail` pointer, we can instantly access the first node (`tail->next`) and the last node (`tail`). 

However, unlike a Doubly Linked List, we do not have backward links. This leads to a critical performance difference when deleting the last node, as we must traverse the entire list to find the second-to-last node to update the `tail`.

---

## 1️⃣ Case 1: Deletion from the Beginning

Deleting the first node is highly efficient. We simply bypass `tail->next` to point to the second node.
**Time Complexity:** $O(1)$

### Algorithm Steps:
1. Handle empty list and single-node list edge cases.
2. Store the first node in a `temp` pointer: `temp = tail->next;`
3. Update the circular link to bypass the first node: `tail->next = temp->next;`
4. Release the memory: `free(temp);`

### C Code Snippet
```c
void deleteFromBeginning() {
    struct node *temp;
    
    if (tail == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    // Edge Case: Only 1 node in the list
    if (tail->next == tail) {
        temp = tail;
        tail = NULL;
        free(temp);
    } 
    // Normal Case: >1 node
    else {
        temp = tail->next;         // 1. Anchor the first node
        tail->next = temp->next;   // 2. Point tail to the second node
        free(temp);                // 3. Release memory
    }
}
```

## 2️⃣ Case 2: Deletion from the End

> [!WARNING]
> **Exam Concept (The $O(n)$ Trap):** 
> Even though we have a `tail` pointer giving us direct access to the last node, **we still have to traverse the entire list**. Why? Because when we delete `tail`, the *second-to-last* node becomes the new `tail`. Because there are no backward links (`prev`), the only way to find the second-to-last node is to traverse from the beginning.
> **Time Complexity:** $O(n)$

### Algorithm Steps:
1. Traverse the list using two pointers: `current` and `previous`.
2. Stop when `current->next == tail->next` (meaning `current` is on the last node).
3. The `previous` node is now the second-to-last node.
4. Update the circular link: `previous->next = tail->next;`
5. Shift the tail pointer: `tail = previous;`
6. Release memory: `free(current);`

### C Code Snippet
```c
void deleteFromEnd() {
    struct node *current, *previous;
    
    if (tail == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    if (tail->next == tail) {
        // Single node edge case logic here...
        return;
    }
    
    current = tail->next; // Start at first node
    
    // Traverse until current is the last node
    while (current->next != tail->next) {
        previous = current;
        current = current->next;
    }
    
    // Connect second-to-last node to the first node
    previous->next = tail->next;
    
    // Shift tail backwards
    tail = previous;
    
    // Release the old tail
    free(current);
}
```

## 3️⃣ Case 3: Deletion from a Specific Position

This is standard SLL deletion traversal. We must find the node *just before* the target position so we can bypass it.
**Time Complexity:** $O(n)$

### Algorithm Steps:
1. Initialize `current = tail->next;`
2. Traverse to `pos - 1` using a `while` loop.
3. Use a second pointer (`nextnode`) to anchor the node to be deleted.
4. Bypass the node: `current->next = nextnode->next;`
5. Release memory: `free(nextnode);`

### C Code Snippet
```c
void deleteFromPosition() {
    struct node *current, *nextnode;
    int pos, i = 1;
    
    printf("Enter position: ");
    scanf("%d", &pos);
    
    // (Assume length validation & beginning/end edge cases are handled here)
    
    current = tail->next; // Start at first node
    
    // Traverse to the node immediately BEFORE the target position
    while (i < pos - 1) {
        current = current->next;
        i++;
    }
    
    // Anchor the node to be deleted
    nextnode = current->next;
    
    // Bypass the deleted node
    current->next = nextnode->next;
    
    // Release memory
    free(nextnode);
}
```
