# 2.7 Find the Length of a Linked List (Iterative Approach) | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
To find the length (number of nodes) in a Singly Linked List, we must traverse the entire list from the first node to the last. Because we visit every node exactly once, the time complexity for this operation is strictly $O(n)$. 

---

## 🛠️ The Iterative Algorithm

### Algorithm Steps:
1. Check if the list is empty (`head == NULL`). If so, the length is 0.
2. Initialize an integer variable `count = 0`.
3. Create a `temp` pointer and set it to `head`.
4. Traverse the list using a `while` loop. The condition should be `temp != NULL`.
5. Inside the loop, increment the `count` by 1.
6. **Crucial Step:** Move the `temp` pointer to the next node (`temp = temp->next`).
7. Once the loop terminates, `count` will hold the exact length of the linked list.

### C Code Snippet

```c
int getLength() {
    int count = 0;
    struct node *temp;
    
    // Start traversal from the head
    temp = head; 
    
    // Traverse until temp falls off the end of the list
    while (temp != NULL) {
        count++;                // Increment node count
        temp = temp->next;      // Move to the next node
    }
    
    return count;
}
```

---

## 🚨 Exam Warning: The Infinite Loop Trap

When writing traversal code in a high-pressure exam environment, students frequently make the following mistake:

```c
// ❌ FATAL ERROR: Infinite Loop
while (temp != NULL) {
    count++;
}
// You forgot to move the pointer!
```

If you forget to include `temp = temp->next;` inside the loop, the `temp` pointer will never change. It will perpetually point to the first node (which is not `NULL`), resulting in an infinite loop that crashes the program. Always ensure your traversal pointers are updated inside your loops!

---

## ⏱️ Complexity Analysis
- **Time Complexity:** $O(n)$, where $n$ is the number of nodes in the linked list. We cannot skip any nodes; we must process each one to count it.
- **Space Complexity:** $O(1)$, because we only allocate memory for two local variables (`count` and `temp`), regardless of how long the list is.
