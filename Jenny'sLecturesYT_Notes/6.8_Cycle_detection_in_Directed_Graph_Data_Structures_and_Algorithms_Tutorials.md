# 6.8 Cycle detection in Directed Graph | Data Structures and Algorithms Tutorials

## 📌 Executive Summary

Cycle detection in a directed graph using **DFS with coloring** (White/Gray/Black). A cycle exists if DFS finds a **back edge** — an edge to a Gray (currently in-stack) vertex.

---

## 📐 Three-Color Algorithm

| Color | Meaning |
|-------|---------|
| **White** | Not yet visited |
| **Gray** | Currently being processed (in recursion stack) |
| **Black** | Fully processed (all descendants visited) |

### Rule: **Cycle exists ⟺ Edge to a Gray vertex found**

---

## 📐 Algorithm
```c
int color[MAX];  // 0=White, 1=Gray, 2=Black

int DFS(int u) {
    color[u] = 1;  // Mark Gray (in process)

    for each neighbor v of u:
        if (color[v] == 1)      // Gray → BACK EDGE → CYCLE!
            return 1;
        if (color[v] == 0)      // White → recurse
            if (DFS(v)) return 1;

    color[u] = 2;  // Mark Black (done)
    return 0;
}

int hasCycle() {
    for (int i = 0; i < V; i++)
        color[i] = 0;  // All White
    for (int i = 0; i < V; i++)
        if (color[i] == 0)
            if (DFS(i)) return 1;
    return 0;
}
```

---

## 📐 Example

```
Graph: A → B → D
       ↓       ↓
       C ← ← ←

DFS from A:
  A(Gray) → B(Gray) → D(Gray) → C(Gray)?
  C→A? But A is Gray → BACK EDGE → CYCLE! ✅

DFS states:
  A: White → Gray → ... → Black
  B: White → Gray → Black
  D: White → Gray → Black
```

---

## ⚠️ Why Not Just "visited" Array?
> In **directed** graphs, reaching a visited vertex doesn't always mean cycle. It could be a **cross edge** (to a Black vertex). Only edges to **Gray** vertices (ancestors in current path) indicate cycles.

---

## ⏱️ Complexity
- **Time: O(V + E)**
- **Space: O(V)**

---

## 💡 Key Takeaways

- Use **3 colors**: White, Gray, Black
- **Back edge (to Gray)** = cycle
- Edge to Black = NOT a cycle (cross/forward edge)
- Must check all components (graph may be disconnected)
- For **undirected** graphs: simpler — any visited non-parent neighbor = cycle
