# 6.8 Cycle Detection in Directed Graph | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=rKQaZuoUR4M)

---

## 📌 Executive Summary
Detecting a cycle in a **Directed Graph** can be efficiently done using **Depth First Search (DFS)**. The core idea is to track not just whether a node has been visited, but *where* it is in the exploration process. If during traversal we encounter an adjacent node that is currently in our active path (i.e., currently in the DFS stack), it indicates the presence of a **Back Edge**, which strictly implies a cycle.

---

## 🔑 Core Concept: The 3-State Flag Array
Instead of a simple boolean `visited` array, we must maintain a `flag` variable (or color mapping) for each vertex with **three distinct states**:

| Flag Value | State | Meaning in DFS |
| :---: | :--- | :--- |
| **-1** | **Unvisited** | The vertex has never been encountered. |
| **0** | **Visited (In Stack)** | The vertex is currently in the active recursion stack (we are still exploring its descendants). |
| **1** | **Processed (Popped)** | The vertex has been fully explored, popped from the stack, and is a dead end. |

*Note: We also maintain a **Parent Map Table** to trace back and reconstruct the exact path of the cycle if one is found.*

---

## ⚙️ Step-by-Step Algorithm
1. Initialize the `flag` of all vertices to `-1` (unvisited).
2. Choose an arbitrary unvisited source vertex. Push it to the Stack, set its flag to `0`, and set its parent to `null`.
3. Explore the adjacent vertices of the current node (top of the Stack):
   - **Case A (Flag is -1):** The adjacent vertex is unvisited. Push it to the Stack, change its flag to `0`, and record the current node as its parent in the Parent Table.
   - **Case B (Flag is 1):** The adjacent vertex has already been fully explored. Ignore it.
   - **Case C (Flag is 0): 🚨 CYCLE DETECTED!** The adjacent vertex is already in the Stack. This means we have found a path back to an ancestor (a Back Edge). Return `TRUE`.
4. If a vertex has no unvisited adjacent nodes left (a dead end), **pop** it from the Stack and change its flag to **1** (Processed).
5. If the Stack becomes empty and all vertices are processed without encountering a `0` flag, the graph does **not** contain a cycle. Return `FALSE`.

---

## 🔄 Tracing the Cycle (Example)
Imagine a DFS path where $A \rightarrow B \rightarrow D \rightarrow E$. 
- Current Stack contains: `[A, B, D, E]`. All have `flag = 0`.
- The Parent Map shows: Parent(E)=D, Parent(D)=B, Parent(B)=A.
- From node `E`, we find an adjacent edge going to `B`.
- We check the flag of `B` and see it is `0`. **Cycle Detected!**

**How to print the cycle:**
1. The cycle includes the edge $E \rightarrow B$.
2. Trace backwards using the Parent Map starting from $E$:
   - Parent of $E$ is $D$.
   - Parent of $D$ is $B$.
   - We have reached $B$, the node that triggered the cycle.
3. The exact cycle path is: $B \rightarrow D \rightarrow E \rightarrow B$.

---

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Time Complexity** | **O(V + E)** | In the absolute worst case (e.g., the cycle is at the very end of the graph, or there is no cycle), the algorithm must visit every Vertex ($V$) and traverse every Edge ($E$) exactly once, matching standard DFS time. |
| **Space Complexity** | **O(V)** | Required to store the `flag` array, the `parent` array, and the recursion Stack, all of which take space proportional to the number of vertices $V$. |

> **Exam Tip:** If a question asks "How do you detect a cycle in a directed graph using DFS?", your answer **MUST** explicitly mention the 3 states (Unvisited, In-Stack, Fully Processed). If you only mention a boolean `visited` array, you will lose marks, as a boolean array is only sufficient for *Undirected* graphs.
