# 7.12 Counting Sort Algorithm | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=pEJiGC-ObQE)

---

## 📌 Executive Summary
Counting Sort is a highly efficient, **non-comparison-based** sorting algorithm. Instead of comparing elements against each other, it sorts by counting the number of occurrences (frequency) of each distinct element. It can achieve linear time complexity $O(N + K)$, but it has strict limitations: it requires extra memory proportional to the range of the data ($K$), and it natively only handles non-negative integers.

---

## 🔑 Core Concept
1. **Find Maximum ($K$):** Find the maximum value in the given array. The range of elements is assumed to be $0$ to $K$.
2. **Frequency Count:** Create an auxiliary `Count` array of size $K + 1$ (indexes from $0$ to $K$), initialized to all zeros. Iterate through the input array; for every element `x`, increment `Count[x]`.
3. **Cumulative Sum (Positions):** Update the `Count` array by keeping a running total (`Count[i] = Count[i] + Count[i-1]`). This modifies the array so that `Count[x]` now stores the actual *position* (1-based index) of element `x` in the final sorted array.
4. **Build Output Array:** Create an output array `B`. To maintain **stability** (keeping duplicate elements in their original relative order), scan the original array from **right to left**. 
   - Look at element `A[i]`.
   - Go to `Count[A[i]]`.
   - Decrement the count value by 1.
   - Place `A[i]` into array `B` at the newly decremented index.
5. **Copy:** Copy the sorted elements from `B` back to the original array `A`.

---

## 📐 Step-by-Step Trace

**Given Array:** `A = [2, 1, 1, 0, 2, 5, 4, 0, 2, 8, 7, 7, 9, 2, 0, 1, 9]` *(N = 17)*
- Maximum element ($K$) = 9.
- Create `Count` array of size 10 (indexes 0 to 9).

### Step 1: Frequency Count
Count how many times each number appears in `A`.
- `Count = [3, 3, 4, 0, 1, 1, 0, 2, 1, 2]`
  *(e.g., 0 appears 3 times, 1 appears 3 times, 2 appears 4 times, etc.)*

### Step 2: Cumulative Sum (Finding Positions)
Update `Count[i] = Count[i] + Count[i-1]`.
- Index 0: `3`
- Index 1: `3 + 3 = 6`
- Index 2: `6 + 4 = 10`
- Index 3: `10 + 0 = 10`
- ...
- `Updated Count = [3, 6, 10, 10, 11, 12, 12, 14, 15, 17]`
*(This means the last '0' belongs at position 3 (index 2), the last '1' belongs at position 6 (index 5), etc.)*

### Step 3: Build Output Array (Right-to-Left)
Scan `A` from right to left (starting at $A[16] = 9$) to maintain stability.
1. Element is `9`. Look at `Count[9]`, which is `17`. 
   - Decrement `17` to `16`.
   - Place `9` at `B[16]`.
2. Next element is `1`. Look at `Count[1]`, which is `6`.
   - Decrement `6` to `5`.
   - Place `1` at `B[5]`.
3. Next element is `0`. Look at `Count[0]`, which is `3`.
   - Decrement `3` to `2`.
   - Place `0` at `B[2]`.
   - *(Continue until array is fully processed)*

## 💻 C Code Implementation

```c
void countSort(int a[], int n, int k) {
    int count[k + 1];
    int b[n]; // Output array
    int i;
    
    // 1. Initialize Count array with zeros
    for(i = 0; i <= k; i++) {
        count[i] = 0;
    }
    
    // 2. Store the frequency of each element
    for(i = 0; i < n; i++) {
        count[a[i]]++;
    }
    
    // 3. Modify Count array to store actual positions (Cumulative sum)
    for(i = 1; i <= k; i++) {
        count[i] = count[i] + count[i - 1];
    }
    
    // 4. Build the output array B by scanning A from right to left (for stability)
    for(i = n - 1; i >= 0; i--) {
        b[--count[a[i]]] = a[i]; // Pre-decrement gives the exact 0-based index
    }
    
    // 5. Copy the sorted elements back to original array A
    for(i = 0; i < n; i++) {
        a[i] = b[i];
    }
}
```

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Time Complexity** | **O(N + K)** | The algorithm relies on 4 separate, non-nested `for` loops. Two loops iterate over `K` (range) and two iterate over `N` (elements). Thus, time is strictly linear $O(N + K)$ in Best, Worst, and Average cases. |
| **Space Complexity** | **O(N + K)** | We allocate a `Count` array of size $K+1$ and an output array `B` of size $N$. |

### ⚠️ Drawbacks & Limitations
1. **Inefficient for large ranges:** If $N = 100$ but $K = 10,000$, we must create a massive `Count` array of size 10,001 mostly filled with zeros. Counting Sort is only viable when **$K$ is $O(N)$** (i.e., the range is comparable to the number of elements).
2. **Negative Numbers:** It cannot natively sort negative numbers (you cannot have an index like `Count[-5]`). 
   - *Fix:* Find the minimum negative value and add its absolute value to every element to "normalize" the array to start at 0, sort, then subtract it back later.
3. **Floating Point Numbers:** It cannot sort decimal/floating-point numbers, as they cannot map directly to array indices.
