## рџ“Њ Executive Summary
This note provides step-by-step traces for evaluating Postfix expressions using a Stack. It reinforces the critical rule of operand ordering during popping: when an operator arrives, the **first popped element is `A` (Right Operand)** and the **second popped element is `B` (Left Operand)**. The calculation is always **`B operator A`**.

---

## рџ“ђ Trace Example 1: Small Expression

**Expression:** `2 3 1 * + 9 -`
*(Scan from Left $\rightarrow$ Right)*

| Token | Action | Stack State (Top at right) | Calculation / Logic |
| :--- | :--- | :--- | :--- |
| `2` | Push | `2` | Operand |
| `3` | Push | `2, 3` | Operand |
| `1` | Push | `2, 3, 1` | Operand |
| `*` | Pop two | `2` | `A = 1`, `B = 3` $\rightarrow$ `3 * 1 = 3` |
| | Push res | `2, 3` | Push 3 |
| `+` | Pop two | ` ` | `A = 3`, `B = 2` $\rightarrow$ `2 + 3 = 5` |
| | Push res | `5` | Push 5 |
| `9` | Push | `5, 9` | Operand |
| `-` | Pop two | ` ` | `A = 9`, `B = 5` $\rightarrow$ `5 - 9 = -4` |
| | Push res | `-4` | Final Answer: **-4** |

> **рџљЁ Danger Zone:** Notice the `-` operation. If you incorrectly did `A - B` (9 - 5), you would get `4`. The correct operation is always `B - A` (5 - 9 = -4).

## рџ’Ў Summary Algorithm for Exams

```text
Algorithm EvaluatePostfix(P):
1. Create an empty stack S
2. For each character 'c' in string P from Left to Right:
3.    If 'c' is an operand:
4.       Push 'c' onto S
5.    Else if 'c' is an operator (+, -, *, /, ^):
6.       Pop A from S
7.       Pop B from S
8.       Result = B <operator> A
9.       Push Result onto S
10. Return the final remaining element in S
```
