## 📌 Executive Summary
A **2D Array** (or Matrix) is essentially an "array of arrays". 

Because computer memory is strictly 1-Dimensional (a long sequential tape of bytes), a 2D matrix cannot be stored as a physical grid. Instead, it must be flattened using either **Row-Major** or **Column-Major** implementation. Calculating memory addresses for 2D arrays is a highly frequent exam topic.

---

## 🧮 2D Array Address Calculation Formulas

Consider a matrix `A[m][n]` where:
- $m$ = Total number of Rows
- $n$ = Total number of Columns
- $i$ = Target row index
- $j$ = Target column index
- $Size$ = Size of the data type (e.g., 4 bytes for int)
- $Base$ = Base address of the array

### 1. Row-Major Implementation (Default in C)
Stores the matrix row by row. (All Row 0 elements, then all Row 1 elements...).
To find element $A[i][j]$, you must cross $i$ complete rows (each containing $n$ columns), and then cross $j$ elements in the current row.

**Formula (0-Indexed):**
$$ \text{Address of } A[i][j] = \text{Base} + [(i \times n) + j] \times \text{Size} $$

**Formula (1-Indexed):** *(If the question states row/col index starts at 1)*
$$ \text{Address of } A[i][j] = \text{Base} + [((i-1) \times n) + (j-1)] \times \text{Size} $$

### 2. Column-Major Implementation
Stores the matrix column by column. (All Column 0 elements, then all Column 1 elements...).
To find element $A[i][j]$, you must cross $j$ complete columns (each containing $m$ rows), and then cross $i$ elements in the current column.

**Formula (0-Indexed):**
$$ \text{Address of } A[i][j] = \text{Base} + [(j \times m) + i] \times \text{Size} $$

**Formula (1-Indexed):**
$$ \text{Address of } A[i][j] = \text{Base} + [((j-1) \times m) + (i-1)] \times \text{Size} $$

> [!WARNING]
> **Exam Trap:**
> Always read the question carefully to determine if the array is **0-indexed** or **1-indexed**, and pay close attention to whether you need to multiply by $n$ (total columns) or $m$ (total rows). Mixing up Row-Major and Column-Major multipliers is the most common error.
