## 📌 Executive Summary

This lecture covers 2D arrays — declaration, initialization, memory representation (**row-major** and **column-major** order), and address calculation formulas.

---

## 💻 Declaration & Initialization

```c
int arr[3][4];                    // 3 rows, 4 columns
int arr[2][3] = {{1,2,3},{4,5,6}};  // Initialized
int arr[][3] = {{1,2,3},{4,5,6}};   // Rows auto-computed
```

---

## 📐 Memory Representation

### Row-Major Order (C uses this)
Rows stored **one after another** in memory.

```
arr[2][3] = {{1,2,3},{4,5,6}}

Memory: [1][2][3][4][5][6]
         Row 0    Row 1
```

### Address Formula (Row-Major):
```
Address(arr[i][j]) = Base + ((i × n) + j) × size

Where: n = number of columns, size = sizeof(element)
```

### Column-Major Order (Fortran uses this)
Columns stored one after another.

```
Memory: [1][4][2][5][3][6]
         Col0  Col1  Col2
```

### Address Formula (Column-Major):
```
Address(arr[i][j]) = Base + ((j × m) + i) × size

Where: m = number of rows
```

---

## 📐 Example: Address Calculation

```
arr[3][4], Base = 1000, size = 4 bytes (int)

Find address of arr[2][3] in Row-Major:
= 1000 + ((2 × 4) + 3) × 4
= 1000 + (8 + 3) × 4
= 1000 + 44
= 1044
```

---

## 💡 Key Takeaways

- C stores 2D arrays in **Row-Major** order
- Row-Major formula: `Base + (i×cols + j) × size`
- Column-Major formula: `Base + (j×rows + i) × size`
- These formulas are **frequently asked** in exams
