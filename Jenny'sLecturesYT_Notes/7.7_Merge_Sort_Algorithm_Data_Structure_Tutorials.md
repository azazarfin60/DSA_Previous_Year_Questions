## 📌 Executive Summary
Merge Sort is a classic **Divide and Conquer** sorting algorithm. Unlike Quick Sort which does its heavy lifting during the *divide* phase (partitioning), Merge Sort trivially divides the array in half until each element stands alone, and does its heavy lifting during the **Conquer (Merge)** phase, stitching sorted sublists back together.

---

## 🔑 Core Concept
1. **Divide:** Recursively divide the given list into two equal halves (`mid = (lb + ub) / 2`). Continue dividing until every sublist contains only exactly **1 element** (a list of 1 element is inherently sorted).
2. **Conquer (Merge):** Take two adjacent sorted sublists and merge them together to produce a single, larger sorted sublist.
3. **Repeat:** Keep merging sublists upwards through the recursion tree until you have one complete sorted array.

---

## 📐 Step-by-Step Trace

**Given Array:** `A = [15, 5, 24, 8, 1, 3, 16, 10, 20]` *(n = 9)*

### Phase 1: Divide (Top-Down)
We find the `mid` point and split until every element is alone.
- Split into `[15, 5, 24, 8, 1]` and `[3, 16, 10, 20]`
- Left side splits: `[15, 5, 24]` and `[8, 1]`
- `[15, 5, 24]` splits: `[15, 5]` and `[24]`
- `[15, 5]` splits: `[15]` and `[5]`
*(Assume this happens down to single elements for the entire array)*

### Phase 2: Merge (Bottom-Up)
We merge adjacent single elements into sorted pairs, pairs into fours, etc.
- **Merge 1:** `[15]` & `[5]` $\rightarrow$ `[5, 15]`
- **Merge 2:** `[5, 15]` & `[24]` $\rightarrow$ `[5, 15, 24]`
- **Merge 3:** `[8]` & `[1]` $\rightarrow$ `[1, 8]`
- **Merge 4:** `[5, 15, 24]` & `[1, 8]` $\rightarrow$ `[1, 5, 8, 15, 24]` (Left half is now sorted!)
- **Merge 5:** `[3]` & `[16]` $\rightarrow$ `[3, 16]`
- **Merge 6:** `[10]` & `[20]` $\rightarrow$ `[10, 20]`
- **Merge 7:** `[3, 16]` & `[10, 20]` $\rightarrow$ `[3, 10, 16, 20]` (Right half is now sorted!)
- **Final Merge:** `[1, 5, 8, 15, 24]` & `[3, 10, 16, 20]` $\rightarrow$ `[1, 3, 5, 8, 10, 15, 16, 20, 24]`

## 💻 C Code Implementation

### 1. The Recursive Divide Function
```c
void mergeSort(int a[], int lb, int ub) {
    if (lb < ub) {
        int mid = (lb + ub) / 2;
        
        // 1. Divide left half
        mergeSort(a, lb, mid);
        // 2. Divide right half
        mergeSort(a, mid + 1, ub);
        // 3. Merge them together
        merge(a, lb, mid, ub);
    }
}
```

### 2. The Merge Function (The Backbone)
> **Note:** Merging requires an extra temporary array `b[]` to hold the sorted values before copying them back to `a[]`.

```c
void merge(int a[], int lb, int mid, int ub) {
    int i = lb;       // Pointer for left sublist
    int j = mid + 1;  // Pointer for right sublist
    int k = lb;       // Pointer for temporary array b[]
    int b[100];       // Temporary auxiliary array

    // Compare elements from both sublists and copy the smaller one to b[]
    while (i <= mid && j <= ub) {
        if (a[i] <= a[j]) {
            b[k] = a[i];
            i++;
        } else {
            b[k] = a[j];
            j++;
        }
        k++;
    }

    // If the left sublist is exhausted, copy the remaining elements of the right sublist
    if (i > mid) {
        while (j <= ub) {
            b[k] = a[j];
            j++; k++;
        }
    } 
    // If the right sublist is exhausted, copy the remaining elements of the left sublist
    else {
        while (i <= mid) {
            b[k] = a[i];
            i++; k++;
        }
    }

    // Copy the merged elements back from temporary array b[] to original array a[]
    for (k = lb; k <= ub; k++) {
        a[k] = b[k];
    }
}
```

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **Time Complexity (All Cases)** | **O(n log n)** | The array is always divided precisely in half, meaning the recursion tree is always exactly $\log_2(n)$ levels deep. At each level, merging the sublists requires comparing every element, taking $O(n)$ time. Therefore, Total Time = $O(n \log n)$ regardless of the initial order of the array. |
| **Space Complexity** | **O(n)** | Merge sort is **not an in-place sort**. It requires an auxiliary array `b[]` of size `n` to hold the elements during the merging process. |

> **Exam Tip:** If a question asks for a sorting algorithm that *guarantees* $O(n \log n)$ worst-case time, the answer is **Merge Sort** (or Heap Sort), but *never* Quick Sort (which has an $O(n^2)$ worst case). However, Quick Sort is often preferred in practice because it doesn't require the $O(n)$ extra space that Merge Sort needs.
