## 📌 Executive Summary

This lecture explains **why linked lists are needed** (solving array drawbacks of fixed size and memory wastage), how nodes are structured in memory, and the fundamental differences between arrays and linked lists.

---

## 🔑 Why Linked Lists? (Array Drawbacks)
1. **Fixed size** — must declare size at compile time; can't extend
2. **Memory wastage** — declaring size 50 but using only 10 = 40 wasted
3. **Costly resizing** — fresh block + copy all data
4. **Contiguous memory required** — may not always be available

### What is a Linked List?
> A linked list is a **linear data structure** where elements are stored in **non-contiguous memory locations**, connected via **pointers**.

---

## 🔑 Node Structure

```
┌──────────┬──────────┐
│   DATA   │   NEXT   │
│  (value) │ (pointer)│
│  4 bytes │  4 bytes │
└──────────┴──────────┘
    Total: 8 bytes per node (for int)
```

### Memory Representation
```
Head → [4|113] → [3|123] → [8|132] → [7|NULL]
        @102      @113      @123      @132
```

---

## 💻 Node Definition in C

```c
struct node {
    int data;              // Data part
    struct node *next;     // Pointer to next node
};
struct node *head = NULL;  // Head pointer
```

---

## 📊 Array vs Linked List

| Feature | Array | Linked List |
|---------|-------|-------------|
| **Size** | Fixed (compile-time) | Dynamic (runtime) |
| **Memory** | Contiguous | Scattered |
| **Access** | Random — **O(1)** | Sequential — **O(n)** |
| **Insertion** | Shift elements — **O(n)** | Update pointers — **O(1)*** |
| **Deletion** | Shift elements — **O(n)** | Update pointers — **O(1)*** |
| **Extra space** | None | Pointer per node |
| **Binary Search** | ✅ Possible | ❌ Not possible |

---

## 💡 Key Takeaways

- Linked list solves the **fixed size** and **memory wastage** problems of arrays
- Each node = **data + pointer to next node**
- **Head pointer** stores address of first node
- **Sequential access only** — must traverse from head
- Insertion/Deletion is easy — just update pointers
- Defined using `struct` in C
