## 📌 Executive Summary
A **Linked List** is a linear, dynamic data structure consisting of a collection of nodes. Unlike arrays, the elements in a linked list are not stored in contiguous memory locations. Instead, each element points to the next, allowing for efficient memory utilization and dynamic resizing at runtime.

---

## 🛑 The Problem with Arrays
To understand the need for Linked Lists, we must first look at the major drawbacks of **Arrays**:
1. **Fixed Size:** Array sizes must be defined at compile-time. 
   - *Wastage:* If you declare an array of size 50 but only store 10 elements, 40 memory slots are wasted.
   - *Overflow:* If you declare an array of size 10 and want to store an 11th element, you must allocate an entirely new block of memory and copy all existing elements over (a very tedious $O(n)$ operation).
2. **Contiguous Memory Requirement:** Arrays require a single, large contiguous block of memory. Even if enough total free memory exists, if it is scattered, the array allocation will fail.
3. **Costly Insertions/Deletions:** Inserting or deleting an element in the middle of a sorted array requires shifting all subsequent elements, taking $O(n)$ time.

---

## 🔗 The Linked List Solution
Linked Lists solve these problems by allocating memory dynamically for each individual element (called a **Node**) only when it is needed. Because the memory locations are scattered, each node must store the address of the next node to maintain the sequence.

### Structure of a Node
Every node in a singly linked list contains exactly two fields:
1. **Data Part:** Stores the actual information (e.g., integer, character).
2. **Link Part (Pointer):** Stores the memory address of the *next* node in the sequence.

```text
  [ Data | Next ] ---> [ Data | Next ] ---> NULL
```

- **Head Pointer:** A special pointer that stores the address of the very first node. If `Head == NULL`, the list is empty.
- **Null Pointer:** The link part of the last node stores `NULL`, indicating the end of the list.

### C Language Representation
Because a node contains two different data types (an integer and a pointer), we must define a custom data type using a `struct`. Because the pointer stores the address of another node of the exact same type, this is called a **Self-Referential Structure**.

```c
struct node {
    int data;               // Data part
    struct node *next;      // Pointer to the next node
};

struct node *head;          // Pointer to the first node
```

## ⚖️ Linked List Properties (Exam Quick-Fire)

### Advantages ✅
1. **Dynamic Size:** Can grow or shrink dynamically at runtime. No memory wastage.
2. **Efficient Insertions/Deletions:** Inserting or deleting a node doesn't require shifting elements. It only requires updating pointers ($O(1)$ pointer manipulation).
3. **No Contiguous Memory Needed:** Can utilize scattered fragments of free memory.

### Disadvantages ❌
1. **Extra Memory Overhead:** Every node requires extra memory to store the pointer (typically 4 or 8 bytes per node).
2. **No Random Access:** You cannot directly access the $n$-th element (like `arr[5]`). You must start at the `head` and traverse sequentially, taking $O(n)$ time.
3. **Binary Search is Not Possible:** Because you cannot randomly access the middle element in $O(1)$ time, standard Binary Search cannot be applied to a Linked List.
