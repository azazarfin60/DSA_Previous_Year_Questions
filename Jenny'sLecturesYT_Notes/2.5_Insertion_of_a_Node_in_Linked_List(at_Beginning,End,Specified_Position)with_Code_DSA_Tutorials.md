## 📌 Executive Summary
Insertion in a Singly Linked List can occur in three primary locations: at the beginning, at the end, or after a specified position. The golden rule for linked list insertion is: **Always establish the right-side link first before updating the left-side link.** Failing to do so overwrites the only pointer to the rest of the list, causing data loss.

---

## 1️⃣ Case 1: Insertion at the Beginning

Inserting a node at the start of the list is extremely efficient because we already have direct access to the `head` pointer.
**Time Complexity:** $O(1)$

### Algorithm Steps:
1. Create a `newnode`.
2. Assign data to `newnode->data`.
3. Set `newnode->next = head;` (The new node now points to the current first node).
4. Update `head = newnode;` (The head now points to the new node).

### C Code Snippet
```c
void insertAtBeginning() {
    struct node *newnode;
    newnode = (struct node*) malloc(sizeof(struct node));
    
    printf("Enter data: ");
    scanf("%d", &newnode->data);
    
    // Step 3 & 4
    newnode->next = head;
    head = newnode;
}
```

> [!CAUTION]
> **Exam Trap (Order Matters):** If you reverse steps 3 and 4 (i.e., you do `head = newnode` *before* `newnode->next = head`), you overwrite the `head` pointer immediately. You lose the address of the entire original list, causing a memory leak!

## 2️⃣ Case 2: Insertion at the End

To insert at the end, we must traverse the entire list to find the last node.
**Time Complexity:** $O(n)$

### Algorithm Steps:
1. Create a `newnode`. Set its data, and strictly set `newnode->next = NULL;` (since it will be the last node).
2. Create a `temp` pointer starting at `head`.
3. Traverse until `temp` points to the *last* node.
4. Set the last node's next pointer: `temp->next = newnode;`

### C Code Snippet
```c
void insertAtEnd() {
    struct node *newnode, *temp;
    newnode = (struct node*) malloc(sizeof(struct node));
    
    printf("Enter data: ");
    scanf("%d", &newnode->data);
    newnode->next = NULL; 
    
    temp = head;
    // Traverse to the last node
    while (temp->next != NULL) {
        temp = temp->next;
    }
    
    // Link the last node to the new node
    temp->next = newnode;
}
```

> [!WARNING]
> **Exam Trap (Traversal Condition):** Notice the condition is `while(temp->next != NULL)`. Students often mistakenly write `while(temp != NULL)`. If you do that, the loop stops when `temp` literally equals `NULL` (it has fallen off the end of the list). You cannot do `NULL->next = newnode`—this causes a segmentation fault. We must stop *at* the last node, not *after* it!

## 3️⃣ Case 3: Insertion After a Specified Position

We must traverse the list until we reach the specific $k^{th}$ node, then carefully wedge the new node between $k$ and $k+1$.
**Time Complexity:** $O(n)$ (Average case)

### Algorithm Steps:
1. Ask the user for the position `pos`. (Assume 1-based indexing).
2. Check if `pos` is valid.
3. Use a `temp` pointer and an integer `i = 1` to traverse up to `pos`.
4. Create the `newnode` and set its data.
5. **Establish right link:** `newnode->next = temp->next;`
6. **Establish left link:** `temp->next = newnode;`

### C Code Snippet
```c
void insertAfterPosition() {
    struct node *newnode, *temp;
    int pos, i = 1;
    
    printf("Enter position after which to insert: ");
    scanf("%d", &pos);
    
    // Assume we have checked validity of 'pos' against list length
    
    temp = head;
    // Traverse until temp is pointing to the given 'pos' node
    while (i < pos) {
        temp = temp->next;
        i++;
    }
    
    newnode = (struct node*) malloc(sizeof(struct node));
    printf("Enter data: ");
    scanf("%d", &newnode->data);
    
    // Step 5: Right link FIRST
    newnode->next = temp->next;
    
    // Step 6: Left link SECOND
    temp->next = newnode;
}
```

> [!CAUTION]
> **Exam Trap (Link Disconnection):** In step 5, if you update `temp->next = newnode` first, you overwrite the address of node $k+1$. You permanently lose the remainder of your list. Always link the new node to the right side of the list before you cut the left side to point to the new node.
