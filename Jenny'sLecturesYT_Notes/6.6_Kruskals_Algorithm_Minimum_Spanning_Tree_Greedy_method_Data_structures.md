# 6.6 Kruskal's Algorithm for Minimum Spanning Tree | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=1N2D3mC26X8)

---

## 📌 Executive Summary
**Kruskal's Algorithm** is a fundamental **Greedy Algorithm** used to find the Minimum Spanning Tree (MST) of a weighted, undirected graph. Unlike Prim's algorithm which grows a single tree from a root node, Kruskal's algorithm looks at the entire graph globally, sorts all the edges, and builds the MST by picking the absolute smallest edges first—provided they do not form a cycle.

---

## ⚙️ Step-by-Step Algorithm

### Step 1: Preprocessing (Graph Simplification)
Before applying the core algorithm, you must clean up the graph:
1. **Remove all loops:** If there is an edge starting and ending at the exact same vertex (a self-loop), delete it regardless of its weight.
2. **Remove parallel edges:** If there are multiple edges connecting the exact same two vertices (e.g., $A \rightarrow B$ with weights 5 and 10), **keep only the edge with the minimum weight** (keep 5, remove 10).

### Step 2: Sorting Edges
Extract all the remaining edges from the simplified graph and arrange them in **increasing order (ascending)** based on their edge weights. 

### Step 3: Edge Selection
Iterate through your sorted list of edges from the smallest to largest:
1. Pick the edge with the minimum weight.
2. Add it to the MST.
3. **Cycle Check:** Does adding this edge create a closed loop/cycle with the edges already present in your MST?
   - **If NO:** Keep the edge.
   - **If YES:** Discard the edge permanently and move to the next edge in the list.
4. Stop the process when you have successfully added exactly **$N - 1$ edges** to the MST (where $N$ is the total number of vertices in the original graph).

## 📐 Algorithm Trace Example
*(Based on a 6-Vertex Graph trace from the transcript)*

**Sorted Edges (After Step 1 & 2):**
1. BD = 2
2. DE = 2
3. AC = 3
4. CD = 3
5. BE = 4
6. AB = 5
7. BC = 6
8. AF = 7
9. FC = 8

**Selection Trace (Step 3):**
1. Pick **BD (2)** $\rightarrow$ No cycle. **(Keep)**
2. Pick **DE (2)** $\rightarrow$ No cycle. **(Keep)**
3. Pick **AC (3)** $\rightarrow$ No cycle. **(Keep)**
4. Pick **CD (3)** $\rightarrow$ No cycle. **(Keep)**
5. Pick **BE (4)** $\rightarrow$ **CYCLE FORMED** (B-D-E-B). *(Discard)*
6. Pick **AB (5)** $\rightarrow$ **CYCLE FORMED** (A-C-D-B-A). *(Discard)*
7. Pick **BC (6)** $\rightarrow$ **CYCLE FORMED** (B-C-D-B). *(Discard)*
8. Pick **AF (7)** $\rightarrow$ No cycle. **(Keep)**

**Result:** We have selected 5 edges (BD, DE, AC, CD, AF). Since $N = 6$, $N-1 = 5$. The algorithm terminates successfully. The final MST contains 6 vertices and 5 edges.

## ⏱️ Complexity Analysis

| Implementation | Time Complexity | Reason |
| :--- | :--- | :--- |
| **Sorting Edges** | **O(E log E)** or **O(E log V)** | Standard sorting algorithms (like Quick Sort / Merge Sort) take $O(E \log E)$. Since $E \le V^2$, $\log(E) \le 2\log(V)$, making it equivalent to $O(E \log V)$. |
| **Cycle Detection (Disjoint Set/Union-Find)**| **O(E log V)** | Using the Disjoint Set (Union-Find) data structure to check for cycles takes $O(\log V)$ per edge. |
| **Total Time Complexity** | **O(E log V)** | The sorting step dominates the time complexity. |

> **Exam Tip:** In exams, always create a literal table of your sorted edges and put a distinct ✅ (tick) or ❌ (cross) next to each edge as you test them for cycles. State explicitly why an edge was rejected (e.g., "Rejected BE(4) as it forms cycle B-D-E"). Examiners look for this explicit trace table to award full marks. Also remember that Kruskal's is better for **sparse graphs** while Prim's is better for dense graphs.
