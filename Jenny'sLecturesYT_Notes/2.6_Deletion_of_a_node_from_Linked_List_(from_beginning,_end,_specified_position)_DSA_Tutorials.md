## 📌 Executive Summary

This lecture covers **three types of deletion** from a singly linked list. Each uses `free()` to release memory. End/position deletion requires **two pointers** (current and previous).

---

## 💻 1. Delete from Beginning

```c
void deleteFromBeginning() {
    if (head == NULL) { printf("List is empty!\n"); return; }
    struct node *temp = head;
    head = head->next;    // Move head forward
    free(temp);           // Free old first node
}
```
**Time: O(1)**

---

## 💻 2. Delete from End

Two pointers: `temp` (current) and `prev` (previous)

```c
void deleteFromEnd() {
    if (head == NULL) { printf("Empty!\n"); return; }
    struct node *temp = head, *prev;

    if (temp->next == NULL) {   // Only one node
        head = NULL;
        free(temp);
        return;
    }

    while (temp->next != NULL) {
        prev = temp;             // prev trails behind
        temp = temp->next;
    }
    prev->next = NULL;           // Disconnect last
    free(temp);                  // Free last node
}
```
**Time: O(n)**

---

## 💻 3. Delete from Specified Position

```c
void deleteFromPosition(int pos) {
    struct node *temp = head, *nextNode;
    int i = 1;

    if (pos == 1) { deleteFromBeginning(); return; }

    while (i < pos - 1) {
        temp = temp->next;       // Go to node before target
        i++;
    }
    nextNode = temp->next;          // Node to delete
    temp->next = nextNode->next;    // Bypass it
    free(nextNode);                 // Free memory
}
```
**Time: O(n)**

---

## ⏱️ Time Complexity

| Operation | Time |
|-----------|------|
| Delete from **beginning** | **O(1)** |
| Delete from **end** | **O(n)** |
| Delete from **position** | **O(n)** |

---

## 💡 Key Takeaways

- Always **free memory** after deletion — avoid memory leaks
- Use **two pointers** for end/position deletion
- Check **empty list** and **single node** as special cases
- Insertion time complexity mirrors deletion complexity
