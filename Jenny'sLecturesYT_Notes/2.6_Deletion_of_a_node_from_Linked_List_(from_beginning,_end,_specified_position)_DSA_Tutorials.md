## 📌 Executive Summary
Deletion in a Singly Linked List involves bypassing the target node by updating pointers. However, merely detaching a node is insufficient—you **must** release the dynamically allocated memory using the `free()` function to prevent memory leaks.

---

## 1️⃣ Case 1: Deletion from the Beginning

This is the simplest operation because we have direct access via the `head` pointer.
**Time Complexity:** $O(1)$

### Algorithm Steps:
1. Ensure the list is not empty (`head != NULL`).
2. Point a `temp` pointer to the `head`.
3. Shift the `head` to point to the second node: `head = head->next`.
4. Free the memory held by `temp`.

### C Code Snippet
```c
void deleteFromBeginning() {
    struct node *temp;
    
    if (head == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    temp = head;            // Temp points to the node to be deleted
    head = head->next;      // Head moves to the second node
    free(temp);             // Release the memory
}
```

## 2️⃣ Case 2: Deletion from the End

To delete the last node, we must traverse the list. Because a singly linked list cannot look backward, we must maintain **two pointers** during traversal: one pointing to the last node (`temp`), and one pointing to the second-to-last node (`prevnode`).
**Time Complexity:** $O(n)$

### Algorithm Steps:
1. Check if the list is empty.
2. If there is only one node (`head->next == NULL`), free `head` and set `head = NULL`.
3. Initialize `temp = head` and `prevnode`.
4. Traverse the list: `while(temp->next != NULL) { prevnode = temp; temp = temp->next; }`
5. At the end of the loop, `temp` points to the last node, and `prevnode` points to the second-to-last node.
6. Sever the link: `prevnode->next = NULL;`
7. Release the memory: `free(temp);`

### C Code Snippet
```c
void deleteFromEnd() {
    struct node *temp, *prevnode;
    
    if (head == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    temp = head;
    
    // Traversing with two pointers
    while (temp->next != NULL) {
        prevnode = temp;
        temp = temp->next;
    }
    
    // If only one node existed in the list
    if (temp == head) {
        head = NULL;
    } else {
        prevnode->next = NULL; // Disconnect the last node
    }
    
    free(temp); // Free the last node
}
```

## 3️⃣ Case 3: Deletion from a Specified Position

To delete the $k^{th}$ node, we must stop our traversal at the $(k-1)^{th}$ node. We then update the $(k-1)^{th}$ node's pointer to leapfrog the $k^{th}$ node and point directly to the $(k+1)^{th}$ node.
**Time Complexity:** $O(n)$ (Average case)

### Algorithm Steps:
1. Traverse the list using a loop `while(i < pos - 1)` to land the `temp` pointer on the $(k-1)^{th}$ node.
2. Create a pointer `nextnode` and set it to the node to be deleted: `nextnode = temp->next`.
3. Leapfrog the `nextnode` by updating the link: `temp->next = nextnode->next`.
4. Free the memory: `free(nextnode)`.

### C Code Snippet
```c
void deleteFromPosition() {
    struct node *temp, *nextnode;
    int pos, i = 1;
    
    printf("Enter position to delete: ");
    scanf("%d", &pos);
    
    temp = head;
    
    // Traverse to the node just BEFORE the one we want to delete (pos - 1)
    while (i < pos - 1) {
        temp = temp->next;
        i++;
    }
    
    // 'nextnode' points to the node to be deleted
    nextnode = temp->next;
    
    // Leapfrog the deleted node
    temp->next = nextnode->next;
    
    // Free the memory
    free(nextnode);
}
```

> [!CAUTION]
> **Exam Trap (The Memory Leak):** Never write `temp->next = temp->next->next` without first storing the middle node's address in a temporary pointer. If you overwrite the pointer directly, you lose access to the detached node, making it impossible to call `free()`. This creates a memory leak, which is heavily penalized in exams!
