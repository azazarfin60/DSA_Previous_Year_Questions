# 9.1 Huffman Coding - Greedy Method | Data Structures Tutorials

📺 [Watch on YouTube](https://www.youtube.com/watch?v=saofdNsZiYY)

---

## 📌 Executive Summary

Huffman Coding is a **greedy algorithm** for lossless data compression. It assigns **variable-length codes** — shorter codes to more frequent characters and longer codes to less frequent ones, using a binary tree.

---

## 📐 Algorithm

1. Create a **leaf node** for each character with its frequency
2. Build a **min-heap** (priority queue) of all nodes
3. While more than one node in heap:
   - Extract **two minimum** frequency nodes
   - Create a new internal node with frequency = sum of both
   - Make the two nodes its left (0) and right (1) children
   - Insert new node back into heap
4. Remaining node = **root** of Huffman Tree
5. Traverse tree: left = 0, right = 1 → codes for each character

---

## 📐 Example

Characters and frequencies:
```
A:5, B:9, C:12, D:13, E:16, F:45
```

### Step-by-step Construction

```
Step 1: Extract min A(5) and B(9) → create node (14)
           14
          /  \
        A:5  B:9

Step 2: Extract C(12) and D(13) → create node (25)
           25
          /  \
       C:12  D:13

Step 3: Extract 14 and E(16) → create node (30)
           30
          /  \
        14   E:16
       / \
     A:5 B:9

Step 4: Extract 25 and 30 → create node (55)
             55
           /    \
         25      30
        / \     / \
     C:12 D:13 14 E:16
               / \
             A:5 B:9

Step 5: Extract F(45) and 55 → create root (100)
                100
              /     \
           F:45      55
                   /    \
                 25      30
                / \     / \
             C:12 D:13 14 E:16
                       / \
                     A:5 B:9
```

### Huffman Codes:
```
F = 0           (1 bit)
C = 100         (3 bits)
D = 101         (3 bits)
A = 1100        (4 bits)
B = 1101        (4 bits)
E = 111         (3 bits)
```

### Verification — Total bits:
```
Without Huffman (3 bits each): 6 chars × 3 = 18 bits per symbol set
With Huffman:
  F: 45×1 + C: 12×3 + D: 13×3 + A: 5×4 + B: 9×4 + E: 16×3
  = 45 + 36 + 39 + 20 + 36 + 48 = 224 bits

vs Fixed 3-bit: (5+9+12+13+16+45) × 3 = 300 bits
Savings: 25.3%!
```

---

## 🔑 Properties of Huffman Codes

| Property | Description |
|----------|-------------|
| **Prefix-free** | No code is a prefix of another → unambiguous decoding |
| **Optimal** | Minimum weighted path length among all prefix codes |
| **Variable length** | Frequent chars → short codes |
| **Greedy** | Always merge two smallest frequencies |

---

## ⏱️ Complexity

| Operation | Time |
|-----------|------|
| Building tree | **O(n log n)** |
| Space | O(n) |

---

## 🎯 Applications
1. **File compression** (ZIP, GZIP)
2. **Image compression** (JPEG uses Huffman)
3. **Text compression**
4. **Network data transmission**

---

## 💡 Key Takeaways

- **Greedy**: always merge two smallest frequencies
- Result is a **prefix-free** binary code
- More frequent = shorter code → **optimal compression**
- Build min-heap → extract two mins → merge → repeat
- **O(n log n)** time complexity
- Very common exam question — practice tree construction!
