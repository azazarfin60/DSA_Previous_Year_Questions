# 2.3 Arrays vs Linked Lists | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=115K8LwP-xY) *(Note: Placeholder link)*

---

## 📌 Executive Summary
There is no absolute "better" data structure; the choice between an Array and a Linked List depends entirely on the specific requirements of your program. If your program relies heavily on **searching and accessing** data randomly, use an Array. If your program relies heavily on **inserting and deleting** data dynamically at the beginning, use a Linked List.

---

## ⚔️ Comparison Table (Exam Focus)

When asked to compare Arrays and Linked Lists in an exam, use this exact table:

| Criteria | Arrays | Linked Lists |
| :--- | :--- | :--- |
| **Memory Allocation** | Static (Compile-time, fixed size) | Dynamic (Run-time, variable size) |
| **Memory Utilization** | Inefficient (Unused indices waste memory) | Efficient (Memory allocated exactly when needed) |
| **Memory Overhead** | Low (Stores only data) | High (Requires extra memory for pointers) |
| **Storage Location** | Contiguous (Continuous memory blocks) | Non-contiguous (Scattered memory blocks) |
| **Access Time** | $O(1)$ Random Access via Index | $O(n)$ Sequential Access (Must traverse from head) |
| **Insertion / Deletion** | Slow $O(n)$ (Requires shifting elements) | Fast $O(1)$ at the beginning (Just updating pointers) |
| **Searching** | Linear Search & Binary Search allowed | Only Linear Search allowed |

---

## 🧠 Detailed Complexity Breakdown

### 1. Cost of Access
- **Array:** Because arrays are stored in contiguous memory blocks, the compiler uses a simple formula to jump directly to any index in constant time: `Base Address + (Index * Size of Data Type)`. **Time: $O(1)$**
- **Linked List:** You cannot randomly jump to the 5th node. You must start at the `head` pointer and follow the links one by one until you reach the 5th node. **Time: $O(n)$**

### 2. Cost of Insertion
- **Array:**
  - **At Beginning/Middle:** You must shift all subsequent elements to the right to make space. **Time: $O(n)$**
  - **At End:** If the array is not full, you can place it directly at the end index. **Time: $O(1)$**
- **Linked List:**
  - **At Beginning:** Create the node, point its `next` to the current first node, and update `head`. No shifting required. **Time: $O(1)$**
  - **At End/Middle:** While the actual pointer manipulation is $O(1)$, you must first *traverse* the list to find the end or the middle node. **Time: $O(n)$**

> **Exam Tip:** If a question asks *"Which data structure is better if you have a massive, complex data type (e.g., a struct taking 100 bytes) and the number of elements fluctuates wildly?"* 
> 
> **Answer:** Linked List. Although linked lists require extra pointer memory (4 or 8 bytes), 8 extra bytes is negligible compared to a 100-byte data block. Using an array could lead to massive memory wastage if you declare 1000 slots but only use 10 (wasting $990 \times 100 = 99,000$ bytes).
