## 📌 Executive Summary

A head-to-head comparison of Arrays and Linked Lists — memory allocation, access patterns, operations, and when to use each.

---

## 📊 Comparison Table

| Feature | Array | Linked List |
|---------|-------|-------------|
| **Memory** | Contiguous, fixed | Non-contiguous, dynamic |
| **Size** | Fixed at compile time | Grows/shrinks at runtime |
| **Access** | **O(1)** random access | **O(n)** sequential |
| **Insertion (beginning)** | O(n) shift | **O(1)** |
| **Insertion (end)** | O(1) if space | O(n) traverse |
| **Deletion** | O(n) shift | O(1) if pointer known |
| **Memory overhead** | None | Extra pointer per node |
| **Cache** | **Excellent** (contiguous) | Poor (scattered) |
| **Memory waste** | If not fully used | No waste |

---

## 🎯 When to Use

| Use Array When | Use Linked List When |
|----------------|---------------------|
| Size is known | Size varies dynamically |
| Need random access | Frequent insert/delete |
| Memory is limited | No random access needed |
| Cache performance matters | Memory fragmented |

---

## 💡 Key Takeaways

- Arrays: O(1) access, poor insert/delete
- Linked Lists: O(1) insert/delete, poor access
- Arrays have better cache locality
- Linked lists waste memory on pointers
