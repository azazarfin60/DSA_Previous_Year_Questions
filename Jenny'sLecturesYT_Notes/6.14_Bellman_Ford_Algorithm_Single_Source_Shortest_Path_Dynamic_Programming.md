# 6.14 Bellman-Ford Algorithm (Single Source Shortest Path) | Data Structures

📺 [Watch on YouTube](https://www.youtube.com/watch?v=FtN3BYH2Zes) *(Note: Placeholder link)*

---

## 📌 Executive Summary
The **Bellman-Ford Algorithm** is a Dynamic Programming approach to solving the Single Source Shortest Path problem. While Dijkstra's Algorithm is greedy and faster, it fails on negative edge weights. Bellman-Ford mathematically guarantees correct shortest paths even if the graph contains **negative edge weights**. However, it is slower and will explicitly report a failure if the graph contains a **Negative Weight Cycle**.

---

## 🔑 Core Concepts & Initialization
Like Dijkstra, Bellman-Ford uses a Distance Array initialized with the Source node at `0` and all other nodes at `\infty`.

**The Relaxation Formula (Same as Dijkstra):**
```c
if (Distance[U] + Cost(U, V) < Distance[V]) {
    Distance[V] = Distance[U] + Cost(U, V);
}
```

**The Core Rule:**
Instead of greedily picking the minimum vertex, Bellman-Ford simply takes **every single edge** in the graph and relaxes them. It repeats this entire process exactly **$N - 1$ times** (where $N$ is the number of vertices).

---

## ⚙️ Step-by-Step Algorithm
1. **Initialize:** Set the distance of the Source vertex to `0` and all others to $\infty$.
2. **List Edges:** Write down a linear list of all edges in the graph (the order does not mathematically matter, though different orders might reach the final state in fewer passes).
3. **The $(N-1)$ Iteration Loop:**
   - Iterate exactly $N - 1$ times.
   - In each iteration, go through your entire list of edges one by one.
   - For every edge $U \rightarrow V$, apply the Relaxation Formula.
   - *Why $N-1$ times?* Because the longest possible shortest path between any two nodes in a graph with $N$ vertices (without looping) can contain at most $N-1$ edges.
4. **The Negative Cycle Check (The $N$-th Iteration):**
   - After completing $N-1$ iterations, the absolute shortest paths are guaranteed to be found—*assuming there are no negative weight cycles*.
   - Run the relaxation loop exactly **one more time** (the $N$-th pass).
   - If **any** distance updates/decreases during this $N$-th pass, it is mathematical proof that the graph contains a **Negative Weight Cycle**. The algorithm must terminate and report that shortest paths cannot be found.

---

## ⚠️ Negative Weight Cycles
A **Negative Weight Cycle** is a cycle in a graph where the sum of all edge weights in that cycle is a negative number (e.g., $A \xrightarrow{7} B \xrightarrow{7} C \xrightarrow{-15} A$. Sum $= -1$).

If you keep traversing this cycle, your total cost will keep decreasing to $-\infty$. Therefore, a "shortest path" does not logically exist. Bellman-Ford's greatest strength is its ability to **detect** these cycles using the $N$-th iteration check, whereas Dijkstra would infinite-loop or return garbage values.

---

## ⏱️ Complexity Analysis

| Metric | Complexity | Reason |
| :--- | :--- | :--- |
| **General Time Complexity** | **O(V $\times$ E)** | We iterate through $E$ edges exactly $V-1$ times. $E \times (V-1) \approx O(V \times E)$. |
| **Worst-Case (Complete Graph)** | **O(V³)** | In a complete graph, the number of edges $E = \frac{V(V-1)}{2}$. Substituting this into $O(V \times E)$ yields $O(V^3)$. |

> **Exam Tip:** Bellman-Ford is heavily tested in theory questions regarding negative edges. 
> - **Question:** "Which algorithm finds shortest paths with negative weights?" $\implies$ **Bellman-Ford**.
> - **Question:** "Does Bellman-Ford work on Negative Weight Cycles?" $\implies$ **NO, it detects them and fails.**
> - **Question:** "Why does it iterate exactly $N-1$ times?" $\implies$ Because the maximum path length without a cycle in an $N$-vertex graph is $N-1$ edges.
