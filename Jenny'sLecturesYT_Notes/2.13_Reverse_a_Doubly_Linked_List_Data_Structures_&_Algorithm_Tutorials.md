## 📌 Executive Summary

Reversing a DLL is simpler than SLL — just **swap `prev` and `next` pointers** of every node, then update head.

---

## 💻 Algorithm

```c
void reverse() {
    struct node *temp = NULL;
    struct node *current = head;

    while (current != NULL) {
        temp = current->prev;           // Save prev
        current->prev = current->next;  // Swap: prev ← next
        current->next = temp;           // Swap: next ← prev
        current = current->prev;        // Move forward (prev is new next)
    }
    if (temp != NULL)
        head = temp->prev;              // Update head
}
```

---

## 🔄 Trace

```
Before: [NULL|5|200] ⇄ [100|6|300] ⇄ [200|8|NULL]

Node 5: swap prev(NULL) ↔ next(200) → [200|5|NULL]
Node 6: swap prev(100) ↔ next(300) → [300|6|100]
Node 8: swap prev(200) ↔ next(NULL) → [NULL|8|200]

After:  [NULL|8|200] ⇄ [300|6|100] ⇄ [200|5|NULL]
        head now → node 8
```

---

## ⏱️ Complexity
- **Time: O(n)** — single traversal
- **Space: O(1)** — in-place

---

## 💡 Key Takeaways

- DLL reversal = **swap prev and next** of each node
- Simpler than SLL reversal (no need for 3 pointers)
- After loop, `head = temp->prev` to set new head
