## 📌 Executive Summary
Reversing a Doubly Linked List (DLL) is conceptually simpler than reversing a Singly Linked List (SLL). Because every node already knows both its predecessor and successor, reversing the list simply means **swapping the `prev` and `next` pointers within every single node**, and finally swapping the global `head` and `tail` pointers.

---

## 🧠 The Swapping Concept

If a node looks like this before reversal:
`[ Prev: 100 | Data: 5 | Next: 300 ]`

After reversal, it must look exactly like this:
`[ Prev: 300 | Data: 5 | Next: 100 ]`

To achieve this without losing our place in the list, we need two pointers:
1. `current`: Points to the node we are currently swapping. Initialized to `head`.
2. `nextnode`: Temporarily stores `current->next` so we know where to go after the swap.

---

## 🛠️ The Iterative Algorithm

### Step 1: Swap Pointers Inside Each Node
Traverse the list using a `while (current != NULL)` loop. Inside the loop:
1. Save the next node: `nextnode = current->next;`
2. Update the forward link to point backward: `current->next = current->prev;`
3. Update the backward link to point forward: `current->prev = nextnode;`
4. Move to the next node to process: `current = nextnode;`

### Step 2: Swap the Global Head and Tail
Once the loop finishes, all internal node links are reversed. However, the `head` is still pointing to what is now the end of the list, and `tail` is pointing to the new beginning. We must swap them.

### C Code Snippet

```c
void reverseDLL() {
    struct node *current, *nextnode;
    
    if (head == NULL) {
        printf("List is empty.\n");
        return;
    }
    
    current = head;
    
    // ---------------------------------------------------
    // Phase 1: Swap 'prev' and 'next' inside EVERY node
    // ---------------------------------------------------
    while (current != NULL) {
        // 1. Secure the remainder of the list
        nextnode = current->next;
        
        // 2. Swap the pointers
        current->next = current->prev;
        current->prev = nextnode;
        
        // 3. Move to the next node in the sequence
        current = nextnode;
    }
    
    // ---------------------------------------------------
    // Phase 2: Swap the global 'head' and 'tail' pointers
    // ---------------------------------------------------
    current = head;     // Borrow 'current' as a temporary variable
    head = tail;
    tail = current;
}
```

> [!WARNING]
> **Exam Trap (Infinite Loop during Reversal):**
> Notice step 4: `current = nextnode;`. 
> A common mistake is writing `current = current->next;`. 
> **Why is this wrong?** Because in step 2, you just overwrote `current->next` with `current->prev`! If you do `current = current->next`, you will travel *backward* to the node you just processed, creating an infinite loop. Always use the secured `nextnode` pointer to move forward.

---

## ⏱️ Complexity Analysis
- **Time Complexity:** $O(n)$. We must visit every single node once to swap its pointers.
- **Space Complexity:** $O(1)$. We only use two temporary pointer variables regardless of list length.
