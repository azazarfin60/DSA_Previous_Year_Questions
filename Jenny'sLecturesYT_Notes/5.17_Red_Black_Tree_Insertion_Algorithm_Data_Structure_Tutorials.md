## 📌 Executive Summary
This lecture details the strict step-by-step algorithm for inserting a node into a Red-Black Tree. It handles Red-Red conflicts by checking the color of the "Uncle" node, which dictates whether you simply Recolor or perform an AVL-style Rotation + Recolor.

---

## 🔑 RBT Insertion Rules

1. **If tree is empty:** Create the new node as root and color it **BLACK**.
2. **If tree is NOT empty:** Insert the new node using standard BST rules and color it **RED**.
3. **Check the Parent of the newly inserted RED node:**
   - **If Parent is BLACK:** You are done! Exit.
   - **If Parent is RED:** You have a Red-Red conflict. Proceed to Rule 4.
4. **Resolve Red-Red Conflict (Check the UNCLE):**
   > *Uncle = The sibling of the Parent node.*
   
   **Case A: Uncle is RED**
   - **Action:** Recolor!
   - Change Parent and Uncle to **BLACK**.
   - Change Grandparent to **RED**.
   - *Exception:* If the Grandparent is the Root, it must stay BLACK.
   - **Recheck:** Since Grandparent is now RED, recheck it against its parent for a new Red-Red conflict.

   **Case B: Uncle is BLACK or NULL (NIL)**
   - **Action:** Rotate + Recolor!
   - Perform the appropriate AVL rotation (LL, RR, LR, RL) on the `Grandparent -> Parent -> NewNode` path.
   - After rotation, the *median* element becomes the local root.
   - **Recolor:** Change the new local root to **BLACK** and its two children to **RED**.

## 📐 Solved Example Trace

**Insertion Sequence:** `10, 18, 7, 15, 16, 30, 25, 40, 60, 2, 1, 70`

### 1. Simple Insertions
- **Insert 10**: Root. Make it **Black**. `[10(B)]`
- **Insert 18**: Red. Parent `10(B)` is Black. Done.
- **Insert 7**: Red. Parent `10(B)` is Black. Done.
```text
      10(B)
     /    \
   7(R)  18(R)
```

### 2. Triggers Case A (Uncle is RED)
- **Insert 15**: Goes to left of `18`. Colored **Red**.
- **Conflict:** `18(R)` and `15(R)`.
- **Check Uncle:** Uncle of `15` is `7`. `7` is **RED**.
- **Action:** Recolor Parent (`18`) and Uncle (`7`) to Black. Grandparent (`10`) becomes Red. But `10` is Root, so it stays Black.
```text
      10(B)
     /    \
   7(B)  18(B)
         /
      15(R)
```

### 3. Triggers Case B (Uncle is NULL) + LR Rotation
- **Insert 16**: Goes to right of `15`. Colored **Red**.
- **Conflict:** `15(R)` and `16(R)`.
- **Check Uncle:** Uncle of `16` is Null (Right child of 18 is Null). Null is **BLACK**.
- **Action:** Rotate + Recolor.
- **Path:** Left of 18, Right of 15 = **LR Rotation**.
- **Rotation:** Median of `{18, 15, 16}` is `16`. `16` becomes the local root. `15` left, `18` right.
- **Recolor:** New root `16` becomes Black. Children `15, 18` become Red.
```text
      10(B)
     /    \
   7(B)  16(B)
         /   \
      15(R)  18(R)
```

### 4. Triggers Case A (Uncle is RED) + Propagation
- **Insert 30**: Right of `18(R)`.
- **Conflict:** `18(R)` and `30(R)`. Uncle is `15(R)` (**RED**).
- **Action:** Recolor. Parent `18` and Uncle `15` become Black. Grandparent `16` becomes Red.
- **Recheck:** `16(R)` parent is `10(B)`. No conflict. Done.
```text
      10(B)
     /    \
   7(B)  16(R)
         /   \
      15(B)  18(B)
               \
               30(R)
```

### 5. More Insertions...
*(The video continues this exact logic for the rest of the array)*
- **Insert 25**: Triggers Case B (RL Rotation) on `{18, 30, 25}`. `25` becomes Black, `18, 30` Red.
- **Insert 40**: Triggers Case A (Recolor `30, 18` to Black, `25` to Red). This causes a cascading conflict between `25(R)` and `16(R)`. Uncle of `25` is `7(B)`. Triggers Case B (RR Rotation) on `{10, 16, 25}`. `16` becomes root (Black), `10(R)` and `25(R)` become children.

## 💡 Key Takeaway
Always memorize the Uncle Rule:
- **Red Uncle = Recolor** (pushes Red up the tree, may cause cascading conflicts).
- **Black/Null Uncle = Rotate & Recolor** (fixes the conflict immediately in that subtree).
