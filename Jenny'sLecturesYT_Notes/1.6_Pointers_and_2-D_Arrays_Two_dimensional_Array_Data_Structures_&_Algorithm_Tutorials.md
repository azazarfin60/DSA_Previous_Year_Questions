## 📌 Executive Summary

This lecture explains the relationship between **pointers** and **2D arrays** in C. A 2D array is treated as an **array of 1D arrays**, and this concept is key to understanding how pointer arithmetic works with multi-dimensional arrays.

---

## 🔑 2D Array = Array of Arrays

For `int a[3][3] = {{6,2,5},{3,8,7},{4,9,1}}`:

```
Logical View:          Memory Layout (Row Major):
  Col  0  1  2
Row 0: 6  2  5         a[0]: [6][2][5]  addresses: 100, 104, 108
Row 1: 3  8  7         a[1]: [3][8][7]  addresses: 112, 116, 120
Row 2: 4  9  1         a[2]: [4][9][1]  addresses: 124, 128, 132
```

- `a[0]`, `a[1]`, `a[2]` are each **1D arrays** (3 elements each)
- `a` is an **array of three 1D arrays**

---

## 📐 Pointer Declarations

```c
int *p;
p = a;         // ❌ INVALID — a points to a 1D array, not int
p = a[0];      // ✅ VALID — a[0] is name of 1D array, points to int
p = &a[0][0];  // ✅ VALID — address of an integer
```

---

## 📐 Address Expressions

| Expression | Returns | Value |
|-----------|---------|-------|
| `a` | Pointer to first 1D array | 100 |
| `a[0]` | Pointer to first element of `a[0]` | 100 |
| `a+1` | Pointer to second 1D array `a[1]` | 112 |
| `*(a+1)` | Base address of `a[1]` | 112 |
| `*(a+1)+2` | Address of `a[1][2]` | 120 |
| `*(*(a+1)+2)` | **Value** of `a[1][2]` | 7 |
| `&a` | Address of entire 2D array | 100 |

---

## 📐 Accessing Elements

### General Formula
```
a[i][j] = *(*(a + i) + j)
```

### Step-by-Step: `*(*(a+1)+2)`
```
a           → pointer to a[0] (address 100)
a+1         → pointer to a[1] (address 112)
*(a+1)      → a[1] itself → base address 112
*(a+1)+2    → 112 + 2×4 = 120
*(*(a+1)+2) → value at 120 = 7  ✅
```

---

## ⚠️ Key Difference: `a+1` vs `&a+1`

```c
a + 1   → adds sizeof(one 1D array) = 3×4 = 12 bytes → 112
&a + 1  → adds sizeof(entire 2D array) = 9×4 = 36 bytes → 136
```

Similarly for 1D array `int a[3]`:
```c
a + 1    → 200 + 4 = 204     (next element)
&a + 1   → 200 + 12 = 212    (past entire array)
```

---

## 💡 Key Takeaways

- 2D array is an **array of 1D arrays** in memory
- `a[i]` is the **name** of the i-th 1D array → returns address of first element
- Use `*(*(a+i)+j)` to access `a[i][j]` via pointer arithmetic
- `a`, `&a[0][0]`, `a[0]` give same numeric value but **different types**
- Memory stored in **row-major order** in C
