# 1.6 Pointers and 2D Arrays | Data Structures

---

## 📌 Executive Summary
Because a 2D Array is an "array of 1D arrays", accessing a 2D array using pointers requires a **double dereference**. 

The base name of a 2D array (e.g., `A`) does not point to a single integer; it points to the **first 1D array** (the entire first row). Understanding this pointer hierarchy is essential for advanced C programming and exam questions.

---

## 🔗 Pointers and 2D Array Equivalencies

### 1. What does `A` return?
- `A` returns the base address of the 2D array, but its type is a **pointer to an array of integers**.
- `A[0]` (or `*A`) returns the base address of the first 1D array. Its type is a **pointer to an integer**.
- `**A` returns the actual integer value at `A[0][0]`.

### 2. Accessing Elements via Pointer Arithmetic
To access element `A[i][j]` using pointers, you must use the following syntax:

$$ A[i][j] \equiv *(*(A + i) + j) $$

**Step-by-Step Breakdown:**
1. `A`: Pointer to the first 1D array (Row 0).
2. `A + i`: Pointer to the $i$-th 1D array (Row $i$).
3. `*(A + i)`: Dereferences the 1D array pointer, giving the base integer address of Row $i$. (This is equivalent to `A[i]`).
4. `*(A + i) + j`: Moves $j$ integer elements deep into Row $i$.
5. `*(*(A + i) + j)`: Dereferences that specific integer address to retrieve the actual data value.

---

## ⚠️ The Advanced `+1` Exam Trap

If you print the addresses, `A`, `A[0]`, and `&A` might all print the exact same number (e.g., `100`). However, their pointer arithmetic scales are completely different:

- **`A[0] + 1` (or `*A + 1`):** Points to the next *integer* in the first row. Adds `sizeof(int)` (4 bytes).
- **`A + 1`:** Points to the next *1D array* (the second row). Adds `sizeof(entire row)` (e.g., $3 \times 4 = 12$ bytes).
- **`&A + 1`:** Points to the memory block immediately following the *entire 2D matrix*. Adds `sizeof(entire matrix)` (e.g., $3 \times 3 \times 4 = 36$ bytes).
