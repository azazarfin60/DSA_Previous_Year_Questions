## 📌 Executive Summary

This lecture covers insertion in DCLL at **beginning** and **end** — both are **O(1)** thanks to `head->prev` giving instant access to the last node.

---

## 💻 Insert at Beginning
```c
void insertAtBeginning(int data) {
    struct node *newNode = (struct node *)malloc(sizeof(struct node));
    newNode->data = data;

    if (head == NULL) {
        head = newNode;
        newNode->next = newNode->prev = head;
        return;
    }
    struct node *last = head->prev;   // O(1)!
    newNode->next = head;
    newNode->prev = last;
    last->next = newNode;
    head->prev = newNode;
    head = newNode;                    // Update head
}
```
**Time: O(1)**

---

## 💻 Insert at End
```c
void insertAtEnd(int data) {
    struct node *newNode = (struct node *)malloc(sizeof(struct node));
    newNode->data = data;

    if (head == NULL) {
        head = newNode;
        newNode->next = newNode->prev = head;
        return;
    }
    struct node *last = head->prev;
    newNode->next = head;
    newNode->prev = last;
    last->next = newNode;
    head->prev = newNode;
    // Don't update head — node is at end
}
```
**Time: O(1)** — no traversal!

---

## 📊 Comparison: Insert at Beginning

| List Type | Time |
|-----------|------|
| SLL | O(1) |
| DLL | O(1) |
| CLL | **O(n)** — must find last node |
| **DCLL** | **O(1)** — `head->prev` = last |

---

## 💡 Key Takeaways

- DCLL insertion at both ends is **O(1)** — major advantage
- `head->prev` eliminates the need to traverse to find last node
- Update 4 pointers for each insertion
- Same code for beginning and end — only difference: update `head` or not
