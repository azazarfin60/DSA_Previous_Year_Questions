## 📌 Executive Summary
Insertion sort is a simple sorting algorithm that builds the final sorted array one item at a time. It is much less efficient on large lists than more advanced algorithms like quicksort, heapsort, or merge sort. However, it is highly efficient for **small data sets** and arrays that are **already mostly sorted** (Best case $O(n)$).

---

## 🔑 Core Concept
1. **Divide the array:** Conceptually divide the array into two parts: a **Sorted Sublist** and an **Unsorted Sublist**.
2. **Initial State:** Assume the first element (index `0`) is already "sorted" (since a list of one element is always sorted). The rest of the array (index `1` to `n-1`) is unsorted.
3. **The Algorithm:**
   - Pick the first element from the unsorted sublist.
   - Store it in a `temp` variable. (This creates a conceptual "hole" or free space in the array).
   - Compare `temp` with the elements in the sorted sublist, moving from **right to left**.
   - If a sorted element is greater than `temp`, shift that element one position to the right.
   - Continue shifting until you find an element smaller than or equal to `temp`, or you reach the beginning of the array.
   - Insert the `temp` value into the newly created "hole".

## 📐 Step-by-Step Trace

**Given Array:** `A = [5, 4, 10, 1, 6, 2]` *(n = 6)*
- Initial Sorted Sublist: `[5]`
- Initial Unsorted Sublist: `[4, 10, 1, 6, 2]`

### Pass 1 (i = 1)
- **Pick:** `4` (store in `temp`).
- **Compare:** Is $5 > 4$? Yes. Shift $5$ to the right. 
- **Insert:** Reached start. Put `4` at index 0.
- **Array:** `[4, 5 | 10, 1, 6, 2]` *(| separates sorted/unsorted)*

### Pass 2 (i = 2)
- **Pick:** `10` (`temp = 10`).
- **Compare:** Is $5 > 10$? No.
- **Insert:** Put `10` right back.
- **Array:** `[4, 5, 10 | 1, 6, 2]`

### Pass 3 (i = 3)
- **Pick:** `1` (`temp = 1`).
- **Compare & Shift:**
  - $10 > 1$? Yes, shift $10$.
  - $5 > 1$? Yes, shift $5$.
  - $4 > 1$? Yes, shift $4$.
- **Insert:** Reached start. Put `1` at index 0.
- **Array:** `[1, 4, 5, 10 | 6, 2]`

### Pass 4 (i = 4)
- **Pick:** `6` (`temp = 6`).
- **Compare & Shift:**
  - $10 > 6$? Yes, shift $10$.
  - $5 > 6$? No. Stop shifting.
- **Insert:** Put `6` after $5$.
- **Array:** `[1, 4, 5, 6, 10 | 2]`

### Pass 5 (i = 5)
- **Pick:** `2` (`temp = 2`).
- **Compare & Shift:**
  - $10 > 2$? Yes, shift $10$.
  - $6 > 2$? Yes, shift $6$.
  - $5 > 2$? Yes, shift $5$.
  - $4 > 2$? Yes, shift $4$.
  - $1 > 2$? No. Stop shifting.
- **Insert:** Put `2` after $1$.
- **Final Array:** `[1, 2, 4, 5, 6, 10]`

## 💻 C Code Implementation

```c
void insertionSort(int a[], int n) {
    int i, j, temp;
    
    // Outer loop traces the unsorted sublist
    for(i = 1; i < n; i++) {
        temp = a[i]; // Pick the element
        j = i - 1;   // Start comparing with the last element of the sorted sublist
        
        // Inner loop: Shift elements of the sorted sublist to the right
        // if they are greater than 'temp'
        while(j >= 0 && a[j] > temp) {
            a[j+1] = a[j]; // Shift element right
            j--;           // Move to the next element on the left
        }
        
        // Insert 'temp' into its correct position
        a[j+1] = temp;
    }
}
```

## ⏱️ Complexity Analysis

| Case | Complexity | Reason |
| :--- | :--- | :--- |
| **Worst Case** | **O(n²)** | Array is in descending order. Every element picked must be compared against *all* elements in the sorted sublist, forcing maximum shifting. |
| **Best Case** | **O(n)** | Array is already sorted. The `while` loop condition `a[j] > temp` fails immediately on the first check. Only the outer loop runs `n-1` times. |
| **Average Case** | **O(n²)** | Elements are in random order. |
| **Space Complexity**| **O(1)** | Sorts in-place; only requires a `temp` variable. |

> **Exam Tip:** If asked "Which sorting algorithm performs best when the array is already sorted?", **Insertion Sort** and **Optimized Bubble Sort** are the correct answers, as their time complexity drops to $O(n)$.
