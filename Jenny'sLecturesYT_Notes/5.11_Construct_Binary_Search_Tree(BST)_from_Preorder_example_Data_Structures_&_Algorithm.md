# 5.11 Construct Binary Search Tree(BST) from Preorder example | Data Structures & Algorithm

📺 [Watch on YouTube](https://www.youtube.com/watch?v=GW63gMgfeS8)

---

## 📌 Executive Summary

BST can be uniquely reconstructed from **Preorder alone** because the BST property (left < root < right) implicitly defines the inorder (sorted order).

---

## 📐 Algorithm

1. First element = **Root**
2. All consecutive elements **< root** = **left subtree**
3. All remaining elements **> root** = **right subtree**
4. Recursively construct subtrees

---

## 📐 Example: Preorder = `[10, 5, 1, 7, 40, 50]`

```
Step 1: Root = 10
        < 10: [5, 1, 7]  → left subtree
        > 10: [40, 50]   → right subtree

Step 2: Left — Preorder [5, 1, 7]
        Root = 5, < 5: [1], > 5: [7]

Step 3: Right — Preorder [40, 50]
        Root = 40, > 40: [50]

Result:
       10
      /  \
     5    40
    / \     \
   1   7    50
```

### Verification (Inorder = sorted):
`1, 5, 7, 10, 40, 50` ✅

---

## 💻 Simple Approach: Insert One by One
```c
struct node* insert(struct node* root, int key) {
    if (root == NULL) {
        struct node* n = (struct node*)malloc(sizeof(struct node));
        n->data = key;
        n->left = n->right = NULL;
        return n;
    }
    if (key < root->data)
        root->left = insert(root->left, key);
    else
        root->right = insert(root->right, key);
    return root;
}
```

**Time: O(n²) worst case (skewed), O(n log n) average**

---

## 💡 Key Takeaways

- BST needs **only Preorder** to reconstruct (inorder = sorted)
- Split: elements < root go left, > root go right
- Simple method: insert preorder elements one by one
