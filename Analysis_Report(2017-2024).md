## 6. Question Pattern Analysis

### Type 1: Definition + Example (02–04 marks)
Guaranteed easy marks in **every paper**.
- "Define linear and non-linear data structure"
- "What is a linked list? Benefits over array?"
- "What are properties of recursion?"

**Strategy:** 1-line definition + 2–3 bullet properties + 1 small example.

### Type 2: Algorithm Trace / Step-by-Step (04–06 marks)
The bulk of the exam — given data, show every step.
- BST construction, Dijkstra's table, Heap build + ops
- Infix→Postfix with stack states, Sorting traces

**Strategy:** Practice on paper. Show initial state → every step → final result.

### Type 3: Pseudocode / Code Writing (03–06 marks)
Increasingly common from 2022 onward.
- Binary search (recursive), BST insertion, Linked list ops
- Tower of Hanoi, Tree height, Linked list reversal, List split

**Strategy:** Memorize clean C-style code with inline comments.

### Type 4: Complexity Analysis (03–05 marks)
- Analyze nested loops, prove Big-O/Ω, solve recurrences
- Compare best/worst/average cases

**Strategy:** Master Theorem 3 cases + substitution method + common complexities.

### Type 5: Comparison / Justification (02–04 marks)
- "Quick sort vs Bubble sort", "Array vs Linked list"
- "Performance analysis vs measurement"

**Strategy:** Comparison table with 4–5 criteria + concluding line.

## 8. Year-over-Year Trend Analysis

1. **2017–2019:** Classical DSA style. Heavy on definitions, traces, theoretical proofs. Full marks = 72.
2. **2020:** Transitional. Introduced code-writing (frequency counts, direct-address tables). Still 72 marks.
3. **2021:** Application-oriented shift. Polynomial addition via LL, cycle detection, Bellman-Ford. Still 72 marks.
4. **2022:** Major format change — 60 marks, CO mapping, code-completion questions (BST insert, tree height), first competitive-programming-style problem (Rotten Oranges BFS).
5. **2023:** CLO mapping introduced, Huffman tree first appearance, linked list split (code writing), Bellman-Ford with negative edges.
6. **2024:** Returns to traditional question styles but under OBE framework. Heavy overlap with 2017/2021. B-tree and doubly linked list first appearances.

```
2017─2019: Theory + Algorithm Traces (traditional, 72 marks)
     │
     ▼
2020─2021: Theory + More Application (transitional, 72 marks)
     │
     ▼
2022─2024: Theory + Code Writing + CO-mapped (OBE era, 60 marks)
```

> [!IMPORTANT]
> Expect more **code-writing** and **code-completion** questions. Prepare working C code for: BST insert, binary search (recursive), linked list operations, tree height, and linked list reversal.

## 11. Key Formulas to Memorize

|---|---|

## 12. Final Exam Strategy

1. **Start with Section A** — more definition/construction questions, easier to score quickly.
2. **Attempt BST + Heap first** — mechanical, guarantees marks if steps are correct.
3. **Graph algorithms** — always draw table/graph at each step. Partial credit for correct intermediate steps.
4. **Code-writing** — use C language, keep simple, add 2–3 inline comments.
5. **Time management** — 30 min per question (3 hrs / 6 questions). Never exceed 35 min on one question.
6. **Definitions** — 1 sentence definition + 2–3 properties + 1 small example.

> [!TIP]
> Solving **2021, 2022, 2023, and 2024** papers thoroughly covers approximately **75–85%** of the likely question space.
